# lib層の責務

## 概要

`lib/`ディレクトリは、**ドメイン非依存の純粋ユーティリティ層**です。すべてのドメインから参照される共通機能を提供します。

## 基本原則

### 1. ドメイン非依存

- **features層への依存禁止**: `lib/`から`@/features/`への依存は原則として禁止です
- **純粋ユーティリティ**: ドメイン固有のロジックを含まない、汎用的な関数のみを配置します

### 2. 配置すべきもの

| カテゴリ | 説明 | 例 |
|---------|------|-----|
| **純粋ユーティリティ** | ドメイン非依存の汎用関数 | `format.ts`, `cn.ts` |
| **環境・設定** | 環境変数、ロギング | `env.ts`, `logger.ts` |
| **共通型・パターン** | Result型、エラーハンドリング | `result.ts` |
| **外部サービス連携** | 分析、広告等（ドメイン非依存部分） | `analytics/` |
| **SEOユーティリティ** | ドメイン非依存のメタデータ生成 | `og-generator.ts` |

### 3. 配置すべきでないもの

| カテゴリ | 理由 | 正しい配置先 |
|---------|------|-------------|
| **ドメイン固有のメタデータ生成** | ドメインロジックが必要 | `features/{domain}/utils/` |
| **ドメイン固有の構造化データ** | ドメイン型・ロジックに依存 | `features/{domain}/utils/` |
| **DBアクセスを伴う処理** | インフラ層の関心事 | `features/{domain}/repositories/` |

## 依存関係の方向性

```
┌─────────────────────────────────────────────────────────────┐
│  lib/ (共有ユーティリティ層)                                   │
│  - ドメイン非依存の純粋ユーティリティ                           │
│  - 他のドメインから参照される                                   │
│  - features/への依存は禁止（原則）                             │
└─────────────────────────────────────────────────────────────┘
                                ↑
                                │ 参照（OK）
                                │
┌─────────────────────────────────────────────────────────────┐
│  features/ (ドメイン層)                                       │
│  - ドメイン固有のビジネスロジック                               │
│  - lib/を参照可能                                             │
│  - 他のfeatures/を参照可能（ただし循環依存禁止）                │
└─────────────────────────────────────────────────────────────┘
```

## チェックリスト

`lib/`に新しいファイルを追加する際は、以下を確認してください：

- [ ] ドメイン固有の型をimportしていない
- [ ] `@/features/` からのimportがない
- [ ] DBアクセスを直接行っていない
- [ ] 純粋関数として実装されている（可能な限り）

## 例外パターン

### 静的マッピングによる参照

`@stats47/area`の`lookupAreaNameSync`のように、静的マッピングのみを参照する場合は、lib層から参照しても問題ありません。これはDBアクセスを伴わないためです。

```typescript
// ✅ OK: 静的マッピングのみを参照
import { lookupAreaNameSync } from "@/features/area";
const areaName = lookupAreaNameSync(areaCode);
```
