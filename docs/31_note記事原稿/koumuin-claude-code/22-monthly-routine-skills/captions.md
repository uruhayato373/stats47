## Instagram — フィード投稿用キャプション (2200 字以内、改行は 2-3 行ずつ)

公務員の月次定型業務は、機械化に向いた仕事が多い。

入力フォーマットが固定 (各課からの報告様式)
出力フォーマットも固定 (上司提出形式・庁内報告形式)
判断より作業が中心 (転記・集計・整形)

Claude Code の `.claude/skills` は、こうした定型業務との相性が抜群です。

スキル化の判定軸は 3 条件。
(1) 月 1 回以上の頻度
(2) 1 回 30 分以上の所要時間
(3) 判断より作業が中心

この 3 つを満たせば、スキル化候補。1 スキル作成に 2-4 時間、ROI 回収は 2-3 ヶ月で達成できます。

失敗パターンは「Skills の中で判断させすぎる」こと。
判断は人間、作業を Skills に分離する設計が成功の鍵です。AI に判断を任せると、エッジケースで間違えて、結局人間が全レビューする羽目になります。

庁内展開時は「他人が読んでも実行できる SKILL.md」を書くことが重要。引継ぎを前提とした記述スタイル (具体的な手順・前提条件・想定エラー対処) を確立すれば、人事異動でも継続します。

note 有料記事 (¥1,200) では、3 条件判定フロー、SKILL.md レビュー観点 12 項目、庁内展開の手順を公開しました。
