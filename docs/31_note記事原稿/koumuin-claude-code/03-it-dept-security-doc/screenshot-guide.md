# 自治体 IT 担当に渡せる Claude Code セキュリティ説明資料 — スクリーンショット撮影ガイド

## 撮影前準備

### macOS 撮影コマンド
- 全画面: `Cmd + Shift + 3`
- 範囲指定: `Cmd + Shift + 4` → 範囲ドラッグ
- ウィンドウ単位: `Cmd + Shift + 4` → Space キー → ウィンドウクリック

### ターミナル設定 (本記事では Word 画面メイン)
- Word / Pages のズーム: 100% で A4 1 ページが画面内に収まる程度に縮小
- 背景色は紙の白 (#FFFFFF) で統一、ダークモード OFF

### マスキング原則 (本記事は最重要)
- **本記事は「実際の提出文書」が題材のため、マスキング失敗が直接事故につながる**
- 自治体名・部署名・申請者氏名・押印欄の印影 → **すべて架空名 (〇〇市総務課 / 担当: 山田太郎) に画像編集で置換**
- 文書内に書かれた電話番号・内線番号・庁内サーバー名 → 完全黒塗り
- 押印欄は印影画像を削除し、空の枠だけ残す
- 「申請日 2026-XX-XX」の具体日付は架空日付に置換 (実際の申請を特定されない)
- ヘッダー / フッターの自動印字 (ファイルパス・作成者名) → Word 設定で OFF にしてから撮影

### 保存先
- `docs/31_note記事原稿/koumuin-claude-code/03-it-dept-security-doc/images/` 配下
- ファイル名: `screenshot-N-<short-desc>.png`

- **マスキング項目**:
  - 件名: 「Claude Code 利用申請書 (〇〇市 総務課 庁内システム担当)」のような架空表記に
  - 申請者氏名・所属コード・押印欄: 架空名 + 印影削除
  - 宛先 (情シス課長名等): 「情報システム課長 殿」のように肩書きのみに
  - データフロー図に庁内サーバー名・IP が出ていれば架空名 (`srv-internal.example.local`) に
- **推奨ファイル名**: `screenshot-1-security-doc-pages.png`
- **撮影手順**:
  1. Word でマスキング済み .docx を開く
  2. 表示 → 1 ページ表示 (ズーム 100%)、ステータスバー / リボン非表示で本文を最大化
  3. 表紙ページを `Cmd + Shift + 4 → Space` でウィンドウ撮影
  4. 次のページにスクロールし、本文ページも同様に撮影
  5. Preview.app で 2 ページを横並びに連結

データ取扱: 個人情報・非公開情報は入力しない (hooks で自動マスキング)

## 撮影後の手順

1. PNG を `03-it-dept-security-doc/images/` に保存
2. ファイルサイズが 500KB 超なら `pngquant --quality=80-90 images/*.png --ext .png --force` で圧縮
3. EXIF メタデータ削除 (撮影端末情報が漏れないように)
4. draft.md 内の `> 📸 [スクリーンショット] ...` マーカーを以下に置換:
   ```markdown

   ```
5. note 投稿前に最終確認: 画像を 100% で拡大して隅々まで実名残存チェック、家族 / 同僚にレビュー依頼
