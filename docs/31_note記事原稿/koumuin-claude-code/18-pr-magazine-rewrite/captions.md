## Instagram — フィード投稿用キャプション (2200 字以内、改行は 2-3 行ずつ)

自治体の広報誌は、各課から集まる原稿を広報担当がリライトして発行されます。

ところが「もっと読みやすく書いてください」と返却すると、各課から「これが正式な表現です」「専門用語は省けません」と返ってきて、何度も往復することに。

「読みやすさ」は主観で議論すると揉めます。

Claude Code で「読みやすさスコア」を定量化すると、議論の土台が変わります。

スコア指標は 4 軸。
(1) 文長平均 (1 文あたりの文字数)
(2) 漢字含有率
(3) 専門用語密度
(4) 受動態率

これらを `scoring.yml` に定義し、Claude Code に原稿を投げると、スコア算出 → リライト案 3 種生成 → 文字数自動調整までを一気通貫で実行します。

`.claude/skills/koho-rewrite/` にスキル化すれば、毎月の発行サイクルに組み込めます。

各課への返却時にスコア表を添付。「主観の押し付け」が「客観基準の共有」に変わり、議論が圧倒的に早くなります。
