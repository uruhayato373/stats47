## Instagram — フィード投稿用キャプション (2200 字以内、改行は 2-3 行ずつ)

災害時、自治体の SNS 発信が「日本語だけ」だと、在留外国人 (技能実習・特定技能・留学生・観光客) に情報が届きません。

避難所の場所、断水の復旧見込み、罹災証明の発行手順——これらが届かないことが、二次被害を生む例が国内外で報告されています。

ところが、災害時に多言語翻訳をかけている時間はない。「翻訳精度」より「リードタイム」が支配的です。誤訳より無発信が致命的。

Claude Code で `.claude/skills/disaster/translate-sns/` を 1 個作れば、日本語原稿から 6 言語 (英・中・韓・ベトナム・ポルトガル・ネパール) + やさしい日本語が 1 プロンプトで完結します。

「やさしい日本語」は同じスキル内で生成。在留外国人 (特に技能実習・特定技能) に最も効きます。出入国管理庁 + 文化庁の「やさしい日本語ガイドライン」を参照しています。

翻訳結果のチェックは「数字・固有名詞・断定/推量」の 3 点だけに絞ります。災害時は速度優先。完璧を目指さない運用設計が現実解です。

LGWAN PC で動かせない場合の運用は、宿直机の個人 PC + テザリングで完結させる構成を記事に書きました。
