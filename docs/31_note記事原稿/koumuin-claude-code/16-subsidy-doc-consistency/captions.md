## Instagram — フィード投稿用キャプション (2200 字以内、改行は 2-3 行ずつ)

補助金申請書類のチェックは、自治体担当者の負荷が高い業務の 1 つ。

国・県・自治体それぞれの様式が混在し、「事業費総額」「実施期間」「申請者名」「補助率」が複数の書類で一致しているか、目視で何度も確認することになります。

Claude Code を使えば、Excel/PDF/Word の書類を中間テキスト化して、整合性キーごとに自動チェックできます。

手順。
(1) 整合性キーを `consistency-keys.yml` に定義 (事業費総額・実施期間・申請者名・補助率など)
(2) 書類群を Claude Code に投げて `/subsidy-check` 一発
(3) 不整合があれば「書類 A は ¥1,000,000、書類 B は ¥1,200,000」と差分レポート

`.claude/skills/subsidy-check/` にスキル化すれば、毎年・毎案件で再利用できます。引継ぎコストも下がります。

国・県の様式差は `appears_in` に書き分けて吸収。最終決裁前の人間チェックは省略しません。

守秘配慮として、申請書原本を Claude に渡すのではなく、値だけ抽出した中間ファイルを渡します。抽出スクリプトは LGWAN 接続端末側で完結させ、Claude には数値とラベルだけが届く設計です。

note 有料記事 (¥800) では、整合性キーの設計、抽出スクリプトのサンプル、不整合レポートのフォーマットを公開しました。
