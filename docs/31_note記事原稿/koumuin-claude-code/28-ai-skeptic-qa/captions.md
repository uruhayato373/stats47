技術論で反論しても通じません。なぜか。
上席の渋りの本音は「技術不信」より、次の 3 動機が主だからです。
(1) 責任不安: 何かあったら誰が責任を取るのか
(2) 世代格差不安: 若手についていけない自分を見せたくない
(3) 序列防衛: 部下に教わる立場になりたくない

これらは技術論で対処できません。動機を読んで対応を変えます。

必ず来る 3 質問は事前準備必須。
(1) 「個人情報はどう守るのか」
(2) 「他自治体の事例はあるのか」
(3) 「失敗したら誰の責任になるのか」

それぞれの回答テンプレを記事に収録しました。

最強の説得は「上席の困りごとを 1 つ Claude Code で解決して見せる」実演です。

渋る上席を「敵」と見ないことが、長期で一番効きます。最終的な味方化を目指します。
