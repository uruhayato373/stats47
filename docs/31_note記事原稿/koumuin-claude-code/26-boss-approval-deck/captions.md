## Instagram — フィード投稿用キャプション (2200 字以内、改行は 2-3 行ずつ)

「Claude Code を業務で使いたい」と上司に提案しても、なかなか OK が出ない。

なぜか。
上司が知りたいのは「機能」ではなく「責任」と「費用対効果」の 2 点だけだからです。

技術仕様を熱く語っても通りません。代わりに、責任 (誰が・いつ・どう止めるか) と費用対効果 (削減時間 / コスト / リスク) を A4 1 枚にまとめます。

説明資料は 2 段構え。
(1) A4 1 枚版: 課長級向け、5 分で読める
(2) 詳細版: 情シス・人事・財政の合議向け、10-15 ページ

決裁ルート上の役職で出し分けます。

「個人情報を入れない設定」(`.claude/settings.json` の `permissions.deny`) を最初の 1 ページに置くと、突破率が跳ね上がります。上司の最大の不安を最初に潰すのがコツ。

他自治体事例ゼロを恐れない。「先進事例なし = チャンス」とむしろ攻めに転換するロジックも記事に書きました。

稟議は段階分割が肝。
「試行運用 (個人 PC 1 ヶ月) → 効果測定 (削減時間レポート) → 本格導入 (庁内展開)」
の 3 段階に分けると、各段階の合議者を絞れて通りやすくなります。
