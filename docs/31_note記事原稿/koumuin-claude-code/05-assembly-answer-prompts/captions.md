## Instagram — フィード投稿用キャプション (2200 字以内、改行は 2-3 行ずつ)

議会答弁の原稿作成は、自治体職員にとって「最も時間と神経を使う仕事」の 1 つ。

過去答弁との整合、関連法令の引用、文体ルールの遵守、想定再質問への備え——ゼロから手書きで 8 時間かかることも珍しくありません。

Claude Code に「攻め (積極姿勢)・守り (慎重姿勢)・折衷 (バランス型)」の 3 パターンで答弁案を出させると、たたき台ができるまでが圧倒的に早くなります。

ポイントは 4 つ。
(1) 一次素材 (関連法令・過去答弁・統計) を `reference/` に集約
(2) 文体ルール (ですます調 / 主語「本市」/ 受身回避) を CLAUDE.md に固定
(3) Subagents 機能で「議員役・事務局役・首長役」の 3 視点から再質問を抽出
(4) 個人名・実議員名・実通告文・部内検討資料は絶対に投げない

最終原稿は必ず人間が責任を持って磨きます。AI 案はあくまでたたき台、政策判断は職員が行うのが大前提です。

note 有料記事 (¥1,200) では、6 タイプの議会質問 (政策的・調査要求・問題追及・要望伝達・確認・パフォーマンス) 別の prompt テンプレと、再質問対応のフローを公開しました。
