## Instagram — フィード投稿用キャプション (2200 字以内、改行は 2-3 行ずつ)

庁内には、誰が作ったか分からない Excel VBA マクロが大量に残っています。

担当者が異動・退職して保守できる人がいない
Office のバージョンアップで動かなくなった
セキュリティ強化で VBA が無効化された
動いているけど中身が読めず、変更できない

これらの VBA を Python に移植する作業は、従来 1 マクロ 1-2 週間がかかる重作業でした。

Claude Code を使えば、3-4 時間で完了します。

工程は 3 つ。
(1) 業務理解 (これは人間が必須)
(2) コード移植 (Claude Code が 70% カバー)
(3) 並列テスト (Claude Code が同等のテストケース生成)

業務理解は人間が必須。「このマクロが何のためにあるか」を AI は推測できません。

移植先は `openpyxl` + `pandas` が現実解。Excel 互換性が高く、Office なし環境でも動きます。

廃止判断も同時に行うのがポイント。
「移植しない (使われていない)」
「業務を見直す (そもそも不要)」
も選択肢に入れます。技術的な移植にこだわると、本来不要な機能を温存することになります。
