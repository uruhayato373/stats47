## 回遊設計 (2026-05-20 更新: マガジン方式へ)

note 公開後の回遊は **note マガジン機能** を主軸にする。

### なぜマガジン方式か

当初は各記事末尾に `../slug/draft.md` 形式の相互リンク (v1 フッタ) を張ったが、この相対パスは
note.com に公開するとリンク切れになる。実 note URL に書き換えるには全記事の公開後にしか
URL が分からず、相互リンクだと最大 31 URL を後から注入する必要があった。

note マガジンに 31 本をまとめれば、note 側が一覧と記事間導線を提供するため、
**注入が必要な URL はマガジン URL 1 個だけ**になる。

### v2 フッタ (マガジン回遊)

全 31 本の末尾に v2 フッタ (`<!-- circulation-footer:v2 -->`) を付与:

- シリーズの説明 + **マガジンへの導線 1 本**
- 未経験者向けに導入記事 #00 を名前で案内 (リンクではなくマガジン内で見つけてもらう)
- マガジン URL は `{{MAGAZINE_URL}}` プレースホルダーで保持

### 公開ワークフロー (再編集サイクルを最小化)

1. note でマガジン「公務員 × Claude Code」を作成 → マガジン URL を取得
2. `inject-magazine-url.cjs <マガジンURL>` で全 31 本の `{{MAGAZINE_URL}}` を実 URL に一括置換
3. 全 31 本を公開 (publish-note。Phase 0 ガードが未注入プレースホルダーを検知して中断)
4. 公開した 31 本をマガジンに追加

マガジン URL は記事公開前に取得できるため、**公開後の再編集サイクルは不要**。

### 実装スクリプト

| スクリプト | 役割 |
|---|---|
| `.claude/scripts/note/add-koumuin-magazine-footer.cjs` | 全 31 本に v2 フッタを付与 (v1 があれば置換、冪等) |
| `.claude/scripts/note/inject-magazine-url.cjs` | `{{MAGAZINE_URL}}` を実マガジン URL に一括置換 |
| `.claude/state/note-published-urls.json` | 公開済み記事の slug→URL 真実源 (publish-note が自動追記) |

旧 `add-koumuin-circulation-footer.cjs` (v1) は使用しない。
