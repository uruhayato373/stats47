## Instagram — フィード投稿用キャプション (2200 字以内、改行は 2-3 行ずつ)

自治体の HP に並ぶ「よくあるご質問 (FAQ)」は、住民が最初に見るページの 1 つ。

ところが「過去 1 年分の問い合わせを集約して FAQ を作る」となると、メール・電話メモを 1 件ずつ読んで分類する作業に膨大な時間がかかります。結果、FAQ が古いまま放置されているケースも少なくありません。

Claude Code に過去の問い合わせデータを読ませると、FAQ を 3 段階で自動生成できます。
(1) 意味クラスタリング (似た質問をまとめる)
(2) FAQ 文生成 (代表質問 + 回答案)
(3) 優先度付け (件数順 / 緊急度順)

回答案は要綱・例規集から引用させて、根拠を明示。所管課決裁前のたたき台として使えます。

月 1 回スキル実行で FAQ の鮮度を保つ運用が現実的。Memory に過去クラスタを蓄積し、新しい問い合わせパターンだけを差分検出する設計が組めます。

公開前は必ず所管課の確認。AI 補助を使ったことを明示する自治体が増えていますが、運用ルールに合わせて掲載方針を決めます。
