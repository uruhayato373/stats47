## Instagram — フィード投稿用キャプション (2200 字以内、改行は 2-3 行ずつ)

公文書の校正は、所属内で「人によって基準が違う」のが悩みのタネ。

「これは送り仮名が違う」「この接続詞は使わない」「ここは受身を避ける」——指摘する人によって観点も基準もバラバラで、新人ほど何度も差し戻されることになります。

Claude Code で `.claude/skills/official-doc-review/` を 1 個作れば、起案文・通知文・要綱・要領・回答書・公告など、すべての公文書に同じ校正基準が適用できます。

基準は 2 段構え。
(1) 内閣告示「公用文作成の考え方」(2022 年版)
(2) 自治体独自規程
両方を `reference/` に読み込ませ、自治体独自ルールが優先される設定にします。

出力は「該当箇所 (20 字)・現状・修正案・根拠 (規程セクション名 + 行番号)」の 4 列 markdown 表。起案文にそのまま貼り付け可能です。

機微情報を含む文書は別記事の個人情報マスキング設定と併用 (hook で送信前ブロック)。スキル化すれば、新人職員に「これ通してから提出して」と git で配って渡せます。チーム全体の差し戻し率が下がる効果も大きい。

note 有料記事 (¥800) では、スキル定義 SKILL.md のサンプルと、公用文表記の判定ルール 30 項目を公開しました。
