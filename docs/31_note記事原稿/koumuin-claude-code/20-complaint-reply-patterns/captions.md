# 苦情メール返信案を 5 パターン出す prompt — SNS 拡散キャプション

## X (旧 Twitter) — 公開時投稿 (140 字以内、本文 URL 別添前提)

苦情メール返信を「謝罪・説明・対応・再発防止」の 4 要素の重み配分違いで 5 パターン生成。全面謝罪型 / 制度説明型 / 中庸型 / 部分受容型 / 毅然型から職員が選ぶ。苦情本文の固有名詞は Hooks で事前マスキング。

#ClaudeCode #公務員 #住民対応

## X — 追い投稿 (公開 1 週間後、別角度の切り口で再投稿、140 字以内)

苦情対応は「謝りすぎ」も「突き放しすぎ」もリスク。5 パターン並べて職員が選ぶ構成にすると、新人でも適切な落としどころが見える。上司決裁前のたたき台用途で、最終文責は職員が持ちます。

#公務員 #苦情対応

## Instagram — フィード投稿用キャプション (2200 字以内、改行は 2-3 行ずつ)

住民からの苦情メール返信は、新人職員に最も負担の大きい仕事の 1 つ。

「謝りすぎると組織として不利な前例になる」「突き放しすぎると炎上する」——どこに落としどころを置くかの判断が難しく、何度も書き直しになります。

Claude Code に苦情メールを投げて、「謝罪・説明・対応・再発防止」の 4 要素の重み配分違いで 5 パターン生成させると、選ぶだけで適切な落としどころが見えます。

パターン分類。
(1) 全面謝罪型: 明らかな自治体側のミス
(2) 制度説明型: 制度上の制約を丁寧に説明
(3) 中庸型: 一部受容しつつ説明
(4) 部分受容型: 限定的に対応
(5) 毅然型: 不当要求への対応

守秘のため、苦情本文の固有名詞 (氏名・住所・電話・口座番号) は Hooks で事前マスキング。マスキング設定は別記事 (#11) で詳しく解説しています。

上司決裁前のたたき台として運用。最終文責は職員。AI 補助を起案文に明示するのが推奨です。
