## Instagram — フィード投稿用キャプション (2200 字以内、改行は 2-3 行ずつ)

公務員が AI を使うときの最大の不安は「個人情報を送ってしまうリスク」です。

住民情報・職員情報・特定個人情報——これらを誤って AI に送ると、情報漏えい事案として処分対象になりかねません。「だから AI は使えない」と諦めていた職員も少なくないはずです。

Claude Code には、このリスクを技術的に防ぐ仕組みがあります。

3 つの設定を組み合わせます。
(1) `PreToolUse` hook で個人情報パターン (電話番号 / マイナンバー / 氏名 + 様 など) を正規表現検出 → Read を技術的ブロック
(2) `settings.json` の `permissions.deny` で「絶対に読まないフォルダ」を固定 (`./personal-data/**` `~/Documents/住民*` など)
(3) `CLAUDE.md` (Memory) に「機微情報の扱い方針」を明示し、AI が自発的に確認するようにする

3 つ揃えると「うっかり個人情報を投げてしまう」リスクを技術的に防げます。二重ガード + 行動規範の構成です。

それでも 100% ではありません。最終的には「個人情報を含むファイルは別フォルダに隔離する」運用が前提です。技術的なガードと運用ルールの両輪が必要です。

本記事は無料で全文公開。3 つの設定の具体的な記述例と、本設定で Claude Code 業務利用の社内承認を取った実例を末尾に掲載しました。
