## Instagram — フィード投稿用キャプション (2200 字以内、改行は 2-3 行ずつ)

e-Stat (政府統計の総合窓口) のデータは便利ですが、ダウンロードした CSV をそのまま使えないことが多い。

3 大障害があります。
(1) セル結合 (見出しが結合されていて構造が崩れている)
(2) ヘッダ複数行 (1 行目だけ読んでも何のデータか分からない)
(3) 注釈混入 (データの途中に「※令和 5 年は推計」などのテキストが混じる)

Claude Code に Python (pandas) を書かせれば、構造把握 → 整形 → 検証まで自動化できます。

特に強いのは「47 都道府県データを地図に載せたい」のような最終目的を伝える指示。パイプライン全体 (取り込み → tidy 変換 → 集計 → 検証 → 可視化用 JSON 出力) が一気に生成されます。

ただし、統計データには「単位の罠」があります。
「千人」と「人」の混在
「年度」と「暦年」の混在
「令和」と「西暦」の混在
これらは人間が必ず検証する必要があります。AI に任せきりにすると、桁が 1000 倍ずれた集計が出てきます。
