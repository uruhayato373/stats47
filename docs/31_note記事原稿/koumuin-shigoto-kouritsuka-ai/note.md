「47 都道府県の統計データを毎日更新する」と聞いて、何人体制をイメージしますか。

答えは、 **1 人 + AI（Claude Code）** です。

そしてこのやり方、たぶん公務員や自治体職員の集計業務にも、ほぼそのまま応用できます。

私が運営している stats47.jp というサイトでは、政府統計（e-Stat）から取ってきたデータを **約 2,000 個の都道府県ランキング** として整理し、毎日自動で更新しています。

このやり方に至るまでの経緯と、公務員業務にどう応用できそうか、書いてみます。


## 1 人で 47 都道府県データを動かす日々

朝、Claude Code に「今日のランキングを更新して」と頼みます。

裏側ではこんなことが起きています。

- e-Stat API から最新データを取得
- 47 都道府県分の数値を計算
- ランキングを再生成（約 2,000 個全部）
- ブログ記事の数値を更新
- R2 ストレージに同期

10 〜 20 分後、見に行くと全部終わっています。

私がやっていることは、結果を眺めて「変な値が混ざってないか」をチェックすることだけ。

実は **約 180 本のブログ記事も、約 148 万件の都道府県別統計データも、すべてこの仕組みで管理しています**。1 人で。


## 半年前まで、半日かけてた集計作業

ここに至る前は、ぜんぜん違いました。

e-Stat（政府統計）のサイトから CSV をダウンロードする。

Excel で結合する。

地域コードの桁が揃ってなくて、 **VLOOKUP がうまく刺さらない**。

「あれ、北海道は `01000` なのに、ある表は `1` で入ってる」。

直して、ようやくグラフが作れる。

1 つの指標で、半日仕事。

5 つの指標を扱う議会資料なら、 **ほぼ丸 1 日**。

「あれ、これ手作業でやる意味あるんだっけ？」と気づいたのが、半年くらい前の話です。


## Claude Code との出会い

きっかけは、個人開発の自動化ツールを探していたことでした。

最初は半信半疑です。

「AI に集計とか任せて大丈夫なんだろうか」

「結局、自分で書いた方が早いんじゃないか」

でも、 **1 週間使ってみて、考えが変わりました**。

スキル（手順書）を書いておけば、それを Claude Code が読んで自走してくれる。

データ取得・整形・チャート生成・記事更新——それぞれを別々のスキルにしておけば、組み合わせて使える。

1 週間後、自動化スキルが 5 個できていました。

3 ヶ月後、 **123 個に増えていました**。

今では、自分が手を動かすのは「最終チェックと、新しい仕組みの設計」だけです。


## 公務員の業務にも、たぶん効く

ここからが本題です。

stats47 でやっていることは、 **公務員業務の構造とほぼ同じ** なんです。

「データソースから定型データを取得し、整形して、報告書 / 資料に流し込む」。

たとえばこんな業務、思い当たりませんか。

- **月次の人口動態報告書** — 同じ表を、年齢階級だけ変えて、毎月作っている
- **議会答弁の資料** — 過去 3 年の答弁を検索して、最新統計と組み合わせる
- **予算執行率のとりまとめ** — 部局別の数字を集約して、月次ダッシュボード化
- **庁内オープンデータの公開準備** — CSV を整形して、メタデータをつけて公開

これらはすべて、 **Claude Code でスキル化できる業務** です。

1 度書けば、翌月以降は引数を変えるだけ。退職や異動で属人化することもありません。

ただし、注意点が 1 つあります。

**個人情報を含むデータは、外部 API に渡さない設計が必須** です。

クラウド型 LLM（私が使っている Claude も含む）に住民データを送るのは、ガバナンス上アウトです。庁内ネットワーク内に閉じた LLM、もしくはオンプレ運用を選ぶ必要があります。

技術的にはまったく可能で、 **「公開データを扱うスキル」と「機密データを扱うスキル」を分ける** だけです。設計のセオリーがあります。


## 1 人で覚えるのが難しい、と感じたら

> ※PR：以下は業務効率化トレーニングのご紹介です。

私は手探りで覚えました。

公式ドキュメントを読んで、試行錯誤して、失敗して、ようやく今のレベルに来ています。

でも正直、 **独学で 100 時間かけるより、効率的に学ぶ選択肢もあります** 。

「AI鬼管理」というサービスでは、Claude Code を使った業務効率化のトレーニングを提供しています。

**無料診断（オンライン面談 30 分）** で、自分の業務にどう Claude Code を当てはめられるか、専門家が一緒に整理してくれます。

無料なので、迷ったら聞くだけでも価値があると思います。「導入を決める前に、まず話だけ聞いてみる」というスタンスでも全然 OK です。

## まとめ

stats47 を 1 人で運用してみて、気づいたことがあります。

自動化で大事なのは、 **「コードを書く能力」より「業務を分解する能力」** だということです。

「データ取得 → 整形 → 可視化 → 公開」をきれいに切り分けられれば、AI はそれぞれを高速にこなしてくれます。

逆に、業務の流れを言葉にできていないと、AI に任せる前に詰まります。

公務員業務こそ、定型化された手順が多く、AI による効率化の余地が大きい領域だと思っています。

1 人で 47 都道府県分のデータを回せるなら、自治体の月次業務も同じ仕組みで回せます。

興味があれば、 stats47.jp で実例を公開しているので、覗いてみてください。
