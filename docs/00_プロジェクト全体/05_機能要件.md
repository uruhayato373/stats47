# 機能要件定義書

## 1. 機能概要

### 1.1 システム全体の機能
stats47は、日本の地域統計データを可視化するWebアプリケーションとして、以下の主要機能を提供する。

## 2. 主要機能一覧

### 2.1 地域選択機能
- **機能名**: RegionSelector
- **概要**: ユーザーが表示したい地域を選択する機能
- **詳細**:
  - 都道府県レベルの選択
  - 市区町村レベルの選択（将来拡張）
  - 地域名による検索機能
  - 地域の階層構造表示

### 2.2 データ取得機能
- **機能名**: EstatDataFetcher
- **概要**: e-Stat APIから統計データを取得する機能
- **詳細**:
  - e-Stat APIとの連携
  - データのキャッシュ機能
  - エラーハンドリング
  - サンプルデータの提供（APIキー未設定時）

### 2.3 データ可視化機能
- **機能名**: StatisticsDisplay
- **概要**: 取得した統計データをグラフやチャートで表示する機能
- **詳細**:
  - 棒グラフ表示
  - 折れ線グラフ表示
  - 円グラフ表示
  - 地域比較表示
  - レスポンシブ対応

### 2.4 ダッシュボード機能
- **機能名**: DashboardPage
- **概要**: 複数の統計データを統合表示するメイン画面
- **詳細**:
  - 地域選択とデータ表示の統合
  - 複数指標の同時表示
  - レイアウトの最適化

## 3. ユーザーストーリー

### 3.1 地域データ閲覧ストーリー
```
As a 地域に興味のあるユーザー
I want to 特定の地域の統計データを確認したい
So that その地域の現状や特徴を理解できる
```

**受け入れ条件**:
- 地域選択画面で都道府県を選択できる
- 選択した地域の統計データが表示される
- データは分かりやすいグラフで可視化される

### 3.2 地域比較ストーリー
```
As a 地域比較に興味のあるユーザー
I want to 複数の地域のデータを比較したい
So that 地域間の違いや特徴を把握できる
```

**受け入れ条件**:
- 複数地域の選択が可能
- 比較用のグラフが表示される
- 地域間の数値差が明確に分かる

## 4. 画面・UI要件

### 4.1 レスポンシブデザイン
- **デスクトップ**: 1200px以上
- **タブレット**: 768px-1199px
- **モバイル**: 767px以下

### 4.2 アクセシビリティ
- WCAG 2.1 AA準拠
- キーボードナビゲーション対応
- スクリーンリーダー対応
- カラーコントラスト比4.5:1以上

### 4.3 UIコンポーネント
- 地域選択ドロップダウン
- データ表示カード
- グラフ・チャートコンポーネント
- ローディングインジケーター
- エラーメッセージ表示

## 5. データ処理要件

### 5.1 データソース
- **主要**: e-Stat API
- **バックアップ**: サンプルデータ（CSV/JSON）

### 5.2 データ形式
- **入力**: JSON（e-Stat API）
- **内部処理**: TypeScript型定義
- **表示**: グラフ用データ変換

### 5.3 データ更新
- リアルタイム更新なし
- 手動更新ボタン提供
- キャッシュ有効期限: 24時間

## 6. API連携要件

### 6.1 e-Stat API連携
- **エンドポイント**: https://api.e-stat.go.jp/rest/3.0/app/json/getStatsData
- **認証**: APIキー認証
- **レート制限**: 1日1,000リクエスト
- **データ形式**: JSON

### 6.2 エラーハンドリング
- API接続エラー時のフォールバック
- タイムアウト処理（30秒）
- ユーザーフレンドリーなエラーメッセージ
