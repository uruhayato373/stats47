## 概要

stats47プロジェクトでは、D3.jsとRechartsの2つのライブラリを使用して可視化を実装します。このガイドでは、どの場面でどちらのライブラリを選択すべきかを判断する基準を提供します。

## D3.js vs Recharts 判断基準

### D3.jsを選択すべきケース

#### 1. 地図ベースの可視化
- **コロプレスマップ**: 都道府県・市区町村の色分け表示
- **インタラクティブ地図**: ズーム・パン・クリック操作
- **地理的データ**: 位置情報を含むデータの可視化

**理由**: D3.jsは地理的データの描画に最適化されており、GeoJSONの処理や地図投影が容易

#### 2. 独自のインタラクション実装が必要
- **カスタムツールチップ**: 複雑な情報表示
- **ドラッグ&ドロップ**: データの並び替え
- **ブラシ選択**: 範囲選択によるフィルタリング
- **複雑なホバー効果**: マルチレイヤー表示

**理由**: D3.jsは低レベルAPIを提供し、完全なカスタマイズが可能

#### 3. 複雑なアニメーション・トランジション
- **データ更新時のスムーズな遷移**
- **複雑なパスアニメーション**
- **段階的な表示アニメーション**
- **物理演算に基づく動き**

**理由**: D3.jsのtransition APIは非常に柔軟で強力

#### 4. 大量データの最適化が必要
- **10,000件以上のデータポイント**
- **リアルタイムデータ更新**
- **仮想スクロール対応**
- **メモリ効率の最適化**

**理由**: D3.jsは手動で最適化を制御できる

#### 5. 独自のチャートタイプ
- **ネットワーク図**
- **サンキー図**
- **ツリーマップ**
- **カスタムレイアウト**

**理由**: D3.jsは任意のSVG要素を自由に配置できる

### Rechartsを選択すべきケース

#### 1. 標準的なチャート表示
- **棒グラフ**: ランキング表示
- **折れ線グラフ**: 時系列データ
- **散布図**: 相関分析
- **円グラフ**: 割合表示

**理由**: Rechartsは標準的なチャートが簡単に実装できる

#### 2. React統合が重要
- **React状態との連携**
- **Props による動的更新**
- **React Hooks との統合**
- **コンポーネントの再利用**

**理由**: RechartsはReact専用に設計されている

#### 3. 迅速な開発が求められる
- **プロトタイプ作成**
- **MVP開発**
- **標準的な要件**
- **短い開発期間**

**理由**: Rechartsは宣言的APIで実装が簡単

#### 4. レスポンシブ対応が重要
- **モバイル表示**
- **画面サイズ自動調整**
- **タッチ操作対応**
- **デバイス別最適化**

**理由**: RechartsのResponsiveContainerが自動対応

#### 5. 保守性を重視
- **チーム開発**
- **コードの可読性**
- **標準的なパターン**
- **ドキュメント充実**

**理由**: Rechartsは学習コストが低く、保守しやすい

## 両方実装する可能性があるケース

同じチャートタイプでも、用途によってライブラリを使い分けることがあります。

### 棒グラフの例

#### Recharts版
**用途**: ダッシュボードの標準表示
```typescript
// シンプルなランキング表示
<BarChart data={rankingData}>
  <Bar dataKey="value" fill="#8884d8" />
</BarChart>
```

**特徴**:
- 迅速な実装
- 標準的な表示
- レスポンシブ対応
- 保守しやすい

#### D3.js版
**用途**: 詳細ページのインタラクティブ表示
```typescript
// 高度なインタラクション付き
const bars = svg.selectAll('.bar')
  .data(data)
  .enter()
  .append('rect')
  .on('click', handleClick)
  .on('mouseover', showTooltip);
```

**特徴**:
- カスタムアニメーション
- 複雑なツールチップ
- ドラッグ&ドロップ
- 完全なカスタマイズ

### 折れ線グラフの例

#### Recharts版
**用途**: 時系列データの標準表示
- ダッシュボードでの概要表示
- 複数系列の比較
- 凡例とツールチップ

#### D3.js版
**用途**: 詳細な時系列分析
- ブラシ選択による期間絞り込み
- ズーム機能
- データポイントの詳細表示

## パフォーマンス比較

| 項目 | D3.js | Recharts | 備考 |
|------|-------|----------|------|
| **初期レンダリング** | 中 | 速 | Rechartsは最適化済み |
| **大量データ** | 最適化次第 | 制限あり | D3.jsは手動最適化可能 |
| **アニメーション** | 自由度高 | 限定的 | D3.jsは任意のアニメーション |
| **バンドルサイズ** | 可変 | 固定 | 使用する機能に依存 |
| **メモリ使用量** | 制御可能 | 自動管理 | D3.jsは手動メモリ管理 |
| **学習コスト** | 高 | 低 | RechartsはReact知識で十分 |
| **カスタマイズ性** | 最高 | 中 | D3.jsは完全制御可能 |

## 実装コスト比較

| 項目 | D3.js | Recharts |
|------|-------|----------|
| **初期実装** | 高 | 低 |
| **カスタマイズ** | 中 | 高 |
| **保守** | 中 | 低 |
| **テスト** | 高 | 中 |
| **ドキュメント** | 高 | 低 |

## 選択フローチャート

```
データの種類は？
├─ 地理的データ → D3.js
├─ 標準的なチャート → Recharts
└─ その他

インタラクション要件は？
├─ 標準的 → Recharts
├─ カスタム → D3.js
└─ なし → Recharts

データ量は？
├─ 大量（10,000件以上）→ D3.js
├─ 中程度（1,000-10,000件）→ 用途次第
└─ 少量（1,000件未満）→ Recharts

開発期間は？
├─ 短期（1週間以内）→ Recharts
├─ 中期（1-2週間）→ 要件次第
└─ 長期（2週間以上）→ D3.js

チームのスキルレベルは？
├─ React中心 → Recharts
├─ JavaScript上級 → D3.js
└─ 学習意欲高 → D3.js
```

## 推奨パターン

### 1. 段階的実装
1. **Phase 1**: Rechartsでプロトタイプ作成
2. **Phase 2**: 要件に応じてD3.jsに移行
3. **Phase 3**: 両方のバージョンを維持

### 2. 用途別使い分け
- **ダッシュボード**: Recharts（標準表示）
- **詳細ページ**: D3.js（インタラクティブ）
- **管理画面**: Recharts（迅速実装）
- **分析画面**: D3.js（高度な機能）

### 3. 共通化
- データ変換ロジックの共通化
- 型定義の共有
- テストパターンの統一
- ドキュメントの統合
