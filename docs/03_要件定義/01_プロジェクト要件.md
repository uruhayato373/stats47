# プロジェクト要件定義書

## 1. プロジェクト概要

### 1.1 プロジェクト名
stats47 - 日本の地域統計データ可視化Webアプリケーション

### 1.2 プロジェクトの背景・目的
日本の地域統計データを分かりやすく可視化し、一般ユーザーが地域の特性やトレンドを直感的に理解できるWebアプリケーションを提供する。

### 1.3 ビジョンとミッション
- **ビジョン**: データドリブンな地域理解を促進し、より良い地域社会の構築に貢献する
- **ミッション**: 複雑な統計データを分かりやすい形で提供し、誰もが地域の現状を理解できるプラットフォームを構築する

### 1.4 技術スタック

#### フロントエンド
- **フレームワーク**: Next.js 15 (App Router)
  - 選定理由: 最新のReact機能とSSR/SSGの最適化
  - バージョン: 15.0.0以上
- **UIライブラリ**: React 19
  - 選定理由: 最新のReact機能（useCallback、useMemo等）の活用
  - バージョン: 19.0.0以上
- **スタイリング**: Tailwind CSS 4
  - 選定理由: ユーティリティファースト、レスポンシブデザインの効率化
  - バージョン: 4.0.0以上
- **言語**: TypeScript
  - 選定理由: 型安全性による開発効率とバグ削減
  - バージョン: 5.0.0以上

#### バックエンド・インフラ
- **ホスティング**: Cloudflare Pages (推奨)
  - 選定理由: グローバルCDN、高速配信、D1データベース統合
  - 代替: Vercel, Netlify
- **データベース**: Cloudflare D1 (SQLite)
  - 選定理由: サーバーレス、グローバル配信、コスト効率
  - 代替: Supabase / PostgreSQL
- **認証**: JWT + bcryptjs
  - 選定理由: シンプルな実装、外部依存の最小化
  - 代替: Supabase Auth, NextAuth.js

#### データ可視化・UI
- **グラフライブラリ**: Recharts
  - 選定理由: React統合、レスポンシブ対応、カスタマイズ性
  - バージョン: 2.8.0以上
- **地図可視化**: D3.js
  - 選定理由: 高度なカスタマイズ、コロプレス地図の実装
  - バージョン: 7.8.0以上
- **状態管理**: Jotai
  - 選定理由: アトミックな状態管理、React 19との親和性
  - バージョン: 2.6.0以上

#### 開発・ビルド
- **ビルドツール**: Turbopack
  - 選定理由: 高速ビルド、Next.js 15との統合
- **リンター**: ESLint
  - 選定理由: コード品質の維持、TypeScript対応
  - 設定: Next.js推奨設定 + カスタムルール
- **フォーマッター**: Prettier
  - 選定理由: コードフォーマットの統一
- **テスト**: Vitest
  - 選定理由: 高速テスト実行、TypeScript対応

## 2. 対象ユーザー

### 2.1 プライマリユーザー
- 地域の現状やトレンドに興味を持つ一般ユーザー
- 地域の統計データを分かりやすく理解したい人
- 地域比較や分析に興味のある人

### 2.2 セカンダリユーザー
- 地域研究を行う学生・研究者
- 地域政策に関わる行政関係者
- 地域ビジネスに関わる企業関係者

## 3. プロジェクトスコープ

### 3.1 含まれる機能
- 地域選択機能（都道府県、市区町村レベル）
- e-Stat APIを活用した統計データ取得
- データの可視化（グラフ、チャート）
- 地域間比較機能
- レスポンシブデザイン対応

### 3.2 含まれない機能
- ユーザー認証・会員機能
- データの編集・更新機能
- リアルタイムデータ更新
- モバイルアプリ

## 4. 成功指標（KPI）

### 4.1 定量的指標
- 月間アクティブユーザー数: 1,000人以上
- ページビュー数: 月間10,000PV以上
- 平均セッション時間: 3分以上
- 直帰率: 60%以下

### 4.2 定性的指標
- ユーザーからのフィードバック評価: 4.0/5.0以上
- 地域データの理解促進効果
- ユーザビリティの向上

## 5. 制約条件

### 5.1 技術的制約
- e-Stat APIの利用制限に準拠
- ブラウザ互換性: Chrome, Firefox, Safari, Edge
- レスポンシブデザイン必須

### 5.2 リソース制約
- 開発期間: 3ヶ月以内
- 開発チーム: 1-2名
- 運用コスト: 月額5,000円以内

### 5.3 法的制約
- 統計データの利用規約遵守
- 個人情報保護法への準拠
- 著作権法への準拠

## 6. リスクと対策

### 6.1 技術リスク
- **リスク**: e-Stat APIの仕様変更
- **対策**: API仕様の定期的な確認とバージョン管理

### 6.2 運用リスク
- **リスク**: サーバー負荷によるパフォーマンス低下
- **対策**: キャッシュ機能の実装とCDNの活用

## 7. 開発環境セットアップ

### 7.1 前提条件

- **Node.js**: 18.x以上（推奨: 20.x）
- **npm**: 9.x以上
- **Git**: 2.30以上
- **Cloudflareアカウント**: D1データベース用

### 7.2 環境構築手順

#### 1. リポジトリのクローン
```bash
git clone <repository-url>
cd stats47
```

#### 2. 依存関係のインストール
```bash
npm install
```

#### 3. 環境変数の設定
```bash
cp env.example .env.local
```

必要な環境変数:
```env
# Cloudflare D1設定
CLOUDFLARE_API_TOKEN=your_api_token_here
CLOUDFLARE_ACCOUNT_ID=your_account_id_here
CLOUDFLARE_D1_DATABASE_ID=your_database_id_here

# e-Stat API設定
ESTAT_API_KEY=your_estat_api_key_here

# アプリケーション設定
NEXT_PUBLIC_SITE_URL=http://localhost:3000
```

#### 4. データベースの初期化
```bash
# D1データベースの作成
npx wrangler d1 create stats47

# スキーマの適用
npx wrangler d1 execute stats47 --remote --file=./database/schemas/main.sql
```

#### 5. 開発サーバーの起動
```bash
npm run dev
```

### 7.3 開発ツール

#### Storybook（コンポーネント開発）
```bash
npm run storybook
# http://localhost:6006 でアクセス
```

#### テスト実行
```bash
# ユニットテスト
npm test

# テストカバレッジ
npm run test:coverage

# E2Eテスト
npm run test:e2e
```
