## JS 削減の具体策

現状: 全ページで JS 1.3~1.6MB（これが LCP 悪化の主因）

| 対策 | 削減見込み | 難易度 |
|---|---|---|
| **route-based code splitting** の徹底（shared chunk の分割） | 200~400KB | 中 |
| **unused exports の tree-shaking** 強化（barrel export の見直し） | 50~100KB | 低 |
| **重いライブラリの代替** (例: date-fns → dayjs, lodash → native) | 50~150KB | 低 |
| **SSR-only コードの client bundle 除外** | 100~200KB | 中 |
| **next/dynamic の追加適用**（viewport 外コンポーネント） | 100~300KB | 低 |
