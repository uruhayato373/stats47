**根本原因（推定）**: 計測タスクは「技術的に新しくない」「単純な API 呼び出し」であるため、相関分析やデザインシステムのような「技術的に面白い」タスクに心理的に負ける。これは意志力の問題ではなく、タスク設計の問題。

**対策案**:
- 計測タスクを「面白くする」: 計測結果のダッシュボード表示や自動レポート生成と組み合わせる
- 物理的にブロック: 計測完了を PR マージの前提条件にする
- タイムボックス: 月曜 9:00-9:30 は計測専用。30 分で終わる量に限定
