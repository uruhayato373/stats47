> **「先週と同じ失敗を繰り返してないか？」**
> SNS 投稿が 6 週連続 Must 入り→未達。今回の対策: **「月曜の最初のタスク」と明記**。ただし、これも意志力依存の側面がある。真の構造的対策は「投稿完了するまで他のタスクに着手しない」ルールの徹底。Must-1 の X 3 件が未完了の状態で Should/Could に進まないこと。
>
> **Could の新規ランキング登録は削除した**。W15 でも Could の技術タスク（DB スキーマクリーンアップ等）が Must の SNS 投稿より先に実行された。Could には SNS/コンテンツ系のみ配置。
>
> **「今週やらないと機会損失になるものは？」**
> 年収ランキング・社会減トレンドはニュース連動だが、1-2 週の猶予あり。Could に配置で適切。デプロイ（Must-3）は W15 の SEO 修正を早期に本番反映するため今週中が望ましい。

> **工数見積もりの妥当性**: Must 3 件は全て S（小）。Should 3 件は S+M+S。合計工数は現実的。ただし、Must-1（X 3 件投稿）が未達になるリスクは「投稿作業自体の難易度」ではなく「他タスクに気を取られる」パターン。タスク順序の厳守が鍵。
