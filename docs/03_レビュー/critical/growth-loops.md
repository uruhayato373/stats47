# stats47 成長ループ分析

生成日: 2026-03-09

## 現状サマリー

| 資産 | 数値 |
|---|---|
| 公開記事数 | 69本 |
| 原稿完成・未公開 | 0本 |
| ランキングページ | 1,511件（充填率 96.9%） |
| カテゴリ | 17 |
| SNS 投稿済みコンテンツ | 55セット（ranking 29 + bar-chart-race 26） |
| YouTube 動画 | 55本（総再生 34,541、登録者 11） |
| AdSense | 4箇所設置、ASP 5社登録 |
| SNS 指標計測 | 未実施（sns_metrics テーブル空） |

**診断**: 技術基盤とスキル群は完成。ボトルネックは**新規記事の継続的な企画・公開**と**計測によるフィードバックループの確立**。

### 主要流入チャネル（推定）

1. **オーガニック検索**: 69記事 + 1,511ランキングページがインデックス対象。現時点で最大のポテンシャル
2. **YouTube**: 55本投稿済み、34,541再生。説明欄からサイトへの誘導が可能
3. **SNS（X/Instagram/TikTok）**: 55コンテンツを投稿済みだが、フォロワー数・エンゲージメント未計測
4. **note.com**: 未確認（投稿済みだが流入計測なし）

### 現在機能しているループ

- **部分的に機能**: コンテンツ SEO ループ（記事は公開済みだが、計測・最適化サイクルが回っていない）
- **初期段階**: クロスプラットフォームループ（SNS → サイトの導線は構築済みだが効果未検証）
- **未起動**: UGC ループ、データ引用ループ（トラフィック規模が前提条件を満たしていない）

## ループ評価

### 1. コンテンツ SEO ループ（推奨度: ★★★★★）

**仕組み**:
```
記事公開 → Google インデックス → 検索流入
    ↑                                    ↓
順位上昇 ← 被リンク増加 ← ユーザーがデータ引用・シェア
```

**評価**:

| 基準 | スコア | 根拠 |
|---|---|---|
| 適合度 | 5/5 | 統計データは検索需要が安定（「都道府県 ○○ ランキング」は常時検索される）。コンテンツメディアの最も自然な成長エンジン |
| 現在の成熟度 | 3/5 | 69記事 + 1,511ランキングページは十分な規模。ただし GSC/GA4 での計測・リライト最適化サイクルが未起動 |
| 実装コスト | S | 記事企画・執筆スキルが完備。新規記事の企画 → 執筆 → 計測 → リライトのプロセスを回すだけ |
| 成長速度 | 中期（2-4ヶ月） | 新規記事のインデックスに2-4週間。リライト効果は1-2ヶ月後に発現 |
| 複利効果 | **極めて高い** | ドメイン権威が上がるほど新規記事も上位表示されやすくなる。記事数が増えるほど内部リンクの回遊価値が上昇 |

**具体的アクション**:
1. バッチ2 カテゴリ企画を `/plan-blog-articles` で生成し、週3-5本ペースで新規記事を公開
2. GSC ベースラインを取得し、順位11-30位の記事をリライト対象に特定
3. FAQ 構造化データ（JSON-LD）を全ランキングページに追加
4. カテゴリページ → 記事、記事 → ランキングの内部リンクを体系的に設置
5. トレンド連動記事（`/discover-trends-all`）で速報性のある記事を月2-3本追加

**ループ係数推定**:
- 記事1本あたりの月間検索流入: 20-100セッション（ロングテール含む）
- 69記事の推定月間オーガニック流入: 1,400-7,000セッション
- 目標（100記事時）: 3,000-15,000セッション/月
- サイクル期間: 2-8週間（記事公開からインデックス・順位安定まで）

### 2. SNS バイラルループ（推奨度: ★★★★☆）

**仕組み**:
```
統計データ投稿（「意外な事実」） → シェア・リポスト
         ↑                              ↓
   サイトでデータ発見 ← 新規ユーザー流入
         ↓
   自発的にシェア → フォロワー増加
```

**評価**:

| 基準 | スコア | 根拠 |
|---|---|---|
| 適合度 | 4/5 | 都道府県ランキングは「意外性」が高く、地元愛を刺激するためシェアされやすい。「○○が1位の県は？」形式はクイズ感覚で拡散する |
| 現在の成熟度 | 2/5 | 55コンテンツ投稿済みだが、エンゲージメント未計測。投稿の質・タイミング最適化が未実施 |
| 実装コスト | S | キャプション生成スキル（`/post-sns-captions`）が完備。投稿作業のみ |
| 成長速度 | 短期（1-2週間） | バイラル投稿は即日効果。ただしフォロワーベースが小さいうちは拡散力が限定的 |
| 複利効果 | 中程度 | フォロワーが増えるほどリーチが拡大するが、アルゴリズム依存度が高い。プラットフォームリスクあり |

**具体的アクション**:
1. X で週3投稿を定着（compare/correlation の新コンテンツを投入）
2. Instagram で週2投稿（カルーセル形式のランキング画像）
3. 投稿時間の A/B テスト（平日昼 vs 夜、週末 vs 平日）
4. 「○○が1位の都道府県は？」クイズ形式の投稿テンプレートを確立
5. `sns_metrics` への指標蓄積を開始し、エンゲージメント率でコンテンツを最適化

**ループ係数推定**:
- 投稿1件あたりの平均リーチ: 500-2,000（X）、300-1,000（Instagram）
- サイト流入変換率: 1-3%（UTM パラメータで計測）
- 投稿1件あたりのサイト流入: 5-60セッション
- シェア率（リポスト/引用）: 2-5%
- サイクル期間: 即日-3日

### 3. クロスプラットフォームループ（推奨度: ★★★★☆）

**仕組み**:
```
YouTube 動画 → 説明欄リンク → サイト流入
     ↓                              ↓
Instagram リール ← 同一素材転用    記事閲覧
     ↓                              ↓
X 投稿 ← 記事からの引用          note.com 記事
     ↓                              ↓
TikTok ← 同一動画転用            被リンク
     ↓
各 SNS フォロワー増 → 次の投稿のリーチ拡大
```

**評価**:

| 基準 | スコア | 根拠 |
|---|---|---|
| 適合度 | 4/5 | Bar Chart Race 動画は YouTube/TikTok/Instagram Reels で転用可能。1素材で4プラットフォームをカバーする効率の良さ |
| 現在の成熟度 | 2/5 | 55本の YouTube 動画と全 SNS 投稿済みだが、プラットフォーム間の誘導（例: YouTube → サイト）の効果が未検証 |
| 実装コスト | S | Remotion による動画生成パイプラインが完備。各 SNS キャプション生成スキルも整備済み |
| 成長速度 | 中期（1-3ヶ月） | 各プラットフォームのアルゴリズムに乗るまで一定の投稿数が必要 |
| 複利効果 | 高い | プラットフォーム間でオーディエンスが相互送客される。1つのプラットフォームでのヒットが他全てに波及 |

**具体的アクション**:
1. YouTube 動画の説明欄に stats47.jp の対応ランキングページ URL を必ず設置
2. YouTube ピン留めコメントでサイトへの誘導メッセージを統一
3. Instagram プロフィールリンクを stats47.jp のランディングページに設定
4. note.com 記事内に stats47.jp への自然なリンクを挿入（月2本）
5. 各プラットフォームの UTM パラメータを統一し、流入元別の効果を GA4 で計測

**ループ係数推定**:
- YouTube 動画1本あたりの月間再生: 628回（34,541 / 55本）
- YouTube → サイト流入変換率: 0.5-2%（説明欄クリック率）
- YouTube 経由月間流入: 170-690セッション（全動画合計）
- クロスプラットフォーム乗数: 1.5-3x（1つのヒットが他に波及）

### 4. UGC（ユーザー生成コンテンツ）ループ（推奨度: ★★★☆☆）

**仕組み**:
```
ユーザーがランキングデータを発見
         ↓
ブログ・記事・レポートで引用（出典: stats47.jp）
         ↓
被リンク増加 → ドメイン権威向上 → 検索順位上昇
         ↓
新規ユーザーがデータ発見 → 引用サイクル継続
```

**評価**:

| 基準 | スコア | 根拠 |
|---|---|---|
| 適合度 | 4/5 | 統計データは引用されやすいコンテンツの代表格。「出典: stats47.jp」の被リンクは SEO に直結 |
| 現在の成熟度 | 1/5 | 現時点でトラフィック規模が小さく、引用される機会が限定的。CSV ダウンロード機能も未実装 |
| 実装コスト | M | CSV ダウンロード機能の実装、埋め込みチャートウィジェットの開発が必要 |
| 成長速度 | 長期（3-6ヶ月） | トラフィック規模が前提。月間 PV 1万を超えてから効果が出始める |
| 複利効果 | **極めて高い** | 被リンクは永続的な SEO 資産。一度引用されると削除されにくく、累積効果が大きい |

**具体的アクション**:
1. `/generate-csv` で主要ランキングの CSV を生成・公開（Phase 2 以降）
2. データ引用時の推奨表記を明記（例: 「出典: stats47.jp」テンプレート）
3. Creative Commons ライセンスの採用を検討（引用ハードルを下げる）
4. 埋め込みチャートウィジェット（iframe）の提供（Phase 3 以降）

**ループ係数推定**:
- 月間 PV 1万時の被リンク獲得率: 月0.5-2件
- 被リンク1件あたりの SEO 効果: ドメインレーティング +0.1-0.5
- サイクル期間: 3-12ヶ月（被リンク → 順位上昇 → 新規流入）
- **現時点では起動条件未達**。SEO ループでトラフィックを確保してから着手

### 5. データ引用ループ（推奨度: ★★☆☆☆）

**仕組み**:
```
CSV ダウンロード・埋め込みチャート提供
         ↓
メディア・研究者・自治体が利用
         ↓
クレジット付き引用 → 権威性向上
         ↓
検索順位上昇 + ブランド認知 → 新規ユーザー
```

**評価**:

| 基準 | スコア | 根拠 |
|---|---|---|
| 適合度 | 3/5 | 行政データの「翻訳者」というポジションとは整合するが、e-Stat 自体が無料で利用可能なため差別化が必要 |
| 現在の成熟度 | 1/5 | CSV 生成スキルは存在するが、サイト上でのダウンロード UI 未実装。埋め込みチャートも未提供 |
| 実装コスト | L | ダウンロード UI、利用規約、アクセス管理、API 設計が必要 |
| 成長速度 | 長期（6ヶ月以上） | メディア・研究者への認知浸透に時間がかかる |
| 複利効果 | 高い | データプロバイダーとしての権威性は長期的に蓄積される |

**具体的アクション**:
1. 現時点では優先度低。SEO + SNS ループが軌道に乗ってから検討
2. まずはランキングページ上での「データ引用方法」の案内を追加（低コスト）
3. メディア向けプレスキット（stats47 の紹介 + データ利用ガイド）を準備（Phase 3）

**ループ係数推定**:
- 現時点では推定不能（トラフィック前提条件を満たしていない）
- 月間 PV 10万到達後に再評価

## 推奨実装順序

| 優先度 | ループ | 理由 |
|---|---|---|
| **1. 最優先** | コンテンツ SEO | 記事企画・執筆スキルが完備済み、複利効果最大、NSM への直接貢献度が最も高い |
| **2. 第2優先** | SNS バイラル | SEO を補完する短期的な流入源。スキルが完備済みで追加開発不要 |
| **3. 第3優先** | クロスプラットフォーム | YouTube 55本の既存資産を活用。SEO + SNS と並行して少ない追加コストで実行可能 |
| **4. 中期** | UGC | トラフィック規模が前提。月間 PV 1万達成後に CSV 公開で起動 |
| **5. 長期** | データ引用 | 開発コスト大。月間 PV 10万達成後に本格検討 |

## ループ間のシナジー

```
コンテンツ SEO ループ ←──────────────→ SNS バイラルループ
  │  記事が SNS 投稿のネタになる        │  SNS 流入が記事の初期トラフィックを生む
  │                                      │
  ├──→ クロスプラットフォームループ ←────┤
  │      YouTube 動画 = 記事の動画版       │
  │      note.com = 記事の要約版           │
  │                                        │
  └──→ UGC ループ ←───────────────────────┘
         トラフィック増 → 引用機会増 → 被リンク → SEO 強化
```

**最重要シナジー**: SEO ループと SNS ループは相互補完関係にある。SEO は中期的な安定流入を、SNS は短期的なスパイクを提供する。両方を同時に回すことで、SEO のインデックス待ち期間を SNS 流入で埋め、SNS の一過性を SEO のストック効果で補完する。
