## 収益化候補 (5案)

### 1. アフィリエイト本格化(配置拡大 + 計測ループ) - 推奨 ★★★★★

#### 仕組み
- 既存 24件の affiliate_ads を活用
- 現状 `blog-bottom` 20件配置 → ブログ流入が少なく機会損失中
- **`/ranking/*`, `/category/*`, `/areas/*` への配置追加**(Top20 流入ページに合わせる)
- A8.net レポート週次取得 → 記事別 EPC 計測 → 配置・訴求文の A/B 最適化

#### 収益レンジ (月次・3ヶ月後想定)
- 悲観: ¥1,000 (配置追加のみ、計測未整備)
- **標準: ¥10,000** (計測ループ稼働、上位記事に商材マッチ)
- 楽観: ¥30,000-50,000 (記事 LTV 上位を AI 等高単価商材へ展開)

#### 実装コスト
- **M (中)**: 1-2 週
  - `location_code` に `ranking-bottom`, `category-bottom`, `area-bottom` 追加
  - 既存 `affiliate_ads` テーブルへの INSERT のみ
  - 週次 A8.net レポート取得スクリプト + improvement-log
- 既存スキル雛形(`adsense-improvement`)を流用可能

#### UX 影響
- **低-中**: バナー型、記事末尾配置で本文閲覧を阻害しない
- 注意: `/areas/*` `/ranking/*` のサイドバー sticky は離脱要因になりうる(A/B 必須)

#### リスク
- A8.net 成約データ取得の自動化難易度(管理画面スクレイピング or 手動 CSV)
- 商材ミスマッチで CTR 上がるが CVR 0 のリスク → 配置開始時はクリック計測重視
- 既存 `affiliate-strategy.md` メモリで指摘された「socialsecurity→chocoZAP」等のミスマッチを先に解消

#### 検証実験
- **Week 1-2**: `/ranking/*` 配置を Top10 ページに追加、クリック数を Cloudflare logs で計測
- **Week 3-4**: A8.net 成約レポートを取得(週次手動 CSV → スクリプト化)
- **Week 5-8**: EPC 上位 / 下位を見て商材入替・配置調整
- **成功基準**: 8 週後に月 ¥5,000 突破
- **失敗判定**: 8 週後に月 ¥1,000 未満 → 商材選定・配置を全面見直し

### 2. AI ツール × 統計実務の特化記事(ユーザー要望) - 推奨 ★★★★☆

#### 仕組み
- 「ChatGPT で e-Stat データを分析する 7 つの手順」「Claude × Excel 統計表の整形術」のような実用記事を 3-5 本投下
- 記事内に **複数 AI 関連アフィリエイトを比較配置**:
  - AI スクール (SHIFT AI / DMM 生成 AI CAMP / Aidemy 等、1件 5,000-15,000円)
  - Udemy 講座(1件 1,000-3,000円、CVR 高)
  - Notion AI / ChatGPT Plus 紹介(月額系、低単価高 CVR)
- Desktop 67% のユーザー層 = 業務で AI を試したい層に刺さる仮説

#### 収益レンジ (月次・3ヶ月後想定)
- 悲観: ¥500 (記事流入が伸びず)
- 標準: ¥3,000-8,000 (記事 2-3 本が Organic Search 流入)
- 楽観: ¥15,000-30,000 (検索順位上位 + AI スクール成約 1-2件/月)

#### 実装コスト
- **M (中)**: 記事 3 本 = 2-3 週
  - 既存 affiliate_ads に AI 系商材を追加 INSERT (ConoHa AI Canvas は既登録)
  - 記事は MDX で書く、`affiliate` frontmatter で関連商材紐付け
  - SEO キーワード: 「e-Stat AI」「統計データ ChatGPT」等のロングテール

#### UX 影響
- **低**: 記事本体が実用情報、アフィリエイトは自然な比較表示
- 統計サイトとしての公共性を損なわない(公的データの活用法を解説する文脈)

#### リスク
- **公務員職場 PC は外部 AI ツール制限あり**(自治体多数)→ 標準ケースで CVR が下がる可能性
- ターゲットを「公務員専用」に絞ると母数が小さく、「データを業務で扱う人全般」に広げた方が ROI 高い
- 競合ブログ多数(AI ツール紹介はレッドオーシャン)→ stats47 の「統計サイト発」という文脈が差別化要素

#### 検証実験
- **Week 1-3**: 記事 1 本目を投下(「e-Stat データを ChatGPT で分析する手順」)
- **Week 4-6**: GSC で検索順位確認、クリック数測定
- **Week 7-12**: 上位記事の affiliate CVR を見て横展開 or 撤退判断
- **成功基準**: 1 記事あたり月 PV > 200 + 月成約 ≥ 1件
- **失敗判定**: 12 週後に Organic Search クリック < 50/月 → SEO キーワード再選定

### 3. AdSense 最適化(配置 + 自動広告 + PSI 連携) - 推奨 ★★★☆☆

#### 仕組み
- 現状 RPM ¥26 / CTR 1.25% は業界平均(¥100-200 / 1.5-2%)より低い
- 配置調整(記事中・サイドバー・サムネ周辺)で **RPM 2-3 倍化を狙う**
- **LCP 改善が前提**(現在 LCP 16秒で広告表示前に離脱)
- 自動広告 ON / OFF の A/B、手動 Anchor 広告の試行

#### 収益レンジ (月次・3ヶ月後想定)
- 悲観: ¥200 (PSI 改善遅延)
- 標準: ¥1,000-3,000 (RPM ¥50-80、PV 微増)
- 楽観: ¥5,000 (PSI 改善 + PV 2倍化が連動)

#### 実装コスト
- **S (小)**: AdSense 管理画面操作 + 配置タグ調整
- ただし **PSI 改善** (LCP 16秒 → 2.5秒以下) は別タスクで L 工数

#### UX 影響
- **中**: 広告枚数増は離脱要因。ユーザビリティ重視で配置上限を設定

#### リスク
- AdSense はクリック単価が低い(統計コンテンツは商業価値低い)→ **上限 RPM ¥100 程度**
- LCP 改善が進まないと収益も伸びない(現状の最大ボトルネック)

#### 検証実験
- **Week 1**: 自動広告 ON、Anchor 広告試行
- **Week 2-4**: RPM / CTR / Viewability を週次計測(既存 adsense-improvement スキル活用)
- **成功基準**: 4 週後に RPM ¥50 超え
- **失敗判定**: RPM ¥30 以下のまま → 配置タグを全面見直し

### 4. データ販売 / API プレミアム(法人向け) - 検討 ★★★☆☆

#### 仕組み
- 47都道府県の完全データセット(area_profiles 22,487行、correlations 36,620行)を CSV / API で販売
- ターゲット: シンクタンク、地方自治体の調査担当、コンサル、不動産・人材業界の調査部門
- 単価: CSV ¥3,000-10,000 / 個別レポート ¥30,000-100,000 / API 月額 ¥10,000-50,000

#### 収益レンジ (月次・6ヶ月後想定)
- 悲観: ¥0 (需要なし)
- 標準: ¥10,000-30,000 (月 1-3件成約)
- 楽観: ¥100,000+ (法人 API 契約 1件)

#### 実装コスト
- **L (大)**: ランディングページ + Stripe 決済 + データ提供フロー
- 営業活動(リーチ施策)が必要

#### UX 影響
- **無**: 既存ユーザーには影響なし、新規動線

#### リスク
- **需要不確実性が最大**:e-Stat は無料公開、有料化の価値訴求が必要
- 競合(帝国データバンク・東洋経済等)に対して価格・差別化を明確化する必要
- 個人開発のため法人取引対応コスト(契約書・請求書)が重い

#### 検証実験
- **Phase A (1ヶ月)**: 簡易 LP + Twitter / 自治体向け Web フォーム告知で**需要ヒアリング**(5-10件)
- **Phase B (需要確認後)**: 最小 PoC (Stripe 決済 + CSV ダウンロード) を 1 商品で実装
- **成功基準**: 3ヶ月で問い合わせ 5件 + 成約 1件
- **失敗判定**: 問い合わせ < 2件 → 撤退

## 注意事項・制約

1. **PSI 改善は最優先の前提条件**
   - LCP 5-19秒のままだと、何を導入しても天井が低い
   - 別タスクとして `/performance-improvement` を並行で回す

2. **公共性の維持**
   - 基本データ(ランキング・カテゴリ・area_profiles)は無料維持
   - 有料化はプレミアム機能(API・詳細分析・カスタムレポート)に限定

3. **個人開発の運用負荷**
   - improvement-log 系スキルがすでに 5 系統稼働中
   - 新規 affiliate-improvement 追加で 6 系統に → 週次運用 1.5h 想定
   - 自動化(GitHub Actions)で運用負荷を抑える

4. **公務員ターゲットは検証対象であって決め打ちにしない**
   - Desktop 67% は業務利用の傍証だが「公務員」とは限らない
   - GA4 Demographics 機能で年齢・性別・職業推定を別途確認
   - 当面は「データを業務で扱う人全般」に広く訴求し、CVR データで絞り込む
