# 非機能要件定義書

## 1. パフォーマンス要件

### 1.1 応答時間
- **ページ表示時間**: 3秒以内
- **データ取得時間**: 5秒以内
- **グラフ描画時間**: 2秒以内
- **地域選択応答時間**: 1秒以内

### 1.2 スループット
- **同時接続数**: 100ユーザー
- **1日あたりのリクエスト数**: 10,000リクエスト
- **ピーク時のリクエスト数**: 1,000リクエスト/時間

### 1.3 リソース使用量
- **メモリ使用量**: 512MB以下
- **CPU使用率**: 70%以下
- **ディスク使用量**: 1GB以下

## 2. セキュリティ要件

### 2.1 認証・認可
- **APIキー管理**: 環境変数での安全な管理
- **アクセス制御**: 公開データのみアクセス可能
- **セッション管理**: 不要（認証機能なし）

### 2.2 データ保護
- **個人情報**: 収集・保存しない
- **APIキー**: クライアントサイドに露出しない
- **通信暗号化**: HTTPS必須

### 2.3 脆弱性対策
- **XSS対策**: 入力値のサニタイズ
- **CSRF対策**: トークン検証
- **SQLインジェクション**: パラメータ化クエリ使用

## 3. 可用性・信頼性要件

### 3.1 稼働率
- **目標稼働率**: 99.0%以上
- **月間ダウンタイム**: 7.2時間以下
- **計画メンテナンス**: 月1回、2時間以内

### 3.2 障害対応
- **障害検知時間**: 5分以内
- **復旧時間**: 30分以内
- **バックアップ**: 日次自動バックアップ

### 3.3 冗長性
- **サーバー冗長化**: 不要（単一サーバー）
- **データバックアップ**: 日次
- **CDN活用**: 静的リソース配信

## 4. 保守性・拡張性要件

### 4.1 コード品質
- **TypeScript使用**: 100%
- **ESLint準拠**: 必須
- **テストカバレッジ**: 80%以上
- **コードレビュー**: 必須

### 4.2 アーキテクチャ
- **コンポーネント設計**: 再利用可能な設計
- **状態管理**: React Hooks使用
- **API設計**: RESTful準拠
- **データベース**: 不要（API連携のみ）

### 4.3 拡張性
- **新機能追加**: 既存コードへの影響最小化
- **スケーラビリティ**: 水平スケーリング対応
- **API拡張**: バージョニング対応

## 5. ユーザビリティ要件

### 5.1 操作性
- **直感的なUI**: 初回利用でも操作可能
- **レスポンシブデザイン**: 全デバイス対応
- **アクセシビリティ**: WCAG 2.1 AA準拠

### 5.2 表示要件
- **文字サイズ**: 16px以上（モバイル14px以上）
- **カラーコントラスト**: 4.5:1以上
- **フォント**: システムフォント使用

### 5.3 エラー処理
- **エラーメッセージ**: 分かりやすい日本語表示
- **ローディング表示**: 処理中であることを明示
- **フォールバック**: APIエラー時の代替表示

## 6. 互換性要件

### 6.1 ブラウザ対応
- **Chrome**: 最新版 + 2バージョン
- **Firefox**: 最新版 + 2バージョン
- **Safari**: 最新版 + 2バージョン
- **Edge**: 最新版 + 2バージョン

### 6.2 デバイス対応
- **デスクトップ**: 1200px以上
- **タブレット**: 768px-1199px
- **モバイル**: 320px-767px

### 6.3 OS対応
- **Windows**: 10以上
- **macOS**: 10.15以上
- **iOS**: 13以上
- **Android**: 8.0以上

## 7. 運用要件

### 7.1 監視・ログ
- **アクセスログ**: 全リクエスト記録
- **エラーログ**: エラー発生時の詳細記録
- **パフォーマンス監視**: 応答時間監視
- **アラート**: 異常検知時の通知

### 7.2 バックアップ・復旧
- **データバックアップ**: 日次自動実行
- **設定バックアップ**: 変更時即座に実行
- **復旧手順**: 文書化済み
- **テスト**: 月1回復旧テスト実行

### 7.3 メンテナンス
- **定期メンテナンス**: 月1回
- **緊急メンテナンス**: 必要時
- **通知**: メンテナンス1週間前に告知
- **ロールバック**: 問題発生時の即座実行

## 8. 法的・規制要件

### 8.1 個人情報保護
- **個人情報**: 収集・保存しない
- **Cookie**: 必要最小限の使用
- **プライバシーポリシー**: 明示

### 8.2 著作権・知的財産
- **統計データ**: 政府統計の利用規約準拠
- **API利用**: e-Stat API利用規約準拠
- **ライセンス**: オープンソースライセンス準拠

### 8.3 アクセシビリティ
- **法的要件**: 障害者差別解消法準拠
- **技術基準**: WCAG 2.1 AA準拠
- **テスト**: アクセシビリティテスト実施
