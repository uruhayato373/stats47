## 1. 顧客は誰か / 顧客に関する洞察

### Primary: ペルソナ 3 (行政・自治体関係者)

**人物像** (既存 [`01_プロジェクト定義.md`](./01_プロジェクト定義.md) §5 ペルソナ 3 を継承):
- 30〜50 歳、自治体職員・議会事務局・地域シンクタンク
- 業務時間 Desktop 利用、e-Stat 操作経験あり
- 予算要望・政策立案・議会答弁の根拠データを必要とする

**stats47 内における到達経路 (実データ)**:

| Signal | 値 (W19) | 出典 |
|---|---|---|
| 「経常収支比率 自治体 ランキング」 | CTR **100%** (1 click / 1 impression)、Position 18 | GSC W19 snapshot |
| 「県庁 年収 ランキング」 | impressions 1、Position 91 | GSC W19 snapshot |
| `/blog/tags/{公務員,地方公務員,行政,行政職員,行政事件,環境行政,行政需要,行政サービス}` | **全 8 ページがインデックス保留** | URL Inspection (2025-12〜2026-04) |
| Desktop 比率 | **67%** (Mobile 31% / Tablet 1%) | GA4 W19 |
| 平均滞在時間 | 154 秒 | GA4 W19 |

**洞察**:
1. **needs は実在**: 「経常収支比率 自治体」CTR 100% は noise ではなく強い意図 (Position 18 でも click)
2. **流入経路が未整備**: 既に 8 つの公務員タグページが存在するが**全部インデックス保留** = Google が内部リンク弱で価値を認識していない
3. **KAZU 自身が Voice of Customer**: 元県庁 20 年なので、ペインを内省で導出できる (一次経験 = E-E-A-T の Experience)
4. **業務時間 Desktop アクセス** が支配的 (67%) → 業務利用層の傍証

### Secondary (並走): ペルソナ 1 (雑学・エンタメ層)

実 GSC top クエリ (W20):「うどん消費量」「健康寿命」「平均身長」「鶏肉消費量」「バナナ消費量」「パン消費量」「パスタ消費量」「ガソリン消費量」 — **完全にペルソナ 1**。

→ **現状の流入の主体はペルソナ 1**。これを切り捨てると NSM (engaged sessions) と AdSense ベースが崩壊する。**Two-track 並走** で維持する。

## 2. 顧客の主要な問題 (ペルソナ 3 focus)

KAZU 元県庁経験 + GSC signal + 既存 [`01_プロジェクト定義.md`](./01_プロジェクト定義.md) §5 ペルソナ 3 統合:

| # | ペイン | 根拠 |
|---|---|---|
| P1 | **e-Stat の検索 UI が複雑で目的データに辿り着けない** | KAZU 一次経験、ペルソナ 0 ペイン記述と一致 |
| P2 | **CSV ヘッダー複雑・年次バラバラで Excel 加工に半日** | KAZU 一次経験 |
| P3 | **会議資料・議会答弁に使える見栄えのグラフを短時間で作りたい** | ペルソナ 3 ペイン (既存 doc) |
| P4 | **47 都道府県横断比較を短時間で準備したい** | 「経常収支比率 自治体 ランキング」CTR 100% |
| P5 | **データ定義・調査手法の変更点を知らずに引用するのが不安** | 公的機関の特性 (調査年 / 主体 / 手法の表記が複雑) |
| P6 | **AI (ChatGPT/Claude) の業務活用を学びたいが、自治体 PC は外部 AI 制限** | [`02_収益化戦略.md`](./02_収益化戦略.md) §戦略 2 リスク |

P6 は両刃の剣 (制限ある自治体多数) だが、**個人 PC・自宅学習** の文脈で AI スキル関連商材 (Udemy / SHIFT AI 等) との親和性が高い。

## 3. 解決策 / なぜこれが正しいか / 根拠データ

### 解決策のマッピング

| ペイン | 解決策 | stats47 既存資産 |
|---|---|---|
| P1, P2 | 探せる + 加工不要のランキング・area_profiles を提供 | 331 ランキング / 22,487 行 area_profiles (既に運用中) |
| P3 | チャート画像・CSV ダウンロードを資料転用可能に | Remotion / Satori 画像生成済 (転用導線は弱い) |
| P4 | 47 都道府県横断ランキング + 偏差値・相関 | 既に運用中 |
| P5 | KAZU 注釈「このデータの読み方」段階的追加 | 一部記事のみ。テンプレ化要 |
| P6 | 公務員 × AI 活用記事シリーズ (Claude Code / ChatGPT × e-Stat) | 進行中: [`koumuin-claude-code-estat-automation/article.md`](../21_ブログ記事原稿/koumuin-claude-code-estat-automation/article.md) / [`koumuin-shigoto-kouritsuka-ai/note.md`](../31_note記事原稿/koumuin-shigoto-kouritsuka-ai/note.md) |

### なぜこれが正しいか (3 つの根拠)

1. **競合が「行政の翻訳者」ポジションを取っていない**
   - とどラン: 相関分析・読み物軸、行政視点なし
   - UUB: 行政アーカイブ軸、可視化弱、加工視点なし
   - e-Stat / RESAS: 一次データ提供のみ、業務効率化視点なし
   - → KAZU 元県庁 20 年は **代替不可能な moat** ([`01_プロジェクト定義.md`](./01_プロジェクト定義.md) §6 競合分析)

2. **needs の signal が実在 (推測ではない)**
   - 「経常収支比率 自治体 ランキング」CTR 100% (W19)
   - 8 つの公務員タグページが既に存在 (放置されているだけで、コンテンツ資産はある)
   - Desktop 67% は業務利用の傍証

3. **既存資産で解決可能 (新規開発の必要が小さい)**
   - 331 ランキング・22,487 行 area_profiles はそのまま流用可能
   - チャート画像・CSV 出力も既存
   - **不足しているのは「ペルソナ 3 向けの導線・記事・内部リンク」のみ**

### 反証可能性 (この仮説が間違っていたら)

- P1〜P5 への解は既に存在するため、**3 ヶ月後にペルソナ 3 流入が増えていなければ「需要を作るのが難しい (Acquisition の問題)」のではなく「公務員はそもそも自分の業務効率化のためにブログを読みに来ない (Channel mismatch)」可能性が高い**
- その場合は「公務員職場での口コミ / 自治体内インフラ (LGWAN・庁内ポータル) 経由のディストリビューション」を再設計する必要がある

## 4. 顧客体験 / 一番大事なメリット

### One-line value proposition

> **「e-Stat で半日かかる『探す + 加工』が 10 秒で終わる。出典付き・グラフ付きでそのまま資料に貼れる」**

### ペルソナ 3 の理想体験フロー

| Step | 体験 | 既存 / 必要 |
|---|---|---|
| 1. 発見 | 「経常収支比率 自治体 ランキング」「県庁 年収 ランキング」「e-Stat ChatGPT」等で着地 | **流入経路強化が必要** (公務員系記事追加) |
| 2. 比較 | 47 都道府県横断ランキング + 自県位置を 5 秒で把握 | **既存** (`/ranking/*`) |
| 3. 加工 | チャート PNG / CSV をダウンロード → 会議資料に貼る | **既存だが導線が弱い** (DL ボタン UX 改善) |
| 4. 学習 | KAZU の「このデータの読み方」注釈で安心して引用 | **テンプレ化が必要** |
| 5. 拡張 | 公務員 × AI シリーズで業務効率化スキル獲得 (note 有料 or アフィリエイト) | **進行中** (記事 2 本、 SEO 着地待ち) |

### 一番大事なメリット (1 つ選ぶなら)

**「KAZU が県庁 20 年で痛感した『使えないデータ』を、後輩公務員が二度と経験しなくて済むようにすること」** — これは Voice of Customer ではなく Founder's narrative。SEO ではなく「公務員職場での口コミ」「note フォロワー」「X コミュニティ」で広がる文脈。
