# 機能要件定義書

## 1. 機能概要

### 1.1 システム全体の機能

stats47 は、日本の地域統計データを可視化する Web アプリケーションとして、以下の主要機能を提供する。

### 1.2 MVP 機能の定義

本ドキュメントでは、各機能に以下の優先度マーカーを付与する：

- **🔴 MVP 必須**: Phase 1（0-3 ヶ月）で実装必須の機能
- **🟡 Phase 2**: Phase 2（3-6 ヶ月）で実装する機能
- **🔵 Phase 3**: Phase 3（6-12 ヶ月）で実装する機能

詳細な実装計画は[フェーズ 1 実装計画](../03_実装計画/01_フェーズ1_基盤整備.md)を参照。

## 2. 主要機能一覧

### 2.1 地域選択機能 🔴

- **Phase**: MVP 必須（Phase 1）
- **機能名**: RegionSelector
- **概要**: ユーザーが表示したい地域を選択する機能
- **MVP 範囲**:
  - 都道府県レベルの選択（47 都道府県）
  - ドロップダウン形式の選択 UI
  - 地域名による検索機能
  - レスポンシブ対応
- **Phase 2 以降**:
  - 市区町村レベルの選択
  - 地域の階層構造表示

### 2.2 e-Stat API 連携機能 🔴

- **Phase**: MVP 必須（Phase 1）
- **機能名**: EstatDataFetcher
- **概要**: e-Stat API から統計データを取得する機能
- **MVP 範囲**:
  - e-Stat API 認証設定
  - データ取得 API の実装
  - エラーハンドリング
  - サンプルデータのフォールバック（API 未設定時）
- **Phase 2 以降**:
  - メタ情報管理機能（キャッシュ・正規化）
  - 高度なキャッシュ戦略（ページング/分割取得）

### 2.3 データ可視化機能 🔴

- **Phase**: MVP 必須（Phase 1）
- **機能名**: StatisticsDisplay
- **概要**: 取得した統計データをグラフやチャートで表示する機能
- **MVP 範囲**:
  - 棒グラフ表示
  - 折れ線グラフ表示
  - データ表形式表示
  - レスポンシブ対応
- **Phase 2 以降**:
  - 円グラフ表示
  - 地域比較表示
  - インタラクティブ機能

### 2.4 ダッシュボード機能 🔴

- **Phase**: MVP 必須（Phase 1）
- **機能名**: DashboardPage
- **概要**: 複数の統計データを統合表示するメイン画面
- **MVP 範囲**:
  - 地域選択とデータ表示の統合
  - 基本的なレイアウト
- **Phase 2 以降**:
  - 複数指標の同時表示
  - レイアウトのカスタマイズ
  - 地域プロファイル機能との統合（地域選択時に自動表示）

### 2.5 データエクスポート機能 🟡

- **Phase**: Phase 2（3-6 ヶ月）
- **機能名**: DataExport
- **概要**: 可視化中のデータを CSV/JSON 形式でダウンロードできる
- **実装内容**:
  - 出力形式: CSV（text/csv）、JSON（application/json）
  - 表示状態の引き継ぎ: フィルタ/ソート/期間/地域選択を反映
  - 上限: レコード件数・生成時間の上限（例: 100k rows / 15s）
  - レート制限: 認証状態に応じた制限を適用
  - 監査: 取得ログ（userId, format, rows, time）を記録

### 2.6 ランキング表示機能 🔴

- **Phase**: MVP 必須（Phase 1）
- **機能名**: RankingDisplay
- **概要**: 都道府県の統計データをランキング形式で表示する機能
- **MVP 範囲**:
  - 47 都道府県の統計データを昇順/降順で並び替え
  - トップ 10/ワースト 10 の表示切り替え
  - 順位番号の表示（1 位、2 位...）
  - ランキングテーブル形式の表示
  - レスポンシブ対応
- **Phase 2 以降**:
  - 前年対比での順位変化表示（↑↓ 矢印）
  - 分位・パーセンタイル表示
  - 市区町村レベルのランキング
  - 地域プロファイル機能へのデータ提供（強み検出用）

### 2.7 地域間比較機能 🟡

- **Phase**: Phase 2（3-6 ヶ月）
- **機能名**: RegionComparison
- **概要**: 複数地域のデータを並列比較表示する機能
- **実装内容**:
  - 2 地域の直接比較（並列グラフ表示）
  - 最大 5 地域までの同時比較
  - 比較用の専用可視化（並列棒グラフ、レーダーチャート）
  - 差分・比率の自動計算表示
  - 比較結果のハイライト表示
- **Phase 3 以降**:
  - カスタム地域グループの保存
  - 時系列での地域比較
  - 比較レポートの自動生成

### 2.8 相関分析機能 🔵

- **Phase**: Phase 3（6-12 ヶ月）
- **機能名**: CorrelationAnalysis
- **概要**: 2 つの統計指標間の相関関係を分析・可視化する機能
- **実装内容**:
  - **相関係数の計算**: ピアソン相関係数（-1.0 〜 +1.0）
  - **散布図の表示**: 2 軸の散布図で関係性を可視化
  - **回帰直線の表示**: 線形回帰直線と R² 値の表示
  - **相関強度の判定**: 強い相関/中程度/弱い相関/無相関の自動判定
  - **地域ラベル表示**: 各プロット点に都道府県名を表示
  - **外れ値のハイライト**: 統計的外れ値の強調表示
- **対応データ**:
  - 都道府県レベルの統計データ（47 プロット）
  - 同一年度のデータのみ（時系列相関は Phase 4）
- **UI 要素**:
  - 統計指標選択（X 軸・Y 軸）
  - 相関係数・p 値の表示
  - インタラクティブ散布図（ズーム・パン対応）
- **Phase 4 以降**:
  - 時系列相関分析（ラグ相関）
  - 3 変数以上の多変量相関分析
  - 偏相関係数の計算
  - 相関行列のヒートマップ表示
  - AI による相関関係の自動発見・推奨

### 2.9 時系列分析機能 🟡

- **Phase**: Phase 2（3-6 ヶ月）
- **機能名**: TimeSeriesAnalysis
- **概要**: 複数年度の統計データの推移を分析・可視化する機能
- **実装内容**:
  - **複数年度データの折れ線グラフ**: 時系列の可視化
  - **年平均成長率（CAGR）の自動計算**: 成長トレンドの定量化
  - **トレンドラインの表示**: 移動平均、線形回帰
  - **前年比・前年同期比の計算**: 変化率の自動計算
  - **期間選択機能**: ユーザーが分析期間を指定
  - **複数地域の時系列比較**: 最大 5 地域の推移を並列表示
- **対応データ**:
  - 年次データ（最低 3 年分、推奨 10 年分）
  - 月次・四半期データ（将来対応）
- **UI 要素**:
  - 期間選択スライダー（開始年〜終了年）
  - トレンドライン表示切替（ON/OFF）
  - グラフ種類選択（折れ線/エリアチャート）
  - CAGR 値の表示
  - データポイントのツールチップ表示
- **Phase 3 以降**:
  - 季節調整（季節性データ）
  - 指数平滑法による予測
  - 自己回帰モデル（ARIMA）
  - 複数指標の同時時系列比較

### 2.10 地図可視化機能（コロプレス地図）🟡

- **Phase**: Phase 2（3-6 ヶ月）
- **機能名**: ChoroplethMap
- **概要**: 統計データを地図上で色分け表示する機能

### 2.11 地域プロファイル機能 🟡

- **Phase**: Phase 2（3-6 ヶ月）
- **機能名**: RegionProfile
- **概要**: 選択された地域の統計プロファイルを自動生成・表示する機能
- **実装内容**:
  - **総合プロファイル表示**:
    - 人口・経済・環境など主要指標の概要
    - カテゴリ別の代表的な統計値
  - **地域の強み自動検出**:
    - 全国ランキング上位（トップ10）の統計項目を自動抽出
    - ランク順に表示（例：「製造業出荷額 全国3位」）
    - 各項目の全国平均との比較
  - **地域特性のビジュアル化**:
    - レーダーチャート（多次元評価）
    - 類似地域の提案（統計的類似性に基づく）
- **表示場所**: 既存ダッシュボードページ内（地域選択時に自動表示）
- **Phase 3 以降の拡張**:
  - 時系列での地域特性変化
  - AI による地域分析レポート自動生成
  - 地域間の詳細比較レポート

### 2.12 カテゴリ・タグ分類機能 🔴

- **Phase**: MVP 必須（Phase 1）
- **機能名**: CategoryTagging & Search
- **概要**: 統計データとブログコンテンツをカテゴリ・タグで分類・検索・フィルタリングする機能
- **MVP 範囲**:
  - **カテゴリナビゲーション**:
    - サイドバーでのカテゴリ階層表示
    - カテゴリ・サブカテゴリでの絞り込み
    - カテゴリアイコンと色の視覚的表現
  - **統計データの検索・フィルタリング**:
    - カテゴリ別の統計データ一覧表示
    - 複合フィルタ（カテゴリ+地域+期間）
    - 検索結果の件数表示
  - **タグ機能**:
    - 統計データへのタグ付け
    - タグによる絞り込み
    - タグクラウド表示
  - **ブログコンテンツの分類**:
    - カテゴリ別ブログ記事一覧
    - タグベースの関連記事表示
    - カテゴリ・タグでのブログ検索
- **Phase 2 以降**:
  - カスタムタグの追加（ユーザー生成タグ）
  - 保存済み検索条件
  - レコメンデーション（タグベースの関連コンテンツ自動提案）
  - 複数タグの AND/OR 検索

### 2.13 統合検索機能 🟡

- **Phase**: Phase 2（3-6 ヶ月）
- **機能名**: UnifiedSearch
- **概要**: 統計データ、ブログコンテンツ、カテゴリなど全コンテンツを横断する高度な検索機能
- **実装内容**:
  - **全文検索**:
    - 統計データ（統計名、説明文、メタデータ）
    - ブログコンテンツ（タイトル、本文、タグ、カテゴリ）
    - カテゴリ・サブカテゴリ名
  - **高度な検索演算子**:
    - AND検索（複数キーワードすべて含む）
    - OR検索（複数キーワードいずれか含む）
    - NOT検索（特定キーワードを除外）
    - フレーズ検索（完全一致）
    - ワイルドカード検索
  - **検索補助機能**:
    - オートコンプリート（入力候補の自動表示）
    - サジェスト機能（関連キーワードの提案）
    - スペルチェック（入力ミスの修正提案）
    - 検索履歴の保存・再利用
  - **検索結果の表示**:
    - コンテンツタイプ別のグループ化（統計/ブログ/カテゴリ）
    - 関連度スコアによるソート
    - ハイライト表示（検索語を強調）
    - ファセット検索（検索結果の絞り込み）
    - ページネーション（大量結果の分割表示）
  - **検索フィルタ**:
    - コンテンツタイプでの絞り込み
    - 日付範囲での絞り込み
    - 地域での絞り込み
    - カテゴリでの絞り込み
- **Phase 3 以降**:
  - 自然言語検索（AIによる意図解釈）
  - 音声検索
  - 画像検索
  - 保存済み検索のアラート通知
  - 検索分析ダッシュボード
- **実装内容**:
  - **都道府県境界の地図表示**: 歴史的行政区域データセット使用
  - **データ形式**: TopoJSON（GeoNLP プロジェクトから直接取得）
  - **統計値による色分け**: カラースケール（連続値/分位数）
  - **インタラクティブ機能**: ズーム、パン、ホバー
  - **ツールチップ表示**: 地域名、統計値、順位
  - **凡例とカラーバー**: 値の範囲を明示
  - **カラースキーム選択**: Sequential（段階）、Diverging（発散）
  - **クリックで詳細表示**: 地域をクリックして詳細データ表示
  - **ベースマップ機能**: 国土地理院タイル（標準地図、淡色地図、白地図、航空写真）
  - **ベースマップ切り替え**: ユーザーが背景地図を選択可能
  - **ズーム・パン機能**: 地図の拡大縮小、移動操作
- **データソース**:
  - **地理データ（行政区域）**: 歴史的行政区域データセット β 版（CODH、CC BY 4.0）
  - **地理データ（施設・災害等）**: 国土数値情報（国土交通省、CC BY 4.0）
  - **ベースマップ**: 国土地理院タイル（国土地理院、国土地理院コンテンツ利用規約）
  - **統計データ**: e-Stat API
- **対応データ**:
  - 都道府県レベル（47 都道府県）
  - 市区町村レベル（Phase 3）
  - 連続値データ（人口、GDP など）
  - 比率データ（高齢化率、失業率など）
- **Phase 2 実装内容**:
  - 現在の都道府県境界表示
  - 基本的なコロプレス地図機能
- **Phase 3 以降の拡張**:
  - 市区町村レベルの地図表示
  - 時系列アニメーション（1889 年〜現在）
  - 施設情報の重ね合わせ（国土数値情報）
    - 医療機関、教育機関、公共施設
  - 災害リスク情報の表示（国土数値情報）
    - 浸水想定区域、土砂災害警戒区域
  - 交通情報の重ね合わせ（国土数値情報）
    - 駅、道路

## 3. ユーザーストーリー

### 3.1 地域データ閲覧ストーリー

```
As a 地域に興味のあるユーザー
I want to 特定の地域の統計データを確認したい
So that その地域の現状や特徴を理解できる
```

**受け入れ条件**:

- 地域選択画面で都道府県を選択できる
- 選択した地域の統計データが表示される
- データは分かりやすいグラフで可視化される

### 3.2 地域比較ストーリー

```
As a 地域比較に興味のあるユーザー
I want to 複数の地域のデータを比較したい
So that 地域間の違いや特徴を把握できる
```

**受け入れ条件**:

- 複数地域の選択が可能（最大 5 地域まで）
- 比較用のグラフが表示される（並列棒グラフ、レーダーチャート）
- 地域間の数値差が明確に分かる（差分・比率の自動計算表示）
- ランキング表示機能も含む（順位・トップ 10/ワースト 10）

**Phase 情報**:

- **Phase 1（MVP）**: ランキング表示機能（2.6）
- **Phase 2**: 地域間比較機能（2.7）の基本機能
- **Phase 3**: 高度な比較機能（カスタムグループ、時系列比較）

> 定義: 本ドキュメントにおける「地域比較」にはランキング表現を含む。
> 対応範囲: 並び替え（昇順/降順）、トップ N/ボトム N、順位表示、順位変化（前年対比）、分位・パーセンタイル。

### 3.3 可視化データのダウンロード

- 一般ユーザーとして、グラフ/テーブルで閲覧中のデータを CSV/JSON でダウンロードして、外部ツールで再分析したい。

受け入れ条件（AC）:

- 現在のフィルタ/ソート/期間/地域選択が反映されたデータを CSV/JSON で取得できる
- 文字コードは UTF-8、CSV はヘッダ行を含む
- サイズ上限・タイムアウト超過時はエラーメッセージを表示
- ファイル名は `{category}_{subcategory}_{area}_{yyyyMM}` 形式

### 3.4 統計値の相関関係分析ストーリー

```
As a データアナリスト/研究者
I want to 2つの統計指標間の相関関係を分析したい
So that 地域特性や因果関係の仮説を立てられる
```

**受け入れ条件（AC）**:

- X 軸・Y 軸に異なる統計指標を選択できる
- 散布図が表示され、各点が都道府県を表す
- 相関係数（r）と p 値が表示される
- 回帰直線が表示される
- 相関の強度が分かりやすく表示される（強い/中程度/弱い/無相関）
- プロット点をホバーすると都道府県名と数値が表示される
- レスポンシブ対応（モバイルでも閲覧可能）

**Phase 情報**:

- **Phase 3**: 基本的な相関分析機能（2 変数、同一年度）
- **Phase 4**: 高度な相関分析（時系列、多変量、AI 推奨）

### 3.5 時系列データ分析ストーリー

```
As a 政策立案者/研究者
I want to 統計データの長期的な推移を分析したい
So that トレンドを把握し、将来の計画を立てられる
```

**受け入れ条件（AC）**:

- 複数年度のデータを選択できる（最低 3 年、最大利用可能年数）
- 折れ線グラフで推移が表示される
- 年平均成長率（CAGR）が自動計算される
- トレンドライン（移動平均/線形回帰）が表示される
- 前年比/前年同期比が表示される
- 複数地域の比較が可能（最大 5 地域）
- データポイントをホバーすると詳細値が表示される

**Phase 情報**:

- **Phase 2**: 基本的な時系列分析（折れ線、CAGR、トレンドライン）
- **Phase 3**: 高度な時系列分析（季節調整、予測モデル）

### 3.6 地図上でのデータ可視化ストーリー

```
As a 一般ユーザー/ビジネスパーソン
I want to 統計データを地図上で視覚的に確認したい
So that 地域間の差異や分布パターンを直感的に理解できる
```

**受け入れ条件（AC）**:

- 都道府県が色分けされた地図が表示される
- 色の濃淡で統計値の大小が分かる
- 地域をホバーすると地域名と統計値が表示される
- 凡例で色と値の対応が明示される
- ズーム・パン操作が可能
- 統計指標と年度を選択できる
- モバイルでも表示可能（レスポンシブ対応）

**Phase 情報**:

- **Phase 2**: 基本的な地図可視化（都道府県レベル、静的表示）
- **Phase 3**: 高度な地図可視化（市区町村レベル、アニメーション）

### 3.7 地域プロファイル閲覧ストーリー

```
As a 地域研究者/ビジネスパーソン
I want to 特定地域の統計的な強みや特徴を包括的に理解したい
So that 地域の特性を素早く把握し、意思決定に活用できる
```

**受け入れ条件（AC）**:

- 地域選択後、自動的にプロファイルセクションが表示される
- 「地域の強み」セクションに全国トップ10入りの統計項目がリスト表示される
- 各統計項目に順位・値・全国平均が表示される
- 主要カテゴリ（人口、経済、環境など）の代表指標が表示される
- レーダーチャートで多次元評価が可視化される
- 統計的に類似する地域が3-5件提案される
- モバイルでも閲覧可能（レスポンシブ対応）

**Phase 情報**:

- **Phase 2**: 基本的な地域プロファイル機能（強み検出、総合プロファイル）
- **Phase 3**: 高度な分析機能（時系列変化、AI レポート）

### 3.8 カテゴリ・タグによる検索ストーリー

**受け入れ条件（AC）**:

- 検索結果に該当件数が表示される
- カテゴリとタグの組み合わせで検索できる
- ブログ記事もカテゴリ・タグで絞り込める
- モバイルでも使いやすいUI（レスポンシブ対応）

### 3.9 統合検索ストーリー


**受け入れ条件（AC）**:

## 8. ドメイン別 機能要件（DDD 準拠）

> 本章は `docs/01_技術設計/01_システム概要/DDDドメイン分類設計書.md` に準拠し、ドメインごとの機能（ユースケース）を要件化する。

### 8.1 ランキング（Analytics）🔴

- **Phase**: MVP 必須（Phase 1）
- **機能**: 統一ランキング取得（GetUnifiedRanking）
  - **ユーザーストーリー**: サブカテゴリのランキングを閲覧し、地域比較と順位を理解したい
  - **受け入れ条件（AC）**:
    - 入力: subcategoryId, areaLevel, timeCode
    - 出力: items[id, areaCode, value, rank, year], metadata, statistics
    - 並び替え/トップ N/順位変化（前年対比）に対応
    - 上限/タイムアウト/エラー表示を定義
    - **MVP 範囲**: 47 都道府県の昇順/降順並び替え、トップ 10/ワースト 10 表示
    - **Phase 2 以降**: 前年対比順位変化、分位・パーセンタイル、市区町村レベル

### 8.2 可視化（Visualization）

- 機能: グラフ描画設定生成（BuildChartConfig）
  - ユーザーストーリー: 指定データを適切な可視化（棒/線/円/地図）で表示したい
  - AC: 軸/凡例/ツールチップ/アクセシビリティ（コントラスト/ARIA）を満たす設定を生成

### 8.3 データ統合（Data Integration / e-Stat）

- 機能: メタ情報取得（GetMetaInfo）
  - AC: 表タイトル/周期/地域種別/分類軸を正規化しキャッシュ、一貫した内部モデルで供給
- 機能: 統計データ取得（GetStatsData）
  - AC: 表 ID・地域・期間・分類で取得し、可視化用へ整形。分割取得・再試行・キャッシュを実装

### 8.4 カテゴリ管理（Category）🔴

- **Phase**: MVP 必須（Phase 1）
- **機能**: カテゴリ/サブカテゴリ管理（CRUD/並び順/公開設定）
  - **ユーザーストーリー**: カテゴリとサブカテゴリを管理し、統計データを分類したい
  - **受け入れ条件（AC）**:
    - 作成/編集/削除、並び順変更、非公開設定、参照整合（削除時の代替割当）
- **機能**: カテゴリナビゲーション（CategoryNavigation）
  - **ユーザーストーリー**: カテゴリ階層をナビゲートして目的の統計を見つけたい
  - **受け入れ条件（AC）**:
    - 入力: なし（全カテゴリ取得）またはcategoryId（特定カテゴリ取得）
    - 出力: categories[id, name, icon, color, subcategories[]]
    - カテゴリ階層の表示順序制御
    - アクティブカテゴリのハイライト
    - パンくずリスト生成
- **機能**: 統計データ検索・フィルタリング（SearchAndFilter）
  - **ユーザーストーリー**: 複数条件で統計データを絞り込みたい
  - **受け入れ条件（AC）**:
    - 入力: filters { categoryId?, subcategoryId?, areaCode?, timeCode?, tags[]? }
    - 出力: results[items[], totalCount, appliedFilters]
    - 複合条件での絞り込み（AND条件）
    - 検索結果の件数表示
    - フィルタ条件のリセット機能
    - **パフォーマンス**: 1秒以内に検索結果を返す
- **機能**: タグ管理（TagManagement）
  - **ユーザーストーリー**: 統計データやブログにタグを付けて分類したい
  - **受け入れ条件（AC）**:
    - 入力: contentId, tags[]
    - 出力: taggedContent { contentId, tags[], relatedContent[] }
    - タグの追加・削除
    - タグクラウド生成（使用頻度順）
    - タグによる関連コンテンツ取得
    - **タグ上限**: 1コンテンツあたり最大10タグ

### 8.5 地域管理（Area）

- 機能: 地域階層/コード管理
  - AC: 都道府県/市区町村の整合性、5 桁コード検証、親子関係の解決

### 8.6 エクスポート（Export）

- 機能: データエクスポート（CSV/JSON）
  - AC: 現在のビュー（filter/sort/selection/period）を反映し、UTF-8、ヘッダ付 CSV/JSON を提供
  - 非機能: 上限（例: 100k/15s）、レート制限、監査ログ（userId, format, rows, time）

### 8.7 認証（Auth）

- 機能: 認証/セッション管理（閲覧・管理権限）
  - AC: 管理系機能（カテゴリ管理/大量エクスポート）には適切なロールが必要

### 8.8 相関分析（Analytics - Correlation）🔵

- **Phase**: Phase 3（6-12 ヶ月）
- **機能**: 相関係数計算（CalculateCorrelation）
  - **ユーザーストーリー**: 2 つの統計指標の相関を分析し、関係性の強さを理解したい
  - **受け入れ条件（AC）**:
    - 入力: indicatorX, indicatorY, areaLevel, timeCode
    - 出力: correlationCoefficient (r), pValue, scatterData[areaCode, valueX, valueY], regressionLine[slope, intercept]
    - 統計的有意性の検定（p < 0.05）
    - 外れ値検出（Z-score > 3）
    - エラーハンドリング（データ不足、分散 0 など）
    - **計算方式**: ピアソン相関係数
    - **データ要件**: 最低 10 地域以上のデータ（47 都道府県推奨）

### 8.9 時系列分析（Analytics - TimeSeries）🟡

- **Phase**: Phase 2（3-6 ヶ月）
- **機能**: 時系列データ取得・分析（GetTimeSeriesData, CalculateTrend）
  - **ユーザーストーリー**: 統計指標の長期推移を分析し、トレンドを理解したい
  - **受け入れ条件（AC）**:
    - 入力: indicatorId, areaCode, startYear, endYear
    - 出力: timeSeriesData[year, value], cagr, trendLine[type, equation], yearOverYearChange[]
    - CAGR 計算: `((最終値/初期値)^(1/年数) - 1) × 100`
    - トレンドライン: 線形回帰（最小二乗法）、移動平均（SMA）
    - 前年比計算: `((当年値 - 前年値) / 前年値) × 100`
    - データ検証: 最低 3 年分のデータが必要
    - 欠損値処理: 線形補間または前方埋め

### 8.10 地図可視化（Visualization - Map）🟡

- **Phase**: Phase 2（3-6 ヶ月）
- **機能**: 地図データ生成（GenerateChoroplethData）
  - **ユーザーストーリー**: 統計データを地図上で色分け表示し、地域分布を理解したい
  - **受け入れ条件（AC）**:
    - 入力: indicatorId, timeCode, colorScheme, classificationMethod
    - 出力: geoData[areaCode, value, color, rank], legend[min, max, breaks], metadata
    - 地理データ: 歴史的行政区域データセット β 版（CODH）の TopoJSON 形式
    - カラースケール: Sequential（viridis, blues など）、Diverging（RdBu など）
    - 分類方法: 等間隔、分位数（5 分位など）、自然分類（Jenks）
    - データソース表記: 「歴史的行政区域データセット β 版（CODH 作成）」の表示
    - アクセシビリティ: カラーコントラスト比 4.5:1 以上、色覚異常対応

### 8.11 地域プロファイル（Analytics - RegionProfile）🟡

- **Phase**: Phase 2（3-6 ヶ月）
- **機能**: 地域プロファイル生成（GenerateRegionProfile）
  - **ユーザーストーリー**: 選択地域の統計的特徴を包括的に理解したい
  - **受け入れ条件（AC）**:
    - 入力: areaCode, profileType (comprehensive | strengths | comparison)
    - 出力: 
      - profile { basicInfo, keyIndicators[category, value, rank, nationalAvg], strengths[indicator, rank, value], radarData, similarRegions[] }
    - **強み検出ロジック**: 全統計項目を走査し、全国ランキングトップ10入りの項目を抽出
    - **類似地域検出**: ユークリッド距離またはコサイン類似度で上位3-5地域を算出
    - **データ要件**: 最低10カテゴリ以上の統計データ
    - **パフォーマンス**: 3秒以内にプロファイル生成
    - **キャッシュ戦略**: 地域×期間単位で24時間キャッシュ

### 8.12 統合検索（Search）🟡

- **Phase**: Phase 2（3-6 ヶ月）
- **機能**: 全文検索エンジン（FullTextSearch）
  - **ユーザーストーリー**: キーワードで全コンテンツを検索したい
  - **受け入れ条件（AC）**:
    - 入力: query (string), filters { contentTypes[]?, dateRange?, areaCode?, categoryId? }, options { page?, limit?, sortBy? }
    - 出力: searchResults { items[{ type, id, title, snippet, score, highlightedText }], totalCount, facets, suggestions }
    - **検索対象**: 統計データ、ブログ、カテゴリ、サブカテゴリ
    - **検索演算子**: AND (スペース区切り), OR (|), NOT (-), フレーズ ("..."), ワイルドカード (*)
    - **スコアリング**: TF-IDF、BM25、または類似アルゴリズム
    - **ハイライト**: マッチした部分を<mark>タグで囲む
    - **パフォーマンス**: 検索実行2秒以内、オートコンプリート200ms以内
- **機能**: オートコンプリート（AutoComplete）
  - **ユーザーストーリー**: 入力中に検索候補を確認したい
  - **受け入れ条件（AC）**:
    - 入力: partialQuery (string), limit (number, default: 10)
    - 出力: suggestions[{ text, type, score }]
    - 入力3文字以上で候補表示
    - 使用頻度と関連度を考慮
    - **パフォーマンス**: 200ms以内にレスポンス
- **機能**: 検索履歴管理（SearchHistory）
  - **ユーザーストーリー**: 過去の検索を再利用したい
  - **受け入れ条件（AC）**:
    - 入力: userId (optional)
    - 出力: history[{ query, timestamp, resultCount }]
    - ローカルストレージに保存（最大50件）
    - 認証ユーザーはサーバー保存可能
    - プライバシー考慮（削除機能）
