# 収益化戦略

> **[2026-05-16 update]** ペルソナ 3 (行政・自治体関係者) を Primary、ペルソナ 1 (雑学・エンタメ層) を Secondary 並走 とする Two-track 戦略を確定。詳細: [`04_ターゲットペルソナ.md`](./04_ターゲットペルソナ.md)

## 目的

Stats47 の収益化は「事業化による利益最大化」ではなく、**サービスを持続的に運営するためのコスト回収と業務効率化スキルの還元**を第一目的とする。サーバー費・ドメイン費・データ取得の工数を広告 + アフィリエイト収益で賄い、個人プロジェクトとして無理なく継続できる状態を目指す。

## 基本方針

> **「まずPVを育てる。収益はPVについてくる」**

ただし 2026-05-16 時点で公務員系 needs (ペルソナ 3) の実証が取れたため、純粋な PV ドリブンではなく **アフィリエイト主軸 + AdSense 補完 + ペルソナ 3 特化記事** の Two-track で進める。

## 収益化候補 (5 案・優先順)

### 1. アフィリエイト本格化 (配置拡大 + 計測ループ) — 推奨 ★★★★★

**仕組み**:
- 既存 24 件の affiliate_ads を活用
- 現状 `blog-bottom` 20 件配置 → ブログ流入が少なく機会損失中
- `/ranking/*`, `/category/*`, `/areas/*` への配置追加 (Top20 流入ページに合わせる)
- A8.net レポート週次取得 → 記事別 EPC 計測 → 配置・訴求文の A/B 最適化

**収益レンジ (月次・3ヶ月後想定)**:
- 悲観: ¥1,000 (配置追加のみ、計測未整備)
- 標準: ¥10,000 (計測ループ稼働、上位記事に商材マッチ)
- 楽観: ¥30,000-50,000 (記事 LTV 上位を AI 等高単価商材へ展開)

**実装コスト**: M (1-2 週)。既存 `adsense-improvement` を雛形に `affiliate-improvement` スキル新設。

**UX 影響**: 低-中。バナー型、記事末尾配置。`/areas/*` `/ranking/*` のサイドバー sticky は A/B 必須。

**リスク**:
- A8.net 成約データ取得の自動化難易度
- 商材ミスマッチで CTR 上がるが CVR 0 のリスク
- memory `affiliate-strategy.md` の「socialsecurity → chocoZAP」等のミスマッチを先に解消

**検証**:
- Week 1-2: `/ranking/*` 配置を Top10 ページに追加、クリック数を Cloudflare logs で計測
- Week 3-4: A8.net 成約レポートを取得 (週次手動 CSV → スクリプト化)
- Week 5-8: EPC 上位 / 下位を見て商材入替・配置調整
- 成功基準: 8 週後に月 ¥5,000 突破
- 失敗判定: 8 週後に月 ¥1,000 未満 → 商材選定・配置を全面見直し

### 2. AI ツール × 統計実務の特化記事 — 推奨 ★★★★☆

**仕組み**:
- 「ChatGPT で e-Stat データを分析する 7 つの手順」「Claude × Excel 統計表の整形術」のような実用記事を 3-5 本投下
- 記事内に AI 関連アフィリエイトを比較配置:
  - AI スクール (SHIFT AI / DMM 生成 AI CAMP / Aidemy 等、1 件 5,000-15,000 円)
  - Udemy 講座 (1 件 1,000-3,000 円、CVR 高)
  - Notion AI / ChatGPT Plus 紹介 (月額系、低単価高 CVR)
- Desktop 67% の業務利用層 + ペルソナ 3 (公務員) に刺さる仮説

**収益レンジ (月次・3ヶ月後想定)**:
- 悲観: ¥500 / 標準: ¥3,000-8,000 / 楽観: ¥15,000-30,000

**実装コスト**: M (記事 3 本 = 2-3 週)。記事は MDX で書く、`affiliate` frontmatter で関連商材紐付け。

**リスク**:
- 公務員職場 PC は外部 AI ツール制限あり → 標準ケースで CVR が下がる可能性
- 競合ブログ多数 → stats47 の「統計サイト発」という文脈が差別化要素

**検証**:
- Week 1-3: 記事 1 本目を投下
- Week 4-6: GSC で検索順位確認
- Week 7-12: 上位記事の affiliate CVR を見て横展開 or 撤退
- 成功基準: 1 記事あたり月 PV > 200 + 月成約 ≥ 1 件
- 失敗判定: 12 週後に Organic Search クリック < 50/月 → SEO キーワード再選定

### 3. AdSense 最適化 (配置 + 自動広告 + PSI 連携) — 推奨 ★★★☆☆

**仕組み**:
- 現状 RPM ¥26 / CTR 1.25% は業界平均 (¥100-200 / 1.5-2%) より低い
- 配置調整で RPM 2-3 倍化を狙う
- **LCP 改善が前提** (現在 LCP 16 秒で広告表示前に離脱)
- 自動広告 ON / OFF の A/B、手動 Anchor 広告の試行

**収益レンジ (月次・3ヶ月後想定)**:
- 悲観: ¥200 / 標準: ¥1,000-3,000 / 楽観: ¥5,000

**実装コスト**: S (AdSense 管理画面操作)。ただし PSI 改善 (LCP 16秒 → 2.5秒) は別タスクで L 工数。

**リスク**:
- AdSense はクリック単価が低い (統計コンテンツは商業価値低い)
- LCP 改善が進まないと収益も伸びない

### 4. データ販売 / API プレミアム (法人向け) — 検討 ★★★☆☆

**仕組み**:
- 47 都道府県の完全データセット (area_profiles 22,487 行、correlations 36,620 行) を CSV / API で販売
- ターゲット: シンクタンク、地方自治体の調査担当、コンサル、不動産・人材業界の調査部門
- 単価: CSV ¥3,000-10,000 / 個別レポート ¥30,000-100,000 / API 月額 ¥10,000-50,000

**収益レンジ (月次・6ヶ月後想定)**:
- 悲観: ¥0 / 標準: ¥10,000-30,000 / 楽観: ¥100,000+

**実装コスト**: L (ランディングページ + Stripe 決済 + データ提供フロー)。営業活動も必要。

**リスク**:
- 需要不確実性が最大: e-Stat は無料公開、有料化の価値訴求が必要
- 競合 (帝国データバンク・東洋経済等) との価格・差別化を明確化する必要
- 個人開発のため法人取引対応コスト (契約書・請求書) が重い

**検証**:
- Phase A (1ヶ月): 簡易 LP + Twitter / 自治体向け Web フォーム告知で需要ヒアリング (5-10 件)
- Phase B (需要確認後): 最小 PoC (Stripe 決済 + CSV ダウンロード) を 1 商品で実装
- 成功基準: 3 ヶ月で問い合わせ 5 件 + 成約 1 件
- 失敗判定: 問い合わせ < 2 件 → 撤退

### 5. note プレミアム / 詳細レポート (個人向け) — 後回し ★★☆☆☆

**仕組み**: 既存記事の深堀り版を note 有料記事 (¥500-2,000) として販売。

**収益レンジ (月次)**: 悲観 ¥0 / 標準 ¥1,000-3,000 / 楽観 ¥10,000

**判断**: 3 のあとに余力があれば検討。今は優先度低い。

### 優先度マトリクス

| 戦略 | 収益ポテンシャル | 実装容易性 | UX影響 | 推奨順位 |
|---|---|---|---|---|
| 1. アフィリエイト本格化 | ★★★★ (月¥10K-30K) | M | 低-中 | **1位** |
| 2. AI × 統計実務記事 | ★★★ (月¥3K-15K) | M | 低 | **2位** |
| 3. AdSense 最適化 | ★★ (月¥1K-3K) | S | 中 | **3位** |
| 4. データ販売 | ★★★★ (月¥10K-100K) | L | 無 | 4位 (検証から) |
| 5. note プレミアム | ★ (月¥1K-3K) | S | 低 | 5位 (保留) |

## やらないこと

### 戦略上やらないこと

| やらないこと | 理由 |
|---|---|
| 複数収益源の同時立ち上げ (4 つ以上) | リソース分散で全てが中途半端になる。Two-track + 1〜2 補助まで |
| B2G 向けのプロアクティブ営業 | 予算 cycle・入札プロセスのコストが個人運営に対して過大 |
| 有料会員制・サブスクリプション | PV が少ない段階では母数が足りない |
| 高単価アフィリエイトの SEO 勝負 | 大手企業が独占するレッドオーシャン |
| note での技術チュートリアル有料販売 | 2026-03-04 検討で「現段階では着手すべきでない」と結論 (無料代替充実、機会コスト過大、ブランド不一致) |
| 総合スコアリングランキング | 異なる次元の指標を合成した恣意的ランキングは公的統計可視化のコアバリューを毀損 |

### リソース配分のガードレール

| やらないこと | 理由 |
|---|---|
| 月に 2 つ以上の新データセットを追加 | 既存コンテンツの収益最適化を優先 |
| 自動化パイプラインのリファクタリングに週 3 時間以上 | 技術的に面白い作業への逃避を防ぐ |
| PV 1,000 未満のページのデザイン改善 | 投資対効果が低い |
| ユーザー要望 3 件未満の新機能開発 | 需要が未検証 |

## 注意事項・制約

1. **PSI 改善は最優先の前提条件**: LCP 5-19 秒のままだと、何を導入しても天井が低い。別タスクとして PSI 改善を並行で回す
2. **公共性の維持**: 基本データ (ランキング・カテゴリ・area_profiles) は無料維持。有料化はプレミアム機能に限定
3. **個人開発の運用負荷**: improvement-log 系スキルがすでに 5 系統稼働中。新規 affiliate-improvement 追加で 6 系統 → 週次運用 1.5h 想定。自動化 (GitHub Actions) で運用負荷を抑える
4. **公務員ペルソナを Primary、Two-track 並走**: GSC「経常収支比率 自治体 ランキング」CTR 100% (W19) 等の signal を確認しペルソナ 3 needs 実在を確認。但し現状流入の主体はペルソナ 1 のため、Track A (SEO) と Track B (公務員特化) の Two-track で並走。詳細・KPI・撤退基準: [`04_ターゲットペルソナ.md`](./04_ターゲットペルソナ.md)
