# 収益化戦略

## 目的

Stats47 の収益化は「事業化による利益最大化」ではなく、**サービスを持続的に運営するためのコスト回収**を第一目的とする。サーバー費・ドメイン費・データ取得の工数を広告収益で賄い、個人プロジェクトとして無理なく継続できる状態を目指す。

---

## 基本方針

> **「まずPVを育てる。収益はPVについてくる」**

Phase 1（PV基盤構築）→ Phase 2（収益多様化）の2段階で進める。複数収益源を同時に追わない。

### 固定費と損益分岐点

| 項目 | 月額 |
|---|---|
| Cloudflare（Pages + D1 + R2） | 約500円 |
| ドメイン（stats47.jp） | 約300円 |
| その他（API利用等） | 約200円 |
| **合計** | **約1,000円** |

| PV単価 | 損益分岐PV |
|---|---|
| 0.3円（楽観） | 3,333 PV/月 |
| 0.1円（悲観: 統計サイト実績値） | 10,000 PV/月 |

---

## Phase 1: Google AdSense（現在〜PV基盤構築）

### 概要

- **手段**: Google AdSense（ディスプレイ広告）
- **目標**: 月間10万PV到達
- UXを損なわない広告配置（コンテンツ下部・サイドバー中心）
- Core Web Vitals を維持した上での広告最適化

### KPI

| 指標 | 3ヶ月 | 6ヶ月 | 1年 |
|---|---|---|---|
| 月間PV | 3万 | 10万 | 20万 |
| AdSense月収益（楽観: 0.3円/PV） | 9,000円 | 3万円 | 6万円 |
| AdSense月収益（悲観: 0.1円/PV） | 3,000円 | 1万円 | 2万円 |
| SNSフォロワー合計 | 500人 | 3,000人 | 10,000人 |

### 撤退基準

Phase 1 開始から 3 ヶ月後に AdSense + アフィリエイトの合計が **月 1,000 円未満** の場合、収益化施策を一時停止し PV 基盤構築に全リソースを振り向ける。

### リスクと対策

| リスク | 対策 |
|---|---|
| AI Overview・ゼロクリック検索の拡大 | SNS（X）からの直接流入を並行強化 |
| AdSense単価の下落 | 早期にnoteアカウントを育て、Phase 2の準備を進める |
| Google広告ポリシー変更・アカウント停止 | データAPI・ウィジェット等、広告非依存の収益モデルの技術検証 |

### 3ヶ月レビュー: Phase 2の小規模検証

Phase 1開始から3ヶ月後、Phase 2移行条件未達でも以下を実施する:
1. noteに統計調査ガイド（A シリーズ）を4本投稿し、stats47.jp への流入効果を UTM で計測する
2. アフィリエイトリンクを3ページに試験設置しCTR/CVRを計測する

これにより Phase 2 の商品設計に実データを反映できる。

---

## Phase 2: 収益多様化

Phase 1の3ヶ月レビュー結果を踏まえ、以下を**選択的に**展開する。優先度は順番通り。

---

### 2-1. アフィリエイト（低リスク・高い親和性）

#### 方針

「情報収集意図」のファクトページではなく、**「行動意図」のあるコンテンツ**に絞って設置する。ランキングデータを「選択の意思決定を助けるコンテンツ」に再構成し、自然な文脈でアフィリエイトを配置する。

#### 第一候補: Uターン・Iターン転職ガイド

1記事で検証してから横展開を判断する。

| 項目 | 内容 |
|---|---|
| テーマ | 地方移住の「実質年収」を都道府県別に試算 |
| ターゲット | 東京・大阪圏から地方転職を検討中の20〜40代 |
| 切り口 | 給与 × 物価 × 家賃の3変数で実質購買力を可視化 |
| ASP案件 | 転職エージェント（リクルート、doda）、引越し見積もり |
| 使用データ | 県民所得、給与月額、消費者物価地域差指数、家賃、有効求人倍率等（すべて既存） |

**SEOカニバリ対策**: 既存ランキングページ（単一指標のファクト検索）とアフィリエイト記事（複合・行動意図のロングテール）でキーワードを棲み分け、クラスター構造で相互リンクする。note 有料記事とのカニバリ対策は「2-3. note有料記事」セクション参照。

**重要**: 「総合スコアリング」は作らない。異なる次元の指標を合成した恣意的ランキングは、stats47の「公的統計の正確な可視化」というコアバリューを毀損する。読者が自分の優先軸で判断できる比較型構成にする。

#### 収益シミュレーション

| 項目 | 値 |
|---|---|
| 名目単価 | 2,000円（新規サイト向け通常報酬） |
| 承認率 | 70% |
| 実質単価 | 1,400円 |

| 記事PV/月 | CTR | CVR | 月間収益 |
|---|---|---|---|
| 3,000 | 5% | 2% | 4,200円 |
| 5,000 | 5% | 2% | 7,000円 |
| 10,000 | 5% | 2% | 14,000円 |

執筆10時間の場合、月3,000PV（月4,200円）で約2.4ヶ月回収。月1,000PV未満ならROI不良。

#### 実行ステップ

1. **ASP登録**: A8.net、もしもアフィリエイトに登録。転職・引越し案件に提携申請
2. **記事執筆・公開**: 既存データの再構成のみ。追加データ取得不要
3. **KPI計測（2週間後）**: PV > 1,000/月ペース → 継続。CTR < 1% → CTA改善。成約0件が1ヶ月 → 施策停止
4. **横展開判断（1ヶ月後）**: 全指標が閾値以上なら2記事目（引越し or 子育て）を制作

#### 横展開候補

| テーマ | ターゲット | ASP案件 |
|---|---|---|
| 引越し先選び総合ガイド | 転勤・結婚・リモート移行の20〜40代 | 引越し見積もり、賃貸検索 |
| 子育て世帯の引越し先ガイド | 出産前後〜小学校入学前の子育て世帯 | 引越し見積もり、学資保険 |
| 自動車保険の都道府県別リスク比較 | 保険見直し検討中のドライバー | 自動車保険一括見積もり |

### 2-2. CSV有料販売

#### CSVの付加価値（e-Stat生データとの差別化）

e-Statのデータは無料だが、以下の課題がある:
- フォーマットがバラバラ（統計ごとに構造が異なる）
- 年度間の比較が困難（基準改定、市町村合併の補正なし）
- 47都道府県の横断比較に加工が必要

stats47のCSVが提供する付加価値:
- **整形済み**: 全指標で統一フォーマット
- **偏差値・順位付き**: 自分で計算不要
- **複数指標の横断CSV**: 1クリックで取得
- **地域区分・カテゴリ分類済み**

ターゲット: コンサル・自治体職員・学生・ジャーナリスト
価値提案: 「数時間の加工作業を数百円で省ける」

#### AdSense + CSV販売の両立

AdSenseを捨てる必要はない。両立が最適:

```
stats47.jp → 無料閲覧（AdSense広告付き）← PV収益の柱
           → CSV有料DL（広告非表示のプレミアム体験）← 単価の柱
```

- **一般ユーザー**: 無料閲覧 + 広告 → AdSense収益
- **仕事で使うユーザー**: CSV購入 → 直接収益 + 広告非表示の満足感

#### 収益比較

| | AdSense | CSV販売 |
|---|---|---|
| 月10万PVの場合 | 月1〜3万円 | 月10件 × 500円 = 5,000円 |
| 必要なPV規模 | 大量PVが必要 | 少数の「本気ユーザー」でよい |
| 立ち上がり | PVさえあれば即収益 | 購入者が来るまでゼロ |

#### 注意点

- **決済インフラが必要**: Clerk + Stripe での実装を想定 → [実装ガイド](../90_課題管理/clerk-billing-implementation.md)
- **CSV単価設定**: 300〜1,000円/ファイルが相場感
- **Clerk Billing はBeta（2026年3月時点）**: USD のみ、日本円未対応

### 2-3. note記事（無料 + 有料）

note は「調査」を軸にした無料記事（A: 統計調査ガイド / B: データリテラシー）で「統計を知る・活用する」ブランドを構築し、stats47.jp への被リンク獲得 + 有料コンテンツ（C/D/E シリーズ）への自然なファネルを形成する。

**重要**: note にランキング記事・データストーリーは投稿しない（ブログとのカニバリ防止）。

戦略の詳細（コンセプト・ファネル構造・SEO カニバリ対策・投稿ルール・スケジュール・KPI・記事企画）は **[note 戦略](../30_note記事企画/note戦略.md)** に一元管理。

#### 技術チュートリアル有料販売: 非推奨（2026-03-04 検討結果）

D3.jsコロプレス地図・Claude Code × e-Stat API等の技術チュートリアルをnoteで有料販売するアイデアを3名体制で検討し、**全員一致で「現段階では着手すべきでない」と結論**。

| 却下理由 | 詳細 |
|---|---|
| 既存方針との矛盾 | 「複数収益源の同時立ち上げはやらない」に違反。noteの信頼構築役割を壊す |
| 無料代替の充実 | D3.js・e-Stat APIの無料チュートリアルがQiita/Zennに10件超存在 |
| 機会コスト | 30〜40時間の執筆工数で無料note記事5〜8本 or ランキング5〜10ページ追加が可能 |
| ブランド不一致 | データエンタメのターゲット（雑学好き・自治体職員）とエンジニア向けチュートリアルは別ペルソナ |
| Claude Code依存 | 年176回のCHANGELOG更新。3ヶ月で陳腐化リスク。メンテコストが売上を上回る |
| 収益の天井 | 年間10〜30万円が現実的。投下時間に見合わない |

**代替案**: 技術ネタはstats47ブログに無料記事として掲載し、「D3.js 日本地図」「e-Stat API」のSEOキーワードで流入獲得に活用する。有料化するならZennの方がプラットフォーム特性に合致するが、Phase 2以降に条件付きで検討。

---

### 2-4. B2B/B2G（インバウンドのみ）

- プロアクティブな営業は行わない
- メディア掲載・SNSバズがきっかけで問い合わせが来た場合のみ対応
- 対応はスポットのカスタム分析・データ提供に限定
- B2B向けの権威性構築・リード獲得 → [マーケティング戦略](./03_マーケティング戦略.md#b2bマーケティングと権威性構築戦略)

## やらないこと

### 戦略上やらないこと

| やらないこと | 理由 |
|---|---|
| 複数収益源の同時立ち上げ | リソース分散で全てが中途半端になる |
| B2G向けのプロアクティブ営業 | 予算cycle・入札プロセスのコストが個人運営に対して過大 |
| 有料会員制・サブスクリプション | PVが少ない段階では母数が足りない |
| 高単価アフィリエイトのSEO勝負 | 大手企業が独占するレッドオーシャン |
| noteでの技術チュートリアル有料販売 | 上記の検討結果参照。Phase 2以降に条件付きで再検討 |

### リソース配分のガードレール

| やらないこと | 理由 |
|---|---|
| 月に2つ以上の新データセットを追加 | 既存コンテンツの収益最適化を優先 |
| 自動化パイプラインのリファクタリングに週3時間以上 | 技術的に面白い作業への逃避を防ぐ |
| PV 1,000未満のページのデザイン改善 | 投資対効果が低い |
| ユーザー要望3件未満の新機能開発 | 需要が未検証 |
