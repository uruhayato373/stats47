# プロジェクト定義


## 1. プロジェクト基本情報

| 項目 | 内容 |
|---|---|
| ブランド名 | 統計で見る都道府県 |
| 技術名 | stats47 |
| キャッチフレーズ | 統計をもっと身近に。探すのに半日、加工にもう半日。そんな時代を終わらせたくて、元県庁職員がつくりました。 |
| 開始日 | 2024年10月 |
| 現在フェーズ | 本格運用フェーズ |
| 運営者 | KAZU（元県庁職員20年、現データ可視化コンサルタント） |

### 運営者プロフィール

長年、自治体職員として統計の「作る現場」と「使う現場」の両方に身を置いてきた。作る立場として見てきたのは、多くの職員が膨大な手間と時間をかけ、1件1件の数字に込める真摯な仕事。しかし使う立場になると、その貴重なデータは複雑なサイトの奥底に眠り、**どこに何があるか分からず、見つけても加工しなければ使い物にならない**という現実に直面した。

**「統計を使いたいのに、使えない。」**

この悩みは、自分だけのものではなかった。周囲の同僚も、取引先の企業も、研究者も、同じ壁にぶつかっていた。stats47は、この「探す苦労」と「加工する手間」をゼロにするために生まれたプロジェクトである。公的データの正確性はそのままに、現代のUI/UXで「すぐに使える形」へとリデザインする。統計の現場を知る開発者だからこそできる「データの翻訳」を通じて、**統計をもっと身近に**していく。

---

## 2. ミッション・ビジョン

### ミッション

> **統計をもっと身近に。探す手間、加工する苦労をなくし、誰もがデータを「使える」社会をつくる。**
> 行政が手間と時間をかけて作り上げた統計の価値を、「データの翻訳者」として届け、読者自身が統計を使いこなせるようになるまでを支援する

### ビジョン

> **「ランキングで笑った人が、e-Stat を自分で読める人になる」**
> エンタメで興味を持ち → 調査の仕組みを知り → データを自分の意思決定に使う。stats47 はその全過程を、かつて統計の現場にいた開発者が支える

### 存在意義（Why）

統計データは行政白書・新聞・政策資料に溢れているにもかかわらず、多くの人は「どこにあるか分からない」「見つけても加工が大変」と感じて素通りしている。一方で「焼肉消費量ランキング」のような身近な切り口なら、統計はSNSで拡散されるほどの力を持つ。**この「探せない」と「使えない」の壁を取り除き、統計をもっと身近にすることが、stats47の存在意義だ。**

統計を「作る苦労」と「使う不便」の両方を知る開発者が、行政と市民の間に立つ「翻訳者」として、データの意味を文脈とともに届ける。単なる数値の可視化ツールではなく、**統計をもっと身近にするための総合プラットフォーム**を目指す。

- **知る**: ランキング・チャート・地図で「統計って面白い」と気づく（stats47.jp）
- **理解する**: 調査の仕組み・データの読み方を学ぶ（note 無料記事）
- **活用する**: 加工済みデータ・分析手法で仕事や意思決定に使う（note 有料記事・CSV）

## 3. コンテンツファネル

### stats47.jp 内のファネル（3ステップ）

```
【Step 1: 雑学・エンタメで入口を作る】
  焼肉消費量・清酒消費量・コンビニ数など
  → 「統計って面白いかも」という最初の体験
        ↓
【Step 2: 地元愛・自分ごと化で深める】
  「自分の県は何位？」「自県の強みはどこ？」
  → 「統計で地元が見える」という発見
        ↓
【Step 3: 統計リテラシーの獲得】
  白書・新聞・政策資料の統計グラフも怖くない
  → 社会的意思決定に参加できる市民の育成
```

### クロスプラットフォーム・ファネル（note連携）

```
【認知】note 無料記事 → 【興味】stats47.jp → 【購入】note 有料記事
```

note は「統計を知る・活用する」のブランド構築と stats47.jp への被リンク獲得を担う。ファネル構造・投稿ルール・記事企画の詳細は **[note 戦略](../30_note記事企画/note戦略.md)** に一元管理。

## 4. コアバリュー

1. **わかりやすさ**: 専門知識ゼロでも楽しめる入口を作る
2. **信頼性**: 公的統計（e-Stat等）に基づく正確なデータと出典の明示
3. **中立性**: 客観的なデータ提示。「不都合な真実」も隠さない
4. **無料アクセス**: 基本データへのアクセスは永続的に無料で提供
5. **経験知**: 統計の現場を知るからこそ書ける「このデータの読み方」。調査手法の変更点や定義の落とし穴など、行政職員ならではの視点を提供する
6. **活用支援**: 「見て終わり」ではなく、読者が自分でデータを使えるようになるまでを支援する。調査の仕組み・データの探し方・分析手法を提供する

## 5. ターゲットユーザー（ペルソナ）

### ペルソナ 0: データ活用に悩む実務者（プロジェクトの原点）

**人物像**: 30〜50歳。仕事や研究で統計データを使いたいと考えている会社員・公務員・研究者・学生。「データを活用したい意志はあるが、入手・加工のハードルに阻まれている」層。

**典型的な行動パターン**:
1. 上司や取引先から「〇〇のデータ出して」と言われる
2. Google検索 →「e-Stat」に辿り着くが、どの調査に欲しいデータがあるか分からない
3. やっと見つけても、CSVのフォーマットが複雑で加工に数時間かかる
4. 結局「ランキング 都道府県」で検索し直して、誰かがまとめた二次情報に頼る

**ペインポイント**: どの統計調査に、どんなデータが含まれているか分からない / e-Statの検索UIが使いにくく、目的のデータに辿り着けない / CSVをダウンロードしても、ヘッダーが複雑・年次がバラバラで加工に時間がかかる / 加工したデータが正しいのか、定義や調査方法の落とし穴がないか不安

**stats47 の価値**: 加工済み・正規化済みのデータがすぐに使える（「探す＋加工」の時間をゼロに） / 出典・調査手法・定義の注意点が明記されていて安心して引用できる / グラフや地図がそのまま資料に使える品質 / 「このデータの読み方」注釈で、統計リテラシーが自然に身につく

**収益化**: AdSense + note有料記事（加工済みCSV・分析レポート）

---

### ペルソナ 1: 雑学・エンタメ層（入口・拡散の主役）

**人物像**: 25〜45歳・会社員・SNS利用者。ランキング・雑学コンテンツを好む。統計は「難しいもの」と避けてきたが、面白い切り口なら見る。

**どう届くか**: TikTok・Instagram Reels・Xのアルゴリズム経由。

**ペインポイント**: 統計が難しく感じる / 地元データがどこにあるか分からない / グラフが多くて何が重要か分からない

**stats47 の価値**: ランキング・偏差値・相関という分かりやすい切り口 / 「自分の県は何位？」と思わず確認したくなるコンテンツ / SNSでシェアしたくなる驚きのデータ

**収益化**: Google AdSense（PV連動）

---

### ペルソナ 2: 地元愛・深掘り層（エンゲージメント・リピートの主役）

**人物像**: 30〜50歳・地元愛が強い・地方在住または出身者。自分の地元が全国でどういう位置にあるかを知りたい。行政資料や地域ニュースは読む。

**どう届くか**: Instagram「〇〇県の強み5指標」カルーセル / Google検索「〇〇県 ランキング」。

**ペインポイント**: 地元の良さをデータで説明したいが出典が分からない / 白書の統計グラフを何となく読み飛ばす / 地域の良い面も悪い面も客観的に知りたい

**stats47 の価値**: 47都道府県の横断比較 / 「不都合な真実」も含めた中立的な提示 / 統計の読み方が自然に身につくコンテンツ設計

**収益化**: 継続訪問 → AdSense収益・SNSシェアによるリーチ拡大

---

### ペルソナ 3: 行政・自治体関係者（専門的ユーザー・権威性の源泉）

**人物像**: 30〜50歳・自治体職員・議会事務局・地域シンクタンク。予算要望や政策立案の根拠としてデータが必要。e-Statは使えるが加工に時間がかかる。

**どう届くか**: Google検索「〇〇 都道府県別 統計」/ 同僚・庁内からの口コミ。

**ペインポイント**: e-Statからのデータ取得・加工に手間がかかる / 会議資料に使える見栄えの良いグラフが欲しい / 他県との比較データを短時間で準備したい

**stats47 の価値**: 加工不要でそのまま資料に使えるグラフ・表 / 行政用語の適切な注釈 / データの定義・調査手法の変更点に関する注意喚起

**収益化**: 信頼性向上 → 被リンク獲得・E-E-A-T強化 → 検索流入の底上げ

---

### ペルソナ 4: ビジネス・マーケター層（実務的ユーザー）

**人物像**: 25〜45歳・マーケター・経営企画・コンサルタント。出店計画やターゲット選定のため、地域別の経済・消費データを必要とする。

**どう届くか**: Google検索「都道府県別 〇〇 ランキング」/ SNSでのデータ引用経由。

**ペインポイント**: 複数の統計を掛け合わせた分析がしたいが自力では手間 / 信頼できる出典付きのデータが必要 / ビジュアルで説得力のある資料を作りたい

**stats47 の価値**: 複数指標の相関分析・地域間比較機能 / 出典明記による資料としての信頼性 / CSV・画像ダウンロードによる二次利用のしやすさ

**収益化**: AdSense + 将来的なプレミアムデータパック検討

## 6. 市場ポジショニング（3C分析）

### 6.1 Customer（市場・顧客）

**市場トレンド**:
- 統計データの需要は「調べる」から**「引用して発信する」**へシフト。SNSで「ドヤれる根拠」としての統計需要が拡大
- AI Overview（SGE）の普及により、単なる数値テーブルを置くだけのサイトは検索結果から消滅リスク
- ユーザーが求めるのは「数字」ではなく、その数字が示す**「意味」と「文脈」**

**検索行動の変化**:
- 「〇〇県 〇〇 ランキング」等の比較検索は依然として高需要
- 「なぜ〇〇県は1位なのか」等の因果・背景を問う検索が増加
- スマホで閲覧 → そのままSNSでシェアする導線が主流

### 6.2 Competitor（競合）

| 競合 | セグメント | 強み | 弱点 |
|---|---|---|---|
| **とどラン** (todo-ran.com) | 相関分析・ストーリーテリング型 | 人口比正規化の徹底、相関係数による多角的分析、著者の深い考察による「読み物」としての価値 | UIの古さ、分析が属人的で更新に限界、モバイル体験が弱い |
| **都道府県市区町村** (uub.jp) | アーカイブ・行政粒度型 | 全1,741自治体の網羅、明治からの合併変遷履歴、毎週更新の首長情報、「落書き帳」コミュニティ | ビジュアル化が弱い、ライト層には敷居が高い、データの可視化機能がない |
| **AI検索** (Perplexity, Gemini等) | 即答型 | サイト訪問不要で回答生成、自然言語での質問に対応 | データ出典が曖昧、ビジュアル精度が低い、「引用元」になれるサイトは逆に恩恵を受ける |
| **e-Stat・RESAS** (公的機関) | 一次データ提供型 | 圧倒的データ量・公的信頼性 | 使いにくいUI、加工が前提、エンタメ性ゼロ |

**競合の成功要因（学ぶべき点）**:
- **とどラン**: 「データ → 解釈 → 物語」の流れが検索エンジンに高く評価されている。相関分析による内部リンク網が強固
- **UUB**: ニッチだが代替不可能な一次情報（変遷履歴）が強力な堀（Moat）。毎週更新による鮮度シグナルがクロール頻度を維持

### 6.3 Company（自社）

**強み**:
- **モダンUI/UX**: 競合が持たない「触っていて気持ちいい」操作性。モバイル対応も万全
- **ドメイン知識（E-E-A-Tの「Experience」）**: 統計を「作る側」と「使う側」の両方を知る、他サイト運営者にない経験
- **開発スピード**: 個人開発 + AI活用による、トレンド・時事への即応性
- **自動化パイプライン**: Remotion/Satori によるSNSコンテンツの自動生成体制
- **ビジュアル力**: 地図UI・タイルマップ・チャートの表現力で競合を圧倒

**弱み**:
- **文脈（Context）の不足**: 「なぜこの県が1位なのか」という解釈・背景説明が不足。「ツール」に留まり「メディア」になりきれていない
- **ロングテールSEOの未整備**: 市区町村名を含む検索クエリへの対応が弱い
- **動的更新シグナルの弱さ**: 年度更新はあるが、UUBのような毎週のニュース性がない
- **被リンク（バックリンク）の不足**: 学術・行政機関からの引用がまだ少ない

**勝ち筋: 「行政の翻訳者」ポジション**:

> stats47が目指すのは、単なる統計の可視化ツールではなく、**「日本で最も引用しやすい、統計のビジュアル・インテリジェンス・ハブ」**である。とどランの分析力とUUBの網羅性を、stats47のUI/UXとドメイン知識で統合する。

## 7. スコープ

### 含まれるもの

- e-Stat API中心の政府統計データ取得・可視化
- 47都道府県の統計データ比較・ランキング機能
- エンタメコンテンツ（雑学シリーズ）による入口設計
- グラフ・チャート・コロプレス地図による可視化
- 統計データの背景・解釈を解説するブログ機能
- SNS向け画像・動画の自動生成（Remotion/Satori）
- Google AdSenseによる広告収益

### 含まれないもの

- 民間企業の独自統計データ（将来的には検討）
- リアルタイムデータ（政府統計は定期更新に依存）
- 個人情報の収集・管理

## 10. リスク管理

| リスク | 軽減策 |
|---|---|
| **個人開発の持続性（最大）**: 一人が燃え尽きる | AIツール・Remotionでコンテンツ自動化。「人がいなくても動くパイプライン」を完備 |
| **PVが伸びない**: エンタメコンテンツがアルゴリズムに乗らない | 手動10〜20本で「当たる法則」を検証してから量産に切り替える |
| **API制限**: e-Stat仕様変更 | ローカルキャッシュ・補完データの整備 |
| **データ品質**: 統計の不整合・欠損 | データ品質チェックの自動化 |
| **競合強化**: 公的機関・民間大手の機能強化 | 「エンタメ→地元愛」ファネル設計自体が差別化となり部分的に代替不可 |
| **AI検索による流入減**: AI Overviewがサイト訪問を代替 | 構造化データ・FAQ対応で「引用元」ポジションを確保。AIに引用されるサイトを目指す |

## 12. ブランド・トーン&マナー

### 文章スタイル

- **丁寧だが堅苦しくない**: 「です・ます」調、専門用語には解説を添える
- **具体的でわかりやすい**: 抽象的な表現を避け、具体例を示す
- **中立的**: 特定の地域を過度に持ち上げたり貶めたりしない

### 推奨する表現

- 「データから読み取れること」
- 「統計で見ると」
- 「他の地域と比較すると」

### 避けるべき表現

- 過度に専門的な統計用語（解説なし）
- センセーショナルな表現
- 断定的すぎる主張

## 13. SEO・コンテンツ戦略

### E-E-A-T強化（最優先）

- **Experience（経験）の可視化**: 運営者の自治体職員経験をAboutページ・各ページの注釈で明示
- **元担当者の視点**: 各指標ページに「このデータの読み方」注釈を段階的に配置（調査手法の変更点、定義の注意点など）
- **トピッククラスター構築**: テーマ別ピラーページ（例：「日本の人口減少」）→ 個別ランキングページへの内部リンク集中

### 構造化データ

- 全指標ページにDataset JSON-LDマークアップを適用 → Googleデータセット検索でのインデックス確保
- FAQ構造化データで「〇〇県で一番〇〇なのは？」等のAI Overview引用率を向上

### コンテンツの質的転換
