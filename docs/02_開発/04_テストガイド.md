# e-Stat API テストガイド

## 概要

本ガイドでは、e-Stat API を使用するアプリケーションのテスト戦略について説明します。API 依存を排除し、効率的な開発・テストを実現するためのモックデータ戦略を中心に解説します。

## 目的

1. **API 依存の排除**: API キーやネットワーク接続なしで開発・テスト可能
2. **開発速度の向上**: API 制限を気にせず高速に反復開発
3. **再現性の確保**: 同じデータで一貫したテストとレビューが可能
4. **オフライン開発**: ネットワーク環境に依存しない開発
5. **コスト削減**: 開発中の API 呼び出しコストを削減

## e-Stat API の種類と特徴

### 1. GET_META_INFO（メタ情報）

統計表の構造情報を取得する API。

**用途**:

- 統計表の階層構造（カテゴリ、地域、年次）の取得
- データの可視化に必要なメタデータの取得

**特徴**:

- 都道府県版と市区町村版で階層構造が異なる
- データサイズ: 小（数 KB〜数十 KB）

**レスポンス例**:

```json
{
  "GET_META_INFO": {
    "RESULT": { "STATUS": 0 },
    "PARAMETER": { "statsDataId": "0000010101" },
    "METADATA_INF": {
      "CLASS_OBJ": [
        {
          "@id": "cat01",
          "CLASS": [{ "@code": "A1101", "@name": "総人口" }]
        }
      ]
    }
  }
}
```

### 2. GET_STATS_DATA（統計データ）

実際の統計数値データを取得する API。

**用途**:

- ランキング表示
- グラフ・チャートの描画
- 統計分析

**特徴**:

- 都道府県版（47 地域）と市区町村版（1,800+地域）でサイズが大きく異なる
- データサイズ: 中程度（数百 KB〜1 MB）

**レスポンス例**:

```json
{
  "GET_STATS_DATA": {
    "RESULT": { "STATUS": 0 },
    "STATISTICAL_DATA": {
      "DATA_INF": {
        "VALUE": [
          {
            "@area": "13000",
            "@cat01": "A1101",
            "@time": "2020100000",
            "$": "13921000"
          }
        ]
      }
    }
  }
}
```

### 3. GET_STATS_LIST（統計表リスト）

利用可能な統計表の一覧を取得する API。

**用途**:

- 統計表の検索・探索
- 統計表 ID の取得
- メタデータの収集

**特徴**:

- 検索条件に応じて結果が変動
- データサイズ: 中（数十 KB〜数百 KB）

## 地域レベルによる分類

### 都道府県データ

**パラメータ**: `lvArea=1`（全国）または`lvArea=2`（都道府県）

**特徴**:

- 47 都道府県（areaCode: 01000-47000）
- データサイズ: 中程度
- 処理速度: 高速
- 用途: 一般的な統計分析、ランキング表示

**データ例**:

```json
{
  "areas": [
    { "areaCode": "01000", "areaName": "北海道" },
    { "areaCode": "02000", "areaName": "青森県" },
    { "areaCode": "13000", "areaName": "東京都" }
  ]
}
```

### 市区町村データ

**パラメータ**: `lvArea=3`（市区町村）

**特徴**:

- 約 1,800 市区町村（areaCode: 01100-47999）
- データサイズ: 大（都道府県の約 40 倍）
- 処理速度: 中程度
- 用途: 詳細な地域分析、市区町村別ランキング

**データ例**:

```json
{
  "areas": [
    { "areaCode": "01100", "areaName": "札幌市" },
    { "areaCode": "01202", "areaName": "函館市" },
    { "areaCode": "13101", "areaName": "千代田区" }
  ]
}
```

```typescript
const nextConfig: NextConfig = {
  // 本番ビルドではモックデータを除外
  webpack: (config, { dev, isServer }) => {
    if (!dev && !isServer) {
      // 本番ビルド時にモックデータへの参照を削除

        ...config.resolve.alias,
        "@/data/mock": false,

  },

  // 静的ファイルの最適化
  experimental: {
    optimizePackageImports: ["@/lib/mock-data"],
  },
};

export default nextConfig;
```

### トラブルシューティング

#### Q1: デプロイ時にモックデータが含まれてしまう

**確認事項**:

1. `wrangler.toml`の`exclude`設定
2. `next.config.ts`の webpack 設定
3. 環境変数`NODE_ENV`の設定

**解決策**:

```bash
# ビルドサイズを確認
npm run build
npm run mock:size-check

# デプロイ前チェックを実行
npm run pre-deploy-check
```

#### Q2: 開発環境でモックデータが読み込めない

**確認事項**:

1. `.env.local`の設定
2. `data/mock/`ディレクトリの存在
3. ファイルパーミッション

**解決策**:

```bash
# モックデータのダウンロード
npm run mock:download -- 0000010101 prefecture

# 環境変数の確認
echo $USE_MOCK_DATA
```

#### Q3: Git 管理でファイルサイズが大きすぎる

**対策**:

1. Git LFS の使用
2. データの圧縮
3. フィクスチャのみ Git 管理

```bash
# Git LFSの設定
git lfs track "data/mock/**/*.json"
git add .gitattributes
git commit -m "Add Git LFS tracking for mock data"
```

## モックデータローダーの実装

### 基本ローダー

```typescript
// src/lib/mock-data/loader.ts
import fs from "fs/promises";
import path from "path";

export type MockDataType = "metainfo" | "statsdata" | "statslist";
export type AreaLevel = "prefecture" | "municipality";
export type DataSize = "minimal" | "small" | "medium";

export class MockDataLoader {
  private static readonly MOCK_DATA_DIR = path.join(
    process.cwd(),
    "data",
    "mock"
  );

  /**
   * メタ情報を読み込む
   */
  static async loadMetaInfo(
    statsDataId: string,
    areaLevel: AreaLevel = "prefecture"
  ): Promise<any> {
    const filePath = path.join(
      this.MOCK_DATA_DIR,
      "metainfo",
      areaLevel,
      `${statsDataId}.json`
    );
    return this.loadJsonFile(filePath);
  }

  /**
   * 統計データを読み込む
   */
  static async loadStatsData(
    statsDataId: string,
    areaLevel: AreaLevel = "prefecture"
  ): Promise<any> {
    const filePath = path.join(
      this.MOCK_DATA_DIR,
      "statsdata",
      areaLevel,
      `${statsDataId}.json`
    );
    return this.loadJsonFile(filePath);
  }

  /**
   * 統計表リストを読み込む
   */
  static async loadStatsList(queryName: string): Promise<any> {
    const filePath = path.join(
      this.MOCK_DATA_DIR,
      "statslist",
      `${queryName}.json`
    );
    return this.loadJsonFile(filePath);
  }

  /**
   * フィクスチャを読み込む
   */
  static async loadFixture(
    dataType: MockDataType,
    size: DataSize
  ): Promise<any> {
    const filePath = path.join(
      this.MOCK_DATA_DIR,
      "fixtures",
      size,
      `${dataType}.json`
    );
    return this.loadJsonFile(filePath);
  }

  /**
   * カタログを読み込む
   */
  static async loadCatalog(): Promise<any> {
    const filePath = path.join(this.MOCK_DATA_DIR, "catalog.json");
    return this.loadJsonFile(filePath);
  }

  private static async loadJsonFile(filePath: string): Promise<any> {
    try {
      const content = await fs.readFile(filePath, "utf-8");
      return JSON.parse(content);
    } catch (error) {
      throw new Error(
        `モックデータが見つかりません: ${filePath}\n` +
          `データをダウンロードしてください: npm run mock:download -- ${path.basename(
            filePath,
            ".json"
          )}`
      );
    }
  }
}
```

### 地域レベル別ローダー

```typescript
// src/lib/mock-data/area-specific-loader.ts
import { MockDataLoader } from "./loader";

export class AreaSpecificLoader {
  /**
   * 都道府県データを読み込む
   */
  static async loadPrefectureData(statsDataId: string) {
    return {
      metainfo: await MockDataLoader.loadMetaInfo(statsDataId, "prefecture"),
      statsdata: await MockDataLoader.loadStatsData(statsDataId, "prefecture"),
    };
  }

  /**
   * 市区町村データを読み込む
   */
  static async loadMunicipalityData(statsDataId: string) {
    return {
      metainfo: await MockDataLoader.loadMetaInfo(statsDataId, "municipality"),
      statsdata: await MockDataLoader.loadStatsData(
        statsDataId,
        "municipality"
      ),
    };
  }

  /**
   * 地域レベルを自動判定してデータを読み込む
   */
  static async loadData(
    statsDataId: string,
    areaLevel?: "prefecture" | "municipality"
  ) {
    if (areaLevel) {
      return areaLevel === "prefecture"
        ? this.loadPrefectureData(statsDataId)
        : this.loadMunicipalityData(statsDataId);
    }

    // 自動判定（統計表IDから判定）
    const isMunicipality = this.isMunicipalityStatsDataId(statsDataId);
    return isMunicipality
      ? this.loadMunicipalityData(statsDataId)
      : this.loadPrefectureData(statsDataId);
  }

  private static isMunicipalityStatsDataId(statsDataId: string): boolean {
    // 統計表IDのパターンから市区町村データかどうかを判定
    // 実際の実装では、カタログファイルやデータベースから判定
    return statsDataId.includes("municipality") || statsDataId.endsWith("_m");
  }
}
```

```typescript
      "0000010101",
```

## CLI ツール

### ダウンロードスクリプト

```typescript
// scripts/download-mock-data.ts
#!/usr/bin/env tsx

import { EstatAPIClient } from '@/lib/estat-api';
import fs from 'fs/promises';
import path from 'path';

interface DownloadOptions {
  statsDataId: string;
  areaLevel: 'prefecture' | 'municipality';
  dataTypes: ('metainfo' | 'statsdata')[];
}

async function downloadMockData(options: DownloadOptions) {
  const { statsDataId, areaLevel, dataTypes } = options;
  const client = new EstatAPIClient();

  console.log(`📥 ダウンロード中: ${statsDataId} (${areaLevel})`);

  for (const dataType of dataTypes) {
    try {
      let data;

      if (dataType === 'metainfo') {
        data = await client.getMetaInfo({ statsDataId });
      } else if (dataType === 'statsdata') {
        const lvArea = areaLevel === 'prefecture' ? '2' : '3';
        data = await client.getStatsData({
          statsDataId,
          lvArea
        });
      }

      const filePath = path.join(
        process.cwd(),
        'data',
        'mock',
        dataType,
        areaLevel,
        `${statsDataId}.json`
      );

      await fs.mkdir(path.dirname(filePath), { recursive: true });
      await fs.writeFile(filePath, JSON.stringify(data, null, 2));

      console.log(`✅ ${dataType} 保存完了: ${filePath}`);
    } catch (error) {
      console.error(`❌ ${dataType} エラー:`, error);
    }
  }
}

async function main() {
  const args = process.argv.slice(2);

  if (args.length < 2) {
    console.error('使用方法: npm run mock:download -- <statsDataId> <areaLevel> [dataTypes...]');
    console.error('例: npm run mock:download -- 0000010101 prefecture metainfo statsdata');
    process.exit(1);
  }

  const [statsDataId, areaLevel, ...dataTypes] = args;

  if (!['prefecture', 'municipality'].includes(areaLevel)) {
    console.error('areaLevelは "prefecture" または "municipality" を指定してください');
    process.exit(1);
  }

  const validDataTypes = dataTypes.length > 0
    ? dataTypes.filter(dt => ['metainfo', 'statsdata'].includes(dt))
    : ['metainfo', 'statsdata'];

  await downloadMockData({
    statsDataId,
    areaLevel: areaLevel as 'prefecture' | 'municipality',
    dataTypes: validDataTypes as ('metainfo' | 'statsdata')[],
  });

  console.log('🎉 ダウンロードが完了しました');
}

main().catch(console.error);
```

### 2. データの検証

```typescript
// src/lib/mock-data/validator.ts
import Ajv from "ajv";

const ajv = new Ajv();

const metainfoSchema = {
  type: "object",
  required: ["GET_META_INFO"],
  properties: {
    GET_META_INFO: {
      type: "object",
      required: ["RESULT", "METADATA_INF"],
      properties: {
        RESULT: { type: "object" },
        METADATA_INF: { type: "object" },
      },
    },
  },
};

const statsdataSchema = {
  type: "object",
  required: ["GET_STATS_DATA"],
  properties: {
    GET_STATS_DATA: {
      type: "object",
      required: ["RESULT", "STATISTICAL_DATA"],
      properties: {
        RESULT: { type: "object" },
        STATISTICAL_DATA: { type: "object" },
      },
    },
  },
};

export function validateMockData(
  data: any,
  type: "metainfo" | "statsdata"
): boolean {
  const schema = type === "metainfo" ? metainfoSchema : statsdataSchema;
  const validate = ajv.compile(schema);
  const valid = validate(data);

  if (!valid) {
    console.error("Validation errors:", validate.errors);
  }

  return valid;
}
```

## トラブルシューティング

### Q1: モックデータが見つからないエラー

```
Error: モックデータが見つかりません: data/mock/statsdata/prefecture/0000010101.json
```

**解決策**:

```bash
# 特定のデータをダウンロード
npm run mock:download -- 0000010101 prefecture

# フィクスチャを使用
npm run mock:generate-fixtures
```

### Q2: 都道府県と市区町村データの使い分けがわからない

**確認事項**:

1. 統計表 ID から地域レベルを判定
2. `lvArea`パラメータの設定
3. カタログファイルの確認

### Q3: パフォーマンステストでメモリ不足

**対策**:

1. データサイズを段階的に増加
2. メモリ使用量の監視
3. ガベージコレクションの実行

### Q4: Storybook でデータが読み込めない

**確認事項**:

1. `.storybook/main.ts`の webpack 設定
2. モックデータのパス設定
3. Storybook の再起動
