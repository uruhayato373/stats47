# 🌏 Next.js × e-Stat API × Cloudflare（D1 / R2）  
都道府県ランキング・ダッシュボード開発環境設計ガイド

---

## 🎯 プロジェクト概要

- **Next.js 15**: フロントエンドおよびAPIルートを統合
- **e-Stat API**: 都道府県データを取得
- **Cloudflare D1**: 統計データ・ランキングキャッシュ管理（SQLite互換）
- **Cloudflare R2**: 画像やJSONなどの静的アセット保存
- **Cloudflare Pages**: デプロイ環境

複数の開発環境（API接続あり／なし）を切り替えながら、デザイン、開発、ステージング、本番を安全に運用するためのベストプラクティスをまとめます。

---

## 🏗️ 環境構成

| 環境名 | 用途 | 特徴 |
|---------|------|------|
| **development** | ローカル開発 | e-Stat API接続（またはモック） |
| **mock** | デザイン検証 | API非接続、スタブデータ使用 |
| **staging** | ステージング | D1/R2接続あり、本番と同構成 |
| **production** | 本番 | 実運用データ、最適化ビルド |

## ⚙️ 設定コード例

### `/src/lib/config.ts`

```typescript
export const config = {
  env: process.env.NEXT_PUBLIC_ENV || 'development',
  useMock: process.env.NEXT_PUBLIC_USE_MOCK === 'true',
  
  estat: {
    baseUrl: process.env.ESTAT_API_BASE_URL || 'https://api.e-stat.go.jp/rest/3.0/app',
    apiKey: process.env.ESTAT_API_KEY,
  },
  
  cloudflare: {
    d1: process.env.CLOUDFLARE_D1_DATABASE_ID,
    r2: process.env.CLOUDFLARE_R2_BUCKET,
    accountId: process.env.CLOUDFLARE_ACCOUNT_ID,
    apiToken: process.env.CLOUDFLARE_API_TOKEN,
  },
  
  mock: {
    dataPath: process.env.MOCK_DATA_PATH || 'data/mock',
  },
} as const;

export type Config = typeof config;
```

### `/src/lib/api/estat.ts`

```typescript
import { config } from '@/lib/config';
import mockData from '@/data/mock/prefecture-stats.json';

export async function getPrefectureStats() {
  if (config.useMock) {
    console.log('Using mock data for development');
    return mockData;
  }

  const res = await fetch(
    `${config.estat.baseUrl}/getStatsData?appId=${config.estat.apiKey}&...`
  );
  
  if (!res.ok) {
    throw new Error('Failed to fetch e-Stat data');
  }
  
  return res.json();
}
```

## 🗺️ Cloudflare D1 設計

### 環境別構成

|環境|D1の種類|用途|
|---|---|---|
|local|ローカルSQLite (D1互換)|スキーマ開発・Seedデータ|
|staging|Cloudflare D1|機能テスト|
|production|Cloudflare D1|本番稼働|

### `wrangler.toml`

```toml
name = "stats47"
main = "src/worker.ts"
compatibility_date = "2024-01-15"

# ビルド設定
[build]
command = "npm run build"
cwd = "."

# デプロイ除外設定
[build.upload]
exclude = [
  "data/mock/**/*.json",
  "data/mock/metainfo/**",
  "data/mock/statsdata/**",
  "data/mock/statslist/**",
  "docs/**",
  "scripts/**",
  "tests/**",
  "**/__tests__/**",
  "**/*.test.ts",
  "**/*.test.tsx",
  "**/*.spec.ts",
  "**/*.spec.tsx",
  "vitest.config.ts",
  "**/__debug__/**",
]

# ローカル環境
[env.local]
NODE_ENV = "development"
ESTAT_API_KEY = "your_estat_api_key_here"

[[env.local.d1_databases]]
binding = "STATS47_DB"
database_name = "stats47"
database_id = "e6533698-d05a-475b-9f39-5558703feef7"
migrations_dir = "database/migrations"

# ステージング環境
[env.staging]
NODE_ENV = "production"

[[env.staging.d1_databases]]
binding = "STATS47_DB"
database_name = "stats47_staging"
database_id = "${STAGING_D1_DATABASE_ID}"
migrations_dir = "database/migrations"

# 本番環境
[env.production]
NODE_ENV = "production"

[[env.production.d1_databases]]
binding = "STATS47_DB"
database_name = "stats47_production"
database_id = "${PROD_D1_DATABASE_ID}"
migrations_dir = "database/migrations"
```

### DB接続例

```typescript
export const runtime = 'edge';

export async function getDB() {
  if (process.env.NEXT_PUBLIC_ENV === 'local') {
    return (await import('@cloudflare/workers-types')).env.STATS47_DB;
  } else {
    const response = await fetch(
      `https://api.cloudflare.com/client/v4/accounts/${process.env.CLOUDFLARE_ACCOUNT_ID}/d1/database/${process.env.CLOUDFLARE_D1_DATABASE_ID}/query`,
      { 
        headers: { 
          'Authorization': `Bearer ${process.env.CLOUDFLARE_API_TOKEN}` 
        } 
      }
    );
    return response;
  }
}
```

### ローカル操作コマンド

## 🧩 マイグレーション運用

|項目|内容|
|---|---|
|スキーマ管理|`database/schemas/` をGit管理|
|Seedデータ|`database/seeds/`|
|マイグレーション|`npx wrangler d1 migrations apply`|
|同期方向|`local → staging → production`|

> 本番環境では「自動migrate禁止」。  
> PRマージ後、CI/CDで明示的に実行。

### マイグレーション実行例

```bash
# ローカル環境
npx wrangler d1 migrations apply --env local

# ステージング環境
npx wrangler d1 migrations apply --env staging

# 本番環境
npx wrangler d1 migrations apply --env production
```

## 🪵 ブランチ戦略

|ブランチ|用途|環境|備考|
|---|---|---|---|
|**main**|本番運用|production|Cloudflare Pages本番デプロイ|
|**develop**|統合テスト|staging|ステージングデプロイ|
|**feature/***|機能開発|local|ローカルD1を使用|

### 開発フロー

1. `feature/estat-chart` で開発（ローカルD1 / .env.development）
    
2. `develop` へPR（staging D1に自動マイグレーション）
    
3. 確認後、`main` にマージ（production D1へ反映）

### ブランチ作成例

```bash
# 機能開発ブランチ
git checkout -b feature/estat-chart

# バグ修正ブランチ
git checkout -b fix/ranking-display

# ホットフィックスブランチ
git checkout -b hotfix/critical-bug
```

## 🔐 Secrets管理

|項目|管理方法|
|---|---|
|e-Stat APIキー|Cloudflare Secrets (`wrangler secret put`)|
|Cloudflare APIトークン|同上|
|環境判定|`NEXT_PUBLIC_ENV`|

> `.env.*` はローカルのみ、Secretsはクラウド専用。

### Secrets設定例

```bash
# ステージング環境のSecrets
wrangler secret put ESTAT_API_KEY --env staging
wrangler secret put CLOUDFLARE_API_TOKEN --env staging

# 本番環境のSecrets
wrangler secret put ESTAT_API_KEY --env production
wrangler secret put CLOUDFLARE_API_TOKEN --env production
```

## ✅ ベストプラクティスまとめ

|項目|推奨方針|
|---|---|
|D1の使い分け|local / staging / production で明確に分離|
|開発モード|`.env.*` と `NEXT_PUBLIC_ENV` で切替|
|スタブデータ|`NEXT_PUBLIC_USE_MOCK=true` でAPI非依存化|
|ブランチ戦略|`feature → develop → main`|
|データ同期|local → staging → production|
|機密情報|Cloudflare Secrets で一元管理|
|マイグレーション|CI/CD内で明示的に実行|
|デザイン確認|mock環境 or Storybookで実施|

## 🚀 推奨次ステップ

1. **GitHub Actions で `develop` / `main` ブランチに応じて Cloudflare Pages に自動デプロイ**
