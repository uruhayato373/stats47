# 🌏 Next.js × e-Stat API × Cloudflare（D1 / R2）  
都道府県ランキング・ダッシュボード開発環境設計ガイド

---

## 🎯 プロジェクト概要

- **Next.js 15**: フロントエンドおよびAPIルートを統合
- **e-Stat API**: 都道府県データを取得
- **Cloudflare D1**: 統計データ・ランキングキャッシュ管理（SQLite互換）
- **Cloudflare R2**: 画像やJSONなどの静的アセット保存
- **Cloudflare Pages**: デプロイ環境

複数の開発環境（API接続あり／なし）を切り替えながら、デザイン、開発、ステージング、本番を安全に運用するためのベストプラクティスをまとめます。

---

## 🏗️ 環境構成

| 環境名 | 用途 | 特徴 |
|---------|------|------|
| **development** | ローカル開発 | e-Stat API接続（またはモック）、ローカルD1使用 |
| **mock** | デザイン検証 | API非接続、DB非接続、モックデータ使用 |
| **staging** | ステージング | D1/R2接続あり、本番と同構成 |
| **production** | 本番 | 実運用データ、最適化ビルド |

## ⚙️ 設定コード例

### `/src/lib/config.ts`

```typescript
export const config = {
  env: process.env.NEXT_PUBLIC_ENV || 'development',
  useMock: process.env.NEXT_PUBLIC_USE_MOCK === 'true',
  
  estat: {
    baseUrl: process.env.ESTAT_API_BASE_URL || 'https://api.e-stat.go.jp/rest/3.0/app',
    apiKey: process.env.ESTAT_API_KEY,
  },
  
  cloudflare: {
    d1: process.env.CLOUDFLARE_D1_DATABASE_ID,
    r2: process.env.CLOUDFLARE_R2_BUCKET,
    accountId: process.env.CLOUDFLARE_ACCOUNT_ID,
    apiToken: process.env.CLOUDFLARE_API_TOKEN,
  },
  
  mock: {
    dataPath: process.env.MOCK_DATA_PATH || 'data/mock',
  },
} as const;

export type Config = typeof config;
```

### `/src/lib/api/estat.ts`

```typescript
import { config } from '@/lib/config';
import mockData from '@/data/mock/prefecture-stats.json';

export async function getPrefectureStats() {
  if (config.useMock) {
    console.log('Using mock data for development');
    return mockData;
  }

  const res = await fetch(
    `${config.estat.baseUrl}/getStatsData?appId=${config.estat.apiKey}&...`
  );
  
  if (!res.ok) {
    throw new Error('Failed to fetch e-Stat data');
  }
  
  return res.json();
}
```

### DB接続例

```typescript
export const runtime = 'edge';

export async function getDB() {
  if (process.env.NEXT_PUBLIC_ENV === 'local') {
    return (await import('@cloudflare/workers-types')).env.STATS47_DB;
  } else {
    const response = await fetch(
      `https://api.cloudflare.com/client/v4/accounts/${process.env.CLOUDFLARE_ACCOUNT_ID}/d1/database/${process.env.CLOUDFLARE_D1_DATABASE_ID}/query`,
      { 
        headers: { 
          'Authorization': `Bearer ${process.env.CLOUDFLARE_API_TOKEN}` 
        } 
      }
    );
    return response;
  }
}
```

## 🔄 データベース同期戦略

### スキーマの同期（マイグレーション）

#### ローカル → ステージング → 本番

```bash
# 1. ローカルでマイグレーションファイルを作成
npx wrangler d1 migrations create stats47 "add_new_column"

# 2. マイグレーションファイルを編集
# database/migrations/0001_add_new_column.sql

# 3. ローカルで適用・テスト
npx wrangler d1 migrations apply stats47 --local

# 4. ステージングに適用
npx wrangler d1 migrations apply stats47 --env staging

# 5. 本番に適用（慎重に！）
npx wrangler d1 migrations apply stats47 --env production
```

### データの同期（エクスポート/インポート）

#### 方法1: SQLダンプを使用

```bash
# ローカルD1からデータをエクスポート
npx wrangler d1 execute stats47 --local --command ".dump" > database/backups/local_dump.sql

# ステージングにインポート
npx wrangler d1 execute stats47 --env staging --file=database/backups/local_dump.sql

# 本番にインポート（非推奨・緊急時のみ）
npx wrangler d1 execute stats47 --env production --file=database/backups/local_dump.sql
```

#### 方法2: 特定テーブルのみエクスポート

```bash
# 特定テーブルをJSON形式でエクスポート
npx wrangler d1 execute stats47 --local --command "SELECT * FROM ranking_items;" --json > database/backups/ranking_items.json

# テーブルをSQL形式でエクスポート
npx wrangler d1 execute stats47 --local --command "SELECT sql FROM sqlite_master WHERE type='table' AND name='ranking_items';"
```

### データ同期の推奨フロー

#### 1. **開発環境（local）**
```
e-Stat API → ローカルD1 → 開発・テスト
```
- e-Stat APIからデータ取得
- ローカルD1に保存
- 開発・テスト実施

#### 2. **ステージング環境（staging）**
```
e-Stat API → Cloudflare D1 (staging) → 統合テスト
```
- マイグレーションのみ適用（スキーマ同期）
- e-Stat APIから直接データ取得
- ローカルデータは同期しない

#### 3. **本番環境（production）**
```
e-Stat API → Cloudflare D1 (production) → 本番運用
```
- マイグレーション適用
- e-Stat APIから直接データ取得
- ステージングデータは同期しない

### データベース管理スクリプト

```bash
# ローカルDBのダンプ
npm run db:dump:local

# ステージングDBのダンプ
npm run db:dump:staging

# マイグレーション適用
npm run db:migrations:local      # ローカル
npm run db:migrations:staging    # ステージング
npm run db:migrations:production # 本番
```

### 重要な注意事項

⚠️ **データ同期の原則**
- **スキーマ（マイグレーション）**: `local → staging → production` の順で同期
- **データ**: 各環境で独立して管理
- **ローカルデータを本番に同期しない**: 各環境でe-Stat APIから取得

✅ **推奨される方法**
- 各環境でe-Stat APIから直接データ取得
- データベースはキャッシュとして使用
- マイグレーション（スキーマ変更）のみを同期
- データのバックアップは定期的に実施

❌ **避けるべき方法**
- ローカルの開発データを本番にコピー
- 本番データをローカルで直接編集
- マイグレーションなしでのスキーマ変更

## 🧠 モックデータ構成

```
data/mock/
├── metainfo/                    # メタ情報モックデータ
│   ├── prefecture/              # 都道府県メタ情報
│   │   └── 0000010101.json      # 都道府県別人口データ
│   └── municipality/            # 市区町村メタ情報
│       └── 0000020201.json      # 市区町村別人口データ
├── statsdata/                   # 統計データモックデータ
│   ├── prefecture/              # 都道府県統計データ
│   │   └── 0000010101_A1101.json # 都道府県別人口統計
│   └── municipality/            # 市区町村統計データ
│       └── 0000020201_A1101.json # 市区町村別人口統計
├── statslist/                   # 統計表リストモックデータ（空）
├── catalog.json                 # データカタログ
└── README.md                    # モックデータ管理ガイド
```

`NEXT_PUBLIC_USE_MOCK=true` の場合のみ利用。  
Storybookやデザイン検証でも共通使用可能。

> **用語の統一**: 本ドキュメントでは「モックデータ」と「スタブデータ」は同じ意味で使用しています。

### モックデータの特徴

- **e-Stat APIの実際のレスポンス形式**を再現
- **都道府県・市区町村別**の階層構造
- **メタ情報と統計データ**を分離管理
- **データベース接続不要**（ローカルJSONファイルのみ）
- **Git管理対象**（複数端末での共有）
- **デプロイ時は除外**（Cloudflare Pages）

### データフロー比較

| 環境 | データソース | データベース | 用途 |
|------|-------------|-------------|------|
| **development** | e-Stat API | ローカルD1 | 開発・テスト |
| **mock** | ローカルJSON | なし | デザイン検証 |
| **staging** | e-Stat API | Cloudflare D1 | 統合テスト |
| **production** | e-Stat API | Cloudflare D1 | 本番運用 |

## 🧭 CI/CD・環境切替例

### cross-envを使用した環境変数管理

本プロジェクトでは`cross-env`パッケージを使用して、クロスプラットフォーム対応の環境変数設定を実現しています。

#### cross-envを使用する理由

- **クロスプラットフォーム対応**: Windows、macOS、Linuxで同じコマンドが動作
- **環境変数の統一**: OSごとの環境変数設定の違いを吸収
- **Next.js対応**: Next.jsの`--env-file`オプション非対応を解決
- **シンプルな記述**: コマンドライン上で直接環境変数を設定

### 開発環境

```bash
# 開発（API接続あり）
npm run dev:api
# → cross-env NODE_ENV=development next dev --turbopack

# デザイン確認（モックデータ）
npm run dev:mock
# → cross-env NODE_ENV=development NEXT_PUBLIC_USE_MOCK=true next dev --turbopack

# ステージング環境での開発
npm run dev:staging
# → cross-env NODE_ENV=production NEXT_PUBLIC_ENV=staging next dev --turbopack

# 本番環境での開発
npm run dev:production
# → cross-env NODE_ENV=production NEXT_PUBLIC_ENV=production next dev --turbopack
```

### デプロイ

```bash
# ステージングデプロイ
npx wrangler pages deploy --env staging

# 本番デプロイ
npx wrangler pages deploy --env production
```
