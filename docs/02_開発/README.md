## 開発フロー

### 1. 新機能開発
1. [コーディング規約](./01_コーディング規約.md)を確認
2. [コンポーネントガイド](./02_コンポーネントガイド.md)に従って設計
3. [テストガイド](./04_テストガイド.md)に従ってテスト作成
4. [Storybookガイド](./05_Storybookガイド.md)に従ってストーリー作成

### 2. スタイリング
1. [スタイリングガイド](./03_スタイリングガイド.md)を参照
2. Tailwind CSSのクラス命名規則に従う
3. レスポンシブデザインを考慮

### 3. デプロイ前
1. [デプロイメントガイド](./06_デプロイメントガイド.md)を確認
2. [環境変数設定ガイド](./07_環境変数設定ガイド.md)で設定を確認
3. [パフォーマンス最適化ガイド](./08_パフォーマンス最適化ガイド.md)で最適化を実施

## 重要な原則

### 🎯 一貫性
- プロジェクト全体で統一されたコーディング規約
- コンポーネント設計パターンの統一
- 命名規則の一貫性

### 🧪 品質
- テストカバレッジの維持
- Storybookでの視覚的テスト
- アクセシビリティの考慮

### ⚡ パフォーマンス
- バンドルサイズの最適化
- レンダリングパフォーマンスの向上
- キャッシュ戦略の活用

### 🔒 セキュリティ
- 環境変数の適切な管理
- API キーの保護
- 入力値の検証
