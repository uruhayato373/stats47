## 結論

**数量的にはほぼ同等**（stats47: 1,574 vs todo-ran: ~1,500+）だが、**カバー領域が大きく異なる**。

- stats47 は **e-Stat（社会生活統計指標・家計調査）に強く**、家計消費・行政財政・人口統計で圧倒的
- todo-ran は **独自集計データに強く**、店舗分布・スポーツ・芸能人出身地・選挙など e-Stat にないデータが充実
- **最も大きなギャップは「店舗分布」**（~200項目）で、e-Stat からは取得不可能
- **e-Stat API で埋められるギャップ**（農産物品種別・漁獲量・車両保有・趣味行動等）は **約150項目**あり、追加投入の余地が大きい

### Phase 3: 差別化戦略の検討（工数: 大 / 要判断）

todo-ran.com が強く、e-Stat API では取得できない領域。追加するかどうかは戦略判断が必要。

#### 3-1. 店舗分布データ（~200件）— 要独自データソース

**選択肢**:

| 方針 | メリット | デメリット |
|---|---|---|
| **A. 追加しない** | 工数ゼロ。e-Stat統計の深掘りに集中 | todo-ranとの最大差分が残る |
| **B. 経済センサスベースで追加** | e-Stat APIで取得可能。「飲食店数」「小売店数」等の業種大分類は既にある | チェーン店別の粒度は出せない |
| **C. 外部APIで収集** | Google Places API、ホットペッパー API 等でチェーン店舗数を集計 | API利用料・スクレイピング規約・データ更新コストが大きい |

**推奨: B（経済センサスベース）を Phase 2 と並行して追加**。業種小分類（日本料理店、中華料理店、ラーメン店、理容室、美容室、書店等）は経済センサスから取得可能で、todo-ran.com の「その他店舗」と部分的に重なる。

#### 3-2. スポーツ・出身地データ（~60件）— 要独自集計

- 甲子園勝利数、プロ野球選手出身地、Jリーガー出身地等は **公式統計には存在しない**
- Wikipedia やスポーツ参考書サイトからの手動集計が必要
- **推奨: 当面は見送り**。ニッチだが更新・メンテナンスコストが高い

#### 3-3. 選挙得票率の詳細（~20件）— 要個別統計

- Phase 1 で投票率は追加可能だが、**政党別得票率**は社会・人口統計体系には含まれない
- 総務省の選挙関連データ（e-Stat に一部あり）から取得する必要がある
- **推奨: Phase 2 完了後に検討**
