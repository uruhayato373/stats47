# YouTube → サイト誘導戦略

## 現状（2026-03-10）

| 項目 | 値 |
|---|---|
| 登録者数 | 18（+7 増加傾向） |
| 総再生回数 | 35,142 |
| 動画数 | 58（Shorts 42 + 通常 16） |
| Shorts 総再生 | 32,283（平均 769/本） |
| 通常動画 総再生 | 11,972（平均 748/本） |
| 説明欄にサイトリンクあり | 25/58 本（43%） |

## 課題

1. **33 本の動画にサイトリンクがない** — 流入機会の損失
2. **Shorts の説明欄はほぼ見られない** — 2 タップ必要で視認性が極めて低い
3. **通常動画の説明欄が未統一** — テンプレートなし、リンクの書き方がバラバラ
4. **YouTube → サイトの流入導線が弱い** — チャンネルページのリンク設定も未最適化

## プラットフォーム別リンク制約

| 手段 | 通常動画 | Shorts |
|---|---|---|
| 説明欄リンク | **クリック可能。効果大** | 書けるがほぼ見られない（`...` → 説明 の 2 タップ） |
| 固定コメント | クリック可能 | **コメント内 URL はクリッカブルにならない** |
| カード / 終了画面 | 使える | **使えない** |
| チャンネルページリンク | プロフィールに表示 | プロフィールに表示 |
| 動画内テロップ | 有効 | 有効（ただし URL を打つ人は少ない） |

**結論**: Shorts でのサイト誘導は構造的に難しい。通常動画を誘導の主軸にする。

## 戦略: Shorts → 登録 → 通常動画 → サイト

```
Shorts（認知・登録者獲得）
  ↓ チャンネル登録
通常動画（再生時間 + サイト誘導）
  ↓ 説明欄リンク / カード / 終了画面
stats47.jp（詳細データ閲覧 + 回遊）
```

Shorts の役割は「登録者を増やすこと」に絞り、サイト誘導は通常動画に集中する。

### Shorts での最低限の誘導

- 説明欄の先頭に `📊 stats47.jp` と書く（ブランド認知のみ。クリック期待しない）
- 動画末尾に 1〜2 秒「詳しくは stats47.jp」テロップを表示
- チャンネルページのリンクに stats47.jp を設定

### 通常動画での本格誘導

- 説明欄テンプレートでリンクを統一
- カード（動画中ポップアップ）でサイトリンク ※要件: チャンネルが YPP に参加済み or 1,000 登録者以上
- 終了画面で関連動画 + サイトリンク ※同上
- 動画内テロップ「データの詳細は説明欄のリンクから」

```
{動画の説明文 2〜3行}

https://stats47.jp/ranking/{ranking-key}?utm_source=youtube&utm_medium=social&utm_campaign={ranking-key}&utm_content=long-video

━━━━━━━━━━━━━━━━━
📊 統計で見る都道府県 | stats47
🌐 https://stats47.jp?utm_source=youtube&utm_medium=social&utm_campaign=channel
━━━━━━━━━━━━━━━━━

#都道府県ランキング #統計データ #stats47
```

**ポイント**:
- 個別ランキング URL を「もっと見る」の折りたたみ前（先頭 2〜3 行以内）に配置
- UTM パラメータで流入元を計測（`utm_content=long-video` で Shorts と区別）
- チャンネル共通リンクは末尾に固定

## 既存動画の説明欄一括更新

### 問題

YouTube Data API の `videos.update` で説明欄を一括更新できるが、**サービスアカウント認証では不可**。チャンネルオーナーの OAuth 2.0 認証が必要。

### OAuth 2.0 セットアップ手順

1. **Google Cloud Console** で OAuth 2.0 クライアント ID を作成
   - アプリケーションの種類: 「デスクトップ アプリ」
   - プロジェクト: 既存の `stats47` プロジェクトを使用
   - OAuth 同意画面: 外部（テストユーザーに自分を追加）

2. **クライアントシークレット JSON をダウンロード**
   - `client_secret_*.json` としてリポジトリルートに配置
   - `.gitignore` に追加

3. **初回認証フロー**（ブラウザが開く、1 回だけ）
   ```javascript
   const { google } = require("googleapis");
   const { authenticate } = require("@google-cloud/local-auth");

   const auth = await authenticate({
     keyfilePath: "client_secret_*.json",
     scopes: ["https://www.googleapis.com/auth/youtube"],
   });
   // トークンを保存 → 以降はリフレッシュトークンで自動更新
   ```

4. **一括更新スキル `/update-youtube-descriptions`** を作成
   - 全動画を取得
   - 説明欄がテンプレートに沿っていない動画を検出
   - ranking key を動画タイトルから推定 or 手動マッピング
   - `videos.update` で説明欄を上書き

### 必要なスコープ

| スコープ | 用途 |
|---|---|
| `youtube.readonly` | 動画一覧・統計取得（現在のサービスアカウントで対応済み） |
| `youtube` | 動画メタデータの更新（OAuth 2.0 必須） |
| `youtube.upload` | 動画アップロード（将来的に自動投稿する場合） |

### 実装優先度

| 優先度 | タスク | 効果 |
|---|---|---|
| **P0** | チャンネルページのリンクに stats47.jp を設定 | 全動画共通。YouTube Studio で即時設定可 |
| **P0** | 今後の通常動画は投稿時にテンプレートを適用 | 追加工数ほぼゼロ。`/post-youtube` スキルに組み込む |
| **P1** | OAuth 2.0 セットアップ + 既存 16 本の通常動画の説明欄更新 | 流入導線の整備 |
| **P2** | 既存 42 本の Shorts 説明欄更新 | 効果は限定的だが統一感 |
| **P2** | `/update-youtube-descriptions` スキル作成 | 今後の運用効率化 |

## 通常動画戦略との接続

### 現在の通常動画（16 本）

- 主に Bar Chart Race の横長版やランキング解説動画
- 再生数トップは「交通事故」2,117 回、「高齢化率」1,394 回
- **説明欄にリンクがある動画の方が再生数が高い傾向**（因果か相関かは不明）

### 通常動画を増やす意義

1. **サイト誘導の唯一の有効チャネル** — Shorts ではリンクが機能しない
2. **YPP 再生時間要件** — 4,000 時間/12ヶ月の達成には通常動画が必須
3. **広告 RPM** — 通常動画は Shorts の 50〜100 倍の収益単価
4. **カード・終了画面** — YPP 加入後はこれが最強のサイト誘導手段

### Shorts → 通常動画の導線

- Shorts のコメントや説明欄で「詳しい解説動画はこちら」と通常動画に誘導
- 通常動画の説明欄でサイトに誘導
- **Shorts で話題になったテーマを通常動画で深掘り**するのが最も自然な流れ

例:
```
Shorts: 「合計特殊出生率ランキング 43年の変遷」（2,105 再生）
  ↓ 関連動画として制作
通常動画: 「なぜ沖縄の出生率は高いのか？ 47都道府県データ分析」
  ↓ 説明欄
stats47.jp/ranking/total-fertility-rate
```

### 通常動画の制作方針（既存企画書との整合）

- `youtube-通常動画企画.md`: 左カード+地図、右タイルマップの 2 カラム構成
- `youtube-カウントダウン動画企画.md`: 47→1 位のカウントダウン形式
- いずれも **説明欄テンプレートを必須適用**
- 動画内にも「詳しくは説明欄のリンクから」テロップを挿入
