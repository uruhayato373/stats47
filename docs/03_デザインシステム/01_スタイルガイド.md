# デザインガイド

## 1. 概要

このドキュメントは、stats47 プロジェクトにおけるデザインシステムの統合ガイドです。一貫性のある UI/UX を提供し、アクセシビリティを確保することを目的としています。

本ガイドでは以下の内容を定義します：

- 配色システム（カラーパレット、テーマ、コンポーネント別配色）
- フォント色システム（階層構造、使用指針、実装方法）
- アクセシビリティ要件（コントラスト比、色覚多様性への配慮）

---

### 3.2 使用指針

#### テキスト階層の選択基準

**Primary (`styles.text.primary`)**

- ページタイトル
- セクション見出し (h1, h2, h3)
- 重要な情報や CTA
- ナビゲーションの現在位置

**Secondary (`styles.text.secondary`)**

- 本文テキスト
- 段落内容
- ボタンラベル
- フォームラベル

**Tertiary (`styles.text.tertiary`)**

- 補助説明文
- キャプション
- メタ情報 (日付、作成者など)
- プレースホルダーテキスト

**Muted (`styles.text.muted`)**

- 非活性状態のテキスト
- ヒント文字
- フッター情報
- 無効化された要素

## 4. アクセシビリティ

### 4.1 コントラスト比基準

#### WCAG 2.1 AA 要件

- **通常テキスト** (14pt 未満): 4.5:1 以上
- **大きなテキスト** (18pt 以上 / 14pt 太字以上): 3:1 以上
- **UI 要素**: 3:1 以上

### 4.2 本プロジェクトの実装

## 5. レスポンシブデザイン実装ガイド

Next.js 15 と Tailwind CSS を使用したブログサイトでレスポンシブデザインを効果的に実装するための包括的なガイドです。

### 5.1 レスポンシブデザインの基本原則

レスポンシブデザインを検討する際には、以下の原則を念頭に置くことが重要です：

1. **モバイルファースト**: 小さい画面サイズから設計を始め、徐々に大きな画面に拡張していく
2. **流動的なレイアウト**: 固定幅ではなく、パーセンテージやビューポート単位を使用
3. **柔軟なメディア**: 画像やビデオが画面サイズに応じて適切に表示されるようにする
4. **ブレイクポイント**: 異なる画面サイズに対応するためのメディアクエリを適切に設定
5. **コンテンツの優先順位付け**: 小さな画面では最も重要なコンテンツを優先表示

### 5.2 Tailwind CSS のレスポンシブユーティリティ

Tailwind CSS は強力なレスポンシブユーティリティを提供しており、Next.js 15 との組み合わせは非常に効果的です。

#### ブレイクポイント

Tailwind CSS のデフォルトブレイクポイント：

```
sm: 640px（小型タブレット、大型モバイル）
md: 768px（タブレット）
lg: 1024px（ノートPC、小型デスクトップ）
xl: 1280px（デスクトップ）
2xl: 1536px（大型デスクトップ）
```

これらのブレイクポイントは必要に応じてカスタマイズできます：

```javascript
// tailwind.config.js
module.exports = {
  theme: {
    screens: {
      mobile: "375px",
      tablet: "640px",
      laptop: "1024px",
      desktop: "1280px",
    },
  },
};
```

#### レスポンシブクラスの使用方法

Tailwind CSS では、接頭辞を使用して特定のブレイクポイントでのみ適用されるスタイルを指定できます：

```html
<div class="w-full md:w-1/2 lg:w-1/3 xl:w-1/4">
  <!-- この要素は小さい画面では幅100%、mdサイズでは50%、lgサイズでは33.33%、xlサイズでは25%になります -->
</div>
```

```tsx
"use client";

import { useEffect, useState } from "react";
import MobileView from "./MobileView";
import DesktopView from "./DesktopView";

export default function ResponsiveComponent() {
  const [isMobile, setIsMobile] = useState(false);

  useEffect(() => {
    const checkIsMobile = () => {
      setIsMobile(window.innerWidth < 768);
    };

    checkIsMobile();
    window.addEventListener("resize", checkIsMobile);

    return () => {
      window.removeEventListener("resize", checkIsMobile);
    };
  }, []);

  return isMobile ? <MobileView /> : <DesktopView />;
}
```

4. **スクリーンリーダー対応**：適切な ARIA ラベルと意味のある HTML 構造を使用

## 6. Alert コンポーネントのスタイリング

### 6.1 概要

Alert コンポーネントは、成功・エラー・情報・警告などのメッセージを統一されたスタイルで表示するための共通コンポーネントです。アプリケーション全体で一貫したメッセージ表示を実現します。

### 6.2 基本的なスタイリング

#### 6.2.1 共通スタイル

```tsx
// 基本構造
<div className="p-4 border rounded-lg">
  <div className="flex items-start gap-3">
    {/* アイコン */}
    <div className="w-4 h-4 flex-shrink-0">{/* アイコンコンテンツ */}</div>
    {/* メッセージ */}
    <div className="flex-1">
      <p className="text-sm font-medium">{message}</p>
    </div>
    {/* 閉じるボタン（オプション） */}
  </div>
</div>
```

#### 6.2.2 タイプ別の色設定

```tsx
// Success (成功)
const successClasses =
  "bg-green-50 border-green-200 text-green-800 dark:bg-green-900/20 dark:border-green-800 dark:text-green-300";

// Error (エラー)
const errorClasses =
  "bg-red-50 border-red-200 text-red-800 dark:bg-red-900/20 dark:border-red-800 dark:text-red-300";

// Info (情報)
const infoClasses =
  "bg-blue-50 border-blue-200 text-blue-800 dark:bg-blue-900/20 dark:border-blue-800 dark:text-blue-300";

// Warning (警告)
const warningClasses =
  "bg-yellow-50 border-yellow-200 text-yellow-800 dark:bg-yellow-900/20 dark:border-yellow-800 dark:text-yellow-300";
```

### 6.3 使用例

#### 6.3.1 基本的な使用

```tsx
import { Alert } from "@/components/atoms/Alert";

// 成功メッセージ
<Alert type="success" message="保存が完了しました" />

// エラーメッセージ
<Alert type="error" message="エラーが発生しました" />

// 情報メッセージ
<Alert type="info" message="新しい機能が利用可能です" />

// 警告メッセージ
<Alert type="warning" message="この操作は取り消せません" />
```

#### 6.3.2 カスタマイズ例

```tsx
// アイコンなし
<Alert type="success" message="メッセージ" showIcon={false} />

// カスタムアイコン
<Alert type="success" message="メッセージ" icon={Heart} />

// 閉じるボタン付き
<Alert
  type="info"
  message="メッセージ"
  onDismiss={() => setShow(false)}
/>

// カスタムスタイル
<Alert
  type="error"
  message="メッセージ"
  className="border-2 border-dashed mb-4"
/>
```

### 6.4 レスポンシブ対応

Alert コンポーネントは基本的にレスポンシブ対応されていますが、必要に応じてカスタムクラスで調整できます：

```tsx
// モバイルで小さく、デスクトップで大きく
<Alert
  type="info"
  message="メッセージ"
  className="p-2 md:p-4 text-xs md:text-sm"
/>

// モバイルで全幅、デスクトップで制限幅
<Alert
  type="warning"
  message="メッセージ"
  className="w-full md:max-w-md"
/>
```

### 6.5 アクセシビリティ

Alert コンポーネントは以下のアクセシビリティ機能を提供します：

```tsx
// ARIA属性
<div role="alert" aria-live="polite">
  {/* コンテンツ */}
</div>

// 閉じるボタンのアクセシビリティ
<button aria-label="アラートを閉じる">
  <X className="w-4 h-4" />
</button>
```

### 6.6 ダークモード対応

すべての Alert タイプでダークモードに対応：

```tsx
// ライトモード
<div className="bg-green-50 border-green-200 text-green-800">
  // ダークモード
  <div className="dark:bg-green-900/20 dark:border-green-800 dark:text-green-300">
```

### 6.7 ベストプラクティス

1. **適切なタイプの選択**: メッセージの内容に応じて適切なタイプを選択
2. **簡潔なメッセージ**: ユーザーが理解しやすい簡潔なメッセージを作成
3. **適切な配置**: 重要なメッセージは目立つ場所に配置
4. **一貫性の維持**: アプリケーション全体で統一されたスタイルを使用
5. **アクセシビリティの考慮**: スクリーンリーダーユーザーにも配慮

### 6.8 トラブルシューティング

#### よくある問題

1. **メッセージが長すぎる場合**

   ```tsx
   // 解決策: メッセージを短くするか、複数行に対応
   <Alert type="info" message="短いメッセージ" />
   ```

2. **色が見づらい場合**

   ```tsx
   // 解決策: カスタムクラスで色を調整
   <Alert
     type="error"
     message="メッセージ"
     className="bg-red-100 border-red-300 text-red-900"
   />
   ```

3. **アイコンが表示されない場合**
   ```tsx
   // 解決策: showIconプロパティを確認
   <Alert type="success" message="メッセージ" showIcon={true} />
   ```

詳細な使用方法については、[Alert コンポーネントドキュメント](./components/atoms/alert.md)を参照してください。

## 5. タイポグラフィ

プロジェクトでは**Geist**フォントファミリーを使用して、一貫性のあるタイポグラフィシステムを提供しています。

### 5.1 フォントファミリー

**メインフォント:**

```css
font-family: var(--font-geist-sans), "Hiragino Sans",
  "Hiragino Kaku Gothic ProN", "Yu Gothic", "YuGothic", "Meiryo", sans-serif;
```

**モノスペースフォント:**

```css
font-family: var(--font-geist-mono), "SF Mono", "Monaco", "Inconsolata",
  "Roboto Mono", monospace;
```

### 5.2 フォントサイズ階層

```tsx
// キャプション、補助テキスト
<span className="text-xs">12px - キャプション</span>

// 小さなボタン、ラベル
<span className="text-sm">14px - ラベル</span>

// 本文テキスト（デフォルト）
<span className="text-base">16px - 本文</span>

// 強調テキスト
<span className="text-lg">18px - 強調</span>

// 小見出し
<span className="text-xl">20px - 小見出し</span>

// サブヘッディング
<span className="text-2xl">24px - サブヘッディング</span>

// ページタイトル
<span className="text-3xl">30px - ページタイトル</span>

// 大見出し
<span className="text-4xl">36px - 大見出し</span>
```

### 5.3 見出しスタイル

**H1 - ページタイトル:**

```tsx
<h1 className="text-3xl font-bold leading-tight text-gray-900 dark:text-neutral-100">
  ページタイトル
</h1>
```

**H2 - セクションタイトル:**

```tsx
<h2 className="text-2xl font-semibold leading-snug text-gray-800 dark:text-neutral-200">
  セクションタイトル
</h2>
```

**H3 - サブセクション:**

```tsx
<h3 className="text-xl font-medium leading-snug text-gray-700 dark:text-neutral-300">
  サブセクション
</h3>
```

### 5.4 本文テキスト

**基本の段落:**

```tsx
<p className="text-base leading-normal text-gray-700 dark:text-neutral-300">
  これは基本的な段落テキストです。読みやすさを重視した行間と文字サイズを使用しています。
</p>
```

**読みやすい段落:**

```tsx
<p className="text-base leading-relaxed text-gray-700 dark:text-neutral-300">
  長文の場合は、leading-relaxed を使用して読みやすさを向上させます。
</p>
```

### 5.5 特殊用途のテキスト

**インラインコード:**

```tsx
<code className="font-mono text-sm bg-gray-100 dark:bg-neutral-800 px-2 py-1 rounded">
  console.log()
</code>
```

**数値表示:**

```tsx
<div className="text-3xl font-bold text-gray-900 dark:text-neutral-100">
  126,000,000
</div>
```

**キャプション:**

```tsx
<figcaption className="text-sm text-gray-500 dark:text-neutral-500 text-center mt-2">
  図表の説明文
</figcaption>
```

### 5.6 アクセシビリティ

**最小フォントサイズ:**

- 本文: 16px (`text-base`) 以上
- キャプション: 12px (`text-xs`) 以上
- UI 要素: 14px (`text-sm`) 以上

**コントラスト比:**

- 通常テキスト: 4.5:1 以上のコントラスト比
- 大きなテキスト: 3:1 以上のコントラスト比

詳細な使用方法については、[タイポグラフィシステムガイド](./03_タイポグラフィシステム.md)を参照してください。
