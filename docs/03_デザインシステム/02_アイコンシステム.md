# アイコンシステムガイド

## 1. 概要

### 1.1 Lucide React について

stats47 プロジェクトでは、**Lucide React**をアイコンライブラリとして使用しています。Lucide React は、Feather Icons の後継として開発されたモダンなアイコンライブラリです。

**主な特徴:**

- **一貫性のあるデザイン**: 統一されたストローク幅とスタイル
- **TypeScript 完全対応**: 型安全性を提供
- **Tree-shaking 最適化**: 必要なアイコンのみをバンドル
- **アクティブなメンテナンス**: 定期的なアップデートと新機能追加
- **MIT ライセンス**: 商用利用可能

### 1.2 なぜ Lucide React を使うのか

**技術的な利点:**

- **パフォーマンス**: Tree-shaking によりバンドルサイズを最小化
- **型安全性**: TypeScript の型定義が完備
- **カスタマイズ性**: SVG ベースで色やサイズを柔軟に変更可能
- **アクセシビリティ**: ARIA 属性との組み合わせが容易

**デザイン的な利点:**

- **視覚的一貫性**: 統一されたデザイン言語
- **スケーラビリティ**: ベクター形式で任意のサイズに対応
- **モダンな外観**: 現代的で洗練されたデザイン

### 1.3 インストールとセットアップ

```bash
npm install lucide-react
```

**基本的なインポート:**

```tsx
import { Search, RefreshCw, Download } from "lucide-react";
```

## 2. 基本的な使用方法

### 2.1 インポートと使用

**推奨されるインポート方法:**

```tsx
// ✅ 良い例: 必要なアイコンのみインポート
import { Search, RefreshCw, Download } from "lucide-react";

// ❌ 悪い例: 全てをインポート（バンドルサイズが大きくなる）
import * as Icons from "lucide-react";
```

**基本的な使用:**

```tsx
import { Search } from "lucide-react";

function SearchButton() {
  return (
    <button>
      <Search className="w-4 h-4" />
      検索
    </button>
  );
}
```

### 2.2 サイズのカスタマイズ

**標準サイズ:**

```tsx
// 小: 16px
<Search className="w-4 h-4" />

// 中: 20px (デフォルト)
<Search className="w-5 h-5" />

// 大: 24px
<Search className="w-6 h-6" />

// 特大: 32px
<Search className="w-8 h-8" />
```

**カスタムサイズ:**

```tsx
// 任意のサイズ
<Search className="w-12 h-12" />
<Search size={48} />
```

### 2.3 色の適用

**テーマ対応の色使用:**

```tsx
// Primary色
<Search className="w-5 h-5 text-indigo-600 dark:text-indigo-400" />

// Secondary色
<Search className="w-5 h-5 text-gray-600 dark:text-neutral-400" />

// Success色
<Search className="w-5 h-5 text-green-600 dark:text-green-400" />

// Error色
<Search className="w-5 h-5 text-red-600 dark:text-red-400" />

// Warning色
<Search className="w-5 h-5 text-yellow-600 dark:text-yellow-400" />
```

### 2.4 ストロークの調整

**ストローク幅の変更:**

```tsx
// 細いストローク
<Search strokeWidth={1} />

// 標準ストローク（デフォルト）
<Search strokeWidth={2} />

// 太いストローク
<Search strokeWidth={3} />
```

## 4. デザイン原則

### 4.1 アイコン選択の基準

**意味の明確性:**

- ユーザーが直感的に理解できるアイコンを選択
- 文化的な違いを考慮
- 複数の意味を持つアイコンは避ける

**視覚的一貫性:**

- 同じ機能には同じアイコンを使用
- サイズとスタイルを統一
- 色の使用を一貫させる

**アクセシビリティ:**

- 十分なコントラスト比を確保
- 適切なサイズ（最小 16px）
- 意味のあるアイコンにはラベルを提供

## 5. アクセシビリティ

### 5.1 ARIA 属性

**装飾的なアイコン:**

```tsx
// 装飾目的のみのアイコン
<Search className="w-4 h-4" aria-hidden="true" />
```

**意味のあるアイコン:**

```tsx
// 情報を伝えるアイコン
<AlertCircle className="w-4 h-4" role="img" aria-label="エラーが発生しました" />
```

**ボタン内のアイコン:**

```tsx
// ボタンにaria-labelを設定
<button aria-label="検索">
  <Search className="w-4 h-4" aria-hidden="true" />
</button>

// または、アイコンとテキストの両方がある場合
<button>
  <Search className="w-4 h-4" aria-hidden="true" />
  検索
</button>
```

### 5.2 スクリーンリーダー対応

## 6. ベストプラクティス

### 6.1 パフォーマンス最適化

**Tree-shaking の活用:**

```tsx
// ✅ 良い例: 必要なアイコンのみインポート
import { Search, RefreshCw, Download } from "lucide-react";

// ❌ 悪い例: 全てをインポート
import * as Icons from "lucide-react";
```

**動的インポート:**

```tsx
// 大量のアイコンを使用する場合
const IconComponent = lazy(() =>
  import("lucide-react").then((module) => ({
    default: module[iconName],
  }))
);
```

### 6.2 再利用可能なアイコンコンポーネント

**カスタムアイコンコンポーネント:**

```tsx
// src/components/atoms/Icon/Icon.tsx
interface IconProps {
  name: string;
  size?: "sm" | "md" | "lg" | "xl";
  color?: "primary" | "secondary" | "success" | "error" | "warning";
  className?: string;
}

const sizeMap = {
  sm: "w-4 h-4",
  md: "w-5 h-5",
  lg: "w-6 h-6",
  xl: "w-8 h-8",
};

const colorMap = {
  primary: "text-indigo-600 dark:text-indigo-400",
  secondary: "text-gray-600 dark:text-neutral-400",
  success: "text-green-600 dark:text-green-400",
  error: "text-red-600 dark:text-red-400",
  warning: "text-yellow-600 dark:text-yellow-400",
};

export const Icon: React.FC<IconProps> = ({
  name,
  size = "md",
  color = "secondary",
  className = "",
}) => {
  const IconComponent = iconMap[name as keyof typeof iconMap];

  if (!IconComponent) {
    console.warn(`Icon "${name}" not found`);
    return null;
  }

  return (
    <IconComponent
      className={`${sizeMap[size]} ${colorMap[color]} ${className}`}
    />
  );
};
```

### 6.3 命名規則

**ファイル名:**

- PascalCase: `CategoryIcon.tsx`
- 機能を表す名前: `ExportButton.tsx`

**コンポーネント名:**

- PascalCase: `CategoryIcon`
- 明確な役割: `LoadingButton`

**アイコン名:**

- Lucide React の公式名を使用: `RefreshCw`, `ChevronDown`
- 一貫性を保つ: 同じ機能には同じアイコン名

### 6.4 よくある間違い

**❌ 避けるべきパターン:**

```tsx
// 1. 不適切なサイズ
<Search className="w-3 h-3" /> // 小さすぎる

// 2. 一貫性のない色使用
<Search className="text-red-500" /> // カラーシステムにない色

// 3. アクセシビリティの欠如
<Search className="w-4 h-4" /> // aria-labelがない

// 4. パフォーマンスの問題
import * as Icons from "lucide-react"; // 全インポート

// 5. 意味の不明確なアイコン
<Heart className="w-4 h-4" /> // 用途が不明
```

**✅ 推奨パターン:**

```tsx
// 1. 適切なサイズ
<Search className="w-4 h-4" />

// 2. 一貫した色使用
<Search className="text-indigo-600 dark:text-indigo-400" />

// 3. アクセシビリティ対応
<button aria-label="検索">
  <Search className="w-4 h-4" aria-hidden="true" />
</button>

// 4. 効率的なインポート
import { Search } from "lucide-react";

// 5. 明確な意味
<Heart className="w-4 h-4" aria-label="お気に入り" />
```

## 8. トラブルシューティング

### 8.1 よくある問題

**問題 1: アイコンが表示されない**

**原因:** インポートエラーまたはアイコン名の間違い

**解決策:**

```tsx
// インポートを確認
import { Search } from "lucide-react";

// アイコン名を確認（大文字小文字を正確に）
<Search className="w-4 h-4" />;
```

**問題 2: アイコンのサイズが期待と異なる**

**原因:** CSS クラスの競合または Tailwind の設定

**解決策:**

```tsx
// 明示的にサイズを指定
<Search size={20} />

// または、より具体的なクラス
<Search className="!w-5 !h-5" />
```

**問題 3: ダークモードで色が正しく表示されない**

**原因:** ダークモード用のクラスが不足

**解決策:**

```tsx
// ダークモード対応の色クラスを使用
<Search className="text-gray-600 dark:text-neutral-400" />
```

**問題 4: アクセシビリティエラー**

**原因:** ARIA 属性の不足

**解決策:**

```tsx
// 適切なARIA属性を追加
<button aria-label="検索">
  <Search className="w-4 h-4" aria-hidden="true" />
</button>
```

### 8.2 デバッグ方法

**コンソールでの確認:**

```tsx

```
