# 統計項目サブカテゴリ構造提案書

## エグゼクティブサマリー

本提案書は、約5,900件の統計項目を含むCSVデータの分析結果に基づき、ダッシュボードとランキング機能における統計項目の分類体系を提案するものです。

### 主要な発見

- **総項目数**: 5,936項目
- **大カテゴリ数**: 13カテゴリ（A～M、および特殊カテゴリ#）
- **サブカテゴリ総数**: 91サブカテゴリ

### 提案の要点

1. 既存のコード体系（A～M）を活かした階層構造
2. ユーザーの関心領域に基づく論理的な分類
3. ダッシュボード表示に適したカテゴリサイズ（最小2項目～最大808項目）
4. 検索性と発見性を両立させた設計

## 提案するカテゴリ構造

### 階層構造の設計原則

1. **3階層構造**: 大カテゴリ → サブカテゴリ → 統計項目
2. **ユーザー視点**: 専門用語を避け、一般ユーザーが理解しやすい名称
3. **バランス**: 各サブカテゴリの項目数を適度に保つ（10～100項目程度）
4. **拡張性**: 将来の統計項目追加に対応できる柔軟な構造

## 実装時の考慮事項

### 1. ユーザーインターフェース設計

#### カテゴリナビゲーション
- **3階層のドリルダウン**: 大カテゴリ → サブカテゴリ → 統計項目の順に絞り込める
- **ブレッドクラム**: 現在位置を常に表示
- **サイドバーまたはメガメニュー**: 全カテゴリを一覧できるナビゲーション
- **カテゴリアイコン**: 各大カテゴリに視覚的に識別しやすいアイコンを付与

#### 検索機能
- **全文検索**: 統計項目名、カテゴリ名で検索
- **サジェスト機能**: 入力中に候補を表示
- **フィルター**: カテゴリ、単位、地域タイプでフィルタリング
- **タグ検索**: 人気のあるキーワードをタグとして表示（例: 高齢化、少子化、経済成長など）

#### ダッシュボード表示
- **カテゴリ別ダッシュボード**: 各大カテゴリごとに専用のダッシュボードページを作成
- **カスタムダッシュボード**: ユーザーが関心のある統計項目を自由に組み合わせて独自のダッシュボードを作成できる機能
- **レスポンシブデザイン**: PC、タブレット、スマートフォンで最適な表示

#### ランキング機能
- **is_rankingフラグの活用**: CSVデータのis_rankingカラムが1の項目のみをランキング対象とする
- **複数のランキング軸**: 最新値、前年比増加率、過去5年平均など
- **ランキング絞り込み**: カテゴリ、サブカテゴリでランキングを絞り込める

### 2. データ構造設計

#### カテゴリマスタテーブル
```sql
CREATE TABLE categories (
  category_id VARCHAR(10) PRIMARY KEY, -- 例: 'A', 'B', 'C'
  category_name VARCHAR(100) NOT NULL, -- 例: '人口・世帯'
  category_name_en VARCHAR(100), -- 英語名（多言語対応用）
  icon_name VARCHAR(50), -- アイコン識別子
  display_order INT NOT NULL,
  description TEXT,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

#### サブカテゴリマスタテーブル
```sql
CREATE TABLE subcategories (
  subcategory_id VARCHAR(10) PRIMARY KEY, -- 例: 'A1', 'A2'
  category_id VARCHAR(10) NOT NULL,
  subcategory_name VARCHAR(100) NOT NULL, -- 例: '人口統計'
  subcategory_name_en VARCHAR(100),
  display_order INT NOT NULL,
  description TEXT,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  FOREIGN KEY (category_id) REFERENCES categories(category_id)
);
```

#### 統計項目マッピングテーブル
既存のCSVデータに加えて、カテゴリ・サブカテゴリとのマッピング情報を追加
```sql
ALTER TABLE stats_mappings ADD COLUMN category_id VARCHAR(10);
ALTER TABLE stats_mappings ADD COLUMN subcategory_id VARCHAR(10);
ALTER TABLE stats_mappings ADD FOREIGN KEY (category_id) REFERENCES categories(category_id);
ALTER TABLE stats_mappings ADD FOREIGN KEY (subcategory_id) REFERENCES subcategories(subcategory_id);
```

### 3. パフォーマンス最適化

#### インデックス設計
- category_id、subcategory_idにインデックスを作成
- item_nameに全文検索インデックスを作成
- is_rankingにインデックスを作成（ランキング機能の高速化）

#### キャッシュ戦略
- カテゴリ・サブカテゴリマスタはRedisなどのインメモリキャッシュに格納
- ランキングデータは定期的に計算してキャッシュ
- ダッシュボードのグラフデータもキャッシュして高速化

#### ページネーション
- 統計項目一覧は100件ごとにページネーション
- 無限スクロールとページネーションのハイブリッド方式も検討

### 4. アクセシビリティ

- **キーボード操作**: カテゴリ選択、統計項目選択をキーボードだけで操作可能に
- **スクリーンリーダー対応**: 適切なARIA属性を付与
- **カラーコントラスト**: WCAG 2.1 AA基準に準拠
- **代替テキスト**: グラフや図表には適切なalt属性やaria-labelを付与

### 5. 多言語対応

- **日本語/英語**: カテゴリ名、サブカテゴリ名、統計項目名を日英で提供
- **コード体系の維持**: cat01コード（A1101など）は言語に依存しないため、多言語対応の基盤として活用

### 6. 段階的な実装計画

#### フェーズ1: 基本機能（MVP）
- 13の大カテゴリ表示
- サブカテゴリ一覧表示
- 統計項目一覧表示（カテゴリ・サブカテゴリでフィルタリング可能）
- 基本的なランキング機能（is_ranking=1の項目のみ）

#### フェーズ2: 検索・発見機能の強化
- 全文検索機能
- タグ検索
- 関連統計項目のレコメンド
- 人気ランキング（アクセス数に基づく）

#### フェーズ3: ダッシュボードの充実
- カテゴリ別ダッシュボード
- カスタムダッシュボード作成機能
- グラフの種類追加（折れ線、棒、円、面積、マップなど）
- データのエクスポート機能（CSV, Excel, PDF）

#### フェーズ4: 高度な分析機能
- 統計項目間の相関分析
- 時系列予測
- 地域クラスタリング
- AIによる洞察提示

### 7. 保守性・拡張性

#### カテゴリ構造の柔軟性
- カテゴリ・サブカテゴリはDBで管理し、コードの変更なしに追加・変更可能に
- cat01コードとの紐付けもDBで管理

#### 統計項目の追加
- 新しい統計項目はcat01コードに基づいて自動的にカテゴリ・サブカテゴリに割り当て
- マッピングルールをDBまたは設定ファイルで管理

#### モニタリング
- カテゴリ別アクセス数の計測
- 検索キーワードの分析
- ユーザー行動の追跡（Google Analyticsなど）

## まとめ

本提案は、約5,900件の統計項目を13の大カテゴリと91のサブカテゴリに体系化し、ユーザーが目的の統計を迅速に発見・利用できる構造を提案しました。

### 提案の強み

1. **既存のコード体系（A～M）を活かした自然な分類**
2. **ユーザー視点の階層設計**: 専門用語を避け、わかりやすいカテゴリ名を採用
3. **バランスの取れた項目分布**: 各サブカテゴリの項目数が適度（2～808項目）
4. **拡張性**: 新しい統計項目の追加に柔軟に対応可能
5. **実装の具体性**: データベース設計、UI設計、段階的実装計画まで網羅

### 次のステップ

1. **ステークホルダーレビュー**: 本提案をプロジェクト関係者でレビュー
2. **ユーザビリティテスト**: プロトタイプを作成し、実際のユーザーでテスト
3. **カテゴリ名の最終決定**: ユーザビリティテストの結果を踏まえて調整
4. **実装着手**: フェーズ1（MVP）から段階的に実装

本提案が、stats47プロジェクトの成功に貢献することを期待しています。
