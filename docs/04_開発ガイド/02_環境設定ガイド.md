# 環境設定ガイド

## 概要

このドキュメントは、stats47 プロジェクトの開発環境をセットアップするための実践的な手順を提供します。開発者が実際に環境を構築する際に必要な具体的なコマンドと設定手順を記載しています。

## 1. 前提条件

### 必要なツール

```bash
# Node.js (18.x以上)
node --version

# npm
npm --version

# Git
git --version

# Wrangler (Cloudflare CLI)
npm install -g wrangler
wrangler --version
```

### 必要なアカウント

- **Cloudflare アカウント**: D1 データベースと Pages の使用
- **e-Stat API キー**: 統計データの取得

## 2. プロジェクトのセットアップ

### 2.1 リポジトリのクローン

```bash
git clone https://github.com/your-org/stats47.git
cd stats47
```

### 2.2 依存関係のインストール

```bash
npm install
```

### 2.3 環境変数ファイルの作成

#### 開発環境用

```bash
# .env.developmentファイルを作成
cp env.development.example .env.development
```

**.env.development の設定:**

```bash
# 環境設定
NEXT_PUBLIC_ENV=development
NODE_ENV=development

# NextAuth設定（認証機能に必要）
NEXTAUTH_SECRET=<生成されたシークレット>
NEXTAUTH_URL=http://localhost:3000

# e-Stat API設定
ESTAT_API_BASE_URL=https://api.e-stat.go.jp/rest/3.0/app
NEXT_PUBLIC_ESTAT_APP_ID=your_estat_app_id

# ローカルD1設定
LOCAL_D1_PATH=.wrangler/state/v3/d1/miniflare-D1DatabaseObject/...
```

#### モック環境用

```bash
# .env.mockファイルを作成
cp env.mock.example .env.mock
```

**.env.mock の設定:**

```bash
# 環境設定
NEXT_PUBLIC_ENV=mock
NODE_ENV=development

# NextAuth設定（認証機能に必要）
NEXTAUTH_SECRET=<生成されたシークレット>
NEXTAUTH_URL=http://localhost:3000

# モックデータ設定
MOCK_DATA_PATH=data/mock
```

#### NEXTAUTH_SECRET の生成方法

**方法1: Node.jsで生成（推奨）:**
```bash
node -e "console.log(require('crypto').randomBytes(32).toString('base64'))"
```

**方法2: OpenSSLで生成:**
```bash
openssl rand -base64 32
```

**セキュリティ注意:**
- 各環境で異なるシークレットを使用
- シークレットは最低32文字
- 本番環境のシークレットは特に厳重に管理
- `.env.*` ファイルは Git にコミットしない

#### 環境別設定ファイル

各環境用の設定例は以下のファイルを参照してください：

- **開発環境**: `env.development.example`
- **モック環境**: `env.mock.example`
- **ステージング環境**: `env.staging.example`
- **本番環境**: `env.production.example`

これらのファイルをコピーして `.env.*` ファイルを作成し、必要に応じて値を調整してください。

## 3. 環境別設定

### 3.1 モック環境（デザイン検証用）

**用途**: API 接続なしでデザイン検証

```bash
# .env.mock
NEXT_PUBLIC_ENV=mock
NODE_ENV=development
```

**起動方法**:

```bash
npm run dev:mock
```

**特徴**:

- JSON ファイルからデータを読み込み
- データベース接続不要
- オフライン開発可能
- 認証機能が動作（ログイン/ログアウト状態の確認可能）

**認証機能:**
- ログイン/ログアウト機能が動作
- 管理画面のデザイン検証が可能

**テストアカウント:**
- 管理者: admin@stats47.local / admin123
- 一般ユーザー: user@stats47.local / user123

### 3.2 開発環境（ローカル開発用）

**用途**: ローカル D1 を使用した開発

```bash
# .env.development
NEXT_PUBLIC_ENV=development
NODE_ENV=development
ESTAT_API_KEY=your_estat_api_key_here
```

**起動方法**:

```bash
npm run dev:local
```

**特徴**:

- ローカル D1（SQLite）を使用
- e-Stat API に接続
- 開発・テストに最適

### 3.3 ステージング環境（統合テスト用）

**用途**: 本番データで統合テスト

```bash
# .env.staging
NEXT_PUBLIC_ENV=staging
NODE_ENV=production
ESTAT_API_KEY=${STAGING_ESTAT_API_KEY}
CLOUDFLARE_D1_DATABASE_ID=${STAGING_D1_DATABASE_ID}
CLOUDFLARE_ACCOUNT_ID=${CLOUDFLARE_ACCOUNT_ID}
CLOUDFLARE_API_TOKEN=${CLOUDFLARE_API_TOKEN}
```

**起動方法**:

```bash
npm run build:staging
npm run start
```

### 3.4 開発ツール（管理画面）

**用途**: e-Stat API 関連の管理ツール

**アクセス方法**:

```bash
# 開発環境で起動後
http://localhost:3000/admin/dev-tools/estat-api
```

**特徴**:

- 認証必須（管理画面配下）
- e-Stat API 関連ツール
  - stats-data: 統計データ取得
  - stats-list: 統計リスト取得
  - meta-info: メタ情報取得
- 本番環境では開発者のみアクセス可能

**環境別アクセス制御**:

| 環境            | 一般ページ | 管理画面 | 開発ツール |
| --------------- | ---------- | -------- | ---------- |
| **development** | 自由       | 認証必須 | 認証必須   |
| **staging**     | 自由       | 認証必須 | 認証必須   |
| **production**  | 自由       | 認証必須 | 開発者のみ |

## 4. Cloudflare D1 のセットアップ

### 4.1 Wrangler の認証

```bash
wrangler login
```

### 4.2 データベースの作成

```bash
# 開発用（ローカル）
wrangler d1 create stats47

# ステージング用
wrangler d1 create stats47_staging

# 本番用
wrangler d1 create stats47_production
```

### 4.3 マイグレーションの実行

```bash
# 開発環境（ローカルD1）
npx wrangler d1 migrations apply stats47 --local

# ステージング環境（リモートD1）
npx wrangler d1 migrations apply stats47_staging --env staging --remote

# 本番環境（リモートD1）
npx wrangler d1 migrations apply stats47 --remote
```

## 5. 環境切り替えコマンド

### 5.1 開発用コマンド

```bash
# モック環境で開発
npm run dev:mock

# ローカルD1環境で開発
npm run dev:local

# ステージング環境でビルド
npm run build:staging

# 本番環境でビルド
npm run build:production
```

### 5.2 データベース操作コマンド

```bash
# ローカルD1の状態確認
npx wrangler d1 list

# リモートD1の状態確認
npx wrangler d1 list --remote

# ローカルD1でクエリ実行
npx wrangler d1 execute stats47 --local --command "SELECT * FROM users LIMIT 5;"

# リモートD1でクエリ実行
npx wrangler d1 execute stats47 --remote --command "SELECT * FROM users LIMIT 5;"
```

### 6.1 環境変数が読み込まれない

```bash
# 現在の環境変数を確認
echo $NEXT_PUBLIC_ENV

# .envファイルの内容を確認
cat .env.development
```

### 6.2 データベース接続エラー

```bash
# ローカルD1の状態確認
npx wrangler d1 list

# リモートD1の状態確認
npx wrangler d1 list --remote

# データベースIDの確認
npx wrangler d1 list --json
```

### 6.3 データが表示されない

```bash

# 期待される出力例：

# [development] Fetching estat metainfo from database...
# [staging] Fetching estat metainfo from database...
```

## 7. よく使用するコマンド

### 7.1 開発関連

```bash
# 開発サーバー起動（モック環境）
npm run dev:mock

# 開発サーバー起動（ローカルD1）
npm run dev:local

# ビルド（ステージング）
npm run build:staging

# ビルド（本番）
npm run build:production
```

### 7.2 データベース関連

```bash
# マイグレーション作成
npx wrangler d1 migrations create stats47 "add_new_table"

# マイグレーション適用（ローカル）
npx wrangler d1 migrations apply stats47 --local

# マイグレーション適用（リモート）
npx wrangler d1 migrations apply stats47 --remote
```

```bash
# ステージングデプロイ
npx wrangler pages deploy --env staging

# 本番デプロイ

```

## 8. 環境別の特徴

| 環境            | データソース  | データベース | 用途         | 起動コマンド               |
| --------------- | ------------- | ------------ | ------------ | -------------------------- |
| **mock**        | JSON ファイル | なし         | デザイン検証 | `npm run dev:mock`         |
| **development** | e-Stat API    | ローカル D1  | 開発・テスト | `npm run dev:local`        |
| **staging**     | e-Stat API    | リモート D1  | 統合テスト   | `npm run build:staging`    |
| **production**  | e-Stat API    | リモート D1  | 本番運用     | `npm run build:production` |

## 9. セキュリティ注意事項

### 9.1 環境変数の管理

- `.env.*`ファイルは Git にコミットしない
- 機密情報は Cloudflare Secrets で管理
- 本番環境の API キーは慎重に管理

### 9.2 データベースアクセス

- 本番環境のデータベースは読み取り中心
- 書き込み操作は慎重に実行
- マイグレーションは段階的に適用
