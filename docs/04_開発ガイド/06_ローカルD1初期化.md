# ローカル D1 初期化ガイド

## 概要

開発環境でのローカル D1 データベースの初期化方法とトラブルシューティングを説明します。

## 前提条件

- Node.js がインストールされていること
- Wrangler CLI がインストールされていること（プロジェクトにインストール済み）
- `better-sqlite3` がインストールされていること

## 初期化方法

### 1. 初回セットアップ

初回セットアップ時は、以下のコマンドを実行してローカル D1 データベースを初期化します:

```bash
npm run db:init:local
```

このコマンドは以下の処理を自動的に実行します:

1. Wrangler CLI を使用してローカル D1 を起動
2. `database/schemas/main.sql` のスキーマを適用
3. `database/migrations/` のマイグレーションファイルを適用
4. データベースの状態を確認

### 2. データベースのリセット

ローカル D1 データベースを完全にリセットして再初期化する場合:

```bash
npm run db:reset:local
```

**注意**: このコマンドは `.wrangler/state/v3/d1/` ディレクトリを削除してから再初期化します。既存のデータはすべて失われます。

### 3. 既存データベースへのマイグレーション適用

データベースが既に存在する場合でも、新しいマイグレーションを適用できます:

```bash
npm run db:migrations:local
```

### 4. シードデータの投入

開発環境でサンプルデータを使用する場合は、シードファイルを適用します:

```bash
APPLY_SEEDS=true npm run db:reset:local
```

または、個別にシードファイルを適用:

```bash
# ユーザーシード
npx wrangler d1 execute stats47 --local --file=database/seeds/users_seed.sql

# カテゴリシード
npx wrangler d1 execute stats47 --local --file=database/seeds/categories_seed.sql

# e-Statメタ情報シード
npx wrangler d1 execute stats47 --local --file=database/seeds/estat_metainfo_seed.sql
```

#### 利用可能なシードファイル

以下のシードファイルが利用可能です:

- `users_seed.sql` - 開発用ユーザー（admin, testuser）
- `categories_seed.sql` - カテゴリとサブカテゴリ
- `estat_metainfo_seed.sql` - e-Stat メタ情報（統計表 ID とタイトル）
- `ranking_groups_seed.sql` - ランキンググループ
- `ranking_items_seed.sql` - ランキングアイテム
- `widget_templates_seed.sql` - ウィジェットテンプレート
- `dashboard_configs_seed.sql` - ダッシュボード設定
- `dashboard_widgets_seed.sql` - ダッシュボードウィジェット

#### e-Stat メタ情報シードファイルの生成

現在のデータベースから`estat_metainfo`テーブルのデータをエクスポートしてシードファイルを作成する場合:

```bash
python3 scripts/generate_estat_metainfo_seed.py
```

このスクリプトは、ローカル D1 データベースから`estat_metainfo`テーブルの全データを取得し、`database/seeds/estat_metainfo_seed.sql`を生成します。

生成されたシードファイルは、べき等性を保つため`INSERT OR REPLACE`を使用しているため、何度実行しても同じ結果になります。

## トラブルシューティング

### エラー: "Cannot open database because the directory does not exist"

**原因**: ローカル D1 データベースのディレクトリが存在しない

**解決方法**: 以下のコマンドでデータベースを初期化してください:

```bash
npm run db:init:local
```

### エラー: "Local D1 database file does not exist"

**原因**: データベースファイルが存在しない、または破損している

**解決方法**: データベースをリセットしてください:

```bash
npm run db:reset:local
```

### エラー: "Wrangler CLI is not installed"

**原因**: Wrangler CLI がプロジェクトにインストールされていない

**解決方法**: 以下のコマンドでインストール:

```bash
npm install
```

または:

```bash
npm install -g wrangler
```

### マイグレーションエラーの解決

マイグレーションの適用に失敗した場合:

1. エラーメッセージを確認
2. マイグレーションファイルの構文を確認
3. 必要に応じてデータベースをリセット:

```bash
npm run db:reset:local
```

## データベースの状態確認

### データベースファイルの場所

ローカル D1 データベースファイルは以下の場所に保存されます:

```
.wrangler/state/v3/d1/miniflare-D1DatabaseObject/b3c084603f7be30f18c6a887d294217e3e6b2010e83e9d09910eae3515a26884.sqlite
```

### テーブル一覧の確認

以下のコマンドでテーブル一覧を確認できます:

```bash
npx wrangler d1 execute stats47 --local --command "SELECT name FROM sqlite_master WHERE type='table' ORDER BY name;"
```

### スキーマの確認

特定のテーブルのスキーマを確認:

```bash
npx wrangler d1 execute stats47 --local --command ".schema users"
```

### データの確認

テーブルのデータを確認:

```bash
npx wrangler d1 execute stats47 --local --command "SELECT * FROM users LIMIT 10;"
```

## 開発ワークフロー

### 日常的な開発

1. 開発サーバーの起動:

   ```bash
   npm run dev
   ```

2. マイグレーションの適用:

   ```bash
   npm run db:migrations:local
   ```

3. データベースの確認:
   ```bash
   npx wrangler d1 execute stats47 --local --command "SELECT COUNT(*) FROM ranking_items;"
   ```

### 新しい開発環境のセットアップ

他の開発者と共有する環境をセットアップする場合:

1. データベースの初期化:

   ```bash
   npm run db:init:local
   ```

2. シードデータの投入:

   ```bash
   APPLY_SEEDS=true npm run db:reset:local
   ```

   または、個別にシードファイルを適用:

   ```bash
   # 基本データ（ユーザー、カテゴリ）
   npx wrangler d1 execute stats47 --local --file=database/seeds/users_seed.sql
   npx wrangler d1 execute stats47 --local --file=database/seeds/categories_seed.sql

   # e-Statメタ情報（他の開発環境と共有したい場合）
   npx wrangler d1 execute stats47 --local --file=database/seeds/estat_metainfo_seed.sql
   ```

### 現在のデータベースをシードファイルとして共有

現在のデータベースのデータを他の開発環境で使用したい場合:

1. 現在のデータからシードファイルを生成:

   ```bash
   # e-Statメタ情報の場合
   python3 scripts/generate_estat_metainfo_seed.py
   ```

2. 生成されたシードファイルを Git にコミット:

   ```bash
   git add database/seeds/estat_metainfo_seed.sql
   git commit -m "Add estat_metainfo seed data"
   ```

3. 他の開発者がシードファイルを適用:

   ```bash
   npx wrangler d1 execute stats47 --local --file=database/seeds/estat_metainfo_seed.sql
   ```

### 開発終了時のデータベース保持

ローカル D1 データベースは `.wrangler/` ディレクトリに保存されるため、開発サーバーを停止してもデータは保持されます。

`.wrangler/` ディレクトリは `.gitignore` に含まれているため、Git にはコミットされません。

## バックアップとリストア

### ローカルデータベースのバックアップ

以下のコマンドでローカル D1 のバックアップを取得:

```bash
npm run db:dump:local
```

バックアップファイルは `database/backups/local_dump.sql` に保存されます。

### リストア方法

現在の方法では、データベースをリセットしてからインポートする方法が簡単です:

```bash
npm run db:reset:local
npm run db:migrations:local
```
