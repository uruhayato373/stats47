## 概要

本ドキュメントでは、都道府県ランキングとコロプレス地図の視覚化機能における**データアーキテクチャの推奨設計**を定義します。

### 設計の核心的な問い

**「estat-api など複数のデータソースからランキングデータを取得する際、どのようなアーキテクチャで設計すべきか?」**

2 つの主要なアプローチ:

- **オプション A**: データソースから取得 → ランキング形式に変換 → 保存（データ中心）
- **オプション B**: ランキングドメインでデータソースパラメータを保存 → 必要時に取得・変換（メタデータ中心）

### 推奨設計

**「データ中心アプローチ + 計算層」**

この設計は、複数のデータソースを組み合わせたランキング計算を可能にするため、オプション A（データ中心）を採用し、計算定義層を追加したものです：

- 生データを R2 に保存（地域レベル別、パラメータハッシュ別）
- 計算定義を D1 の`ranking_calculations`テーブルで管理（直接・計算・集計）
- 初回取得時に生データを R2 に保存
- ランキング取得時に計算実行・結果キャッシュ
- 複数データソース組み合わせによる計算ランキングが可能

**R2 ストレージ構造**:

```
estat_raw_data/
  ├── estat/
      ├── prefecture/
      │   └── 0000010102_B1101_00000/
      │       └── 2020.json
      └── city/
          └── 0000010102_B1101_01000/
              └── 2020.json

ranking_values/
  └── population/
      ├── prefecture/2020.json
      └── city/2020.json
```

## 要件定義

### 機能要件

1. **マルチデータソース対応**

   - e-Stat API
   - 気象庁データ
   - カスタム CSV
   - その他外部 API（世界銀行、OECD など）

2. **時系列データ対応**

   - 全年度のデータを保持
   - 年度選択によるランキング切り替え
   - 時系列トレンド分析機能

3. **地域レベル対応**

   - 都道府県レベル
   - 市区町村レベル
   - 両方の階層構造を保持

4. **視覚化機能**
   - コロプレス地図
   - ランキングテーブル
   - 統計グラフ

### 非機能要件

1. **パフォーマンス**

   - 初回ロード: 3 秒以内
   - キャッシュヒット時: 500ms 以内
   - 年度切り替え: 1 秒以内

2. **スケーラビリティ**

   - 都道府県レベル: D1 データベース（〜5KB/年度）
   - 市区町村レベル: R2 ストレージ（〜200KB/年度）
   - 複数年度: R2 ストレージ（全年度保持）

3. **可用性**

   - キャッシュによる外部 API 障害への耐性
   - データソース追加時のゼロダウンタイム
   - 段階的な移行が可能

4. **保守性**
   - データソースの追加が容易
   - 変換ロジックの独立性
   - テストの容易性

### データ戦略

**ハイブリッドキャッシング戦略**:

- **初回取得**: API → 変換 → R2 保存
- **2 回目以降**: R2 から直接取得（API 呼び出しなし）
- **更新**: 手動トリガーまたは TTL ベース（将来的にバッチ処理）

**ストレージ分担**:

- **D1**: ランキング設定・メタデータのみ（高速クエリ用）
  - ranking_items, data_sources, data_source_metadata
  - ranking_groups, ranking_group_items, subcategory_ranking_items
- **R2**: ランキング値データ（大容量・全年度）
  - すべての ranking_values（都道府県・市区町村レベル）

### 主要コンポーネント

#### 1. Ranking Domain（中核ドメイン）

**責務**:

- ランキングデータの取得・管理
- 可視化設定の保持 syuu
- ビジネスロジックの実装

**主要エンティティ**:

```typescript
// ランキング項目（データソース非依存）
interface RankingItem {
  rankingKey: string; // 一意識別子
  label: string; // 表示名
  name: string; // 正式名称
  unit: string; // 単位
  dataSourceId: string; // データソースID
  mapColorScheme: string; // 色スキーム
  rankingDirection: "asc" | "desc";
}

// ランキングデータポイント
interface RankingDataPoint {
  areaCode: string;
  areaName: string;
  areaType: "prefecture" | "city";
  parentAreaCode?: string;
  parentAreaName?: string;
  value: number;
  rawValue?: number;
  displayValue?: string;
  rank: number;
  rankInParent?: number;
  percentile?: number;
  timeCode: string;
  timeName: string;
  dataQuality?: DataPointQuality;
}
```

#### 2. Data Source Metadata（データ取得・計算定義層）

**責務**:

- ランキングデータの取得方法定義
- 複数データソースの組み合わせによる計算ランキング
- 地域レベル別のパラメータ管理

**主要エンティティ**:

```typescript
// データソースメタデータ（拡張版）
interface DataSourceMetadata {
  id: number;
  rankingItemId: number;
  dataSourceId: string;
  areaType: "prefecture" | "city" | "national";
  calculationType: "direct" | "ratio" | "aggregate";
  metadata: DirectMetadata | RatioMetadata;
}

// 直接ランキング: 1つのデータソースから直接取得
interface DirectMetadata {
  stats_data_id: string;
  cd_cat01: string;
  cd_area?: string;
}

// 計算ランキング（比率型）: 分子÷分母×係数
interface RatioMetadata {
  numerator: {
    source_key: string; // 列参照用キー
    stats_data_id: string;
    cd_cat01: string;
  };
  denominator: {
    source_key: string; // 列参照用キー
    stats_data_id: string;
    cd_cat01: string;
  };
  multiplier: number; // 例: 1000（人口密度）
  decimal_places?: number; // 小数点以下桁数
}
```

**テーブル定義**:

##### data_sources（データソース定義）

| カラム名    | データ型 | 制約        | デフォルト値      | 説明             |
| ----------- | -------- | ----------- | ----------------- | ---------------- |
| id          | TEXT     | PRIMARY KEY | -                 | データソース ID  |
| name        | TEXT     | NOT NULL    | -                 | データソース名   |
| description | TEXT     | -           | NULL              | 説明             |
| base_url    | TEXT     | -           | NULL              | ベース URL       |
| api_version | TEXT     | -           | NULL              | API バージョン   |
| is_active   | BOOLEAN  | -           | 1                 | アクティブフラグ |
| created_at  | DATETIME | -           | CURRENT_TIMESTAMP | 作成日時         |
| updated_at  | DATETIME | -           | CURRENT_TIMESTAMP | 更新日時         |

**インデックス**: なし

**初期データ例**:

| id     | name           | description              | base_url                 |
| ------ | -------------- | ------------------------ | ------------------------ |
| estat  | e-Stat         | 政府統計の総合窓口       | https://api.e-stat.go.jp |
| custom | カスタムデータ | ユーザー定義データソース | NULL                     |

##### data_source_metadata（データソース固有メタデータ - 拡張版）

| カラム名             | データ型     | 制約                        | デフォルト値            | 説明                |
| ---------------- | -------- | ------------------------- | ----------------- | ----------------- |
| id               | INTEGER  | PRIMARY KEY AUTOINCREMENT | -                 | メタデータ ID          |
| ranking_item_id  | INTEGER  | NOT NULL                  | -                 | ランキング項目 ID        |
| data_source_id   | TEXT     | NOT NULL                  | -                 | データソース ID         |
| area_type        | TEXT     | NOT NULL                  | -                 | 地域レベル             |
| calculation_type | TEXT     | NOT NULL                  | 'direct'          | 計算タイプ             |
| metadata         | TEXT     | NOT NULL                  | -                 | データソースパラメータ（JSON） |
| created_at       | DATETIME | -                         | CURRENT_TIMESTAMP | 作成日時              |
| updated_at       | DATETIME | -                         | CURRENT_TIMESTAMP | 更新日時              |

**外部キー**:

- `ranking_item_id` → `ranking_items(id)` ON DELETE CASCADE
- `data_source_id` → `data_sources(id)`

**UNIQUE 制約**: `(ranking_item_id, data_source_id, area_type)`

**CHECK 制約**:

- `area_type IN ('prefecture', 'city', 'national')`
- `calculation_type IN ('direct', 'ratio', 'aggregate')`

**インデックス**:

- `idx_data_source_metadata_ranking` ON data_source_metadata(ranking_item_id)
- `idx_data_source_metadata_source` ON data_source_metadata(data_source_id)
- `idx_data_source_metadata_area` ON data_source_metadata(area_type)

**metadata JSON 構造例**:

**1. 直接ランキング（calculation_type='direct', area_type='prefecture'）**:

```json
{
  "stats_data_id": "0000010102",
  "cd_cat01": "B1101",
  "cd_area": "00000"
}
```

**2. 計算ランキング（calculation_type='ratio', area_type='prefecture'）**:

```json
{
  "numerator": {
    "source_key": "population",
    "stats_data_id": "0000010102",
    "cd_cat01": "A1101"
  },
  "denominator": {
    "source_key": "area",
    "stats_data_id": "0000020101",
    "cd_cat01": "B1101"
  },
  "multiplier": 1000,
  "decimal_places": 2
}
```

**データ例**:

| ranking_item_id | data_source_id | area_type  | calculation_type | metadata                                                     |
| --------------- | -------------- | ---------- | ---------------- | ------------------------------------------------------------ |
| 1               | estat          | prefecture | direct           | `{"stats_data_id":"xxx","cd_cat01":"yyy","cd_area":"00000"}` |
| 1               | estat          | city       | direct           | `{"stats_data_id":"xxx","cd_cat01":"yyy","cd_area":"01000"}` |
| 2               | estat          | prefecture | ratio            | `{"numerator":{...},"denominator":{...},"multiplier":1000}`  |
| 2               | estat          | city       | ratio            | `{"numerator":{...},"denominator":{...},"multiplier":1000}`  |

#### 3. Adapter Layer（データ取得・変換）

**責務**:

- 外部データソースからのデータ取得
- 統一フォーマット（UnifiedRankingData）への変換
- エラーハンドリング

**インターフェース**:

```typescript
interface RankingDataAdapter {
  readonly sourceId: string;
  readonly sourceName: string;

  fetchAndTransform(params: AdapterFetchParams): Promise<UnifiedRankingData>;
  getAvailableYears(
    rankingKey: string,
    level: TargetAreaLevel
  ): Promise<string[]>;
}
```

#### 4. Cache/Storage Layer（永続化）

**責務**:

- 変換後データの保存
- 高速データ取得
- ストレージ管理

**ストレージ分担**:

```typescript
// D1: メタデータのみ（高速クエリ用）
interface D1Storage {
  // ランキング項目定義
  getRankingItem(rankingKey: string): Promise<RankingItem>;
  getRankingItems(): Promise<RankingItem[]>;

  // データソースメタデータ
  getDataSourceMetadata(rankingKey: string): Promise<Record<string, unknown>>;

  // ランキンググループ
  getRankingGroups(subcategoryId: string): Promise<RankingGroup[]>;

  // 注意: ranking_valuesはR2に保存（D1にはアクセスしない）
}

// R2: すべてのランキング値データ（大容量ストレージ）
interface R2Storage {
  // ランキング値の保存・取得
  saveUnifiedData(
    rankingKey: string,
    timeCode: string,
    data: UnifiedRankingData
  ): Promise<void>;
  getUnifiedData(
    rankingKey: string,
    timeCode: string
  ): Promise<UnifiedRankingData>;

  // 都道府県レベル・市区町村レベル両方を含む
  // 全年度のデータを保持
}
```

**データ保存先**:

- **D1**: ranking_items, data_sources, data_source_metadata, ranking_groups, ranking_group_items, subcategory_ranking_items
- **R2**: すべての ranking_values（都道府県・市区町村レベル両方）
