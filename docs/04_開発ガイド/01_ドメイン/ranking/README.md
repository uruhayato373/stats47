## 主要な設計判断

### ハイブリッドアプローチの採用

**問い**: データソースからデータをどう管理するか?

**答え**: 変換後データ保存 + メタデータ管理のハイブリッド

**理由**:
- 初回取得後はキャッシュから高速取得（パフォーマンス）
- メタデータでデータソースパラメータを保持（柔軟性）
- 必要時にアダプター経由で再取得可能（保守性）

### 階層型ストレージ

**D1 Database**:
- 用途: ランキング項目定義、データソース設定、グループ定義
- 特徴: 低レイテンシ（10-50ms）、SQLクエリ可能

**R2 Storage**:
- 用途: すべてのランキング値データ（都道府県・市区町村レベル）
- 特徴: 大容量対応、低コスト、HTTP アクセス

### Adapter Pattern

新しいデータソースの追加が容易：
- e-Stat API
- 気象庁 API
- CSV/Excelファイル
- 将来: RESAS、世界銀行、OECD など
