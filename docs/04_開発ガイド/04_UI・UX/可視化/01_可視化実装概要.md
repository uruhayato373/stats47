# 可視化実装概要

## 概要

stats47プロジェクトでは、統計データの可視化に複数のライブラリを組み合わせて使用しています。このドキュメントでは、使用するライブラリの選定理由と全体アーキテクチャについて説明します。

## 使用ライブラリ一覧

### 1. Recharts（チャート・グラフ）

**用途**: 折れ線グラフ、棒グラフ、円グラフ、散布図などの統計グラフ

**選定理由**:
- React専用設計でNext.jsとの親和性が高い
- TypeScriptサポートが充実
- レスポンシブ対応が標準で提供
- カスタマイズ性が高い
- コミュニティが活発でドキュメントが充実

**バージョン**: `^2.8.0`

### 2. Leaflet（地図）

**用途**: コロプレスマップ、インタラクティブ地図

**選定理由**:
- 軽量で高速
- 豊富なプラグインエコシステム
- TopoJSON/GeoJSONサポート
- モバイル対応が優秀
- 国土地理院タイルとの相性が良い

**バージョン**: `^1.9.4`

### 3. D3.js（データ操作・カスタム可視化）

**用途**: 複雑なデータ変換、カスタムチャート、色スケール生成

**選定理由**:
- 強力なデータ操作機能
- 柔軟な可視化表現
- 他のライブラリとの連携が容易
- 統計計算機能が豊富

**バージョン**: `^7.8.5`

### データフロー

```
統計データ (e-Stat API)
    ↓
データ変換 (D3.js utilities)
    ↓
チャートコンポーネント (Recharts)
    ↓
ユーザーインターフェース
```

```
地理データ (TopoJSON)
    ↓
データ変換 (D3.js geo utilities)
    ↓
地図コンポーネント (Leaflet)
    ↓
ユーザーインターフェース
```

## パフォーマンス考慮事項

### 1. データサイズ最適化

- **地理データ**: 用途に応じて解像度を選択（low/medium/high/full）
- **チャートデータ**: 必要に応じてデータサンプリング
- **キャッシュ**: R2ストレージを活用した地理データキャッシュ

### 2. レンダリング最適化

- **遅延読み込み**: チャート・地図の遅延初期化
- **メモ化**: React.memo、useMemo、useCallbackの活用
- **仮想化**: 大量データの仮想スクロール

### 3. バンドルサイズ最適化

- **Tree Shaking**: 使用しないライブラリ機能の除外
- **動的インポート**: 必要時のみライブラリを読み込み
- **CDN**: 外部CDNの活用検討

## アクセシビリティ対応

### 1. 色覚異常者への配慮

- 色以外の視覚的手がかり（パターン、形状）
- 高コントラストモード対応
- カラーブラインドフレンドリーな色パレット

### 2. スクリーンリーダー対応

- 適切なARIAラベル
- データテーブルの代替提供
- キーボードナビゲーション対応
