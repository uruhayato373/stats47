# フック実装ガイド

## 概要

stats47プロジェクトにおけるカスタムフックの設計原則、実装パターン、配置ルールについて説明します。

## フックの配置ルール

### グローバルフック（`src/hooks/`）

**配置条件**:
- 複数のコンポーネントから使用される
- ドメインロジックやビジネスロジックを持つ
- 汎用的な機能を提供する
- グローバルな状態管理と連携する

**例**:
```typescript
// src/hooks/useTheme.ts - テーマ管理
// src/hooks/useAuthModal.ts - 認証モーダル管理
// src/hooks/useUserMenu.ts - ユーザーメニュー管理
```

### コンポーネントローカルフック

**配置条件**:
- そのコンポーネント内でのみ使用される
- コンポーネントの実装詳細である
- コンポーネントと密結合している
- 他のコンポーネントから参照する必要がない

**例**:
```typescript
// src/components/atoms/Button/hooks/useButtonState.ts
// src/components/molecules/Form/hooks/useFormValidation.ts
```

## フック設計原則

### 1. 単一責務の原則

各フックは一つの明確な責務を持つべきです。

```typescript
// ✅ 良い例: テーマ管理のみ
export function useTheme() {
  const { theme, setTheme, systemTheme, resolvedTheme } = useNextTheme();
  const currentTheme = resolvedTheme || theme;

  return {
    theme: currentTheme,
    setTheme,
    systemTheme,
    toggleTheme: () => {
      setTheme(currentTheme === "light" ? "dark" : "light");
    },
  };
}

// ❌ 悪い例: 複数の責務を持つ
export function useThemeAndAuth() {
  // テーマ管理 + 認証管理 = 責務が混在
}
```

### 2. 型安全性

TypeScriptを活用して型安全なフックを実装します。

```typescript
interface UseAuthModalReturn {
  isOpen: boolean;
  open: () => void;
  close: () => void;
}

export function useAuthModal(): UseAuthModalReturn {
  const [isOpen, setIsOpen] = useState(false);

  return {
    isOpen,
    open: () => setIsOpen(true),
    close: () => setIsOpen(false),
  };
}
```

### 3. パフォーマンス考慮

不要な再レンダリングを避けるため、適切なメモ化を使用します。

```typescript
export function useUserMenu() {
  const [isOpen, setIsOpen] = useState(false);
  const dropdownRef = useRef<HTMLDivElement>(null);

  // 外部クリックで閉じる処理をメモ化
  const handleClickOutside = useCallback((event: MouseEvent) => {
    if (
      dropdownRef.current &&
      !dropdownRef.current.contains(event.target as Node)
    ) {
      setIsOpen(false);
    }
  }, []);

  useEffect(() => {
    document.addEventListener("mousedown", handleClickOutside);
    return () => {
      document.removeEventListener("mousedown", handleClickOutside);
    };
  }, [handleClickOutside]);

  return {
    isOpen,
    toggle: () => setIsOpen(!isOpen),
    close: () => setIsOpen(false),
    dropdownRef,
  };
}
```

## 実装パターン

### 1. 状態管理フック

```typescript
// src/hooks/useAuthModal.ts
"use client";

import { useState, useEffect } from "react";

/**
 * 認証モーダルの状態とURLパラメータ処理を管理するカスタムフック
 */
export function useAuthModal() {
  const [isOpen, setIsOpen] = useState(false);

  // URLパラメータでモーダルを自動表示
  useEffect(() => {
    const params = new URLSearchParams(window.location.search);
    if (params.get("auth") === "true") {
      setIsOpen(true);
      // URLからパラメータを削除（ブラウザ履歴を汚さないため）
      const url = new URL(window.location.href);
      url.searchParams.delete("auth");
      window.history.replaceState({}, "", url.toString());
    }
  }, []);

  return {
    isOpen,
    open: () => setIsOpen(true),
    close: () => setIsOpen(false),
  };
}
```

### 3. 外部ライブラリラッパーフック

```typescript
// src/hooks/useTheme.ts
"use client";

import { useTheme as useNextTheme } from "next-themes";

/**
 * next-themes をラップしたカスタムフック
 * 
 * プロジェクト固有の拡張が必要な場合は、このフックで行います。
 * 現時点では next-themes をそのまま再エクスポート。
 */
export function useTheme() {
  const { theme, setTheme, systemTheme, resolvedTheme } = useNextTheme();
  const currentTheme = resolvedTheme || theme;

  return {
    theme: currentTheme, // 実際に適用されているテーマ
    setTheme,
    systemTheme,
    toggleTheme: () => {
      setTheme(currentTheme === "light" ? "dark" : "light");
    },
  };
}
```

## ベストプラクティス

### 1. 命名規則

- フック名は`use`で始める
- 機能を明確に表現する名前を使用
- 略語は避け、フルネームを使用

```typescript
// ✅ 良い例
useAuthModal()
useUserMenu()
useTheme()

// ❌ 悪い例
useAuth()
useMenu()
useT()
```

### 2. ドキュメント

各フックにはJSDocコメントを追加します。

```typescript
/**
 * 認証モーダルの状態とURLパラメータ処理を管理するカスタムフック
 * 
 * @returns {Object} モーダルの状態と操作関数
 * @returns {boolean} isOpen - モーダルの開閉状態
 * @returns {Function} open - モーダルを開く関数
 * @returns {Function} close - モーダルを閉じる関数
 */
export function useAuthModal() {
  // 実装...
}
```

### 3. エラーハンドリング

適切なエラーハンドリングを実装します。

```typescript
export function useApiData<T>(url: string) {
  const [data, setData] = useState<T | null>(null);
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState<string | null>(null);

  useEffect(() => {
    const fetchData = async () => {
      try {
        setLoading(true);
        setError(null);
        const response = await fetch(url);
        
        if (!response.ok) {
          throw new Error(`HTTP error! status: ${response.status}`);
        }
        
        const result = await response.json();
        setData(result);
      } catch (err) {
        setError(err instanceof Error ? err.message : 'An error occurred');
      } finally {
        setLoading(false);
      }
    };

    fetchData();
  }, [url]);

  return { data, loading, error };
}
```

### 4. テスト可能性

フックはテストしやすいように設計します。

```typescript
// テストしやすいフック
export function useCounter(initialValue: number = 0) {
  const [count, setCount] = useState(initialValue);

  const increment = useCallback(() => setCount(prev => prev + 1), []);
  const decrement = useCallback(() => setCount(prev => prev - 1), []);
  const reset = useCallback(() => setCount(initialValue), [initialValue]);

  return {
    count,
    increment,
    decrement,
    reset,
  };
}
```

## フックの移動ガイド

### コンポーネントローカル → グローバル

フックが複数のコンポーネントで使用されるようになった場合、グローバルに移動します。

**手順**:
1. `src/hooks/`にフックファイルを移動
2. インポートパスを更新
3. 古いファイルを削除
4. TypeScriptコンパイルエラーを確認

**例**:
```typescript
// 移動前: src/components/Header/hooks/useAuthModal.ts
import { useAuthModal } from "./hooks";

// 移動後: src/hooks/useAuthModal.ts
import { useAuthModal } from "@/hooks/useAuthModal";
```

## トラブルシューティング

### よくある問題

1. **フックのルール違反**
   - 条件分岐内でフックを呼び出している
   - ループ内でフックを呼び出している

2. **無限再レンダリング**
   - useEffectの依存配列が不適切
   - オブジェクトや関数が毎回新しく作成されている

3. **メモリリーク**
   - イベントリスナーが適切にクリーンアップされていない
   - タイマーが適切にクリアされていない

### デバッグ方法

```typescript
// フックの動作をデバッグ
export function useDebugHook() {
  const [state, setState] = useState(initialValue);
  
  useEffect(() => {
    console.log('Hook state changed:', state);
  }, [state]);
  
  return { state, setState };
}
```
