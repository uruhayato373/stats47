# フォント実装ガイド

## 1. 概要

このガイドでは、stats47プロジェクトで使用しているInter + Noto Sans JPフォントの実装方法について説明します。

### 1.1 フォント構成

- **欧文フォント**: Inter（Google Fonts）
- **日本語フォント**: Noto Sans JP（Google Fonts）
- **モノスペースフォント**: Geist Mono（Google Fonts）

### 1.2 主な特徴

- **ハイブリッド構成**: 欧文と日本語で最適なフォントを選択
- **変数フォント対応**: パフォーマンスに優れた実装
- **完全無料**: 商用利用可能なオープンソース
- **高品質**: Googleが開発した信頼性の高いフォント

## 2. パッケージのインストール

### 2.1 必要なパッケージ

**追加パッケージ:**
- なし（Next.jsの`next/font/google`に含まれる）

**削除可能パッケージ:**
- `geist`（段階的に削除可能）

### 2.2 パッケージの確認

```bash
# 現在のパッケージを確認
npm list geist

# geistパッケージを削除（オプション）
npm uninstall geist
```

## 3. layout.tsxの設定

### 3.1 フォントのインポート

```tsx
// src/app/layout.tsx
import { Inter } from "next/font/google";
import { Noto_Sans_JP } from "next/font/google";
import { Geist_Mono } from "next/font/google";
```

### 3.2 フォント設定

```tsx
// Inter（欧文フォント）の設定
const inter = Inter({
  subsets: ["latin"],
  variable: "--font-inter",
  display: "swap",
  preload: true,
});

// Noto Sans JP（日本語フォント）の設定
const notoSansJP = Noto_Sans_JP({
  weight: ["400", "500", "600", "700"],
  subsets: ["latin"],
  variable: "--font-noto-sans-jp",
  display: "swap",
  preload: true,
});

// Geist Mono（モノスペースフォント）の設定
const geistMono = Geist_Mono({
  subsets: ["latin"],
  variable: "--font-geist-mono",
  display: "swap",
});
```

### 3.3 HTML要素への適用

```tsx
export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html 
      lang="ja" 
      className={`${inter.variable} ${notoSansJP.variable} ${geistMono.variable}`}
    >
      <body className={inter.className}>
        {children}
      </body>
    </html>
  );
}
```

## 4. CSS変数の定義

### 4.1 globals.cssの設定

```css
/* src/app/globals.css */
@theme inline {
  --font-sans: var(--font-inter), var(--font-noto-sans-jp);
  --font-mono: var(--font-geist-mono);
}

/* フォントフォールバック */
body {
  font-family: var(--font-inter), var(--font-noto-sans-jp), 
    -apple-system, BlinkMacSystemFont, "Segoe UI",
    "Hiragino Sans", "Yu Gothic", "Meiryo", sans-serif;
}
```

### 4.2 Tailwind CSSの設定

```js
// tailwind.config.ts
module.exports = {
  theme: {
    extend: {
      fontFamily: {
        sans: ['var(--font-inter)', 'var(--font-noto-sans-jp)', 'sans-serif'],
        mono: ['var(--font-geist-mono)', 'monospace'],
      },
    },
  },
}
```

## 5. コンポーネントでの使用方法

### 5.1 基本的な使用方法

```tsx
// デフォルトフォント（Inter + Noto Sans JP）
<p className="text-base">これは基本のテキストです</p>

// 明示的にフォントを指定
<p className="font-sans">Inter + Noto Sans JP フォント</p>

// モノスペースフォント
<code className="font-mono">console.log('Hello World')</code>
```

### 5.2 フォントウェイトの指定

```tsx
// 各ウェイトの使用例
<p className="font-normal">400 - 通常</p>
<p className="font-medium">500 - 中程度</p>
<p className="font-semibold">600 - やや太字</p>
<p className="font-bold">700 - 太字</p>
```

### 5.3 カスタムコンポーネント

```tsx
// src/components/atoms/Typography/Typography.tsx
interface TypographyProps {
  variant?: "h1" | "h2" | "h3" | "body" | "caption";
  children: React.ReactNode;
  className?: string;
}

const variantStyles = {
  h1: "text-3xl font-bold leading-tight",
  h2: "text-2xl font-semibold leading-snug",
  h3: "text-xl font-medium leading-snug",
  body: "text-base leading-normal",
  caption: "text-sm",
};

export const Typography: React.FC<TypographyProps> = ({
  variant = "body",
  children,
  className = "",
}) => {
  const Component = variant.startsWith("h")
    ? (variant as keyof JSX.IntrinsicElements)
    : "p";

  return (
    <Component
      className={`${variantStyles[variant]} ${className}`}
    >
      {children}
    </Component>
  );
};
```

## 6. パフォーマンス最適化

### 6.1 フォントサブセット化

```tsx
// 必要な文字セットのみを読み込み
const inter = Inter({
  subsets: ["latin"], // 欧文のみ
  display: "swap",
});

const notoSansJP = Noto_Sans_JP({
  weight: ["400", "500", "700"], // 必要なウェイトのみ
  subsets: ["latin"],
  display: "swap",
});
```

### 6.2 プリロード設定

```tsx
// 重要なフォントを事前読み込み
const inter = Inter({
  preload: true, // プリロード有効
  display: "swap",
});
```

### 6.3 font-display戦略

```tsx
// フォールバック表示を優先
const inter = Inter({
  display: "swap", // フォールバック表示を優先
});

// より高速化したい場合
const inter = Inter({
  display: "optional", // フォント読み込み失敗時はフォールバック
});
```

### 6.4 変数フォントの活用

```tsx
// 変数フォントを使用（推奨）
const inter = Inter({
  weight: "variable", // 100-900の全ウェイトを1ファイルで
  subsets: ["latin"],
  display: "swap",
});
```

## 7. トラブルシューティング

### 7.1 よくある問題

**問題1: フォントが表示されない**

**原因:** フォントの読み込みエラー

**解決策:**
```tsx
// 1. インポートを確認
import { Inter } from "next/font/google";
import { Noto_Sans_JP } from "next/font/google";

// 2. 変数名を確認
const inter = Inter({
  variable: "--font-inter", // 正しい変数名
});

// 3. HTMLクラスを確認
<html className={`${inter.variable} ${notoSansJP.variable}`}>
```

**問題2: 日本語が正しく表示されない**

**原因:** 日本語フォントの設定不備

**解決策:**
```tsx
// 1. 日本語フォントの設定を確認
const notoSansJP = Noto_Sans_JP({
  weight: ["400", "500", "600", "700"],
  subsets: ["latin"], // 日本語は自動で含まれる
  variable: "--font-noto-sans-jp",
});

// 2. CSS変数の順序を確認
font-family: var(--font-inter), var(--font-noto-sans-jp), sans-serif;
```

**問題3: パフォーマンスが悪い**

**原因:** 不要なウェイトの読み込み

**解決策:**
```tsx
// 必要なウェイトのみを指定
const notoSansJP = Noto_Sans_JP({
  weight: ["400", "700"], // 最小限のウェイト
  subsets: ["latin"],
  display: "swap",
});
```

### 7.2 デバッグ方法

**フォントの確認:**

**Lighthouse での確認:**

## 8. ベストプラクティス

### 8.1 フォント選択の指針

**欧文テキスト:**
- 数字、UI要素 → Inter
- コード、データ → Geist Mono

**日本語テキスト:**
- 見出し、本文 → Noto Sans JP
- キャプション、ラベル → Noto Sans JP

### 8.2 パフォーマンス最適化

1. **必要なウェイトのみ読み込み**
2. **プリロードで重要なフォントを優先**
3. **font-display: swap でFOUTを防止**
4. **変数フォントでファイルサイズ削減**

### 8.3 アクセシビリティ

1. **最小フォントサイズ16px以上**
2. **十分なコントラスト比**
3. **適切な行間（line-height）**
4. **読みやすい文字間隔**
