# 環境別起動ガイド

## 概要

このドキュメントは、stats47 プロジェクトの各環境での起動方法と特徴を説明します。開発者が目的に応じて適切な環境を選択し、効率的に開発を進められるよう実践的な情報を提供します。

## 環境一覧

| 環境            | コマンド                   | データソース  | 用途         |
| --------------- | -------------------------- | ------------- | ------------ |
| **mock**        | `npm run dev:mock`         | JSON ファイル | デザイン検証 |
| **development** | `npm run dev:local`        | ローカル D1   | 開発・テスト |
| **staging**     | `npm run build:staging`    | リモート D1   | 統合テスト   |
| **production**  | `npm run build:production` | リモート D1   | 本番運用     |

## 1. モック環境

### 起動方法

```bash
npm run dev:mock
```

### 特徴

- **データソース**: JSON ファイル（`data/mock/`）
- **データベース**: 接続不要
- **API 接続**: 不要
- **用途**: デザイン検証、UI 開発

### 適用場面

- デザインシステムの検証
- コンポーネントの開発
- オフライン開発
- API 接続が不要な作業

### アクセス先

- アプリケーション: `http://localhost:3000`
- 管理画面: `http://localhost:3000/admin`（認証不要）

## 2. 開発環境（ローカル D1）

### 起動方法

```bash
npm run dev:local
```

### 特徴

- **データソース**: ローカル D1（SQLite）
- **API 接続**: e-Stat API に接続
- **用途**: 開発・テスト

### 前提条件

- e-Stat API キーの設定
- ローカル D1 のセットアップ完了

### 環境変数

```bash
# .env.development
NEXT_PUBLIC_ENV=development
NODE_ENV=development
ESTAT_API_KEY=your_estat_api_key_here
```

### アクセス先

- アプリケーション: `http://localhost:3000`
- 管理画面: `http://localhost:3000/admin`（認証必須）
- 開発ツール: `http://localhost:3000/admin/dev-tools/estat-api`

## 3. ステージング環境

### 起動方法

```bash
npm run build:staging
npm run start
```

### 特徴

- **データソース**: リモート D1（ステージング）
- **API 接続**: e-Stat API に接続
- **用途**: 統合テスト

### 前提条件

- ステージング用 D1 データベースの作成
- Cloudflare 認証設定
- ステージング用 API キー

### 環境変数

```bash
# .env.staging
NEXT_PUBLIC_ENV=staging
NODE_ENV=production
ESTAT_API_KEY=${STAGING_ESTAT_API_KEY}
CLOUDFLARE_D1_DATABASE_ID=${STAGING_D1_DATABASE_ID}
CLOUDFLARE_ACCOUNT_ID=${CLOUDFLARE_ACCOUNT_ID}
CLOUDFLARE_API_TOKEN=${CLOUDFLARE_API_TOKEN}
```

### アクセス先

- アプリケーション: デプロイ先 URL
- 管理画面: デプロイ先 URL/admin（認証必須）
- 開発ツール: デプロイ先 URL/admin/dev-tools/estat-api

## 4. 本番環境

### 起動方法

```bash
npm run build:production
npm run start
```

### 特徴

- **データソース**: リモート D1（本番）
- **API 接続**: e-Stat API に接続
- **用途**: 本番運用

### 前提条件

- 本番用 D1 データベースの作成
- Cloudflare 認証設定
- 本番用 API キー

### 環境変数

```bash
# .env.production
NEXT_PUBLIC_ENV=production
NODE_ENV=production
ESTAT_API_KEY=${PRODUCTION_ESTAT_API_KEY}
CLOUDFLARE_D1_DATABASE_ID=${PRODUCTION_D1_DATABASE_ID}
CLOUDFLARE_ACCOUNT_ID=${CLOUDFLARE_ACCOUNT_ID}
CLOUDFLARE_API_TOKEN=${CLOUDFLARE_API_TOKEN}
```

### アクセス先

- アプリケーション: 本番 URL
- 管理画面: 本番 URL/admin（認証必須）
- 開発ツール: 本番 URL/admin/dev-tools/estat-api（開発者のみ）

## 環境別アクセス制御

| 環境            | 一般ページ | 管理画面 | 開発ツール |
| --------------- | ---------- | -------- | ---------- |
| **mock**        | 自由       | 自由     | 自由       |
| **development** | 自由       | 認証必須 | 認証必須   |
| **staging**     | 自由       | 認証必須 | 認証必須   |
| **production**  | 自由       | 認証必須 | 開発者のみ |

## よく使用するコマンド

### 開発関連

```bash
# モック環境で開発
npm run dev:mock

# ローカルD1環境で開発
npm run dev:local

# ビルド（ステージング）
npm run build:staging

# ビルド（本番）
npm run build:production
```

### データベース関連

```bash
# ローカルD1の状態確認
npx wrangler d1 list

# リモートD1の状態確認
npx wrangler d1 list --remote

# マイグレーション適用（ローカル）
npx wrangler d1 migrations apply stats47 --local

# マイグレーション適用（リモート）
npx wrangler d1 migrations apply stats47 --remote
```

### デプロイ関連

```bash
# ステージングデプロイ
npx wrangler pages deploy --env staging

# 本番デプロイ
npx wrangler pages deploy --env production
```

## トラブルシューティング

### 環境変数が読み込まれない

```bash
# 現在の環境変数を確認
echo $NEXT_PUBLIC_ENV

# .envファイルの内容を確認
cat .env.development
```

### データベース接続エラー

```bash
# ローカルD1の状態確認
npx wrangler d1 list

# リモートD1の状態確認
npx wrangler d1 list --remote

# データベースIDの確認
npx wrangler d1 list --json
```

### データが表示されない

```bash
# コンソールログで環境を確認
# 期待される出力例：
# [mock] Fetching estat metainfo from mock data...
# [development] Fetching estat metainfo from database...
# [staging] Fetching estat metainfo from database...
```

## 環境選択の指針

### モック環境を選ぶ場合

- デザインシステムの検証
- コンポーネントの単体開発
- オフライン環境での作業
- API 接続が不要な作業

### 開発環境を選ぶ場合

- 機能開発・テスト
- データベース操作の検証
- e-Stat API 連携のテスト
- 統合テスト前の検証

### ステージング環境を選ぶ場合

- 本番データでの統合テスト
- パフォーマンステスト
- 本番デプロイ前の最終確認
- チーム内での動作確認

### 本番環境を選ぶ場合

- 本番運用
- 本番データでの動作確認
- 緊急時の調査・修正
