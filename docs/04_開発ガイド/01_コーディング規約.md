# コーディング規約

## 概要

このドキュメントは、stats47 プロジェクト固有のコーディング規約を定義します。プロジェクトで採用している設計パターン（Fetcher/Formatter/Display、Atomic Design、ドメイン駆動設計）に基づいた実践的な規約を提供します。

## 1. プロジェクト概要

### 1.1 採用技術スタック

- **フレームワーク**: Next.js 15 (App Router)
- **UI ライブラリ**: React 19
- **言語**: TypeScript 5.x
- **スタイリング**: Tailwind CSS 4
- **状態管理**: Jotai
- **データフェッチ**: SWR
- **可視化**: Recharts, D3.js

### 1.2 設計パターン

- **Fetcher/Formatter/Display**: データ取得・整形・表示の責務分離
- **Atomic Design**: コンポーネントの階層化設計
- **ドメイン駆動設計**: ビジネスロジックのドメイン別整理
- **Server/Client Components**: レンダリング戦略の使い分け

## 2. ディレクトリ構造規約

### 2.1 Atomic Design の階層

```
src/components/
├── atoms/           # 最小単位のコンポーネント
├── molecules/       # atoms の組み合わせ
├── organisms/       # molecules の組み合わせ
├── templates/       # ページレイアウト
└── pages/          # ページ固有のコンポーネント
```

### 2.2 ドメイン別構成

```
src/lib/
├── domain/         # ドメイン別ビジネスロジック
│   ├── estat-api/  # e-Stat API ドメイン
│   ├── ranking/    # ランキングドメイン
│   └── visualization/ # 可視化ドメイン
└── shared/         # 共通ユーティリティ
```

### 2.3 コンポーネント配置ルール

- **atoms**: 再利用可能な最小単位（Button, Input, Modal など）
- **molecules**: 特定の機能を持つ組み合わせ（SearchForm, DataCard など）
- **organisms**: 複雑な機能を持つ大きなコンポーネント（DataTable, Chart など）
- **templates**: ページレイアウト（DashboardLayout, AuthLayout など）
- **pages**: ページ固有のコンポーネント（HomePage, RankingPage など）

```typescript
});
```

```typescript
// ✅ 推奨（カスタムフック）
export function useRankingData(rankingKey: string, year: string) {
  const { data, error, isLoading, mutate } = useSWR(
    `/api/rankings/data?rankingKey=${rankingKey}&year=${year}`,
    fetcher,
    {
      revalidateOnFocus: false,
      dedupingInterval: 60000,
    }
  );

  return { data, error, isLoading, refetch: mutate };
}
```

## 7. コンポーネント設計規約

### 7.1 Atomic Design の階層別ルール

```typescript
// atoms: 最小単位、props は最小限
export interface ButtonProps {
  children: React.ReactNode;
  onClick?: () => void;
  variant?: "primary" | "secondary";
}

// molecules: 特定の機能を持つ組み合わせ
export interface SearchFormProps {
  onSearch: (query: string) => void;
  placeholder?: string;
}

// organisms: 複雑な機能、複数の molecules を使用
export interface DataTableProps {
  data: any[];
  columns: Column[];
  onRowClick?: (row: any) => void;
}
```

### 7.2 Props の定義方法

```typescript
// ✅ 推奨（明確な型定義）
interface EstatDataDisplayProps {
  data: EstatStatsDataResponse | null;
  loading: boolean;
  error: string | null;
}

// ✅ 推奨（forwardRef の使用）
const Button = forwardRef<HTMLButtonElement, ButtonProps>(
  ({ children, onClick, variant = "primary" }, ref) => {
    return (
      <button ref={ref} onClick={onClick} className={getVariantClass(variant)}>
        {children}
      </button>
    );
  }
);
```

## まとめ

このコーディング規約に従うことで、以下の効果が期待できます：

1. **一貫性**: プロジェクト全体で統一されたコーディングスタイル
2. **可読性**: 明確な命名規則と構造化されたコード
3. **保守性**: 責務分離とモジュール化による変更の局所化
4. **拡張性**: 設計パターンに基づいた拡張しやすい構造
5. **品質**: 型安全性とエラーハンドリングの徹底
