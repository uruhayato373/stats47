# 環境別認証動作ガイド

## 概要

Stats47 プロジェクトでは、開発効率とセキュリティのバランスを取るため、環境ごとに異なる認証動作を実装しています。

## 環境構成

| 環境     | コマンド                     | 認証     | データ | 用途                   |
| -------- | ---------------------------- | -------- | ------ | ---------------------- |
| **Mock** | `npm run dev:mock`           | バイパス | モック | UI 開発・素早い確認    |
| **API**  | `npm run dev:api`            | 必須     | 実 API | 認証テスト・統合テスト |
| **本番** | `npm run build && npm start` | 必須     | 実 API | 本番運用               |

## 詳細動作

### Mock 環境（開発効率重視）

#### 特徴

- **認証バイパス**: 自動的に管理者としてログイン状態
- **モックデータ**: `data/mock/auth/users.json` を使用
- **高速起動**: 認証処理なしで即座にアクセス可能

#### 実装

```typescript
// src/features/auth/lib/auth.ts
export const auth = async () => {
  const isMockEnv = process.env.NEXT_PUBLIC_USE_MOCK === "true";

  if (isMockEnv && process.env.NODE_ENV !== "production") {
    return {
      user: {
        id: "mock-admin",
        name: "モック管理者",
        email: "admin@example.com",
        username: "admin",
        role: "admin" as const,
      },
    };
  }

  return null; // API環境・本番環境では認証必須
};
```

#### 使用方法

```bash
# Mock環境で起動
npm run dev:mock

# ブラウザで直接アクセス
# http://localhost:3000/admin
# → 認証なしで管理者画面にアクセス可能
```

### API 環境（認証テスト）

#### 特徴

- **認証必須**: ログインフローを完全にテスト可能
- **実データ**: 実際の API・データベースを使用
- **統合テスト**: 本番に近い環境での動作確認

#### 実装

```typescript
// API環境では認証チェックを有効化
export const auth = async () => {
  // NextAuth実装（TODO: 有効化予定）
  // 現在は一時的に無効化
  return null;
};
```

#### 使用方法

```bash
# API環境で起動
npm run dev:api

# ブラウザでアクセス
# http://localhost:3000/admin
# → 認証が必要（ログインページにリダイレクト）
```

### 本番環境（セキュリティ重視）

#### 特徴

- **認証必須**: セキュリティを最優先
- **実データ**: 本番データベースを使用
- **パフォーマンス**: 最適化された本番ビルド

#### 実装

```typescript
// 本番環境では必ず認証チェック
export const auth = async () => {
  if (process.env.NODE_ENV === "production") {
    // 本番用NextAuth実装
    return await nextAuth();
  }

  return null;
};
```

## 環境変数

### 設定ファイル

- **開発環境**: `.env.development`
- **ステージング**: `.env.staging`（予定）
- **本番環境**: `.env.production`

### 重要な環境変数

```bash
# 認証設定
AUTH_SECRET=your-secret-key
AUTH_URL=http://localhost:3000

# モック制御
NEXT_PUBLIC_USE_MOCK=true  # Mock環境
NEXT_PUBLIC_USE_MOCK=false # API環境

# データベース設定
DATABASE_URL=your-database-url
```

## 開発ワークフロー

### UI 開発時（Mock 環境）

1. `npm run dev:mock` で起動
2. `/admin` に直接アクセス
3. 認証なしで管理画面を開発・テスト
4. モックデータで UI 動作確認

### 認証機能開発時（API 環境）

1. `npm run dev:api` で起動
2. ログイン機能を実装・テスト
3. 認証フローを確認
4. 実データでの動作確認

### 本番デプロイ前（API 環境）

1. `npm run dev:api` で最終確認
2. 認証機能の動作テスト
3. 実データでの統合テスト
4. パフォーマンス確認

## セキュリティ考慮事項

### Mock 環境の制限

- **本番環境では無効**: `NODE_ENV=production` では認証バイパス無効
- **開発環境のみ**: 本番データへのアクセスは不可
- **一時的な機能**: 本番デプロイ時は自動的に無効化

### 認証バイパスの条件

```typescript
// セキュリティチェック
if (isMockEnv && process.env.NODE_ENV !== "production") {
  // Mock環境かつ本番環境でない場合のみバイパス
  return mockSession;
}
```

## トラブルシューティング

### よくある問題

#### 1. Mock 環境で認証が効かない

```bash
# 環境変数を確認
echo $NEXT_PUBLIC_USE_MOCK
# → "true" であることを確認

# 再起動
npm run dev:mock
```

#### 2. API 環境で認証エラー

```bash
# 環境変数を確認
echo $NEXT_PUBLIC_USE_MOCK
# → "false" であることを確認

# AUTH_SECRETが設定されているか確認
echo $AUTH_SECRET
```

#### 3. 本番環境で認証バイパスが有効

```bash
# NODE_ENVを確認
echo $NODE_ENV
# → "production" であることを確認

# 本番ビルドを再実行
npm run build
npm start
```
