# 環境設定と起動ガイド

## 概要

このドキュメントは、stats47 プロジェクトの環境設定から起動までを一元的に説明します。各環境の特徴、セットアップ手順、起動方法を実践的に提供します。

## 1. 環境一覧

| 環境            | コマンド                   | データソース  | 用途         |
| --------------- | -------------------------- | ------------- | ------------ |
| **development** | `npm run dev:mock`         | JSON ファイル | デザイン検証 |
| **development** | `npm run dev:api`          | e-Stat API    | API 開発     |
| **staging**     | `npm run dev:staging`      | リモート D1   | 統合テスト   |
| **production**  | `npm run build:production` | リモート D1   | 本番運用     |

### モードの切り替え

development 環境では`NEXT_PUBLIC_USE_MOCK`フラグでモードを切り替えます：

- **`npm run dev:mock`**: `NEXT_PUBLIC_USE_MOCK=true` - モックデータを使用
- **`npm run dev:api`**: `NEXT_PUBLIC_USE_MOCK=false` - e-Stat API を使用

## 2. 初回セットアップ

### 2.1 前提条件

```bash
# Node.js (18.x以上)
node --version

# npm
npm --version

# Git
git --version
```

### 2.2 プロジェクトのセットアップ

```bash
# リポジトリのクローン
git clone https://github.com/your-org/stats47.git
cd stats47

# 依存関係のインストール
npm install
```

### 2.3 環境変数ファイルの作成

#### 開発環境用（共通設定）

```bash
# .env.developmentファイルを作成
cp env.development.example .env.development
```

**.env.development の設定:**

```bash
# NextAuth v5設定（Auth.js）
AUTH_SECRET=<生成されたシークレット>
AUTH_URL=http://localhost:3000

# e-Stat API設定
ESTAT_API_BASE_URL=https://api.e-stat.go.jp/rest/3.0/app
NEXT_PUBLIC_ESTAT_APP_ID=your_estat_app_id

# モックデータパス
MOCK_DATA_PATH=data/mock

# 注意: NEXT_PUBLIC_USE_MOCKはpackage.jsonのスクリプトで指定します
# dev:mock → NEXT_PUBLIC_USE_MOCK=true
# dev:api → NEXT_PUBLIC_USE_MOCK=false
```

#### AUTH_SECRET の生成方法

**Node.js で生成（推奨）:**

```bash
node -e "console.log(require('crypto').randomBytes(32).toString('base64'))"
```

**OpenSSL で生成:**

```bash
openssl rand -base64 32
```

**セキュリティ注意:**

- 各環境で異なるシークレットを使用
- シークレットは最低 32 文字
- `.env.*` ファイルは Git にコミットしない

## 3. 環境別詳細

### 3.1 開発環境（モックモード）

#### 起動方法

```bash
npm run dev:mock
```

#### 特徴

- **データソース**: JSON ファイル（`data/mock/`）
- **データベース**: 接続不要
- **API 接続**: 不要
- **認証**: 動作あり（テストアカウント使用）
- **モード**: `NEXT_PUBLIC_USE_MOCK=true`

#### 適用場面

- デザインシステムの検証
- コンポーネントの開発
- オフライン開発
- 管理画面のデザイン確認

#### テストアカウント

- **管理者**: admin@stats47.local / admin123
- **一般ユーザー**: user@stats47.local / user123

#### アクセス先

- アプリケーション: `http://localhost:3000`
- 管理画面: `http://localhost:3000/admin`

### 3.2 開発環境（API モード）

#### 起動方法

```bash
npm run dev:api
```

#### 特徴

- **データソース**: e-Stat API
- **API 接続**: 必要（e-Stat API キー）
- **認証**: 動作あり
- **モード**: `NEXT_PUBLIC_USE_MOCK=false`

#### 前提条件

- e-Stat API キーの取得と設定

#### アクセス先

- アプリケーション: `http://localhost:3000`
- 管理画面: `http://localhost:3000/admin`（認証必須）
- 開発ツール: `http://localhost:3000/admin/dev-tools/estat-api`

### 3.3 ステージング環境（統合テスト用）

#### 起動方法

```bash
npm run dev:staging
```

#### 特徴

- **データソース**: リモート D1（ステージング）
- **API 接続**: e-Stat API に接続
- **用途**: 統合テスト

#### 前提条件

- ステージング用 D1 データベースの作成
- Cloudflare 認証設定

### 3.4 本番環境

#### ビルド方法

```bash
npm run build:production
```

#### 特徴

- **データソース**: リモート D1（本番）
- **用途**: 本番運用

## 4. Cloudflare D1 のセットアップ

### 4.1 Wrangler の認証

```bash
wrangler login
```

### 4.2 データベースの作成

```bash
# 開発用（ローカル）
wrangler d1 create stats47

# ステージング用
wrangler d1 create stats47_staging

# 本番用
wrangler d1 create stats47_production
```

### 4.3 マイグレーションの実行

```bash
# 開発環境（ローカルD1）
npx wrangler d1 migrations apply stats47 --local

# ステージング環境（リモートD1）
npx wrangler d1 migrations apply stats47_staging --env staging --remote

# 本番環境（リモートD1）
npx wrangler d1 migrations apply stats47 --remote
```

| **development** | 自由       | 認証必須 | 認証必須   |
| **staging**     | 自由       | 認証必須 | 認証必須   |

## 7. 環境選択の指針

### モック環境を選ぶ場合

- デザインシステムの検証
- コンポーネントの単体開発
- オフライン環境での作業
- 管理画面の UI デザイン確認

### 開発環境を選ぶ場合

- e-Stat API 連携の開発
- API レスポンスの検証
- データフェッチのテスト

### ステージング環境を選ぶ場合

- 本番データでの統合テスト
- パフォーマンステスト
- 本番デプロイ前の最終確認

### 本番環境を選ぶ場合

- 本番運用
- 本番データでの動作確認

## 8. トラブルシューティング

### 8.1 AUTH_SECRET エラー

```bash
# エラー: [auth][error] MissingSecret

# 原因: .env.developmentファイルが作成されていない、または読み込まれていない

# 解決方法:
# 1. 環境変数ファイルが存在するか確認
ls -la .env.*

# 2. .env.developmentを作成
cp env.development.example .env.development

# 3. AUTH_SECRETが設定されているか確認
cat .env.development | grep AUTH_SECRET

# 4. サーバーを再起動
npm run dev:mock
# または
npm run dev:api
```

### 8.2 環境変数が読み込まれない

```bash
# 現在の環境変数を確認
echo $NEXT_PUBLIC_USE_MOCK

# .envファイルの内容を確認
cat .env.development
```

### 8.3 データベース接続エラー

```bash
# ローカルD1の状態確認
npx wrangler d1 list

# リモートD1の状態確認
npx wrangler d1 list --remote

# データベースIDの確認
npx wrangler d1 list --json
```

### 8.4 データが表示されない

コンソールログで環境を確認してください：

```bash
# 期待される出力例：
# [development] NEXT_PUBLIC_USE_MOCK=true - Fetching estat metainfo from mock data...
# [development] NEXT_PUBLIC_USE_MOCK=false - Fetching estat metainfo from API...
# [staging] Fetching estat metainfo from database...
```

## 9. 技術的な補足

### 9.1 環境変数の読み込み仕組み

プロジェクトでは Next.js の標準的な環境変数読み込み機能を使用しています。

**環境ファイルの読み込み順序:**

1. `.env.development.local` (最優先)
2. `.env.local` (test を除く)
3. `.env.development`
4. `.env`

### 9.2 環境判定の方法

コード内で`process.env.NEXT_PUBLIC_USE_MOCK`を使用してモードを判定：

```typescript
// モックモードの判定例
const useMock = process.env.NEXT_PUBLIC_USE_MOCK === "true";

if (useMock) {
  // モックデータを使用
  return fetchFromJSON();
} else {
  // e-Stat APIを使用
  return fetchFromAPI();
}
```

## 10. セキュリティ注意事項

### 10.1 環境変数の管理

- `.env.*`ファイルは Git にコミットしない
- 機密情報は Cloudflare Secrets で管理
- 本番環境の API キーは慎重に管理
- 各環境で異なる`AUTH_SECRET`を使用

### 10.2 データベースアクセス

- 本番環境のデータベースは読み取り中心
- 書き込み操作は慎重に実行
- マイグレーションは段階的に適用
