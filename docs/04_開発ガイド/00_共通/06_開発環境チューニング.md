# 開発環境チューニング（実務最小）

> 本文書はローカル開発・デザイン検証・DX 改善に特化したガイドです。
> 総論は `docs/04_開発ガイド/パフォーマンス最適化.md`、本番向け最適化は `docs/04_開発ガイド/本番パフォーマンス最適化.md` を参照してください。

## 1. 5 分セットアップ（最短）

```bash
# モック開発（最速）
cp env.mock.example .env.mock
npm run dev:mock

# ローカルD1で開発（API接続あり）
cp env.development.example .env.development
npx wrangler d1 migrations apply stats47 --local
npm run dev:api
```

## 2. モック環境での開発

### 特徴

- データソース: `data/mock/database/*.json`
- DB/API 接続不要、即時起動、オフライン可

### ディレクトリ

```
data/mock/database/
├── ranking_items.json
├── ranking_values.json
└── estat_metainfo.json
```

### 手順

```bash
cp env.mock.example .env.mock
npm run dev:mock
```

### Storybook での利用

```typescript
// .storybook/main.ts
import type { StorybookConfig } from "@storybook/nextjs";

const config: StorybookConfig = {
  env: (config) => ({
    ...config,
    NEXT_PUBLIC_ENV: "mock",
  }),
};

export default config;
```

### モックデータの更新

```bash
npx wrangler d1 execute stats47 --local --command "SELECT * FROM ranking_items LIMIT 10;" --json > data/mock/database/ranking_items.json
npx wrangler d1 execute stats47 --local --command "SELECT * FROM ranking_values LIMIT 50;" --json > data/mock/database/ranking_values.json
npx wrangler d1 execute stats47 --local --command "SELECT * FROM estat_metainfo LIMIT 50;" --json > data/mock/database/estat_metainfo.json
```

## 3. 環境変数と設定

### `.env.development`

```bash
# e-Stat API
ESTAT_API_BASE_URL=https://api.e-stat.go.jp/rest/3.0/app
ESTAT_API_KEY=your-dev-api-key

# 環境
NEXT_PUBLIC_ENV=development
NODE_ENV=development

# モック
MOCK_DATA_PATH=data/mock
```

### `.env.mock`

```bash
NEXT_PUBLIC_ENV=mock
NODE_ENV=development
MOCK_DATA_PATH=data/mock
```

### 設定コード例

```typescript
// /src/infrastructure/config.ts
export const config = {
  env: process.env.NEXT_PUBLIC_ENV || "development",
  estat: {
    baseUrl:
      process.env.ESTAT_API_BASE_URL || "https://api.e-stat.go.jp/rest/3.0/app",
    apiKey: process.env.ESTAT_API_KEY,
  },
  cloudflare: {
    d1: process.env.CLOUDFLARE_D1_DATABASE_ID,
    r2: process.env.CLOUDFLARE_R2_BUCKET,
    accountId: process.env.CLOUDFLARE_ACCOUNT_ID,
    apiToken: process.env.CLOUDFLARE_API_TOKEN,
  },
  mock: {
    dataPath: process.env.MOCK_DATA_PATH || "data/mock",
  },
} as const;
export type Config = typeof config;
```

## 4. ローカル D1（SQLite）運用

### 基本コマンド（development）

```bash
# スキーマ適用
npx wrangler d1 execute stats47 --local --file=./database/schemas/main.sql

# Seed投入
npx wrangler d1 execute stats47 --local --file=./database/seeds/users_seed.sql

# テーブル/データ確認
npx wrangler d1 execute stats47 --local --command "SELECT name FROM sqlite_master WHERE type='table';"
npx wrangler d1 execute stats47 --local --command "SELECT * FROM users LIMIT 5;"
```

### Next.js 起動

```bash
npm run dev:api
```

## 5. CI/CD・開発スクリプト

`cross-env` を利用して環境を切り替えます。

```bash
npm install --save-dev cross-env
```

```json
{
  "scripts": {
    "dev:mock": "cross-env NEXT_PUBLIC_ENV=mock next dev --turbopack",
    "dev:api": "cross-env NODE_ENV=development NEXT_PUBLIC_ENV=development next dev --turbopack"
  }
}
```

## 6. トラブルシューティング（開発）

```bash
# 1) 環境変数が読まれない
echo $NEXT_PUBLIC_ENV && cat .env.development

# 2) データが表示されない
# 期待ログ: [mock] ... / [development] ...

# 3) ローカルD1の状態
npx wrangler d1 list
```
