# D1 データベース実装ガイド

## 概要

このドキュメントは、Cloudflare D1 データベースの**実装と使い方**を説明する実践的なガイドです。スキーマ設計の詳細は[技術設計ドキュメント](../../../01_技術設計/04_インフラ設計/01_データベース設計.md)を参照してください。

## クイックスタート

### 環境設定

| 環境             | データベース                     | 接続方法       | 用途         |
| ---------------- | -------------------------------- | -------------- | ------------ |
| **ローカル開発** | `.wrangler/state/.../xxx.sqlite` | better-sqlite3 | 開発・テスト |
| **ステージング** | Cloudflare D1 (dev)              | REST API       | 統合テスト   |
| **本番**         | Cloudflare D1 (prod)             | REST API       | 本番運用     |

## 基本的なクエリ操作

### SELECT クエリ

#### 単一レコードの取得

```typescript
// 都道府県コードで都道府県を取得
const result = await db
  .prepare("SELECT * FROM users WHERE email = ?")
  .bind(email)
  .first();

console.log(result); // { id: "...", email: "...", name: "..." }
```

#### 複数レコードの取得

```typescript
// 全都道府県を取得
const result = await db
  .prepare("SELECT * FROM estat_metainfo WHERE area_type = ? ORDER BY updated_at DESC")
  .bind("prefecture")
  .all();

console.log(result.results); // [{ stats_data_id: "...", ... }, ...]
console.log(result.meta); // { duration: 10, rows_read: 47, ... }
```

#### JOINクエリ

```typescript
// ランキング項目とデータソースをJOIN
const result = await db
  .prepare(`
    SELECT
      ri.ranking_key,
      ri.label,
      ri.unit,
      ds.name as data_source_name
    FROM ranking_items ri
    INNER JOIN data_sources ds ON ri.data_source_id = ds.id
    WHERE ri.is_active = 1
    ORDER BY ri.label
  `)
  .all();

console.log(result.results);
```

### INSERT クエリ

#### 単一レコードの挿入

```typescript
// 新しいユーザーを作成
const result = await db
  .prepare(`
    INSERT INTO users (id, email, name, role)
    VALUES (?, ?, ?, ?)
  `)
  .bind(userId, email, name, "user")
  .run();

console.log(result.success); // true
console.log(result.meta.last_row_id); // 挿入されたレコードのID
```

#### 複数レコードの一括挿入（バッチ）

```typescript
// 複数のランキングデータを一括挿入
const statements = prefectures.map((pref) =>
  db
    .prepare(`
      INSERT INTO ranking_data (ranking_key, area_code, value)
      VALUES (?, ?, ?)
    `)
    .bind(rankingKey, pref.prefCode, pref.value)
);

const results = await db.batch(statements);
console.log(`${results.length} records inserted`);
```

### UPDATE クエリ

```typescript
// ユーザー情報を更新
const result = await db
  .prepare(`
    UPDATE users
    SET name = ?, updated_at = CURRENT_TIMESTAMP
    WHERE id = ?
  `)
  .bind(newName, userId)
  .run();

console.log(result.meta.changes); // 更新された行数
```

### DELETE クエリ

```typescript
// 古いセッションを削除
const result = await db
  .prepare(`
    DELETE FROM sessions
    WHERE expires < ?
  `)
  .bind(new Date().toISOString())
  .run();

console.log(`${result.meta.changes} sessions deleted`);
```

## 実践的なユースケース

### ユースケース1: ランキング項目の取得

```typescript
// app/api/ranking/items/route.ts
import { getRequestContext } from "@cloudflare/next-on-pages";
import { NextResponse } from "next/server";

export async function GET(request: Request) {
  try {
    const { env } = getRequestContext();
    const db = env.STATS47_DB;

    const result = await db
      .prepare(`
        SELECT
          ri.id,
          ri.ranking_key,
          ri.label,
          ri.name,
          ri.unit,
          ri.map_color_scheme,
          ds.name as data_source_name
        FROM ranking_items ri
        INNER JOIN data_sources ds ON ri.data_source_id = ds.id
        WHERE ri.is_active = 1
        ORDER BY ri.display_order_in_group, ri.label
      `)
      .all();

    return NextResponse.json({
      items: result.results,
      count: result.results.length,
    });
  } catch (error) {
    console.error("Failed to fetch ranking items:", error);
    return NextResponse.json(
      { error: "Failed to fetch ranking items" },
      { status: 500 }
    );
  }
}
```

### ユースケース2: グループ別ランキング項目の取得

```typescript
// グループIDでランキング項目を取得
export async function getRankingItemsByGroup(
  db: D1Database,
  groupId: number
) {
  const result = await db
    .prepare(`
      SELECT
        ri.*,
        rg.name as group_name
      FROM ranking_items ri
      INNER JOIN ranking_groups rg ON ri.group_id = rg.id
      WHERE ri.group_id = ? AND ri.is_active = 1
      ORDER BY ri.display_order_in_group
    `)
    .bind(groupId)
    .all();

  return result.results;
}
```

### ユースケース3: データソースメタデータの取得

```typescript
// ランキング項目の全メタデータを取得
export async function getMetadataForRankingItem(
  db: D1Database,
  rankingItemId: number
) {
  const result = await db
    .prepare(`
      SELECT
        dsm.area_type,
        dsm.calculation_type,
        dsm.metadata,
        ds.name as data_source_name
      FROM data_source_metadata dsm
      INNER JOIN data_sources ds ON dsm.data_source_id = ds.id
      WHERE dsm.ranking_item_id = ?
    `)
    .bind(rankingItemId)
    .all();

  return result.results.map((row) => ({
    ...row,
    metadata: JSON.parse(row.metadata as string),
  }));
}
```

### ユースケース4: トランザクション

```typescript
// 複数のテーブルを更新（トランザクション）
export async function createRankingItemWithMetadata(
  db: D1Database,
  itemData: RankingItemData,
  metadataList: MetadataData[]
) {
  // D1はトランザクションをサポートしていないため、batchを使用
  const statements = [
    // ランキング項目を挿入
    db
      .prepare(`
        INSERT INTO ranking_items (ranking_key, label, name, unit, data_source_id)
        VALUES (?, ?, ?, ?, ?)
      `)
      .bind(
        itemData.rankingKey,
        itemData.label,
        itemData.name,
        itemData.unit,
        itemData.dataSourceId
      ),

    // メタデータを挿入（複数）
    ...metadataList.map((metadata) =>
      db
        .prepare(`
          INSERT INTO data_source_metadata (ranking_item_id, data_source_id, area_type, metadata)
          VALUES (?, ?, ?, ?)
        `)
        .bind(
          itemData.id,
          metadata.dataSourceId,
          metadata.areaType,
          JSON.stringify(metadata.params)
        )
    ),
  ];

  const results = await db.batch(statements);

  // 全ての操作が成功したか確認
  const allSuccess = results.every((r) => r.success);

  if (!allSuccess) {
    throw new Error("Transaction failed");
  }

  return { success: true };
}
```

### ページネーション

```typescript
// ページネーションでデータを取得
export async function getRankingItemsPaginated(
  db: D1Database,
  page: number = 1,
  pageSize: number = 20
) {
  const offset = (page - 1) * pageSize;

  const [items, countResult] = await Promise.all([
    // データ取得
    db
      .prepare(`
        SELECT * FROM ranking_items
        WHERE is_active = 1
        ORDER BY label
        LIMIT ? OFFSET ?
      `)
      .bind(pageSize, offset)
      .all(),

    // 総数取得
    db
      .prepare(`
        SELECT COUNT(*) as total FROM ranking_items
        WHERE is_active = 1
      `)
      .first(),
  ]);

  return {
    items: items.results,
    pagination: {
      page,
      pageSize,
      total: countResult.total,
      totalPages: Math.ceil(countResult.total / pageSize),
    },
  };
}
```

### クエリ実行時間の計測

```typescript
export async function measureQueryPerformance<T>(
  queryFn: () => Promise<T>
): Promise<{ result: T; duration: number }> {
  const start = performance.now();
  const result = await queryFn();
  const duration = performance.now() - start;

  console.log(`Query executed in ${duration.toFixed(2)}ms`);

  return { result, duration };
}

// 使用例
const { result, duration } = await measureQueryPerformance(() =>
  db.prepare("SELECT * FROM ranking_items").all()
);
```

## トラブルシューティング

### データベース接続エラー

**症状**: `D1_ERROR: Database not found`

**原因**: 環境変数が未設定、またはバインディングが正しくない

**解決策**:
1. `wrangler.toml`でD1バインディングを確認
   ```toml
   [[d1_databases]]
   binding = "STATS47_DB"
   database_name = "stats47"
   database_id = "your-database-id"
   ```
2. 環境変数が正しく設定されているか確認

### マイグレーションエラー

**症状**: `Error: Migration already applied`

**原因**: 既に適用されたマイグレーションを再実行しようとしている

**解決策**:
```bash
# マイグレーション状態を確認
npx wrangler d1 migrations list stats47 --local

# 必要に応じてロールバック（注意！）
# D1は自動ロールバックをサポートしていないため、手動でSQLを実行
```

### プリペアドステートメント

エラー

**症状**: `Error: Too many SQL variables`

**原因**: プレースホルダーの数が制限（最大999個）を超えている

**解決策**: バッチ処理を複数回に分割
```typescript
// 大量のレコードを挿入する場合
const BATCH_SIZE = 500;
for (let i = 0; i < records.length; i += BATCH_SIZE) {
  const batch = records.slice(i, i + BATCH_SIZE);
  const statements = batch.map((record) =>
    db.prepare("INSERT INTO ...").bind(...)
  );
  await db.batch(statements);
}
```

### JSONパースエラー

**症状**: `JSON parse error`

**原因**: metadata列に不正なJSON文字列が格納されている

**解決策**: バリデーションを追加
```typescript
function parseMetadata(metadataStr: string): object | null {
  try {
    return JSON.parse(metadataStr);
  } catch (error) {
    console.error("Invalid JSON in metadata:", metadataStr);
    return null;
  }
}
```

## ベストプラクティス

### 1. プリペアドステートメントを常に使用

```typescript
// ❌ 悪い例：SQLインジェクションのリスク
const result = await db.prepare(`SELECT * FROM users WHERE email = '${email}'`).all();

// ✅ 良い例：プリペアドステートメント
const result = await db.prepare("SELECT * FROM users WHERE email = ?").bind(email).all();
```

### 2. エラーハンドリング

```typescript
try {
  const result = await db.prepare("...").bind(...).all();
  return result.results;
} catch (error) {
  // エラー情報をログに記録
  console.error("Database error:", {
    error: error.message,
    stack: error.stack,
    query: "...",
  });

  // ユーザーには安全なエラーメッセージを返す
  throw new Error("Failed to fetch data");
}
```

### 3. 接続の再利用

```typescript
// ❌ 悪い例：毎回接続を取得
export async function getUser() {
  const db = getDb();
  return await db.prepare("...").first();
}

export async function getUsers() {
  const db = getDb();
  return await db.prepare("...").all();
}

// ✅ 良い例：接続を引数として渡す
export async function getUser(db: D1Database, userId: string) {
  return await db.prepare("...").bind(userId).first();
}

export async function getUsers(db: D1Database) {
  return await db.prepare("...").all();
}
```

### 4. 型安全性の確保

```typescript
interface User {
  id: string;
  email: string;
  name: string;
  role: string;
}

export async function getUser(db: D1Database, userId: string): Promise<User | null> {
  const result = await db
    .prepare("SELECT * FROM users WHERE id = ?")
    .bind(userId)
    .first<User>();

  return result;
}
```
