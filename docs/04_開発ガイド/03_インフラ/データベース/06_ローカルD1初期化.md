# ローカル D1 初期化ガイド

## 概要

開発環境でのローカル D1 データベースの初期化方法とトラブルシューティングを説明します。

## 前提条件

- Node.js がインストールされていること
- Wrangler CLI がインストールされていること（プロジェクトにインストール済み）
- `better-sqlite3` がインストールされていること

## 初期化方法

### 1. 初回セットアップ

初回セットアップ時は、以下のコマンドを実行してローカル D1 データベースを初期化します:

```bash
npm run db:init:local
```

このコマンドは以下の処理を自動的に実行します:

1. Wrangler CLI を使用してローカル D1 を起動
2. `database/schemas/main.sql` のスキーマを適用
3. `database/migrations/` のマイグレーションファイルを適用
4. データベースの状態を確認

### 2. データベースのリセット

ローカル D1 データベースを完全にリセットして再初期化する場合:

```bash
npm run db:reset:local
```

**注意**: このコマンドは `.wrangler/state/v3/d1/` ディレクトリを削除してから再初期化します。既存のデータはすべて失われます。

### 3. 既存データベースへのマイグレーション適用

データベースが既に存在する場合でも、新しいマイグレーションを適用できます:

```bash
npm run db:migrations:local
```

## トラブルシューティング

### エラー: "Cannot open database because the directory does not exist"

**原因**: ローカル D1 データベースのディレクトリが存在しない

**解決方法**: 以下のコマンドでデータベースを初期化してください:

```bash
npm run db:init:local
```

### エラー: "Local D1 database file does not exist"

**原因**: データベースファイルが存在しない、または破損している

**解決方法**: データベースをリセットしてください:

```bash
npm run db:reset:local
```

### エラー: "Wrangler CLI is not installed"

**原因**: Wrangler CLI がプロジェクトにインストールされていない

**解決方法**: 以下のコマンドでインストール:

```bash
npm install
```

または:

```bash
npm install -g wrangler
```

### マイグレーションエラーの解決

マイグレーションの適用に失敗した場合:

1. エラーメッセージを確認
2. マイグレーションファイルの構文を確認
3. 必要に応じてデータベースをリセット:

```bash
npm run db:reset:local
```

## データベースの状態確認

### データベースファイルの場所

ローカル D1 データベースファイルは以下の場所に保存されます:

```
.wrangler/state/v3/d1/miniflare-D1DatabaseObject/b3c084603f7be30f18c6a887d294217e3e6b2010e83e9d09910eae3515a26884.sqlite
```

### テーブル一覧の確認

以下のコマンドでテーブル一覧を確認できます:

```bash
npx wrangler d1 execute stats47 --local --command "SELECT name FROM sqlite_master WHERE type='table' ORDER BY name;"
```

### スキーマの確認

特定のテーブルのスキーマを確認:

```bash
npx wrangler d1 execute stats47 --local --command ".schema users"
```

### データの確認

テーブルのデータを確認:

```bash
npx wrangler d1 execute stats47 --local --command "SELECT * FROM users LIMIT 10;"
```

## 開発ワークフロー

### 日常的な開発

1. 開発サーバーの起動:

   ```bash
   npm run dev:api
   ```

2. マイグレーションの適用:

   ```bash
   npm run db:migrations:local
   ```

3. データベースの確認:
   ```bash
   npx wrangler d1 execute stats47 --local --command "SELECT COUNT(*) FROM ranking_items;"
   ```

### 開発終了時のデータベース保持

ローカル D1 データベースは `.wrangler/` ディレクトリに保存されるため、開発サーバーを停止してもデータは保持されます。

`.wrangler/` ディレクトリは `.gitignore` に含まれているため、Git にはコミットされません。

## バックアップとリストア

### ローカルデータベースのバックアップ

以下のコマンドでローカル D1 のバックアップを取得:

```bash
npm run db:dump:local
```

バックアップファイルは `database/backups/local_dump.sql` に保存されます。

### リストア方法

現在の方法では、データベースをリセットしてからインポートする方法が簡単です:

```bash
npm run db:reset:local
npm run db:migrations:local
```

## 関連コマンド

- `npm run db:init:local` - ローカル D1 の初期化
- `npm run db:reset:local` - ローカル D1 のリセット
- `npm run db:migrations:local` - マイグレーションの適用
- `npm run db:dump:local` - データベースのダンプ
