# 環境設定ガイド

## 概要

stats47 プロジェクトにおける環境固有の設定（URL、ドメイン、環境変数等）について説明します。

## 環境別 URL 設定

### 推奨 URL 構成

| 環境                       | URL                           | 用途               |
| -------------------------- | ----------------------------- | ------------------ |
| **開発環境**               | `http://localhost:3000`       | ローカル開発       |
| **開発環境（パブリック）** | `https://dev.stats47.com`     | チーム共有・テスト |
| **ステージング環境**       | `https://staging.stats47.com` | 本番前テスト       |
| **本番環境**               | `https://stats47.com`         | 本番運用           |

### 環境別ドメイン構成

#### メインドメイン

```
stats47.com                    # 本番環境
├── www.stats47.com           # 本番環境（www）
├── dev.stats47.com           # 開発環境
├── staging.stats47.com       # ステージング環境
└── api.stats47.com           # API エンドポイント
```

#### R2 ストレージドメイン

```
api.stats47.com               # メタ情報キャッシュ
├── dev.stats47.com           # 開発用R2
├── staging.stats47.com       # ステージング用R2
└── geoshape.stats47.com      # GeoShapeデータ（既存）
```

### 環境判定の実装

#### 環境設定の取得

```typescript
// src/lib/config/environment.ts
export type Environment = "production" | "staging" | "development" | "mock";

export function getEnvironment(): Environment {
  const env = process.env.NEXT_PUBLIC_ENV;

  if (
    env === "production" ||
    env === "staging" ||
    env === "development" ||
    env === "mock"
  ) {
    return env;
  }

  // デフォルトは本番環境
  return "production";
}

export function isDevelopment(): boolean {
  return getEnvironment() === "development";
}

export function isProduction(): boolean {
  return getEnvironment() === "production";
}

export function isMock(): boolean {
  return getEnvironment() === "mock";
}
```

#### URL 生成の実装

```typescript
// src/lib/config/urls.ts
export function getAppUrl(): string {
  const env = process.env.NEXT_PUBLIC_ENV || "production";

  const urls = {
    production: "https://stats47.com",
    staging: "https://staging.stats47.com",
    development: "https://dev.stats47.com",
    mock: "http://localhost:3000",
  };

  return urls[env] || urls.production;
}

export function getApiUrl(): string {
  const baseUrl = getAppUrl();
  return `${baseUrl}/api`;
}

export function getR2MetaInfoUrl(environment: string = "production"): string {
  const urls = {
    production: "https://api.stats47.com",
    staging: "https://staging.stats47.com",
    development: "https://dev.stats47.com",
    mock: "", // mock環境ではR2を使用しない
  };

  return urls[environment] || urls.production;
}

export function getR2GeoShapeUrl(): string {
  return "https://geoshape.stats47.com";
}
```

## SSL 証明書

### Cloudflare での自動 SSL

Cloudflare を使用している場合、SSL 証明書は自動的に管理されます。

#### SSL 設定の確認

```bash
# SSL設定の確認
curl -I https://stats47.com
curl -I https://dev.stats47.com
curl -I https://staging.stats47.com
```

#### SSL 設定の最適化

Cloudflare ダッシュボードで以下の設定を推奨：

```
SSL/TLS設定:
- 暗号化モード: Full (strict)
- Edge証明書: Universal SSL
- 常時HTTPS: 有効
- HSTS: 有効（本番環境のみ）
```

## トラブルシューティング

### URL 接続エラー

#### 問題: カスタムドメインにアクセスできない

**原因**:

- DNS 設定が正しくない
- プロパゲーション待ち
- Cloudflare の設定ミス

**解決方法**:

```bash
# DNS解決の確認
nslookup dev.stats47.com
nslookup staging.stats47.com
nslookup stats47.com

# Cloudflareの設定確認
npx wrangler pages project list
npx wrangler pages domain list --project-name=stats47-dev
```

#### 問題: HTTPS 接続エラー

**原因**:

- SSL 証明書の問題
- Cloudflare の設定ミス

**解決方法**:

```bash
# SSL証明書の確認
openssl s_client -connect stats47.com:443 -servername stats47.com

# CloudflareのSSL設定確認
# ダッシュボード > SSL/TLS > 概要
```

### ドメイン設定エラー

#### 問題: サブドメインが反映されない

**原因**:

- DNS レコードの設定ミス
- プロパゲーション待ち

**解決方法**:

```bash
# DNSレコードの確認
dig dev.stats47.com
dig staging.stats47.com

# Cloudflareでの設定確認
# ダッシュボード > DNS > レコード
```

#### 問題: R2 カスタムドメインが動作しない

**原因**:

- R2 バケットの設定ミス
- DNS 設定の問題

**解決方法**:

```bash
# R2カスタムドメインの確認
wrangler r2 bucket custom-domain list stats47

# DNS設定の確認
nslookup api.stats47.com
```

### SSL 証明書エラー

#### 問題: SSL 証明書の警告

**原因**:

- 証明書の期限切れ
- 設定の不整合

**解決方法**:

```bash
# 証明書の詳細確認
openssl s_client -connect stats47.com:443 -servername stats47.com -showcerts

# Cloudflareでの再発行
# ダッシュボード > SSL/TLS > Edge証明書 > 再発行
```

## 監視・ログ

### 環境別ログ設定

```typescript
// src/lib/logging/environment.ts
export function logEnvironment(message: string, data?: any) {
  const env = process.env.NEXT_PUBLIC_ENV || "production";

  console.log(`[${env.toUpperCase()}] ${message}`, data);
}

export function logApiCall(endpoint: string, method: string, status: number) {
  logEnvironment(`API Call: ${method} ${endpoint} - ${status}`);
}

export function logError(error: Error, context?: any) {
  const env = process.env.NEXT_PUBLIC_ENV || "production";

  console.error(`[${env.toUpperCase()}] ERROR:`, error, context);
}
```

```typescript
// src/lib/monitoring/environment-metrics.ts
export function trackEnvironmentMetrics(
```
