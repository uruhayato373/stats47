# データベースドキュメント

## 概要

stats47 プロジェクトのデータベースとストレージに関する統合ドキュメントです。Cloudflare D1 データベースと R2 ストレージの設計、実装、運用について包括的に説明しています。

## ドキュメント構成

### 01_D1 データベース概要.md

**用途**: D1 データベースの設計とアーキテクチャの理解

**内容**:

- プロジェクト情報とデータベース環境
- スキーマ設計（テーブル定義、ER 図）
- アーキテクチャ（環境別設定、接続方法、データフロー）
- 設計原則（正規化、インデックス戦略、パフォーマンス考慮）

**対象者**: アーキテクト、設計者、新規参加者

### 02_D1 実装ガイド.md

**用途**: D1 データベースの開発・実装手順

**内容**:

- 環境セットアップ（ローカル開発環境、Wrangler 設定）
- マイグレーション管理（作成手順、実行方法、ロールバック）
- クエリパターン（CRUD 操作、JOIN、トランザクション、N+1 問題回避）
- ベストプラクティス（セキュリティ、パフォーマンス、エラーハンドリング、テスト）

**対象者**: 開発者、実装者

### 06\_ローカル D1 初期化.md

**用途**: ローカル D1 データベースの初期化とトラブルシューティング

**内容**:

- ローカル D1 の初期化方法
- データベースのリセット
- トラブルシューティング（よくあるエラーと解決方法）
- 開発ワークフロー
- バックアップとリストア

**対象者**: 開発者

### 03_R2 ストレージガイド.md

**用途**: R2 ストレージの設計と実装

**内容**:

- 概要（R2 とは、D1 との使い分け、ユースケース）
- セットアップ（バケット作成、アクセス許可、環境別設定）
- 実装パターン（保存・読み込み、キャッシュ戦略、バッチ処理）
- デプロイ（手順、環境別デプロイ、トラブルシューティング）
- ベストプラクティス（命名規則、メタデータ活用、パフォーマンス、コスト最適化）

**対象者**: 開発者、インフラエンジニア

### 04\_環境設定ガイド.md

**用途**: 環境固有の設定（URL、ドメイン、環境変数）

**内容**:

- 環境別 URL 設定（開発・ステージング・本番）
- カスタムドメイン設定（Cloudflare Pages、DNS 設定、SSL 証明書）
- 環境変数管理（ローカル・Cloudflare Pages・wrangler.toml）
- トラブルシューティング（接続エラー、ドメイン設定エラー、SSL 証明書エラー）

**対象者**: インフラエンジニア、DevOps エンジニア

### 05\_運用・保守ガイド.md

**用途**: 運用保守の手順とトラブルシューティング

**内容**:

- 日常運用（バックアップ、モニタリング、ログ確認、パフォーマンス監視）
- トラブルシューティング（接続エラー、マイグレーションエラー、クエリエラー、パフォーマンス問題）
- メンテナンス（データクリーンアップ、インデックス再構築、容量管理）
- 変更履歴（リファクタリング履歴、スキーマ変更履歴、主要な変更点）

**対象者**: 運用エンジニア、SRE

## 技術スタック

### データベース

- **Cloudflare D1**: SQLite ベースのサーバーレスデータベース
- **Wrangler**: D1 の管理・マイグレーションツール
- **better-sqlite3**: ローカル開発用 SQLite クライアント

### ストレージ

- **Cloudflare R2**: S3 互換オブジェクトストレージ
- **AWS SDK**: S3 互換 API でのアクセス
- **カスタムドメイン**: CDN 最適化

### 環境管理

- **Cloudflare Pages**: ホスティング
- **wrangler.toml**: 設定管理
- **環境変数**: 環境別設定
