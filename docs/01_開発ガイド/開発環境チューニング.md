# 開発環境チューニング（実務最小）

## 1. 目的/適用範囲

- ローカル/モック/ステージングでの開発体験と速度を最小労力で改善。

## 2. SWR（開発）設定

- ログを有効化、短い `dedupingInterval`、`revalidateOnFocus: true`。

## 3. ビルド/型チェック

- Turbopack、型チェック分離（`tsc --noEmit`）、Storybook/Playground 併用。

## 4. モック/スタブ

- `NEXT_PUBLIC_ENV=mock`、`data/mock/**` の活用、API 依存を外して高速化。

## 5. ログ/デバッグ

- フェッチャー/フックの onError/onSuccess ログを dev 限定出力。

## 6. チェックリスト（PR 前）

- [ ] SWR キー null 制御あり
- [ ] dev 設定が本番に混入していない
- [ ] 重い処理は `useMemo`/遅延ロード
- [ ] モックで再現できる
