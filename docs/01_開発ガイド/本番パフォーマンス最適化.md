# 本番パフォーマンス最適化（実務最小）

## 1. 目的/適用範囲

- 本番環境の応答時間・スループット・安定性を最小の運用コストで確保する。

## 2. レンダリング戦略

- SSR/ISR: 初期体験が重要・再利用可能なデータは ISR（`revalidate`）を設定。
- CSR+SWR: ユーザー依存/頻更新データは CSR + SWR。

## 3. キャッシュ/ヘッダー

- Edge/CDN を前提に `Cache-Control: public, s-maxage=3600, stale-while-revalidate=86400`。
- API Route はデータ特性ごとに TTL を明示。

## 4. 画像/静的資産

- Next/Image、WebP、適切な `sizes`、遅延ロード、CDN 配信。

## 5. SWR 設定（本番）

- `revalidateOnFocus: false`、`dedupingInterval: 300000`（5 分）を基準にデータ特性で調整。

## 6. 計測/KPI

- Core Web Vitals（LCP/INP/CLS）、API p95、エラー率、キャッシュヒット率。

## 7. セキュリティ/ヘッダー

- HTTPS 強制、CSP、XSS 対策、API Key のサーバ側限定使用。

## 10. Cloudflare D1（staging/production）

> リモート D1 の運用は必ず `--remote` を使用。誤ってローカルに適用しない。

### `wrangler.toml` 抜粋

```toml
[env.staging]
NODE_ENV = "production"
[[env.staging.d1_databases]]
binding = "STATS47_DB"
database_name = "stats47_staging"
database_id = "39f18714-83bf-423c-9956-dcc9e466affb"
migrations_dir = "database/migrations"

[env.production]
NODE_ENV = "production"
[[env.production.d1_databases]]
binding = "STATS47_DB"
database_name = "stats47"
database_id = "${PROD_D1_DATABASE_ID}"
migrations_dir = "database/migrations"
```

### 基本コマンド

```bash
# テーブル一覧
npx wrangler d1 execute stats47_staging --env staging --remote --command "SELECT name FROM sqlite_master WHERE type='table';"
npx wrangler d1 execute stats47 --remote --command "SELECT name FROM sqlite_master WHERE type='table';"

# 読み取り
npx wrangler d1 execute stats47_staging --env staging --remote --command "SELECT * FROM users LIMIT 5;"
npx wrangler d1 execute stats47 --remote --command "SELECT * FROM users LIMIT 5;"

# マイグレーション
npx wrangler d1 migrations apply stats47_staging --env staging --remote
npx wrangler d1 migrations apply stats47 --remote
```

## 11. デプロイ（Pages）

```bash
# ステージング
npx wrangler pages deploy --env staging

# 本番
npx wrangler pages deploy --env production
```

## 12. データ同期（推奨フロー）

```bash
# 本番 → ステージング（エクスポート→変換→インポート）
npx wrangler d1 execute stats47 --remote --command "SELECT * FROM ranking_items;" --json > database/backups/$(date +%Y%m%d)/ranking_items.json
node scripts/import-to-staging.js
npx wrangler d1 execute stats47_staging --env staging --remote --file=database/backups/$(date +%Y%m%d)/import_to_staging.sql
```

## 13. Secrets 管理

```bash
# ステージング
wrangler secret put ESTAT_API_KEY --env staging
wrangler secret put CLOUDFLARE_API_TOKEN --env staging

# 本番
wrangler secret put ESTAT_API_KEY --env production
wrangler secret put CLOUDFLARE_API_TOKEN --env production
```

## 14. 注意事項

- Staging/Production では常に `--remote` を付与
- ローカルデータをリモートへコピーしない（本番データ → ステージングで検証）
- マイグレーションは `local → staging → production` の順で適用
