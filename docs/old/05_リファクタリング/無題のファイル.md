```markdown
## 🎯 プロジェクト概要

- **Next.js**：フロントエンドおよびAPIルートを統合
- **e-Stat API**：都道府県データを取得
- **Cloudflare D1**：統計データ・ランキングキャッシュ管理（SQLite互換）
- **Cloudflare R2**：画像やJSONなどの静的アセット保存
- **Cloudflare Pages**：デプロイ環境

複数の開発環境（API接続あり／なし）を切り替えながら、
デザイン、開発、ステージング、本番を安全に運用するためのベストプラクティスをまとめます。

## 🏗️ 環境構成

| 環境名 | 用途 | 特徴 |
|---------|------|------|
| **development** | ローカル開発 | e-Stat API接続（またはモック） |
| **mock** | デザイン検証 | API非接続、スタブデータ使用 |
| **staging** | ステージング | D1/R2接続あり、本番と同構成 |
| **production** | 本番 | 実運用データ、最適化ビルド |
```

```ts
export const config = {

  },

```

### `/lib/api/estat.ts`

```ts
import { config } from '@/lib/config';
import mockData from '@/data/mock/prefecture-stats.json';

export async function getPrefectureStats() {
  if (config.useMock) return mockData;

  const res = await fetch(`${config.estat.baseUrl}/getStatsData?appId=${config.estat.apiKey}&...`);
  if (!res.ok) throw new Error('Failed to fetch e-Stat data');
  return res.json();
}
```

### DB接続例

```ts
export const runtime = 'edge';

export async function getDB() {
  if (process.env.NEXT_PUBLIC_ENV === 'local') {
    return (await import('@cloudflare/workers-types')).env.DB;
  } else {
    const response = await fetch(
      `https://api.cloudflare.com/client/v4/accounts/${process.env.CLOUDFLARE_ACCOUNT_ID}/d1/database/${process.env.CLOUDFLARE_D1_DATABASE_ID}/query`,
      { headers: { 'Authorization': `Bearer ${process.env.CLOUDFLARE_API_TOKEN}` } }
    );
    return response;
  }
}
```

完璧です。  
以下は、先ほどの**Next.js × Cloudflare D1/R2 × e-Stat API構成**を前提にした  
**GitHub Actions（CI/CD）設定例**です。

Cloudflare PagesとD1を環境別に自動デプロイする仕組みを含み、  
「feature → develop → main」ブランチ戦略に完全対応しています。

---

```yaml
# .github/workflows/deploy.yml
name: 🚀 Deploy Next.js + Cloudflare D1/R2 Dashboard

on:
  push:
    branches:
      - main
      - develop
  pull_request:
    branches:
      - develop

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest

    env:
      NODE_VERSION: 20
      NEXT_PUBLIC_ENV: ${{ github.ref_name == 'main' && 'production' || 'staging' }}
      CLOUDFLARE_ACCOUNT_ID: ${{ secrets.CLOUDFLARE_ACCOUNT_ID }}
      CLOUDFLARE_API_TOKEN: ${{ secrets.CLOUDFLARE_API_TOKEN }}
      ESTAT_API_KEY: ${{ secrets.ESTAT_API_KEY }}
      CLOUDFLARE_D1_DATABASE_ID: ${{ github.ref_name == 'main' && secrets.CLOUDFLARE_D1_PROD_ID || secrets.CLOUDFLARE_D1_STAGING_ID }}
      CLOUDFLARE_R2_BUCKET: ${{ github.ref_name == 'main' && secrets.CLOUDFLARE_R2_PROD_BUCKET || secrets.CLOUDFLARE_R2_STAGING_BUCKET }}

    steps:
      - name: 🧰 Checkout repository
        uses: actions/checkout@v4

      - name: 🟢 Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: ${{ env.NODE_VERSION }}
          cache: 'npm'

      - name: 📦 Install dependencies
        run: npm ci

      - name: 🧱 Build Next.js app
        run: npm run build

      - name: 🧩 Run database migrations (D1)
        run: |
          if [ "${{ github.ref_name }}" == "main" ]; then
            npx wrangler d1 migrations apply --env production
          else
            npx wrangler d1 migrations apply --env staging
          fi
        env:
          CLOUDFLARE_API_TOKEN: ${{ secrets.CLOUDFLARE_API_TOKEN }}

      - name: ☁️ Deploy to Cloudflare Pages
        run: |
          if [ "${{ github.ref_name }}" == "main" ]; then
            npx wrangler pages deploy .vercel/output/static --env production
          else
            npx wrangler pages deploy .vercel/output/static --env staging
          fi
        env:
          CLOUDFLARE_API_TOKEN: ${{ secrets.CLOUDFLARE_API_TOKEN }}
          CLOUDFLARE_ACCOUNT_ID: ${{ secrets.CLOUDFLARE_ACCOUNT_ID }}

  preview:
    if: github.event_name == 'pull_request'
    runs-on: ubuntu-latest
    env:
      CLOUDFLARE_ACCOUNT_ID: ${{ secrets.CLOUDFLARE_ACCOUNT_ID }}
      CLOUDFLARE_API_TOKEN: ${{ secrets.CLOUDFLARE_API_TOKEN }}
      NEXT_PUBLIC_ENV: preview

    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
      - run: npm ci
      - run: npm run build

      - name: 🔍 Deploy preview to Cloudflare Pages
        run: npx wrangler pages deploy .vercel/output/static --branch=${{ github.head_ref }}
        env:
          CLOUDFLARE_API_TOKEN: ${{ secrets.CLOUDFLARE_API_TOKEN }}
```

## 🔍 構成の説明

|ステップ|内容|
|---|---|
|**on.push**|`main` または `develop` への push をトリガー|
|**on.pull_request**|PR時にプレビュー環境を自動デプロイ|
|**build-and-deploy job**|Nodeセットアップ → Next.jsビルド → D1マイグレーション → Cloudflare Pagesデプロイ|
|**環境変数分岐**|`github.ref_name` により `staging` / `production` を自動判定|
|**preview job**|PRブランチ単位で一時的な Cloudflare Pages 環境を作成|

## ⚙️ 開発者フロー（自動化後）

1. **機能追加**  
    → `feature/estat-map` ブランチを作成  
    → `.env.development` でローカル検証（ローカルD1使用）
    
2. **PR作成**  
    → `develop` へ Pull Request  
    → GitHub Actions が Cloudflare Pages にプレビュー環境を自動デプロイ  
    → デザインやデータ確認
    
3. **マージ後**  
    → `develop` ブランチが push されると **staging環境** へ自動反映  
    → D1 ステージングDBへ自動マイグレーション
    
4. **リリース時**  
    → `main` にマージすると **production環境** に自動デプロイ  
    → 本番D1へマイグレーション・R2への保存開始
