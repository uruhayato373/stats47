# プロジェクト要件定義書

## 1. プロジェクト概要

### 1.1 プロジェクト名
- **正式名称**: 統計で見る都道府県
- **技術名**: stats47
- **プロジェクトコード**: STATS47

### 1.2 プロジェクトの背景

#### 1.2.1 社会的背景
- 地方創生・地域活性化への関心の高まり
- データドリブンな意思決定の重要性の増大
- 統計データの可視化ニーズの拡大
- 地域間格差の可視化需要

#### 1.2.2 技術的背景
- e-Stat APIの公開と活用促進
- Web技術の発達（React、D3.js等）
- オープンデータの推進
- クラウドインフラの普及

#### 1.2.3 市場背景
- 地域統計データの需要増加
- 無料で利用できる統計可視化ツールの不足
- 専門知識不要の統計ツールの需要

### 1.3 プロジェクトの目的

#### 1.3.1 主要目的
1. **データアクセスの民主化**: 誰もが地域統計データにアクセスできる環境の提供
2. **地域理解の促進**: 複雑な統計データを分かりやすく可視化
3. **意思決定支援**: データに基づいた地域理解と意思決定の支援
4. **統計リテラシー向上**: 統計データの読み方と活用法の普及

#### 1.3.2 副次的目的
1. **e-Stat API活用促進**: 政府統計の活用事例の提供
2. **地域活性化支援**: 地域の特徴と魅力の発見・発信
3. **研究・教育支援**: 学術研究や教育での統計データ活用
4. **政策立案支援**: 自治体の政策立案におけるデータ活用

### 1.4 プロジェクトスコープ

#### 1.4.1 対象範囲
- **地理的範囲**: 日本全国47都道府県
- **データ範囲**: 政府統計、自治体統計、民間統計
- **時系列範囲**: 過去10年程度のデータ
- **ユーザー範囲**: 一般ユーザー、研究者、自治体職員、企業経営者

#### 1.4.2 対象外
- 市区町村レベルの詳細データ（将来拡張予定）
- リアルタイムデータ
- 予測・推計データ（将来拡張予定）
- 個人情報を含むデータ

## 2. 技術要件

### 2.1 技術スタック

#### 2.1.1 フロントエンド
- **フレームワーク**: Next.js 15
- **UI ライブラリ**: React 19
- **言語**: TypeScript
- **スタイリング**: Tailwind CSS 4
- **データ可視化**: Recharts, D3.js
- **状態管理**: Jotai
- **ビルドツール**: Turbopack

#### 2.1.2 バックエンド
- **API**: Next.js API Routes
- **データベース**: PostgreSQL
- **認証**: NextAuth.js
- **ORM**: Prisma
- **キャッシュ**: Redis（将来実装）

#### 2.1.3 インフラ・デプロイ
- **ホスティング**: Vercel
- **データベース**: Supabase
- **CDN**: Vercel Edge Network
- **監視**: Vercel Analytics
- **ドメイン**: 独自ドメイン（将来実装）

#### 2.1.4 開発・運用
- **バージョン管理**: Git, GitHub
- **CI/CD**: GitHub Actions
- **テスト**: Vitest, Testing Library
- **リンター**: ESLint
- **フォーマッター**: Prettier
- **ドキュメント**: Markdown

### 2.2 パフォーマンス要件

#### 2.2.1 レスポンス時間
- **初期ページ読み込み**: 3秒以内
- **データ表示**: 2秒以内
- **ページ遷移**: 1秒以内
- **API レスポンス**: 1秒以内

#### 2.2.2 スループット
- **同時接続数**: 100ユーザー
- **1日あたりのページビュー**: 1,000PV
- **API リクエスト**: 1日1,000リクエスト

#### 2.2.3 可用性
- **稼働率**: 99%以上
- **ダウンタイム**: 月4時間以内
- **メンテナンス**: 月1回、深夜時間帯

### 2.3 セキュリティ要件

#### 2.3.1 データ保護
- **個人情報**: 収集・保存しない
- **API キー**: 環境変数で管理
- **データベース**: 暗号化接続（SSL/TLS）
- **ログ**: 個人情報を含まない

#### 2.3.2 アクセス制御
- **認証**: NextAuth.js
- **認可**: ロールベースアクセス制御
- **セッション管理**: セキュアなセッション管理
- **CSRF 保護**: CSRF トークン実装

#### 2.3.3 脆弱性対策
- **依存関係**: 定期的な脆弱性チェック
- **セキュリティヘッダー**: 適切なHTTPヘッダー設定
- **入力検証**: すべての入力データの検証
- **SQL インジェクション**: Prisma ORM による対策

## 3. 機能要件

### 3.1 主要機能

#### 3.1.1 データ可視化機能
- **地域選択**: 都道府県レベルの地域選択
- **データ表示**: 棒グラフ、折れ線グラフ、円グラフ
- **地域比較**: 複数地域の同時比較
- **時系列表示**: 過去データとの比較

#### 3.1.2 ブログ機能
- **記事表示**: 統計解説記事の表示
- **地域特集**: 都道府県別の特集記事
- **検索機能**: 記事の検索・フィルタリング
- **関連記事**: 関連記事の自動表示

#### 3.1.3 ユーザー機能
- **認証**: ユーザー登録・ログイン
- **プロフィール**: ユーザー情報管理
- **お気に入り**: 地域・記事のお気に入り機能
- **コメント**: 記事へのコメント機能

### 3.2 管理機能

#### 3.2.1 コンテンツ管理
- **記事管理**: ブログ記事の作成・編集・削除
- **データ管理**: 統計データの更新・管理
- **ユーザー管理**: ユーザーアカウントの管理
- **コメント管理**: コメントの承認・削除

#### 3.2.2 分析機能
- **アクセス解析**: ページビュー・ユーザー行動分析
- **パフォーマンス監視**: システムパフォーマンス監視
- **エラーログ**: エラーログの収集・分析
- **レポート**: 定期レポートの生成
