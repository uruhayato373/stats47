# 統合データプラットフォーム 包括的設計書

## エグゼクティブサマリー

本設計書は、stats47（統計で見る都道府県）プロジェクトを、単一データソース（e-Stat API）から、多様なAPIと統計データ、地図情報を一元的に管理・可視化できる包括的データプラットフォームへと拡張するための設計指針を定めます。

### プロジェクトビジョン

**「日本の地域データを網羅的に集約し、誰もが理解できる形で提供する、日本最大級のオープンデータプラットフォーム」**

### 主要な拡張ポイント

1. **多様なデータソース統合**: 政府統計、地域経済、気象、交通など複数APIの統合
2. **高度な地図機能**: 複数の地図ライブラリを活用した柔軟な可視化
3. **リアルタイムデータ**: 気象や交通などの動的データ表示
4. **市区町村対応**: 全国1,700以上の自治体データへの完全対応
5. **データダウンロード**: CSV、JSON、GeoJSON形式でのデータエクスポート

## 1. 利用可能なAPIとデータソース

### 1.1 統計データAPI

#### 1.1.1 e-Stat API（政府統計総合窓口）★最優先★

**概要**
- 提供元: 総務省統計局・独立行政法人統計センター
- 公式URL: https://www.e-stat.go.jp/api/
- データ形式: XML, JSON, CSV
- 更新頻度: 統計により異なる（月次、年次など）
- 利用料金: 無料（要API登録）

**提供データ**
- 国勢調査（人口、世帯）
- 経済センサス（事業所、企業統計）
- 住宅・土地統計調査
- 農林業センサス
- 労働力調査
- 家計調査
- その他650以上の政府統計

**API制限**
- リクエスト: 1日1,000回まで
- レスポンスサイズ: 最大100,000件/リクエスト

**統合優先度**: ★★★★★（既存実装あり、継続拡充）

**活用例**
- 都道府県別人口動態ダッシュボード
- 産業構造比較グラフ
- 世帯構成トレンド分析

---

#### 1.1.2 RESAS API（地域経済分析システム）★高優先★

**概要**
- 提供元: 内閣府まち・ひと・しごと創生本部事務局、経済産業省
- 公式URL: https://opendata.resas-portal.go.jp/
- データ形式: JSON
- 更新頻度: 定期更新（年次）
- 利用料金: 無料（要API登録）

**提供データ**
- 人口構成・人口動態
- 地域経済循環（産業構造、企業数、従業者数）
- 観光データ（宿泊者数、観光入込数）
- 農業データ（産出額、経営体数）
- 工業データ（製造品出荷額、事業所数）
- 商業データ（販売額、事業所数）
- まちづくりデータ（不動産取引、地価）

**API制限**
- リクエスト: 制限あり（詳細は要確認）
- 市区町村レベルのデータ提供あり

**統合優先度**: ★★★★★（地域経済分析に必須）

**活用例**
- 地域経済ダッシュボード
- 観光統計マップ
- 産業別従業者数推移グラフ
- 製造業出荷額ランキング

---

#### 1.1.3 統計でみる市区町村のすがた（e-Stat統合データ）

**概要**
- 提供元: 総務省統計局
- 公式URL: https://www.stat.go.jp/data/s-sugata/index.html
- データ形式: Excel, CSV
- 更新頻度: 年次（2025年版は6月公開）
- 利用料金: 無料

**提供データ**
- 全国1,747市区町村の主要統計
- 人口、世帯、産業、経済、教育、医療、福祉など約200指標
- 時系列データ（複数年）

**統合優先度**: ★★★★☆（市区町村ダッシュボード強化）

**活用例**
- 市区町村ランキング（任意指標）
- 市区町村間比較ダッシュボード
- 地域特性分析

## 6. データフロー設計

### 6.1 データ取得フロー

#### 6.1.1 SSR（Server-Side Rendering）フロー

```
1. ユーザーリクエスト
   ↓
2. Next.js Server Component
   ↓
3. データサービス呼び出し
   ↓
4. キャッシュチェック (Cloudflare D1/R2)
   ├─ キャッシュヒット → 8. へ
   └─ キャッシュミス
      ↓
5. アダプター選択（データソース判定）
   ↓
6. 外部API呼び出し（並列実行）
   ├─ e-Stat API
   ├─ RESAS API
   ├─ 気象庁API
   └─ 国交省API
   ↓
7. データ変換・正規化（共通形式へ）
   ↓
8. キャッシュ保存（TTL設定）
   ↓
9. HTMLレンダリング（サーバー側）
   ↓
10. レスポンス返却（HTML + JSON）
    ↓
11. クライアント側ハイドレーション
    ↓
12. インタラクティブUIの有効化
```

#### 6.1.2 CSR（Client-Side Rendering）フロー

```
1. ユーザーインタラクション（フィルター変更など）
   ↓
2. SWRによるデータフェッチ
   ↓
3. API Route (/api/data)
   ↓
4. データサービス呼び出し
   ↓
5. キャッシュチェック
   ├─ キャッシュヒット → レスポンス返却
   └─ キャッシュミス
      ↓
6. アダプター実行
   ↓
7. 外部API呼び出し
   ↓
8. データ変換
   ↓
9. キャッシュ保存
   ↓
10. JSON レスポンス返却
    ↓
11. クライアント側レンダリング更新
```

#### 6.2.2 TTL（Time To Live）設定

| データ種別 | TTL | 理由 |
|-----------|-----|------|
| 統計データ（年次） | 24時間 | 年1回更新 |
| 統計データ（月次） | 6時間 | 月1回更新 |
| 地図データ（GeoJSON） | 7日間 | ほぼ変更なし |
| 気象データ | 5分 | 頻繁に更新 |
| 交通データ | 5分 | リアルタイム |
| ランキングデータ | 1時間 | 動的計算結果 |
| メタデータ | 24時間 | ほぼ変更なし |

### 6.3 エラーハンドリング

#### 6.3.1 エラー階層

```typescript
// エラーレベル
enum ErrorLevel {
  CRITICAL = 'critical', // システム全体停止
  ERROR = 'error',       // 機能停止
  WARNING = 'warning',   // 一部機能制限
  INFO = 'info',         // 情報レベル
}

// エラークラス階層
class DashboardError extends Error {
  level: ErrorLevel;
  code: string;
  details: Record<string, any>;
}

class APIError extends DashboardError {
  apiName: string;
  statusCode: number;
}

class DataValidationError extends DashboardError {
  validationErrors: ValidationError[];
}
```

#### 6.3.2 エラー復旧戦略

| エラータイプ | 復旧戦略 | ユーザー表示 |
|------------|---------|------------|
| API タイムアウト | リトライ（3回） | 「データ取得中...」 |
| API レート制限 | キャッシュフォールバック | キャッシュデータ表示 + 警告 |
| データ不整合 | デフォルト値使用 | 「一部データ利用不可」 |
| ネットワークエラー | リトライ + キャッシュ | 「オフラインモード」 |
| 認証エラー | ログイン画面へリダイレクト | エラーメッセージ |
