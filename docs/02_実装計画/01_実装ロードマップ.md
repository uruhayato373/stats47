## 基本方針

1. **配信と計測が最優先** — 新規コンテンツ制作より、既存資産の配信と効果計測に時間を使う
2. **技術開発は凍結** — 新機能・スキル開発・リファクタリングは Sprint 3 以降に限定
3. **週次メトリクス追跡を義務化** — 毎週月曜に GA4/GSC/SNS の数値を記録
4. **計画は 1 Sprint 先まで** — Sprint 完了時の実績データに基づいて次を設計

## 週次メトリクス追跡

スプリントタスクではなく恒常的な運用ルール。

**日曜夜: `/weekly-review`（その週の振り返り・記録）**
**月曜朝: `/weekly-plan`（翌週の計画・メトリクス取得）**

```
月曜に取得するデータ:
- GA4: PV（週次/累計）、ユーザー数、流入チャネル別比率
- GSC: インプレッション、クリック、平均掲載順位、上位クエリ 10 件
- YouTube: 総再生数、登録者数（`/fetch-youtube-data`）
- X: フォロワー数、投稿数（週）、インプレッション合計
- note: フォロワー数、記事数、PV
- AdSense: 推定収益（週次）
- アフィリエイト: クリック数、成約数、収益
```

記録先: GitHub Issue（ラベル `weekly-review`、タイトル `[Weekly Review] YYYY-Www`）+ `.claude/skills/analytics/{ga4,gsc}-improvement/reference/snapshots/YYYY-Www/` CSV
