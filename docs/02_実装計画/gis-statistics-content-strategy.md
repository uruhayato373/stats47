# 統計 × GIS 掛け合わせコンテンツ戦略

## Context

stats47 を「都道府県統計の可視化サイト」から一歩進め、**様々な統計と GIS 情報を掛け合わせて新たな価値を創造するサイト**に育てたい。起点は公務員向けの note マガジン (既存 `koumuin-claude-code` シリーズの延長)。

この戦略は 2026-05-20 のセッションでの議論を元にする。議論で確定した要点:

- **公務員 × GIS は素直な組み合わせ** — 自治体職員は GIS を都市計画・防災・統計分析で実務利用する。KSJ は国交省の公的データなので親和性が高い。`koumuin-claude-code` で読者層と配信チャネルは実証済み。
- **「動いているものを見せる」戦略は有効** — 働く掛け合わせコンテンツは文章説明より圧倒的に伝わり、SNS でも回る。そのページ自体が stats47 の SEO コンテンツになり、note 記事のネタにもなる。コンテンツと配信は対立しない。
- **"demo" ではなく "コンテンツ"** — 作るのは公開・SEO 対象・本番データの恒久ページ。使い捨ての試作物 (demo) ではない。「demo」と呼ぶと品質バー・メンテ前提が下振れするため、本ドキュメントでは「掛け合わせコンテンツ」と呼ぶ。
- **警戒すべきは「汎用化」** — 失敗モードはコンテンツを作ることではなく、「設定可能な多レイヤー GIS ツール (掛け合わせエンジン)」を作り始めること。todo-ran (1,501 指標) / uub (1,843 指標) は網羅性で既に強く、stats47 の差別化は「信頼性 × 網羅性 × Web 送客」であって生の掛け合わせ機能ではない。

## 設計原則 (スコープ規律)

1. **1 問 1 ページ** — 掛け合わせコンテンツは「具体的な 1 つの問い」に答える 1 ページ。ハードコードでよい。
2. **汎用化しない** — レイヤー選択 UI・相関エンジン・設定可能化を作り始めない。膨らませた瞬間に汎用ツール地獄。
3. **コンテンツ = 実ページ** — 掛け合わせコンテンツは stats47 の本番ページ。それが note 記事の題材になる。別物として二重に作らない。
4. **判断ゲートを置く** — コンテンツの反応 (Web 送客・note 反応) を測ってから横展開・ツール化を判断する。需要未検証のまま投資しない。

## キラー掛け合わせ問い (候補)

公務員に刺さり、感情喚起もあり、既存データで実現できる問い:

| 問い | データ | 状態 |
|---|---|---|
| 過疎地域に医療機関がどれだけあるか | A17 過疎地域 × P04 医療機関 | ✅ 実装済み (Phase 2) |
| 日が照る場所に太陽光発電は建っているか | 年間日照時間 (e-Stat) × P03 太陽光発電施設 | 🔜 第 2 弾 企画中 — `gis-cross-02-sunshine-solar.md` |
| 津波浸水想定区域に高齢者福祉施設がどれだけあるか | A40 津波浸水想定 × P14 福祉施設 | ❌ 不採用 — A40 が 8 県分のみ取得 (47 県ランキング不可) |
| 土砂災害警戒区域と小学校区の重なり | A33 土砂災害警戒区域 × A27 小学校区 | 候補 (両方 47 県完備) |
| 豪雪地帯の消防署配置 | A22 豪雪地帯 × P17 消防署 | 候補 (A22 は豪雪地帯のある 23 県のみ) |

第 2 弾 (日照 × 太陽光) は note アクセス実測 (日照時間ランキングが note 1 位・728 ビュー) を根拠に企画。詳細は `gis-cross-02-sunshine-solar.md`。statistic × GIS-point 型で、#1 (GIS-area × GIS-point) とは掛け合わせ方が異なる (空間結合不要)。

第 1 弾の選定経緯: 当初 A40 × P14 (災害 × 福祉) を想定したが、データ実体調査で A40 が 8 県・300MB しか取得されておらず 47 県ランキング不可と判明。両方 47 県完備の **A17 過疎地域 × P04 医療機関** に変更して実装した。

## フェーズ構成

### Phase 1 — 公務員向け GIS note マガジン

`koumuin-claude-code` の延長として「公務員のための GIS 活用」テーマを立ち上げる。

- 記事企画 (backlog): GIS とは / KSJ データの入手 / QGIS 入門 / 統計との重ね合わせ実例 など
- note 有料マガジン化の検討 (自治体職員・自治体に予算あり、koumuin シリーズの課金導線を継承)
- 配置先: `docs/30_note記事企画/backlog/` に記事企画、原稿は `docs/31_note記事原稿/`

### Phase 2 — 掛け合わせコンテンツ #1 (過疎 × 医療) ✅ 実装済み

「過疎地域に医療機関がどれだけあるか」を可視化する公開ページ。

- ページ: `/gis-cross/depopulation-medical` — 47都道府県 choropleth ランキング + 県別オーバーレイ詳細マップ
- データ: A17 過疎地域 (polygon) × P04 医療機関 (point) を空間結合 (point-in-polygon)
- 前処理スクリプト `apps/web/scripts/export-depopulation-medical-snapshot.ts` で 47 県集計 → R2 snapshot
- 既存 `LeafletChoroplethMap` を再利用、公開ページ (SEO 対象、sitemap 登録済み)
- スコープ規律どおり 1 問 1 ページ・ハードコード。レイヤー選択 UI なし
- 実装計画: `~/.claude/plans/stateless-stargazing-teapot.md`

### Phase 3 — コンテンツの note 記事化 + 反応測定

Phase 2 のページを題材に note 記事を書く (「過疎地域の医療空白をデータで見る」等)。

- GA4 で `/gis-cross/depopulation-medical` の流入・滞在を測定
- note 記事の反応 (スキ・コメント・Web 送客 UTM) を測定
- 測定期間と評価基準を記事公開時に決める (`evidence-based-judgment.md` に従い、想定値と検証コマンドを明記)

### Phase 4 — 横展開 or ツール化の判断ゲート

Phase 3 の実測を見て判断:

- **反応あり** → キラー問いリストの残りをコンテンツ化 (Phase 2 を繰り返す、1 問 1 ページ)。横展開がさらに伸びれば、その時点で初めて「レイヤー選択 UI 等の汎用化」を検討。
- **反応なし** → 掛け合わせコンテンツ路線を止め、note マガジン (Phase 1) 単独に絞る。原因仮説と次の検証を記録。

なお「掛け合わせ」とは別系統の低コスト施策として、KSJ の施設 point データ (郵便局・文化施設・公共施設など Tier A ~16 件) を `register-ksj-rankings` で「都道府県別 ○○数」ランキング化できる。これは判断ゲートの対象外で、記事展開と並行してよい補完作業。

## トレードオフ (明示)

この戦略の肝は **スコープ規律**。

- A17 × P04 を 1 ページ・ハードコードで出すなら低リスク・短期間で「動くもの」が出る。
- そこで「レイヤーを選べるように」「相関も出そう」と膨らませた瞬間に汎用ツール地獄に入る。
- 1 問 1 ページで止める覚悟がある限り、この戦略は強い。
