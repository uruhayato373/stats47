# 環境設定ガイド

## 概要

stats47 プロジェクトにおける環境固有の設定（URL、ドメイン、環境変数等）について説明します。

詳細なセットアップ手順については、[環境設定と起動ガイド](../../04_開発ガイド/02_環境設定と起動ガイド.md)を参照してください。

## 環境戦略

### 環境の種類

stats47 プロジェクトでは、以下の 4 つの環境を使用します：

| 環境            | 説明                       | 主な用途                                |
| --------------- | -------------------------- | --------------------------------------- |
| **mock**        | モックデータを使用する環境 | オフライン開発、デザイン検証、Storybook |
| **development** | ローカル開発環境           | ローカル開発、API 開発                  |
| **staging**     | ステージング環境           | 本番前テスト、統合テスト                |
| **production**  | 本番環境                   | 本番運用                                |

### 各環境の目的と特徴

#### Mock 環境

**目的**: オフラインでの開発とデザイン検証

**特徴**:

- **データソース**: JSON ファイル（`data/mock/`）
- **データベース**: 不要
- **API 接続**: 不要
- **認証**: バイパス有効（自動的に管理者としてログイン）
- **用途**: UI 開発、コンポーネント開発、Storybook、デザイン検証

#### Development 環境

**目的**: ローカルでの開発とテスト

**特徴**:

- **データソース**: ローカル D1 Database または e-Stat API
- **データベース**: ローカル D1（`.wrangler/state/v3/d1`）
- **API 接続**: e-Stat API（オプション）
- **認証**: NextAuth（Auth.js）による認証管理
- **用途**: フル機能の開発とテスト

#### Staging 環境

**目的**: 本番前の統合テスト

**特徴**:

- **データソース**: リモート D1 Database（ステージング）
- **データベース**: ステージング用 D1
- **API 接続**: e-Stat API
- **認証**: 本番と同等の認証システム
- **用途**: 統合テスト、パフォーマンステスト、本番前の最終確認

#### Production 環境

**目的**: 本番運用

**特徴**:

- **データソース**: リモート D1 Database（本番）
- **データベース**: 本番用 D1
- **API 接続**: e-Stat API
- **認証**: 本番用認証システム（厳格なセキュリティ）
- **用途**: 本番運用

### 環境別データソース戦略

環境別のデータソース戦略については、[データベース設計](01_データベース設計.md#環境別データソース戦略)を参照してください。

| 環境            | データソース  | 接続方法                    | 用途                      |
| --------------- | ------------- | --------------------------- | ------------------------- |
| **mock**        | JSON ファイル | `data/mock/database/*.json` | オフライン開発、Storybook |
| **development** | ローカル D1   | `.wrangler/state/v3/d1`     | ローカル開発              |
| **staging**     | リモート D1   | Cloudflare D1 API           | 本番前テスト              |
| **production**  | リモート D1   | Cloudflare D1 API           | 本番運用                  |

### モックモードと API モードの切り替え戦略

Development 環境では、`NEXT_PUBLIC_USE_MOCK`フラグにより、モックモードと API モードを切り替えます。

**設計思想**:

- **モックモード** (`NEXT_PUBLIC_USE_MOCK=true`): UI 開発、デザイン検証に最適化
- **API モード** (`NEXT_PUBLIC_USE_MOCK=false`): 実データでの開発とテスト

**切り替え方法**:

- `npm run dev:mock`: モックモード（`NEXT_PUBLIC_USE_MOCK=true`）
- `npm run dev:api`: API モード（`NEXT_PUBLIC_USE_MOCK=false`）

## 環境別 URL 設定

### 推奨 URL 構成

| 環境                       | URL                           | 用途               |
| -------------------------- | ----------------------------- | ------------------ |
| **開発環境**               | `http://localhost:3000`       | ローカル開発       |
| **開発環境（パブリック）** | `https://dev.stats47.com`     | チーム共有・テスト |
| **ステージング環境**       | `https://staging.stats47.com` | 本番前テスト       |
| **本番環境**               | `https://stats47.com`         | 本番運用           |

### 環境別ドメイン構成

#### メインドメイン

```
stats47.com                    # 本番環境
├── www.stats47.com           # 本番環境（www）
├── dev.stats47.com           # 開発環境
├── staging.stats47.com       # ステージング環境
└── api.stats47.com           # API エンドポイント
```

#### R2 ストレージドメイン

```
api.stats47.com               # メタ情報キャッシュ
├── dev.stats47.com           # 開発用R2
├── staging.stats47.com       # ステージング用R2
└── geoshape.stats47.com      # GeoShapeデータ（既存）
```

### 環境判定ロジックの設計

#### 設計思想

環境判定は`src/lib/environment.ts`で一元管理します。

**設計原則**:

- **型安全性**: TypeScript の型システムを活用した環境型の定義
- **一元管理**: 環境判定ロジックを一箇所に集約
- **デフォルト値**: 環境変数が未設定の場合の安全なデフォルト値

#### 環境判定の仕組み

**環境判定フロー**:

```
1. process.env.NEXT_PUBLIC_ENV から環境を取得
2. 有効な環境値（production/staging/development/mock）か検証
3. 無効な場合はデフォルト値（production）を返却
4. モックモード判定（NEXT_PUBLIC_USE_MOCK）と統合
```

**実装場所**: `src/lib/environment.ts`

**主要機能**:

- `resolveEnvironment()`: 現在の実行環境を解決
- `checkMockDataEnabled()`: モックデータモードが有効か判定
- `buildEnvironmentConfig()`: 環境情報とモック設定を統合した設定オブジェクトを構築

#### 環境別の動作制御

環境判定により、以下の動作が制御されます：

- **データソースの選択**: JSON/ローカル D1/リモート D1
- **認証動作**: バイパス有効/無効
- **キャッシュ戦略**: モック/実データ
- **ログレベル**: 開発/本番

## 環境変数設計

### 環境変数の分類

環境変数は以下のカテゴリに分類されます：

| カテゴリ         | 説明                         | 例                                                       |
| ---------------- | ---------------------------- | -------------------------------------------------------- |
| **認証**         | 認証システムの設定           | `AUTH_SECRET`, `AUTH_URL`                                |
| **API**          | 外部 API の設定              | `NEXT_PUBLIC_ESTAT_APP_ID`                               |
| **データソース** | データベース・ストレージ設定 | `CLOUDFLARE_D1_DATABASE_ID`, `CLOUDFLARE_R2_BUCKET_NAME` |
| **環境識別**     | 環境の判別                   | `NEXT_PUBLIC_ENV`, `NODE_ENV`                            |
| **URL 設定**     | アプリケーション・API の URL | `NEXT_PUBLIC_APP_URL`, `NEXT_PUBLIC_R2_METAINFO_URL`     |

### 環境変数の一覧

#### 認証関連

| 環境変数名    | 型       | 説明                                  | 必須                           | 環境   |
| ------------- | -------- | ------------------------------------- | ------------------------------ | ------ |
| `AUTH_SECRET` | `string` | NextAuth（Auth.js）のシークレットキー | 必須（API/Staging/Production） | 全環境 |
| `AUTH_URL`    | `string` | 認証システムの URL                    | 必須（API/Staging/Production） | 全環境 |

#### API 関連

| 環境変数名                 | 型       | 説明                             | 必須                           | 環境                   |
| -------------------------- | -------- | -------------------------------- | ------------------------------ | ---------------------- |
| `NEXT_PUBLIC_ESTAT_APP_ID` | `string` | e-Stat API のアプリケーション ID | 必須（API/Staging/Production） | API/Staging/Production |

#### データソース関連

| 環境変数名                  | 型       | 説明               | 必須                       | 環境               |
| --------------------------- | -------- | ------------------ | -------------------------- | ------------------ |
| `CLOUDFLARE_D1_DATABASE_ID` | `string` | D1 データベース ID | 任意（Staging/Production） | Staging/Production |
| `CLOUDFLARE_R2_BUCKET_NAME` | `string` | R2 バケット名      | 任意（Staging/Production） | Staging/Production |
| `MOCK_DATA_PATH`            | `string` | モックデータのパス | 任意                       | Mock               |

#### 環境識別

| 環境変数名             | 型                                                     | 説明                   | 必須     | 環境        |
| ---------------------- | ------------------------------------------------------ | ---------------------- | -------- | ----------- |
| `NEXT_PUBLIC_ENV`      | `"production" \| "staging" \| "development" \| "mock"` | 実行環境               | 任意     | 全環境      |
| `NODE_ENV`             | `"production" \| "development"`                        | Node.js の実行環境     | 自動設定 | 全環境      |
| `NEXT_PUBLIC_USE_MOCK` | `"true" \| "false"`                                    | モックモード使用フラグ | 任意     | Development |

#### URL 設定

| 環境変数名                    | 型       | 説明                         | 必須 | 環境               |
| ----------------------------- | -------- | ---------------------------- | ---- | ------------------ |
| `NEXT_PUBLIC_APP_URL`         | `string` | アプリケーションのベース URL | 任意 | Staging/Production |
| `NEXT_PUBLIC_R2_METAINFO_URL` | `string` | R2 メタ情報の URL            | 任意 | Staging/Production |
| `NEXT_PUBLIC_R2_GEOSHAPE_URL` | `string` | R2 GeoShape の URL           | 任意 | Staging/Production |

### 命名規則

#### `NEXT_PUBLIC_*` プレフィックス

**意味**: Next.js がクライアント側に公開する環境変数に使用

**設計思想**:

- クライアント側で使用する環境変数は`NEXT_PUBLIC_`プレフィックスが必要
- サーバー側のみで使用する環境変数はプレフィックス不要
- セキュリティのため、機密情報は`NEXT_PUBLIC_`プレフィックスを付けない

**使用例**:

- `NEXT_PUBLIC_ESTAT_APP_ID`: クライアント側から e-Stat API を呼び出す場合
- `AUTH_SECRET`: サーバー側のみで使用するためプレフィックスなし

### セキュリティ考慮事項

1. **機密情報の取り扱い**

   - シークレットキー、API キーは Git にコミットしない
   - `.env.*`ファイルは`.gitignore`に追加
   - 環境変数の値は具体的な値ではなく説明を記載

2. **環境変数の公開範囲**

   - `NEXT_PUBLIC_*`プレフィックス付きはクライアント側に公開される
   - 機密情報は`NEXT_PUBLIC_`プレフィックスを付けない

3. **環境別の値管理**

   - 各環境で異なる値を使用（特にシークレットキー）
   - 開発ガイドを参照して適切に設定

詳細な環境変数の設定手順については、[環境設定と起動ガイド](../../04_開発ガイド/02_環境設定と起動ガイド.md)を参照してください。

## カスタムドメイン設定

### Cloudflare Pages での設定

#### 1. プロジェクト作成

```bash
# 開発環境用プロジェクト
npx wrangler pages project create stats47-dev

# ステージング環境用プロジェクト
npx wrangler pages project create stats47-staging

# 本番環境用プロジェクト
npx wrangler pages project create stats47
```

#### 2. カスタムドメインの設定

```bash
# 開発環境
npx wrangler pages domain add dev.stats47.com --project-name=stats47-dev

# ステージング環境
npx wrangler pages domain add staging.stats47.com --project-name=stats47-staging

# 本番環境
npx wrangler pages domain add stats47.com --project-name=stats47
npx wrangler pages domain add www.stats47.com --project-name=stats47
```

### R2 バケットのカスタムドメイン

#### バケット別ドメイン設定

```bash

# ステージング環境R滑动バケット
wrangler r2 bucket custom-domain add stats47-staging --domain staging.stats47.com

# 開発環境R2バケット
wrangler r2 bucket custom-domain add stats47-development --domain dev.stats47.com
```

```bash
# カスタムドメインの確認
wrangler r2 bucket custom-domain list stats47
wrangler r2 bucket custom-domain list stats47-staging
wrangler r2 bucket custom-domain list stats47-development
```

## SSL 証明書

### Cloudflare での自動 SSL

Cloudflare を使用している場合、SSL 証明書は自動的に管理されます。

**設計方針**:

- **暗号化モード**: Full (strict)
- **Edge 証明書**: Universal SSL
- **常時 HTTPS**: 有効
- **HSTS**: 本番環境のみ有効

トラブルシューティングとデプロイスクリプトの詳細については、[環境設定と起動ガイド](../../04_開発ガイド/02_環境設定と起動ガイド.md)を参照してください。
