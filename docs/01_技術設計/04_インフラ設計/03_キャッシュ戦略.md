# キャッシュ戦略

## 概要

stats47 プロジェクトでは、3 層キャッシュアーキテクチャを採用し、各ドメインの特性に応じた最適なキャッシュ戦略を実装します。これにより、高速なデータアクセスとコスト効率を両立します。

## 3 層キャッシュアーキテクチャ

```
┌─────────────────────────────────────┐
│ L1: Client Cache (SWR/React Query)  │ ← 1-5分
├─────────────────────────────────────┤
│ L2: CDN Cache (Cloudflare CDN)      │ ← 1時間-1日
├─────────────────────────────────────┤
│ L3: R2 Storage Cache                │ ← 1日-30日
└─────────────────────────────────────┘
```

### 各層の役割

#### L1: クライアントキャッシュ

- **技術**: SWR、React Query
- **TTL**: 1-5 分
- **用途**: リアルタイム性が重要なデータ
- **例**: ランキングデータ、検索結果

#### L2: CDN キャッシュ

- **技術**: Cloudflare CDN
- **TTL**: 1 時間-1 日
- **用途**: 静的・準静的データの配信
- **例**: 地域マスタ、地理データ

#### L3: R2 ストレージキャッシュ

- **技術**: Cloudflare R2
- **TTL**: 1 日-30 日
- **用途**: 大容量データの永続キャッシュ
- **例**: TopoJSON、API レスポンス

## ドメイン別キャッシュ戦略

| ドメイン | L1     | L2     | L3     | 理由                       |
| -------- | ------ | ------ | ------ | -------------------------- |
| Area     | 1 時間 | 1 日   | -      | 地域マスタは変更頻度が低い |
| Geoshape | 1 時間 | 1 日   | 30 日  | TopoJSON は大容量          |
| EstatAPI | 5 分   | 1 時間 | 1-7 日 | データ鮮度が重要           |
| Content  | -      | 1 日   | -      | 静的コンテンツ             |
| Ranking  | 5 分   | 1 時間 | 1 日   | ランキング更新頻度に応じる |

### Area ドメイン

**キャッシュ戦略**:

- L1: 1 時間（地域選択 UI での再利用）
- L2: 1 日（CDN 配信による高速化）
- L3: なし（静的 JSON ファイル）

**実装例**:

```typescript
// SWR でのクライアントキャッシュ
const { data: prefectures } = useSWR(
  "/api/areas/prefectures",
  fetcher,
  { revalidateOnFocus: false, dedupingInterval: 3600000 } // 1時間
);
```

### Geoshape ドメイン

**キャッシュ戦略**:

- L1: 1 時間（地図表示での再利用）
- L2: 1 日（CDN 配信）
- L3: 30 日（R2 での永続キャッシュ）

**実装例**:

```typescript
// R2 キャッシュの実装
export class GeoshapeCacheService {
  async getTopoJson(year: string, level: string, resolution: Resolution) {
    const cacheKey = `geoshape:${year}:${level}:${resolution}`;

    // L1: メモリキャッシュ
    if (this.memoryCache.has(cacheKey)) {
      return this.memoryCache.get(cacheKey);
    }

    // L3: R2 キャッシュ
    const cached = await this.r2Repository.get(cacheKey);
    if (cached && !this.isExpired(cached, 30 * 24 * 60 * 60 * 1000)) {
      this.memoryCache.set(cacheKey, cached);
      return cached;
    }

    // 新規取得
    const data = await this.fetchFromGeoshape(year, level, resolution);
    await this.r2Repository.set(cacheKey, data, 30 * 24 * 60 * 60 * 1000);
    this.memoryCache.set(cacheKey, data);
    return data;
  }
}
```

### EstatAPI ドメイン

**キャッシュ戦略**:

- L1: 5 分（リアルタイム性重視）
- L2: 1 時間（CDN 配信）
- L3: API 種別に応じて 1-7 日

**TTL 設定**:

```typescript
const ESTAT_TTL = {
  getMetaInfo: 7 * 24 * 60 * 60 * 1000, // 7日
  getStatsData: 24 * 60 * 60 * 1000, // 24時間
  getStatsList: 7 * 24 * 60 * 60 * 1000, // 7日
} as const;
```

### Content ドメイン

**キャッシュ戦略**:

- L1: なし（静的コンテンツ）
- L2: 1 日（CDN 配信）
- L3: なし（静的ファイル）

### Ranking ドメイン

**キャッシュ戦略**:

- L1: 5 分（ランキング更新に対応）
- L2: 1 時間（CDN 配信）
- L3: 1 日（R2 での永続キャッシュ）

## キャッシュ無効化戦略

### 手動無効化

**管理画面からの操作**:

```typescript
// 管理画面でのキャッシュ無効化
export class CacheManagementService {
  async invalidateCache(pattern: string) {
    // L1: クライアントキャッシュクリア
    await this.clearClientCache(pattern);

    // L2: CDN キャッシュクリア
    await this.clearCDNCache(pattern);

    // L3: R2 キャッシュクリア
    await this.clearR2Cache(pattern);
  }
}
```

### 自動無効化

**TTL 期限切れ**:

```typescript
// TTL チェックと自動削除
export class CacheCleanupService {
  async cleanupExpiredCache() {
    const expiredKeys = await this.findExpiredKeys();
    for (const key of expiredKeys) {
      await this.r2Repository.delete(key);
    }
  }
}
```

## キャッシュキー設計

### 命名規則

```
{domain}:{type}:{identifier}:{version}
```

**例**:

- `area:prefectures:v1`
- `geoshape:topojson:2023:pref:c`
- `estat:getStatsData:A1101:2023:v1`

### バージョン管理

```typescript
// キャッシュキーのバージョン管理
export class CacheKeyBuilder {
  buildKey(domain: string, type: string, identifier: string, version?: string) {
    const v = version || "v1";
    return `${domain}:${type}:${identifier}:${v}`;
  }

  // バージョンアップ時のキー生成
  buildNextVersionKey(key: string) {
    const [domain, type, identifier, version] = key.split(":");
    const nextVersion = this.incrementVersion(version);
    return `${domain}:${type}:${identifier}:${nextVersion}`;
  }
}
```

## パフォーマンス監視

### メトリクス収集

```typescript
// キャッシュヒット率の監視
export class CacheMetrics {
  private hits = new Map<string, number>();
  private misses = new Map<string, number>();

  recordHit(key: string) {
    this.hits.set(key, (this.hits.get(key) || 0) + 1);
  }

  recordMiss(key: string) {
    this.misses.set(key, (this.misses.get(key) || 0) + 1);
  }

  getHitRate(key: string): number {
    const hit = this.hits.get(key) || 0;
    const miss = this.misses.get(key) || 0;
    return hit / (hit + miss);
  }
}
```

### アラート設定

- **キャッシュヒット率低下**: 70% 以下でアラート
- **レスポンス時間増加**: 平均 2 秒以上でアラート
- **エラー率増加**: 5% 以上でアラート

## コスト最適化

### R2 ストレージの活用

- **読み取り無料**: R2 の読み取り課金なしを最大活用
- **圧縮**: gzip 圧縮によるストレージ効率化
- **ライフサイクル**: 古いキャッシュの自動削除

### CDN の効率的利用

- **エッジキャッシュ**: 地理的に分散したキャッシュ
- **プリフェッチ**: 重要なデータの事前読み込み
- **圧縮**: HTTP 圧縮による転送量削減

## 実装例

### 統合キャッシュサービス

```typescript
export class UnifiedCacheService {
  constructor(
    private memoryCache: MemoryCache,
    private cdnCache: CDNCache,
    private r2Cache: R2Cache
  ) {}

  async get<T>(
    key: string,
    fetcher: () => Promise<T>,
    ttl: CacheTTL
  ): Promise<T> {
    // L1: メモリキャッシュ
    const cached = this.memoryCache.get<T>(key);
    if (cached) return cached;

    // L2: CDN キャッシュ
    const cdnCached = await this.cdnCache.get<T>(key);
    if (cdnCached) {
      this.memoryCache.set(key, cdnCached, ttl.l1);
      return cdnCached;
    }

    // L3: R2 キャッシュ
    const r2Cached = await this.r2Cache.get<T>(key);
    if (r2Cached) {
      this.memoryCache.set(key, r2Cached, ttl.l1);
      await this.cdnCache.set(key, r2Cached, ttl.l2);
      return r2Cached;
    }

    // 新規取得
    const data = await fetcher();
    this.memoryCache.set(key, data, ttl.l1);
    await this.cdnCache.set(key, data, ttl.l2);
    await this.r2Cache.set(key, data, ttl.l3);
    return data;
  }
}
```
