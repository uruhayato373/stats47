## データ取得エラー

### エラー: "Ranking item not found"

**症状**: ランキング項目が見つからないエラーが発生する。

**原因**:
- D1データベースに該当のランキング項目が登録されていない
- `ranking_key` のスペルミス

**対処法**:

```sql
-- ランキング項目が存在するか確認
SELECT * FROM ranking_items WHERE ranking_key = 'your-ranking-key';

-- 存在しない場合は登録
INSERT INTO ranking_items (ranking_key, label, name, unit, data_source_id)
VALUES ('your-ranking-key', 'ラベル', '名称', '単位', 'estat');
```

---

### エラー: "Data source metadata not found"

**症状**: データソースメタデータが見つからない。

**原因**:
- `data_source_metadata` テーブルに取得パラメータが登録されていない

**対処法**:

```sql
-- メタデータを確認
SELECT * FROM data_source_metadata WHERE ranking_key = 'your-ranking-key';

-- 存在しない場合は登録
INSERT INTO data_source_metadata (
  ranking_key,
  data_source_id,
  area_type,
  calculation_type,
  metadata
) VALUES (
  'your-ranking-key',
  'estat',
  'prefecture',
  'direct',
  '{"stats_data_id":"0000010102","cd_cat01":"B1101"}'
);
```

---

### エラー: "API request failed: 403"

**症状**: 外部API（e-Stat等）の呼び出しで403エラーが発生する。

**原因**:
- APIキーが無効または期限切れ
- 環境変数が正しく設定されていない

**対処法**:

1. 環境変数を確認:

```bash
# .env.local または wrangler.toml
ESTAT_API_KEY=your-api-key-here
```

2. APIキーが有効か確認:

```bash
curl "https://api.e-stat.go.jp/rest/3.0/app/json/getStatsList?appId=YOUR_API_KEY"
```

3. Cloudflare Workersのシークレットを確認:

```bash
npx wrangler secret list
```

---

### エラー: "Adapter not found"

**症状**: 指定したデータソースのアダプターが見つからない。

**原因**:
- アダプターが `AdapterRegistry` に登録されていない
- データソースIDのタイプミス

**対処法**:

```typescript
// src/features/ranking/adapters/base/adapter-registry.ts
export class AdapterRegistry {
  constructor() {
    // アダプターを登録
    this.register(new EstatRankingAdapter());
    this.register(new YourNewAdapter()); // ← 追加されているか確認
  }
}
```

## パフォーマンス問題

### 問題: データ取得が遅い（3秒以上）

**症状**: ランキングデータの取得に3秒以上かかる。

**原因**:
- キャッシュミス
- 外部API呼び出しのレイテンシ

**対処法**:

1. キャッシュヒット率を確認:

```typescript
// ログから確認
// ✅ [Cache] R2 hit: ... → キャッシュヒット（高速）
// ❌ [Cache] Miss: ... → キャッシュミス（低速）
```

2. 初回取得を手動で実行:

```typescript
// 管理画面から事前にデータを取得
const service = await RankingService.create();
await service.getRankingData('population', '2023');
```

3. バッチ処理で一括キャッシュ:

```typescript
// すべての年度のデータを事前取得
const years = ['2020', '2021', '2022', '2023'];
for (const year of years) {
  await service.getRankingData('population', year);
}
```

### 問題: R2ストレージへのアクセスが遅い

**症状**: R2からのデータ取得に200ms以上かかる。

**原因**:
- R2バケットのリージョンが遠い
- ファイルサイズが大きすぎる

**対処法**:

1. データ圧縮を有効化:

```typescript
// R2保存時にgzip圧縮
import { gzipSync } from 'node:zlib';

const compressed = gzipSync(JSON.stringify(data));
await r2.put(key, compressed, {
  httpMetadata: {
    contentEncoding: 'gzip',
  },
});
```

2. ファイルサイズを確認:

```bash
# R2オブジェクトのサイズを確認
npx wrangler r2 object get stats47-dev/ranking/population/2023.json --file=temp.json
ls -lh temp.json
```

## データ不整合

### 問題: ランキングの順位がおかしい

**症状**: 明らかに大きい値が下位にランクされている。

**原因**:
- `ranking_direction` の設定ミス（ascとdescの逆）
- データ型の不一致（文字列として比較されている）

**対処法**:

1. `ranking_direction` を確認:

```sql
SELECT ranking_key, ranking_direction FROM ranking_items WHERE ranking_key = 'your-key';

-- 必要に応じて修正
UPDATE ranking_items SET ranking_direction = 'desc' WHERE ranking_key = 'your-key';
```

2. データ型を確認:

```typescript
// 数値として扱われているか確認
console.log(typeof dataPoint.value); // → "number" であるべき

// 文字列の場合は変換
dataPoint.value = parseFloat(dataPoint.value);
```

---

### 問題: データが古い

**症状**: 最新年度のデータが表示されない。

**原因**:
- キャッシュが更新されていない
- 外部APIに最新データがまだ公開されていない

**対処法**:

1. キャッシュを強制更新:

```bash
# API経由で更新
curl -X POST http://localhost:3000/api/rankings/refresh \
  -H "Content-Type: application/json" \
  -d '{"rankingKey": "population"}'
```

2. 外部APIの更新状況を確認:

```bash
# e-Stat APIで最新年度を確認
curl "https://api.e-stat.go.jp/rest/3.0/app/json/getStatsData?appId=YOUR_KEY&statsDataId=0000010102"
```


1. ログを確認（コンソール出力）
2. D1データベースの登録状況を確認
3. 環境変数を確認
4. R2ストレージの内容を確認
5. 外部APIの応答を確認

### Q: パフォーマンスを改善したい

**A**: [技術設計ドキュメント](../../../01_技術設計/03_ドメイン設計/10_ranking.md#キャッシュ戦略) のキャッシュ戦略を参照してください。

### Q: 新しいエラーが発生した

**A**: エラーメッセージ、スタックトレース、再現手順を記録し、このドキュメントに追記してください。
