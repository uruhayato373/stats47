## 新しいデータソースの追加

新しい外部データソース（API、CSV等）からランキングデータを取得できるようにする手順。

### ステップ1: データソースを登録

D1データベースに新しいデータソースを登録します。

```sql
-- 例: RESAS APIを追加
INSERT INTO data_sources (id, name, description, base_url, is_active)
VALUES (
  'resas',
  'RESAS',
  '地域経済分析システム',
  'https://opendata.resas-portal.go.jp',
  1
);
```

### ステップ2: アダプターを実装

`src/features/ranking/adapters/` 配下に新しいアダプターを作成します。

**最小実装例**:

```typescript
// src/features/ranking/adapters/resas/resas-ranking-adapter.ts
import type { RankingDataAdapter } from '../base/RankingDataAdapter';

export class ResasRankingAdapter implements RankingDataAdapter {
  readonly sourceId = 'resas';
  readonly sourceName = 'RESAS';

  async fetchAndTransform(params: AdapterFetchParams): Promise<UnifiedRankingData> {
    // 1. RESAS APIを呼び出し
    const rawData = await this.fetchFromResas(params);

    // 2. 統一フォーマットに変換
    const dataPoints = this.transformToUnified(rawData);

    // 3. ランク計算
    const rankedData = this.calculateRanks(dataPoints);

    return {
      metadata: {
        rankingKey: params.rankingKey,
        dataSourceId: this.sourceId,
        // ... その他メタデータ
      },
      values: rankedData,
      quality: this.assessQuality(rankedData),
    };
  }

  async getAvailableYears(rankingKey: string): Promise<string[]> {
    // 利用可能な年度を返す
    return ['2020', '2021', '2022', '2023'];
  }

  async isAvailable(): Promise<boolean> {
    // APIが利用可能かチェック
    return true;
  }
}
```

### ステップ3: アダプターを登録

`AdapterRegistry` に新しいアダプターを登録します。

```typescript
// src/features/ranking/adapters/base/adapter-registry.ts
import { ResasRankingAdapter } from '../resas/resas-ranking-adapter';

export class AdapterRegistry {
  private adapters = new Map<string, RankingDataAdapter>();

  constructor() {
    // 既存アダプター
    this.register(new EstatRankingAdapter());

    // 新しいアダプターを追加
    this.register(new ResasRankingAdapter());
  }
}
```

### ステップ4: テスト

アダプターが正しく動作するかテストします。

```typescript
// tests/features/ranking/adapters/resas-ranking-adapter.test.ts
describe('ResasRankingAdapter', () => {
  it('should fetch and transform data correctly', async () => {
    const adapter = new ResasRankingAdapter();
    const result = await adapter.fetchAndTransform({
      rankingKey: 'gdp-per-capita',
      timeCode: '2023',
      level: 'prefecture',
      sourceSpecific: { /* RESAS固有パラメータ */ },
    });

    expect(result.values).toHaveLength(47);
    expect(result.values[0]).toHaveProperty('rank');
  });
});
```

## ランキング項目の追加

新しい統計指標のランキング項目を追加する手順。

### ステップ1: データソースメタデータを確認

追加したい統計指標がどのデータソースから取得できるか確認します。

**例**: e-Statから「一人当たり県民所得」を取得する場合
- データセットID: `0000030201`
- カテゴリコード: `A0201`

### ステップ2: ランキング項目を登録

D1データベースに新しいランキング項目を登録します。

```sql

INSERT INTO ranking_items (
  ranking_key,

  '一人当たり県民所得（都道府県別）',
  '千円',
  'estat',
  'interpolateYlGn',
  'desc',
  1

-- データソースメタデータを追加
INSERT INTO data_source_metadata (
  ranking_key,
  data_source_id,
  area_type,
  calculation_type,
  metadata
) VALUES (
  'per-capita-income',
  'estat',
  'prefecture',
  'direct',
  '{"stats_data_id":"0000030201","cd_cat01":"A0201","cd_area":"00000"}'

```

### ステップ3: サブカテゴリに紐付け

既存のサブカテゴリにランキング項目を紐付けます。

```sql
INSERT INTO subcategory_ranking_items (
  subcategory_id,
  ranking_item_id,
  display_order,
  is_default

SELECT
  'economy',  -- サブカテゴリID
  id,
  10,  -- 表示順
  0    -- デフォルト選択ではない

WHERE ranking_key = 'per-capita-income';
```

## ランキンググループの作成

関連する複数のランキング項目をグループ化する手順。

### ユースケース

**例**: 「製造業」サブカテゴリ内で以下の項目をグループ化
- 製造品出荷額
- 製造品出荷額（事業所あたり）
- 製造品出荷額（従業者あたり）

### ステップ1: ランキンググループを作成

```sql
INSERT INTO ranking_groups (
  group_key,
  subcategory_id,
  name,
  description,
  display_order,
  is_collapsed
) VALUES (
  'manufacturing-output',
  'manufacturing',
  '製造品出荷額',
  '製造業の出荷額に関する統計',
  1,
  0  -- デフォルトで展開
);
```

### ステップ2: ランキング項目をグループに追加

```sql
-- グループIDを取得
SET @group_id = (SELECT id FROM ranking_groups WHERE group_key = 'manufacturing-output');

-- 1つ目の項目を追加
INSERT INTO ranking_group_items (group_id, ranking_item_id, display_order, is_featured)
SELECT @group_id, id, 1, 1
FROM ranking_items WHERE ranking_key = 'manufacturing-output-total';

-- 2つ目の項目を追加
INSERT INTO ranking_group_items (group_id, ranking_item_id, display_order, is_featured)
SELECT @group_id, id, 2, 0
FROM ranking_items WHERE ranking_key = 'manufacturing-output-per-establishment';

-- 3つ目の項目を追加
INSERT INTO ranking_group_items (group_id, ranking_item_id, display_order, is_featured)
SELECT @group_id, id, 3, 0
FROM ranking_items WHERE ranking_key = 'manufacturing-output-per-employee';
```

### ステップ3: 動作確認

APIを呼び出してグループが正しく表示されるか確認します。

```bash
curl http://localhost:3000/api/rankings/groups/subcategory/manufacturing
```

**期待されるレスポンス**:

```json
{
  "subcategory": {
    "id": "manufacturing",
    "name": "製造業"
  },
  "groups": [
    {
      "id": 1,
      "groupKey": "manufacturing-output",
      "name": "製造品出荷額",
      "items": [
        { "rankingKey": "manufacturing-output-total", "label": "製造品出荷額" },
        { "rankingKey": "manufacturing-output-per-establishment", "label": "製造品出荷額（事業所あたり）" },
        { "rankingKey": "manufacturing-output-per-employee", "label": "製造品出荷額（従業者あたり）" }
      ]
    }
  ]
}
```

## よくある質問

### Q1: 既存のアダプターを参考にしたい

**A**: `src/features/ranking/adapters/estat/` の実装を参考にしてください。最も完成度の高い実装例です。

### Q2: データソースメタデータのJSON構造がわからない

**A**: [技術設計ドキュメント](../../../01_技術設計/03_ドメイン設計/10_ranking.md#データソースメタデータ設計) を参照してください。

### Q3: 計算型ランキング（人口密度など）を追加したい

**A**: `calculation_type` を `'ratio'` に設定し、metadata JSONで分子・分母を指定します。詳細は技術設計ドキュメント参照。
