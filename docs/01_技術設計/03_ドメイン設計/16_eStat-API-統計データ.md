# eStat API 統計データ管理ドメイン設計

## 概要

eStat API 統計データ管理ドメインは、stats47 プロジェクトにおける e-Stat API の `GET_STATS_DATA` エンドポイントを通じた統計データ取得・整形・提供機能を担当する支援ドメインです。

### ビジネス価値

- **統計データの一元取得**: e-Stat API から統計データを統一的に取得
- **データ整形の標準化**: 多次元データを扱いやすい形式に変換
- **パフォーマンス最適化**: キャッシュとフィルタリングによる高速データ配信
- **UI 連携**: ダッシュボードや可視化コンポーネントへのデータ提供

## 責務と主要概念

### 責務

1. **統計データの取得**

   - e-Stat API の `GET_STATS_DATA` エンドポイントから統計データを取得
   - R2 キャッシュからの優先取得
   - エラーハンドリングとリトライ

2. **データ整形**

   - 多次元データをアプリケーションが扱いやすい形式に変換
   - 地域、年次、分類項目の正規化
   - NULL 値と特殊文字の適切な処理

3. **フィルタリング**

   - 地域、年次、分類項目によるデータ絞り込み
   - API パラメータレベルとクライアント側の両方をサポート

4. **キャッシュ管理**

   - R2 ストレージを活用したデータキャッシング
   - キャッシュキーの適切な管理
   - バックグラウンドでのキャッシュ更新

### 主要概念

#### StatsDataId（統計表 ID）

e-Stat の各統計表に付与される一意の識別子。

**具体例**: `0003412313`（国勢調査）、`0000010101`（人口推計）

**制約**: 8-10 桁の数字、e-Stat で定義された ID のみ有効

**用途**: 統計データ取得時の識別子（`statsDataId`パラメータ）、統計表の一意識別

#### API パラメータ

e-Stat API 呼び出しで使用するパラメータ名。

**分類事項パラメータ**:

- **`cdCat01`〜`cdCat15`**: 分類事項 01-15 の絞り込みパラメータ（任意）
- 具体例: `cdCat01=001,002`（総数、男）

**地域パラメータ**:

- **`cdArea`**: 地域コード指定パラメータ（任意）
- 具体例: `cdArea=13000,27000`（東京都、大阪府）

**時間軸パラメータ**:

- **`cdTime`**: 時間軸コード指定パラメータ（任意）
- 具体例: `cdTime=2023000000`（2023 年）

**詳細**: [API パラメータ](#api-パラメータ)セクションを参照

#### API レスポンス構造（生データ）

e-Stat API から返されるそのままのデータ構造。

**EstatStatsDataResponse**:

e-Stat API の統計データ取得 API（`GET_STATS_DATA`）のレスポンス構造。

- **構造**: `GET_STATS_DATA.STATISTICAL_DATA`
- **主要要素**:
  - `CLASS_INF`: 分類情報（`CLASS_OBJ[]`）
  - `DATA_INF`: データ値（`VALUE[]`、`NOTE[]`）
  - `TABLE_INF`: 統計表情報

**DATA_INF.VALUE**:

実際の統計数値データを含む配列。各要素は以下の属性を持つ。

- **`@tab`**: 表章項目コード（必須、例: `"A1101"`）
- **`@area`**: 地域コード（任意、例: `"13000"`）
- **`@time`**: 時間軸コード（任意、例: `"2020"`）
- **`@cat01`〜`@cat15`**: 分類コード（任意、例: `"001"`）
- **`$`**: データ値（数値または特殊文字、例: `"14047594"`、`"***"`）

**詳細**: [API レスポンス構造](#api-レスポンス構造)セクションを参照

#### 変換後のデータ構造

e-Stat API の生データを整形・変換した後のデータ構造。

**FormattedEstatData**:

整形済み統計データの全体構造。

- **構造**: `{ tableInfo, areas, categories, years, values, metadata, notes? }`
- **用途**: アプリケーション内で使用する標準的なデータ形式

**FormattedValue**:

整形済み統計値。生データの`VALUE`を変換した形式。

- **構造**: `{ value, unit, dimensions: { area, time, tab?, cat01?〜cat15? }, rank? }`
- **特徴**:
  - `dimensions`で全次元を統一的に管理（`area`、`time`は必須、`cat01`〜`cat15`はオプション）
  - 特殊文字（`***`、`-`など）は`null`に変換
  - 地域・時間・分類のコードと名称がセットで管理

**詳細**: 型定義は`src/features/estat-api/stats-data/types/`を参照

## ドメインモデル

### 主要エンティティ

#### FormattedEstatData（整形済み統計データ）

e-Stat API の生データを整形・変換した後のデータ構造。アプリケーション内で使用する標準的な形式。

| 属性名       | 型                    | 説明                                 |
| ------------ | --------------------- | ------------------------------------ |
| `tableInfo`  | `FormattedTableInfo`  | 統計表情報（変換後）                 |
| `areas`      | `FormattedArea[]`     | 地域情報の配列（変換後）             |
| `categories` | `FormattedCategory[]` | 分類情報の配列（変換後）             |
| `years`      | `FormattedYear[]`     | 年度情報の配列（変換後）             |
| `values`     | `FormattedValue[]`    | 統計値の配列（変換後）               |
| `metadata`   | `FormattedMetadata`   | メタデータ（処理情報、統計情報など） |
| `notes?`     | `DataNote[]`          | 注記情報（任意）                     |

型定義は`src/features/estat-api/stats-data/types/`を参照してください。

#### FormattedValue（整形済み統計値）

個別の統計値データ。生データの`DATA_INF.VALUE`を整形した形式。

| 属性名       | 型                                                              | 説明                                              |
| ------------ | --------------------------------------------------------------- | ------------------------------------------------- |
| `value`      | `number \| null`                                                | 統計値（数値または null。特殊文字は null に変換） |
| `unit`       | `string \| null`                                                | 単位                                              |
| `dimensions` | `{ area: {...}, time: {...}, tab?: {...}, cat01?: {...}, ... }` | 多次元データの次元情報（統一的に管理）            |
| `rank?`      | `number`                                                        | ランキング順位（オプション）                      |

**`dimensions`の構造**:

- `area`: `{ code, name, level?, parentCode? }`（必須）
- `time`: `{ code, name }`（必須）
- `tab`: `{ code, name, unit? }`（任意）
- `cat01`〜`cat15`: `{ code, name, unit? }`（任意）

型定義は`src/features/estat-api/stats-data/types/`を参照してください。

## アーキテクチャ設計

### レイヤー構造

```
┌─────────────────────────────────────────┐
│     Presentation Layer                  │
│  (ダッシュボード、可視化コンポーネント)  │
└──────────────────┬──────────────────────┘
                   │
┌──────────────────▼──────────────────────┐
│     Service Layer                       │
│  - fetchStatsData()                     │
│  - fetchFormattedStatsData()            │
│  - formatStatsData()                    │
└──────────────────┬──────────────────────┘
                   │
┌──────────────────▼──────────────────────┐
│     Core Layer (HTTP Client)            │
│  - executeHttpRequest                   │
└──────┬───────────────┬──────────────────┘
       │               │
┌──────▼──────┐  ┌─────▼──────┐  ┌──────────────┐
│     e-Stat  │  │     R2     │  │   Memory     │
│     API     │  │  Storage   │  │    Cache     │
│  (External) │  │ (Persistent)│  │ (Temporary)  │
└─────────────┘  └────────────┘  └──────────────┘
```

### データフロー

```
統計データ取得リクエスト
    │
    ▼
Server Action (getStatsDataAction)
    │
    ├─► R2キャッシュ確認
    │       │
    │       ├─► ヒット → オリジナルレスポンス取得
    │       │       │
    │       │       └─► formatStatsData() で整形
    │       │
    │       └─► ミス → e-Stat API
    │                   │
    │                   ▼
    │               fetchStatsData()
    │                   │
    │                   ├─► executeHttpRequest()
    │                   │       │
    │                   │       ▼
    │                   │   e-Stat API
    │                   │       │
    │                   │       ▼
    │                   │   オリジナルレスポンス
    │                   │       │
    │                   ├─► バックグラウンドR2保存（オリジナル）
    │                   │
    │                   └─► formatStatsData() で整形
    │
    └─► 整形済みデータ返却
```

### 実装方針

**Server Action による実装**:

- **Server Action を使用**: すべてのデータ取得は Server Action で実装
- **Route Handler は使用しない**: API Route Handler（`/api/...`）は使用せず、Server Action のみを使用
- **useSWR は使用しない**: サーバーサイドでのデータ取得のため、クライアントサイドの useSWR は使用しない
- **純粋関数で実装**: クラスではなく純粋関数を使用し、テストと再利用性を向上
- **キャッシュ**: Next.js 15 の`use cache`ディレクティブと R2 ストレージを併用

**理由**:

- **型安全性**: TypeScript で完全に型付けされた API
- **パフォーマンス**: サーバーサイドでの実行により、クライアントへの負荷を軽減
- **Next.js 15 の推奨パターン**: Next.js 15 では Server Action が推奨される実装方法
- **関数の純粋性**: 副作用を最小化し、テストが容易で、再利用性が高い
- **キャッシュの効率性**: R2 キャッシュにより API 呼び出しを削減

### 設計パターン

#### 1. 純粋関数パターン

**目的**: データ取得と整形の責務を分離し、テストと再利用性を向上

- **取得関数**: HTTP 通信と生データ取得（`fetchStatsData()`, `fetchFormattedStatsData()`）
- **整形関数**: データの整形と変換（`formatStatsData()`）
- **関数の純粋性**: 副作用を最小化し、同じ入力に対して同じ出力を返す

#### 2. Server Action による実装

**目的**: 型安全性とパフォーマンス向上

- サーバーサイドでのデータ取得
- Next.js 15 の`use cache`ディレクティブによるキャッシュ
- 型安全性の向上

## データソースとストレージ

### データソース

#### e-Stat API（外部サービス）

**エンドポイント**: `/getStatsData`

**用途**: 統計データの取得

**認証**: アプリケーション ID（環境変数 `NEXT_PUBLIC_ESTAT_APP_ID`）

**レート制限**: 1 日あたり 1,000 回、1 時間あたり 100 回（推奨）

#### API パラメータ

**基本パラメータ**:

| パラメータ名    | 必須 | 説明                      | 例            |
| --------------- | ---- | ------------------------- | ------------- |
| `appId`         | ○    | アプリケーション ID       | `YOUR_APP_ID` |
| `statsDataId`   | △    | 統計表 ID（8-10 桁）      | `0003412313`  |
| `cdArea`        | -    | 地域コード                | `13000,27000` |
| `cdTime`        | -    | 時間軸コード              | `2023000000`  |
| `cdCat01-15`    | -    | 分類事項 01-15 の絞り込み | `001,002`     |
| `limit`         | -    | 取得件数（最大 100,000）  | `10000`       |
| `startPosition` | -    | 取得開始位置              | `1`           |

詳細は [e-Stat API 公式ドキュメント](https://www.e-stat.go.jp/api/api-info/api-spec) を参照してください。

#### API レスポンス構造

**基本構造**:

```typescript
interface GetStatsDataResponse {
  GET_STATS_DATA: {
    RESULT: {
      STATUS: number;
      ERROR_MSG?: string;
      DATE: string;
    };
    STATISTICAL_DATA: {
      RESULT_INF: {
        TOTAL_NUMBER: number;
        FROM_NUMBER: number;
        TO_NUMBER: number;
        NEXT_KEY?: number;
      };
      TABLE_INF: TableInfo;
      CLASS_INF: ClassInfo;
      DATA_INF: DataInfo;
    };
  };
}
```

**RESULT（リクエスト結果）**:

| フィールド  | 説明                     | 例                                |
| ----------- | ------------------------ | --------------------------------- |
| `STATUS`    | ステータスコード         | `0`: 正常、`100`以上: エラー      |
| `ERROR_MSG` | エラーメッセージ（任意） | `"正常に終了しました。"`          |
| `DATE`      | レスポンス生成日時       | `"2024-01-15T10:30:00.000+09:00"` |

**RESULT_INF（データ件数情報）**:

| フィールド     | 説明           | 例     |
| -------------- | -------------- | ------ |
| `TOTAL_NUMBER` | 総データ件数   | `2350` |
| `FROM_NUMBER`  | 取得開始位置   | `1`    |
| `TO_NUMBER`    | 取得終了位置   | `1000` |
| `NEXT_KEY`     | 次ページのキー | `1001` |

**TABLE_INF（統計表情報）**:

統計表の基本情報を含みます。メタ情報 API（`GET_META_INFO`）の`TABLE_INF`と同じ構造です。

| フィールド             | 説明           | 例                                       |
| ---------------------- | -------------- | ---------------------------------------- |
| `@id`                  | 統計表 ID      | `"0003412313"`                           |
| `STAT_NAME`            | 統計調査名     | `{"@code": "00200522", "$": "国勢調査"}` |
| `GOV_ORG`              | 作成機関       | `{"@code": "00200", "$": "総務省"}`      |
| `STATISTICS_NAME`      | 統計名称       | `"令和2年国勢調査 人口等基本集計"`       |
| `TITLE`                | 統計表タイトル | `"都道府県，男女別人口"`                 |
| `SURVEY_DATE`          | 調査年月       | `"202010"`                               |
| `OPEN_DATE`            | 公開日         | `"2021-11-30"`                           |
| `UPDATED_DATE`         | 更新日時       | `"2023-12-01"`                           |
| `OVERALL_TOTAL_NUMBER` | 総データ件数   | `"2350"`                                 |
| `CYCLE`                | 周期           | `"5年"`、`"年次"`、`"-"`                 |
| `COLLECT_AREA`         | 集計地域区分   | `"全国"`、`"都道府県"`、`"市区町村"`     |

詳細は [eStat API（メタ情報）サブドメイン設計](./15_eStat-API-メタ情報.md#table_inf統計表情報) を参照してください。

**CLASS_INF（分類情報）**:

統計データの各次元（軸）の詳細情報を提供します。メタ情報 API（`GET_META_INFO`）の`CLASS_INF`と同じ構造です。

**CLASS_OBJ の種類**:

| @id              | 名称     | 説明                             |
| ---------------- | -------- | -------------------------------- |
| `tab`            | 表章項目 | 統計表の主題（人口、世帯数など） |
| `cat01`〜`cat15` | 分類事項 | 男女別、年齢別など（最大 15 個） |
| `area`           | 地域     | 全国、都道府県、市区町村         |
| `time`           | 時間軸   | 年次、月次、日次など             |

**CLASS の属性**:

| 属性          | 説明         | 例                              |
| ------------- | ------------ | ------------------------------- |
| `@code`       | 項目コード   | `"001"`, `"13000"`              |
| `@name`       | 項目名称     | `"総数"`, `"東京都"`            |
| `@unit`       | 単位         | `"人"`, `"%"`                   |
| `@level`      | 階層レベル   | `"1"`: 最上位、`"2"`: 第 2 階層 |
| `@parentCode` | 親項目コード | `"00000"` (全国)                |

詳細は [eStat API（メタ情報）サブドメイン設計](./15_eStat-API-メタ情報.md#class_inf分類情報) を参照してください。

**DATA_INF（データ値）**:

実際の統計数値データを含みます。

| フィールド | 説明                   | 例                                                                          |
| ---------- | ---------------------- | --------------------------------------------------------------------------- |
| `VALUE`    | データ値の配列（必須） | `[{ "@tab": "A1101", "@area": "13000", "@time": "2020", "$": "14047594" }]` |
| `NOTE`     | 注記情報の配列（任意） | `[{ "@char": "***", "$": "該当なし" }]`                                     |

**VALUE の属性**:

| 属性               | 必須 | 説明                   | 例                                            |
| ------------------ | ---- | ---------------------- | --------------------------------------------- |
| `@tab`             | ○    | 表章項目コード（必須） | `"A1101"`                                     |
| `@area`            | -    | 地域コード             | `"13000"`                                     |
| `@time`            | -    | 時間軸コード           | `"2020"`                                      |
| `@cat01`〜`@cat15` | -    | 分類事項コード（任意） | `"001"`                                       |
| `@unit`            | -    | 単位                   | `"人"`                                        |
| `$`                | ○    | 統計値                 | `"14047594"`（数値）または`"***"`（特殊文字） |

**NOTE の属性**:

| 属性    | 説明                          | 例                    |
| ------- | ----------------------------- | --------------------- |
| `@char` | 特殊文字（\*\*\*, -, X など） | `"***"`, `"-"`, `"X"` |
| `$`     | 注記の説明                    | `"該当なし"`          |

詳細な型定義は実装コード（`src/features/estat-api/stats-data/types/`）を参照してください。

### R2 ストレージ構造

R2 ストレージへのキャッシュは以下のディレクトリ構造で管理されます。詳細は [R2 ストレージ設計ドキュメント](../04_インフラ設計/02_R2ストレージ設計.md#estatapi-ドメイン) を参照してください。

```
estat-api/
└── stats-data/                      # 統計データキャッシュ
    └── {statsDataId}/               # 統計表ID別ディレクトリ
        └── {params_hash}.json       # パラメータハッシュベースのファイル名
```

**キー生成規則**:

- **統計データ**: `estat-api/stats-data/{statsDataId}/{params_hash}.json`

**パラメータ**:

- `statsDataId`: 統計表 ID（8-10 桁、例: `0003412313`）
- `params_hash`: パラメータのハッシュ値（`cdArea`、`cdTime`、`cdCat01`〜`cdCat15`などを含むリクエストパラメータから生成）

**例**:

- `estat-api/stats-data/0003412313/a1b2c3d4e5f6g7h8.json`（統計表 ID: `0003412313`、パラメータハッシュ: `a1b2c3d4e5f6g7h8`）
- `estat-api/stats-data/0003000001/f9g0h1i2j3k4l5m6.json`（統計表 ID: `0003000001`、パラメータハッシュ: `f9g0h1i2j3k4l5m6`）

**キャッシュデータ形式**:

オリジナルの API レスポンス（`EstatStatsDataResponse`）を JSON 形式でそのまま保存します。これにより、整形ロジックが変更されても同じオリジナルデータから再整形が可能です。

### キャッシュ戦略

#### 1. R2 ストレージ（永続キャッシュ）

**目的**: 複数リクエスト間でのキャッシュ共有、永続性の確保

**詳細**: R2 ストレージの設計詳細については、[R2 ストレージ設計](../04_インフラ設計/02_R2ストレージ設計.md)を参照してください。

**実装**:

- R2 バケットに統計データを JSON 形式で保存
- **キャッシュデータ**: オリジナルの API レスポンス（`EstatStatsDataResponse`）を保存
- キャッシュヒット時は R2 からオリジナルデータを取得し、必要に応じて整形
- キャッシュミス時は e-Stat API から取得し、バックグラウンドで R2 に保存

**オリジナルデータをキャッシュする理由**:

- **再利用性**: 整形ロジックが変更されても、同じオリジナルデータから再整形可能
- **柔軟性**: 異なる整形フォーマット（例えば、将来的に異なる形式が必要になった場合）に対応可能
- **一貫性**: メタ情報ドメインと同じ設計パターンに統一
- **データ整合性**: e-Stat API から取得した生データをそのまま保存することで、データの完全性を保証

**メリット**:

- サーバー再起動後もデータが保持される
- 複数の Worker インスタンス間でキャッシュを共有
- API 呼び出しを大幅に削減
- 整形ロジックの変更に柔軟に対応

#### 2. Next.js 15 の`use cache`ディレクティブ

**目的**: 関数レベルのキャッシュによる複数リクエスト間での効率化

**実装例**:

```typescript
// src/features/estat-api/stats-data/actions.ts
"use server";

import { fetchFormattedStatsData } from "./services/fetcher";

export async function getStatsDataAction(
  statsDataId: string,
  options?: FetchOptions
) {
  "use cache"; // Next.js 15でのキャッシュ（複数リクエスト間で有効）

  return await fetchFormattedStatsData(statsDataId, options);
}
```

**特徴**:

- 関数レベルのキャッシュ
- 入力パラメータに基づく自動キャッシュ
- 複数リクエスト間で有効

### パフォーマンス考慮事項

**データ取得の最適化**:

- **limit パラメータ**: 必要な件数のみ取得（最大 100,000 件）
- **フィルタリング**: API パラメータでの事前絞り込み
- **並列処理**: 複数統計表の同時取得

**データサイズ**:

| データタイプ   | サイズ      | 備考               |
| -------------- | ----------- | ------------------ |
| 都道府県データ | ~10KB-1MB   | 統計表により異なる |
| 市区町村データ | ~100KB-10MB | 統計表により異なる |
| 時系列データ   | ~50KB-5MB   | 年数により異なる   |

## 主要機能

### 1. 統計データ取得

**機能**: e-Stat API から統計データを取得

**主要関数**:

- `fetchStatsData()`: 統計データ取得（生データ）
- `fetchFormattedStatsData()`: 取得と整形を一括実行

**特徴**:

- R2 キャッシュによる高速化
- エラーハンドリングとリトライ
- Server Action による実装
- 純粋関数による実装

### 2. データ整形

**機能**: 多次元データを整形済みデータに変換

**主要関数**:

- `formatStatsData()`: 統計データの整形

**特徴**:

- 地域情報の正規化（5 桁コードの抽出）
- 分類情報の階層構造のフラット化
- NULL 値と特殊文字の適切な処理
- 純粋関数による実装（同じ入力に対して同じ出力を返す）

### 3. フィルタリング

**機能**: 地域、年次、分類項目によるデータ絞り込み

**対応するフィルタリング**:

- 地域コードによる絞り込み（`cdArea`）
- 時間軸コードによる絞り込み（`cdTime`）
- 分類コードによる絞り込み（`cdCat01`〜`cdCat15`）
- API パラメータレベルとクライアント側の両方をサポート

## 制約と前提条件

### 制約

1. **API レート制限**: e-Stat API のレート制限に準拠
2. **データサイズ**: 1 回のリクエストで最大 100,000 件まで取得可能
3. **パラメータ制限**: 分類事項は最大 15 個（cdCat01-cdCat15）

### 前提条件

1. **API キーの取得**: e-Stat API のアプリケーション ID が必要
2. **ネットワーク接続**: e-Stat API へのアクセス権限
3. **R2 ストレージ**: キャッシュ用の R2 ストレージへのアクセス権限
