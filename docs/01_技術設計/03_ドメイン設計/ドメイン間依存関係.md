# ドメイン間依存関係

## 概要

stats47 プロジェクトにおける各ドメイン間の依存関係と制約について説明します。

## ドメイン依存関係図

```
┌─────────────────────────────────────────────────────────────┐
│                    共有ドメイン (Shared Kernel)              │
│  ┌─────────────┐ ┌─────────────┐ ┌─────────────┐          │
│  │ 共通型定義   │ │ 共通ユーティリティ │ │ 共通定数   │          │
│  └─────────────┘ └─────────────┘ └─────────────┘          │
└─────────────────────────────────────────────────────────────┘
                                ↑
                                │ 参照のみ
┌─────────────────────────────────────────────────────────────┐
│                    コアドメイン (Core Domain)                │
│  ┌─────────────┐ ┌─────────────┐ ┌─────────────┐          │
│  │ 統計分析    │ │ ランキング   │ │ 可視化      │          │
│  │ ドメイン     │ │ ドメイン     │ │ ドメイン     │          │
│  └─────────────┘ └─────────────┘ └─────────────┘          │
└─────────────────────────────────────────────────────────────┘
                                ↑
                                │ 依存

│                  支援ドメイン (Supporting Domain)           │

│  └─────────────┘ └─────────────┘ └─────────────┘          │
```

## 依存関係の詳細

### 1. コアドメイン間の関係

#### 統計分析ドメイン → ランキングドメイン

- **依存内容**: 統計データの分析結果
- **依存方向**: 一方向（統計分析 → ランキング）
- **制約**: ランキングドメインは統計分析ドメインに依存しない

#### ランキングドメイン → 可視化ドメイン

- **依存内容**: ランキングデータの可視化
- **依存方向**: 一方向（ランキング → 可視化）
- **制約**: 可視化ドメインはランキングドメインに依存しない

### 2. コアドメイン → 支援ドメイン

#### 統計分析ドメイン → e-Stat API ドメイン

- **依存内容**: 統計データの取得
- **依存方向**: 一方向（統計分析 → e-Stat API）
- **制約**: e-Stat API ドメインは統計分析ドメインに依存しない

#### ランキングドメイン → 地域管理ドメイン

- **依存内容**: 地域情報の取得
- **依存方向**: 一方向（ランキング → 地域管理）
- **制約**: 地域管理ドメインはランキングドメインに依存しない

#### 可視化ドメイン → ダッシュボードドメイン

- **依存内容**: 可視化コンポーネントの提供
- **依存方向**: 一方向（可視化 → ダッシュボード）
- **制約**: ダッシュボードドメインは可視化ドメインに依存しない

### 3. 支援ドメイン → 汎用ドメイン

#### e-Stat API ドメイン → 認証ドメイン

- **依存内容**: API アクセスの認証
- **依存方向**: 一方向（e-Stat API → 認証）
- **制約**: 認証ドメインは e-Stat API ドメインに依存しない

#### ダッシュボードドメイン → カテゴリ管理ドメイン

- **依存内容**: カテゴリ情報の取得
- **依存方向**: 一方向（ダッシュボード → カテゴリ管理）
- **制約**: カテゴリ管理ドメインはダッシュボードドメインに依存しない

### 4. 共有ドメインへの依存

#### 全ドメイン → 共有ドメイン

- **依存内容**: 共通型定義、ユーティリティ、定数
- **依存方向**: 一方向（各ドメイン → 共有）
- **制約**: 共有ドメインは他のドメインに依存しない

## 依存関係の制約

### 1. 循環依存の禁止

- **原則**: ドメイン間の循環依存は禁止
- **例外**: 共有ドメインのみ他のドメインから参照される
- **検証**: TypeScript の型チェックで循環依存を検出

### 2. 上位ドメインへの依存禁止

- **原則**: 下位ドメインは上位ドメインに依存してはいけない
- **例**: 汎用ドメインは支援ドメインに依存しない
- **例外**: 共有ドメインは全ドメインから参照可能

### 3. 横断的依存の制限

- **原則**: 同階層のドメイン間の依存は最小限に
- **例外**: コアドメイン間の必要最小限の依存のみ許可
- **検証**: アーキテクチャテストで依存関係を検証

## 依存関係の管理

### 1. 型定義による制約

```typescript
// 共有ドメイン
export interface BaseEntity {
  id: string;
  createdAt: Date;
  updatedAt: Date;
}

// コアドメイン
export interface RankingItem extends BaseEntity {
  // ランキング固有のプロパティ
}

// 支援ドメイン
export interface EstatData extends BaseEntity {
  // e-Stat API固有のプロパティ
}
```

### 2. インターフェースによる抽象化

```typescript
// コアドメインが支援ドメインに依存する場合
export interface DataProvider {
  getData(params: DataParams): Promise<DataResult>;
}

// 支援ドメインが実装
export class EstatDataProvider implements DataProvider {
  // 実装
}
```

### 3. 依存性注入による制御

```typescript
// コアドメイン
export class RankingService {
  constructor(
    private dataProvider: DataProvider,
    private regionService: RegionService
  ) {}
}
```

## 依存関係の変更手順

### 1. 依存関係の追加

1. **影響分析**: 追加による影響範囲を分析
2. **設計レビュー**: アーキテクチャチームによるレビュー
3. **実装**: 段階的な実装とテスト
4. **文書更新**: 依存関係図の更新

### 2. 依存関係の削除

1. **影響分析**: 削除による影響範囲を分析
2. **移行計画**: 代替手段の検討と移行計画
3. **実装**: 段階的な削除とテスト
4. **文書更新**: 依存関係図の更新

## 監視・検証

### 1. 静的解析

- **TypeScript**: 型チェックによる循環依存の検出
- **ESLint**: カスタムルールによる依存関係の検証
- **ArchUnit**: アーキテクチャテストによる制約の検証

### 2. 動的監視

- **依存関係グラフ**: 自動生成による可視化
- **メトリクス**: 依存関係の複雑度の監視
- **アラート**: 制約違反の検出と通知
