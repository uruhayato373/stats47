# 技術スタック

## 概要

stats47 プロジェクトで採用している技術スタックの一覧と選定理由をまとめます。

## フロントエンド

### コア技術

- **フレームワーク**: Next.js 15 (App Router)
  - 理由: SSR/SSG/ISR の柔軟な使い分け、App Router による最新のルーティング機能
- **UI ライブラリ**: React 19
  - 理由: Server Components、Suspense、Concurrent Features の活用
- **言語**: TypeScript 5.x
  - 理由: 型安全性による開発効率向上、実行時エラーの削減

### スタイリング・UI

- **スタイリング**: Tailwind CSS 4
  - 理由: ユーティリティファースト、カスタムデザインシステムとの親和性
- **状態管理**: Jotai
  - 理由: 軽量、TypeScript との親和性、コンポーネント粒度での状態管理
- **データフェッチ**: SWR
  - 理由: キャッシング、再検証、エラーハンドリングの自動化

### 可視化・チャート

- **可視化**: Recharts, D3.js
  - Recharts: React との親和性、レスポンシブ対応
  - D3.js: カスタム可視化、高度なデータ操作

### データ可視化

- **Recharts**: 時系列グラフ
- **D3.js**: 高度な可視化、TopoJSON 処理
- **Leaflet**: 地図表示（MIT License）
- **react-leaflet**: Leaflet の React ラッパー
- **topojson-client**: TopoJSON のパース・変換

### 地理データソース

#### 行政区域データ

- **歴史的行政区域データセット β 版**: コロプレス地図用（CODH/GeoNLP プロジェクト、CC BY 4.0）
  - 提供元: 国立情報学研究所 人文学オープンデータ共同利用センター
  - 期間: 1889 年〜現在
  - 形式: TopoJSON, GeoJSON

#### ベースマップ

- **国土地理院タイル**: 背景地図（国土地理院、国土地理院コンテンツ利用規約）
  - 標準地図、淡色地図、白地図、航空写真
  - Leaflet の TileLayer で統合
  - 商用利用可、出典表示必須

#### その他の GIS データ（Phase 3 以降）

- **国土数値情報**: 施設情報、災害リスク、交通情報（国土交通省、CC BY 4.0）
  - 医療機関、教育機関、公共施設
  - 浸水想定区域、土砂災害警戒区域
  - 駅、道路

#### 統計データ

- **e-Stat API**: 統計データ

### ビルド・開発

- **ビルドツール**: Turbopack
  - 理由: Next.js 15 の高速ビルド、Rust ベースの高速化

## バックエンド

### ランタイム・API

- **ランタイム**: Node.js 18.x
  - 理由: Next.js との統合、豊富なエコシステム
- **API**: Next.js API Routes
  - 理由: フロントエンドとの統合、型安全性

### データストレージ

- **データベース**: Cloudflare D1 (SQLite)
  - 理由: サーバーレス、無料枠、SQLite の軽量性
- **オブジェクトストレージ**: Cloudflare R2
  - 理由: S3 互換、低コスト、CDN 統合

### 認証

- **認証**: Auth.js (NextAuth.js)
  - 理由: 多様なプロバイダー対応、セキュリティの自動化

## インフラ・デプロイ

### ホスティング

- **ホスティング**: Cloudflare Pages
  - 理由: 高速 CDN、無料枠、Git 連携
- **CDN**: Cloudflare CDN
  - 理由: グローバル配信、キャッシング最適化
- **ドメイン**: Cloudflare DNS
  - 理由: 高速 DNS、セキュリティ機能

### 監視・分析

- **監視**: Cloudflare Analytics
  - 理由: パフォーマンス監視、セキュリティ分析

## 開発・品質管理

### コード品質

- **リンター**: ESLint
- **フォーマッター**: Prettier
- **型チェック**: TypeScript Compiler

### テスト

- **テストフレームワーク**: Vitest
- **E2E テスト**: Playwright (予定)

### ドキュメント

- **ドキュメント**: Markdown + Obsidian
- **API 仕様**: OpenAPI (予定)

## 技術選定の原則

1. **型安全性**: TypeScript を基盤とした型安全な開発
2. **パフォーマンス**: 高速なビルドとランタイム性能
3. **開発効率**: 開発者体験の向上
4. **コスト効率**: 無料枠を活用した低コスト運用
5. **スケーラビリティ**: 将来の拡張に対応可能な設計
