# D3.js 採用理由

## ステータス
accepted

## 背景

stats47 プロジェクトでは、以下の要件を満たすデータ可視化ライブラリが必要でした：

1. **TopoJSON処理**: 地理データの効率的な処理と変換
2. **高度な可視化**: Rechartsでは実現困難なカスタムチャート
3. **地図可視化**: コロプレスマップ（地域別統計の色分け地図）の描画
4. **データ操作**: 強力なデータ変換・集計機能

## 決定

**D3.js v7.9.0** を採用

## 理由

### 1. TopoJSON処理能力
- TopoJSONからGeoJSONへの効率的な変換
- 地理データの最適化と圧縮
- 地図データの軽量化

### 2. 高度な可視化機能
- カスタムSVGレンダリング
- 複雑なデータ変換と集計
- インタラクティブな可視化

### 3. コロプレスマップ対応
- 地域別統計データの色分け表示
- 地図のズーム・パン機能
- ホバー効果とツールチップ

### 4. データ操作の柔軟性
- 強力なデータ変換・集計機能
- 複雑な計算ロジックの実装
- リアルタイムデータ更新

## 使用箇所

### 1. コロプレスマップ
```typescript
// 都道府県別統計の色分け地図
const choroplethMap = d3.select("#map")
  .selectAll("path")
  .data(topojson.feature(topology, topology.objects.prefectures).features)
  .enter()
  .append("path")
  .attr("d", path)
  .style("fill", d => colorScale(statistics[d.properties.name]));
```

### 2. レーダーチャート
```typescript
// 地域プロファイルのレーダーチャート
const radarChart = d3.select("#radar")
  .append("svg")
  .attr("width", width)
  .attr("height", height);
```

### 3. カスタムグラフ
```typescript
// 時系列データの複雑な可視化
const customChart = d3.select("#chart")
  .selectAll("circle")
  .data(data)
  .enter()
  .append("circle")
  .attr("cx", d => xScale(d.date))
  .attr("cy", d => yScale(d.value));
```

## 代替案の検討

### Chart.js
**メリット:**
- シンプルなAPI
- 軽量
- 学習コストが低い

**デメリット:**
- 地図処理に不向き
- TopoJSON処理機能なし
- カスタマイズ性が限定的

**結論:** 地図可視化要件を満たせないため不採用

### Mapbox GL JS
**メリット:**
- 高性能な地図表示
- 豊富な地図スタイル
- ズーム・パン機能

**デメリット:**
- TopoJSON処理に別ライブラリが必要
- ライセンス費用が発生
- 学習コストが高い

**結論:** コストと複雑性を考慮し不採用

### Recharts + カスタム実装
**メリット:**
- 既存ライブラリとの統合
- 一貫したAPI

**デメリット:**
- 地図可視化の実装が困難
- TopoJSON処理の実装コストが高い
- パフォーマンスの問題

**結論:** 開発効率とパフォーマンスを考慮し不採用

## 結果

この決定により以下の効果が期待されます：

### 1. 開発効率の向上
- 地図可視化の迅速な実装
- 複雑な可視化の簡易実装

### 2. パフォーマンスの最適化
- TopoJSONによる軽量化
- 効率的なレンダリング

### 3. 機能の拡張性
- カスタム可視化の柔軟な実装
- 将来の要件変更への対応

### 4. ユーザー体験の向上
- インタラクティブな地図表示
- 直感的なデータ可視化

- キーボードナビゲーション
