# Cloudflare D1 データベース採用理由

## ステータス
accepted

## 背景

stats47 プロジェクトでは、以下の要件を満たすデータベースが必要でした：

1. **サーバーレス対応**: Cloudflare Workersとの統合
2. **コスト効率**: 無料枠での運用
3. **スケーラビリティ**: 将来の拡張に対応
4. **型安全性**: TypeScriptとの統合
5. **開発効率**: 簡単なセットアップとメンテナンス

## 決定

**Cloudflare D1 (SQLite)** を採用

## 理由

### 1. サーバーレス統合
- **Cloudflare Workers対応**: 完全なサーバーレス環境
- **グローバル配信**: エッジでのデータアクセス
- **自動スケーリング**: トラフィックに応じた自動拡張
- **低レイテンシ**: エッジでの高速データアクセス

### 2. コスト効率
- **無料枠**: 月100,000リクエストまで無料
- **従量課金**: 使用量に応じた課金
- **予測可能なコスト**: 明確な料金体系
- **運用コスト削減**: サーバー管理が不要

### 3. SQLiteの利点
- **軽量**: ファイルベースのデータベース
- **高速**: ローカルでの高速アクセス
- **シンプル**: 複雑な設定が不要
- **移植性**: データの移行が容易

### 4. 開発効率
- **簡単なセットアップ**: 数分での環境構築
- **Wrangler統合**: コマンドラインでの管理
- **マイグレーション**: バージョン管理されたスキーマ変更
- **バックアップ**: 自動バックアップ機能

## 使用箇所

### 1. データベーススキーマ
```sql
-- 統計データテーブル
CREATE TABLE statistics (
  id TEXT PRIMARY KEY,
  region_code TEXT NOT NULL,
  category_code TEXT NOT NULL,
  year INTEGER NOT NULL,
  value REAL,
  unit TEXT,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 地域マスターテーブル
CREATE TABLE regions (
  code TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  prefecture_code TEXT,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- カテゴリマスターテーブル
CREATE TABLE categories (
  code TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  parent_code TEXT,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### 2. データベース接続
```typescript
// lib/database.ts
import { drizzle } from 'drizzle-orm/d1';
import { D1Database } from '@cloudflare/workers-types';

export function getDatabase(env: { DB: D1Database }) {
  return drizzle(env.DB);
}

// API Route
export async function GET(request: Request, { params }: { params: { id: string } }) {
  const db = getDatabase(env);
  const statistics = await db.select().from(statisticsTable).where(eq(statisticsTable.id, params.id));
  return Response.json(statistics);
}
```

### 3. マイグレーション
```sql
-- migrations/0001_create_statistics_table.sql
```

## 代替案の検討

### PostgreSQL (Supabase)
**メリット:**
- 豊富な機能
- 強力なクエリ機能
- リアルタイム機能

**デメリット:**
- コストが高い
- 複雑な設定
- サーバーレス統合が限定的

**結論:** コストとシンプルさを考慮しD1を採用

### MySQL (PlanetScale)
**メリット:**
- 成熟したデータベース
- 豊富な機能
- スケーラビリティ

**デメリット:**
- コストが高い
- 複雑な設定
- サーバーレス統合が限定的

**結論:** コストとシンプルさを考慮しD1を採用

### MongoDB (MongoDB Atlas)
**メリット:**
- ドキュメントベース
- 柔軟なスキーマ
- スケーラビリティ

**デメリット:**
- コストが高い
- 学習コストが高い
- サーバーレス統合が限定的

**結論:** コストとシンプルさを考慮しD1を採用
