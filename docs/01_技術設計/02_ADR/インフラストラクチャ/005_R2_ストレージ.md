# Cloudflare R2 ストレージ採用理由

## ステータス
accepted

## 背景

stats47 プロジェクトでは、以下の要件を満たすオブジェクトストレージが必要でした：

1. **ファイル保存**: 画像、ドキュメント、データファイルの保存
2. **CDN統合**: 高速なファイル配信
3. **コスト効率**: 低コストでの運用
4. **スケーラビリティ**: 将来の拡張に対応
5. **Cloudflare統合**: 既存インフラとの統合

## 決定

**Cloudflare R2** を採用

## 理由

### 1. Cloudflareエコシステム統合
- **Cloudflare Workers対応**: 完全なサーバーレス環境
- **CDN統合**: 自動的なCDN配信
- **グローバル配信**: エッジでの高速配信
- **統一管理**: 単一のダッシュボードでの管理

### 2. コスト効率
- **S3互換**: 既存のS3ツールとの互換性
- **低コスト**: 従来のクラウドストレージより安価
- **無料枠**: 月10GBまで無料
- **従量課金**: 使用量に応じた課金

### 3. パフォーマンス
- **低レイテンシ**: エッジでの高速アクセス
- **自動最適化**: 画像の自動最適化
- **キャッシング**: 効率的なキャッシュ戦略
- **帯域幅最適化**: データ転送の最適化

### 4. 開発効率
- **S3 API互換**: 既存のS3ライブラリが使用可能
- **簡単なセットアップ**: 数分での環境構築
- **Wrangler統合**: コマンドラインでの管理
- **型安全性**: TypeScriptとの統合

## 使用箇所

### 1. ファイルアップロード
```typescript
// lib/r2.ts
import { S3Client, PutObjectCommand } from '@aws-sdk/client-s3';

const r2Client = new S3Client({
  region: 'auto',
  endpoint: `https://${ACCOUNT_ID}.r2.cloudflarestorage.com`,
  credentials: {
    accessKeyId: R2_ACCESS_KEY_ID,
    secretAccessKey: R2_SECRET_ACCESS_KEY,
  },
});

export async function uploadFile(file: File, key: string) {
  const command = new PutObjectCommand({
    Bucket: R2_BUCKET_NAME,
    Key: key,
    Body: file,
    ContentType: file.type,
  });
  
  return await r2Client.send(command);
}
```

### 2. 画像最適化
```typescript
// lib/image-optimization.ts
export function getOptimizedImageUrl(key: string, options: {
  width?: number;
  height?: number;
  quality?: number;
  format?: 'webp' | 'jpeg' | 'png';
}) {
  const params = new URLSearchParams();
  if (options.width) params.set('width', options.width.toString());
  if (options.height) params.set('height', options.height.toString());
  if (options.quality) params.set('quality', options.quality.toString());
  if (options.format) params.set('format', options.format);
  
  return `https://${R2_PUBLIC_URL}/${key}?${params.toString()}`;
}
```

### 3. ファイル管理
```typescript
// lib/file-management.ts
import { S3Client, ListObjectsV2Command, DeleteObjectCommand } from '@aws-sdk/client-s3';

export async function listFiles(prefix?: string) {
  const command = new ListObjectsV2Command({
    Bucket: R2_BUCKET_NAME,
    Prefix: prefix,
  });
  
  const response = await r2Client.send(command);
  return response.Contents || [];
}

export async function deleteFile(key: string) {
  const command = new DeleteObjectCommand({
    Bucket: R2_BUCKET_NAME,
    Key: key,
  });
  
  return await r2Client.send(command);
}
```

### 4. 静的ファイル配信
```typescript
// app/api/files/[key]/route.ts
export async function GET(
  request: Request,
  { params }: { params: { key: string } }
) {
  try {
    const file = await getFileFromR2(params.key);
    
    return new Response(file.Body, {
      headers: {
        'Content-Type': file.ContentType || 'application/octet-stream',
        'Cache-Control': 'public, max-age=31536000', // 1年
      },
    });
  } catch (error) {
    return new Response('File not found', { status: 404 });
  }
}
```

## 代替案の検討

### AWS S3
**メリット:**
- 成熟したサービス
- 豊富な機能
- 広く採用されている

**デメリット:**
- コストが高い
- データ転送料金が高い
- Cloudflareとの統合が限定的

**結論:** コストとCloudflare統合を考慮しR2を採用

### Google Cloud Storage
**メリット:**
- 高性能
- 豊富な機能
- Googleエコシステムとの統合

**デメリット:**
- コストが高い
- Cloudflareとの統合が限定的
- 学習コストが高い

**結論:** コストとCloudflare統合を考慮しR2を採用

### Azure Blob Storage
**メリット:**
- Microsoftエコシステムとの統合
- 企業向け機能
- セキュリティ機能

**デメリット:**
- コストが高い
- Cloudflareとの統合が限定的
- 複雑な設定

**結論:** コストとCloudflare統合を考慮しR2を採用
