# SWR 採用理由

## ステータス
accepted

## 背景

stats47 プロジェクトでは、以下の要件を満たすデータフェッチライブラリが必要でした：

1. **キャッシング**: 効率的なデータキャッシング
2. **再検証**: データの自動更新
3. **エラーハンドリング**: 統一されたエラー処理
4. **ローディング状態**: ローディング状態の管理
5. **型安全性**: TypeScriptとの統合

## 決定

**SWR** を採用

## 理由

### 1. 自動キャッシング
- **メモリキャッシュ**: 同一データの重複取得を防止
- **ストレージキャッシュ**: ブラウザストレージとの統合
- **キャッシュキー**: 柔軟なキャッシュキー管理
- **TTL**: キャッシュの有効期限設定

### 2. 再検証機能
- **自動再検証**: フォーカス時の自動更新
- **間隔再検証**: 定期的なデータ更新
- **手動再検証**: 必要に応じた手動更新
- **楽観的更新**: UIの即座な更新

### 3. エラーハンドリング
- **統一されたエラー処理**: 一貫したエラー管理
- **リトライ機能**: 自動的な再試行
- **フォールバック**: エラー時の代替データ
- **エラー境界**: コンポーネントレベルでのエラー処理

### 4. 開発者体験
- **シンプルなAPI**: 直感的な使用方法
- **TypeScript統合**: 型安全なデータフェッチ
- **DevTools**: デバッグツールの提供
- **軽量**: 小さなバンドルサイズ

## 使用箇所

### 1. 基本的なデータフェッチ
```typescript
import useSWR from 'swr';

function StatisticsPage() {
  const { data, error, isLoading } = useSWR('/api/statistics', fetcher);
  
  if (isLoading) return <Loading />;
  if (error) return <Error message={error.message} />;
  
  return <StatisticsList data={data} />;
}
```

### 2. 条件付きデータフェッチ
```typescript
function CategoryStatistics({ categoryId }: { categoryId: string }) {
  const { data, error } = useSWR(
    categoryId ? `/api/statistics/${categoryId}` : null,
    fetcher
  );
  
  if (!categoryId) return null;
  if (error) return <Error />;
  
  return <CategoryData data={data} />;
}
```

### 3. カスタムフェッチャー
```typescript
const fetcher = async (url: string) => {
  const response = await fetch(url);
  if (!response.ok) {
    throw new Error('データの取得に失敗しました');
  }
  return response.json();
};

function useStatisticsData(regionId: string) {
  return useSWR(
    `/api/statistics/region/${regionId}`,
    fetcher,
    {
      revalidateOnFocus: false,
      revalidateOnReconnect: true,
      refreshInterval: 30000, // 30秒ごとに更新
    }
  );
}
```

### 4. ミューテーション
```typescript
import useSWRMutation from 'swr/mutation';

function useUpdateStatistics() {
  const { trigger, isMutating } = useSWRMutation(
    '/api/statistics',
    async (url, { arg }) => {
      const response = await fetch(url, {
        method: 'PUT',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify(arg),
      });
      return response.json();
    }
  );
  
  return { updateStatistics: trigger, isUpdating: isMutating };
}
```

## 代替案の検討

### TanStack Query (React Query)
**メリット:**
- 豊富な機能
- 強力なキャッシング
- ミューテーション機能

**デメリット:**
- 学習コストが高い
- バンドルサイズが大きい
- 複雑な設定

**結論:** シンプルさを重視しSWRを採用

### Apollo Client
**メリット:**
- GraphQL特化
- 強力なキャッシング
- リアルタイム機能

**デメリット:**
- GraphQL専用
- バンドルサイズが大きい
- 学習コストが高い

**結論:** REST APIを使用するため不採用

### ネイティブfetch
**メリット:**
- 追加の依存関係なし
- 完全な制御

**デメリット:**
- キャッシング機能なし
- エラーハンドリングが複雑
- ローディング状態の管理が困難

**結論:** 機能要件を満たせないため不採用
