## 設計原則

### 1. 関心の分離

- 各ドキュメントは明確な責務を持つ
- 設計（WHAT/WHY）と実装（HOW）を分離

### 2. 一貫性

- プロジェクト全体で統一された設計思想
- ドメイン駆動設計の原則に基づく

### 3. 保守性

- 将来の変更に対応可能な設計
- 明確な依存関係と制約

### 4. パフォーマンス

- 高速な初期表示とユーザー体験
- 適切なキャッシング戦略

## ドキュメントの使い方

### 新規開発時

1. [システムアーキテクチャ](./01_システム概要/システムアーキテクチャ.md)で全体像を把握
2. [DDD ドメイン分類](./03_ドメイン設計/DDDドメイン分類.md)でドメインを特定
3. 該当する設計ドキュメントを参照して実装

### 技術選定時

1. [ADR 一覧](01_技術設計/02_ADR/00_はじめに.md)で過去の決定を確認
2. 新しい ADR を作成して決定を記録
3. 関連ドキュメントを更新

### 問題解決時

1. [エラーハンドリング](./06_設計パターン/エラーハンドリング.md)で統一された対応
2. [パフォーマンス設計](./06_設計パターン/パフォーマンス設計.md)で最適化手法を確認
