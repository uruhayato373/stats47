# プロジェクト構造

## モノレポ構成

```
stats47/
├── apps/
│   ├── web/             # メインの Next.js アプリ (App Router)
│   └── video/           # 動画生成 (Remotion)
├── packages/            # 共有パッケージ
├── docs/                # ドキュメント
└── scripts/             # ユーティリティスクリプト
```

## packages/

| パッケージ | 役割 |
|---|---|
| `database` | D1/SQLite スキーマ (Drizzle ORM)・マイグレーション・シード |
| `visualization` | D3 チャート・地図コンポーネント (React) |
| `r2-storage` | Cloudflare R2 ストレージ抽象化 |
| `estat-api` | e-Stat API ラッパー・データ変換 |
| `ranking` | ランキングアルゴリズム・データ処理 |
| `area` | 地域コード型定義・ユーティリティ |
| `area-profile` | 地域プロフィールデータサービス |
| `category` | カテゴリデータサービス |
| `correlation` | 相関分析ロジック |
| `ai-content` | AI コンテンツ生成 |
| `gis` | GIS ツール (TopoJSON・geoshape) |
| `components` | 共有 UI コンポーネント (Radix UI) |
| `types` | 共有 TypeScript 型定義 |
| `utils` | 汎用ユーティリティ |
| `mock` | テスト用モックデータ生成 |
| `logger` | ロギングユーティリティ |

## apps/web の構造

```
apps/web/src/
├── app/           # App Router ルート定義
├── features/      # 機能別モジュール（主要なコード配置先）
├── components/    # 共有コンポーネント (organisms/molecules)
├── hooks/         # カスタムフック
├── lib/           # ユーティリティ (analytics, metadata, structured-data 等)
├── store/         # 状態管理 (Jotai)
├── config/        # 設定
├── constants/     # 定数
├── types/         # 型定義
├── providers/     # React Provider
└── middleware.ts
```

### ルート構造

```
app/
├── page.tsx                           # トップページ
├── ranking/[rankingKey]/              # ランキング詳細
├── dashboard/[categoryKey]/[areaCode]/ # 地域別ダッシュボード (※)
├── areas/[areaCode]/                  # 地域プロフィール
├── compare/                           # 地域比較
├── correlation/                       # 相関分析
├── search/                            # 検索
├── blog/[slug]/                       # ブログ記事
├── privacy/                           # プライバシーポリシー
└── terms/                             # 利用規約
```

※ dashboard のルートは `[subcategoryKey]` でサブカテゴリ別、`[areaCode]` で地域別に表示。

### features/

機能ごとにモジュールを分割。各 feature は以下のサブディレクトリを持つ。

```
features/{name}/
├── actions/       # Server Actions
├── components/    # UI コンポーネント
├── services/      # データ取得・変換ロジック
├── adapters/      # データ変換アダプター
├── types/         # 型定義
├── utils/         # ユーティリティ
├── hooks/         # フック
└── lib/           # 内部ライブラリ
```

全 feature が全サブディレクトリを持つわけではない。feature 一覧は [DDDドメイン分類](./04_DDDドメイン分類.md) を参照。

### components/

共有 UI コンポーネント。レイアウト系が中心。

```
components/
├── organisms/     # Header, Footer, Sidebar, PageHeader, MobileNavigation
└── molecules/     # ShareButtons, BreadcrumbStructuredDataScript, DatasetStructuredDataScript
```
