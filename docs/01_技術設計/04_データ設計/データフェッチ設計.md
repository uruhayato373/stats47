# データフェッチ設計（層分離・環境抽象）

> 本文書は WHAT/WHY（設計総論）です。useSWR の具体的な実装 HOW は `docs/04_開発ガイド/データフェッチ.md` を参照。

## 1. 設計原則

- 責務分離: キー/フェッチャー/変換/表示の分離
- 型安全: Params/Response/Error を明示
- 環境抽象: Data Access Layer が環境差分を吸収
- キャッシュ前提: 冪等性・再検証戦略を設計に含める

## 2. 層構造

```
UI (components) ── hooks (SWR) ── fetcher ── formatter ── client(HTTP/DB)
```

- hooks は UI から渡された Params を Key に正規化
- fetcher は副作用（HTTP/DB）を一元化
- formatter は型付けと正規化、ドメイン知識の集約点

## 3. 環境別データソース

- `NEXT_PUBLIC_ENV` で mock/development/staging/production を切替
- UI/Hook では環境分岐しない（Data Access Layer で吸収）

## 4. レンダリング戦略の整合

- SSR/ISR を「初速・共有性」が高いデータに適用
- CSR+SWR を「ユーザー依存・頻更新」に適用

## 5. 失敗耐性

- フェッチャーで例外を構造化し UI は文言統一
- 再試行ポリシー（回数/間隔）をデータ特性で設定
