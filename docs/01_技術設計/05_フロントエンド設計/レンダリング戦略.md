# レンダリング戦略

## 概要

stats47 プロジェクトでは、Next.js 15 の App Router を活用して、SSR/ISR/CSR を適切に使い分けるレンダリング戦略を採用しています。

## レンダリング戦略の選択指針

### データ特性による選択

| データ特性             | レンダリング方式 | 適用例                                 |
| ---------------------- | ---------------- | -------------------------------------- |
| 初期体験重要・共有可能 | SSR/ISR          | ランディングページ、ランキング初期表示 |
| ユーザー依存・頻更新   | CSR+SWR          | 絞り込み、個別設定                     |
| 静的・変更頻度低       | SSG              | カテゴリ一覧、ヘルプページ             |

### 判断基準

```
データの特性を分析
    ↓
「CPU/IO コスト高 or 初速重要」？
    ↓ YES
    SSR/ISR を適用
    ↓ NO
「ユーザー固有/頻更新」？
    ↓ YES
    CSR+SWR を適用
    ↓ NO
    SSG を適用
```

## レンダリング方式の詳細

### 1. Server-Side Rendering (SSR)

#### 適用場面

- 初期体験が重要なページ
- SEO が重要なページ
- 共有可能なデータの表示

#### 実装例

```typescript
// app/(stats)/[category]/[subcategory]/page.tsx
export default async function SubcategoryPage({
  params,
}: {
  params: { category: string; subcategory: string };
}) {
  // サーバーサイドでデータ取得
  const data = await getSubcategoryData(params.category, params.subcategory);

  return (
    <div>
      <h1>{data.title}</h1>
      <RankingDisplay data={data.rankings} />
    </div>
  );
}
```

#### メリット

- 初期表示が高速
- SEO に有利
- 共有可能なデータの最適化

#### デメリット

- サーバー負荷が高い
- ユーザー固有データには不向き

### 2. Incremental Static Regeneration (ISR)

#### 適用場面

- 定期的に更新されるデータ
- キャッシュ可能なデータ
- パフォーマンスと更新のバランス

#### 実装例

```typescript
// app/(stats)/[category]/[subcategory]/ranking/[rankingKey]/page.tsx
export async function generateStaticParams() {
  const rankings = await getRankingKeys();
  return rankings.map((ranking) => ({
    rankingKey: ranking.key,
  }));
}

export const revalidate = 3600; // 1時間ごとに再生成

export default async function RankingPage({
  params,
}: {
  params: { rankingKey: string };
}) {
  const data = await getRankingData(params.rankingKey);

  return <RankingDisplay data={data} />;
}
```

#### メリット

- 静的生成の高速性
- 定期的な更新
- CDN キャッシュの活用

#### デメリット

- 更新の遅延
- 複雑な設定

### 3. Client-Side Rendering (CSR)

#### 適用場面

- ユーザー固有のデータ
- 頻繁に更新されるデータ
- インタラクティブな機能

#### 実装例

```typescript
"use client";

// app/(stats)/[category]/[subcategory]/area/[areaCode]/page.tsx
export default function AreaDashboard({
  params,
}: {
  params: { areaCode: string };
}) {
  const { data, error, isLoading } = useSWR(
    `/api/dashboard/${params.areaCode}`,
    fetcher
  );

  if (isLoading) return <LoadingSpinner />;
  if (error) return <ErrorMessage error={error} />;

  return <Dashboard data={data} />;
}
```

#### メリット

- リアルタイム更新
- ユーザー固有データの最適化
- インタラクティブな機能

#### デメリット

- 初期表示が遅い
- SEO に不利
- JavaScript 依存

## ハイブリッド戦略

### 1. SSR + CSR の組み合わせ

```typescript
// app/(stats)/[category]/[subcategory]/page.tsx
export default async function SubcategoryPage({
  params,
}: {
  params: { category: string; subcategory: string };
}) {
  // 初期データをSSRで取得
  const initialData = await getSubcategoryData(
    params.category,
    params.subcategory
  );

  return (
    <div>
      <h1>{initialData.title}</h1>
      {/* 初期データを表示 */}
      <RankingDisplay data={initialData.rankings} />

      {/* インタラクティブな部分はCSR */}
      <InteractiveFilters initialData={initialData} />
    </div>
  );
}
```

### 2. ISR + CSR の組み合わせ

```typescript
// app/(stats)/[category]/[subcategory]/ranking/[rankingKey]/page.tsx
export const revalidate = 3600; // 1時間ごとに再生成

export default async function RankingPage({
  params,
}: {
  params: { rankingKey: string };
}) {
  // 基本データをISRで取得
  const baseData = await getRankingData(params.rankingKey);

  return (
    <div>
      <h1>{baseData.title}</h1>
      {/* 基本データを表示 */}
      <RankingTable data={baseData.rankings} />

      {/* 動的な部分はCSR */}
      <DynamicFilters baseData={baseData} />
    </div>
  );
}
```

// Server Component
export default async function Page() {

## キャッシング戦略

### 1. Next.js キャッシング

```typescript
// データキャッシュ
export default async function Page() {
  const data = await fetch("/api/data", {
    next: { revalidate: 3600 }, // 1時間キャッシュ
  });

  return <Content data={data} />;
}

// ルートキャッシュ
export const revalidate = 3600; // 1時間ごとに再生成
```

### 2. SWR キャッシング

```typescript
"use client";

export function DataComponent() {
  const { data, error } = useSWR("/api/data", fetcher, {
    revalidateOnFocus: false,
    revalidateOnReconnect: true,
    dedupingInterval: 60000, // 1分間の重複排除
    errorRetryCount: 3,
    errorRetryInterval: 5000,
  });

  return <Content data={data} />;
}
```

## エラーハンドリング

### 1. Server Component エラー

```typescript
// app/error.tsx
"use client";

export default function Error({
  error,
  reset,
}: {
  error: Error & { digest?: string };
  reset: () => void;
}) {
  return (
    <div>
      <h2>Something went wrong!</h2>
      <button onClick={() => reset()}>Try again</button>
    </div>
  );
}
```

### 2. Client Component エラー

```typescript
"use client";

export function DataComponent() {
  const { data, error } = useSWR("/api/data", fetcher);

  if (error) {
    return <ErrorMessage error={error} />;
  }

  return <Content data={data} />;
}
```
