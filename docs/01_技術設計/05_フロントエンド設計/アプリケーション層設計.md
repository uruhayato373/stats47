# アプリケーション層設計

## 概要

アプリケーション層は、複数のドメインを統合してユーザーのユースケースを実現する層です。ドメイン層のサービスを組み合わせて、具体的なビジネス機能を提供します。

### 役割

- **ドメイン統合**: 複数のドメインサービスを組み合わせてユースケースを実現
- **トランザクション管理**: 複数ドメインにまたがる操作の整合性保証
- **DTO変換**: ドメインオブジェクトとプレゼンテーション層の間のデータ変換
- **ユースケース実装**: 具体的なビジネス機能の実装

## AreaProfile（地域プロファイル）

### 概要

地域プロファイル機能は、複数のコアドメイン（Ranking、Comparison、TimeSeries）と支援ドメイン（Area、Geographic）を統合して、地域の総合的なプロファイルを生成します。

### 責務

- 地域の基本情報とランキングデータの統合
- 時系列トレンドの分析と表示
- 類似地域の検出と表示
- 地域の強み・特徴の分析

### AreaProfileService

複数のドメインサービスを統合して地域プロファイルを生成するアプリケーションサービス。

```typescript
export class AreaProfileService {
  constructor(
    // コアドメイン
    private rankingService: RankingService,
    private comparisonService: ComparisonService,
    private timeSeriesService: TimeSeriesService,
    // 支援ドメイン
    private areaService: AreaService,
    private geographicService: GeographicService
  ) {}

  async generateAreaProfile(areaCode: string): Promise<AreaProfileDto> {
    // 1. 基本情報を取得
    const basicInfo = await this.areaService.getAreaInfo(areaCode);
    
    // 2. ランキングデータを取得
    const topRankings = await this.rankingService.getTopRankings(areaCode, 10);
    
    // 3. 時系列トレンドを取得
    const trends = await this.timeSeriesService.getKeyTrends(areaCode);
    
    // 4. 類似地域を検出
    const similarAreas = await this.comparisonService.findSimilarAreas(areaCode);
    
    // 5. 地理情報を取得
    const geoInfo = await this.geographicService.getAreaGeoInfo(areaCode);

    return {
      areaCode,
      basicInfo,
      topRankings,
      trends,
      similarAreas,
      geoInfo
    };
  }

  async compareAreas(areaCode1: string, areaCode2: string): Promise<AreaComparisonDto> {
    const profile1 = await this.generateAreaProfile(areaCode1);
    const profile2 = await this.generateAreaProfile(areaCode2);
    
    const comparison = await this.comparisonService.compareAreaProfiles(profile1, profile2);
    
    return {
      area1: profile1,
      area2: profile2,
      comparison
    };
  }
}
```

### AreaProfileQuery

地域プロファイルの検索・フィルタリング機能を提供するクエリサービス。

```typescript
export class AreaProfileQuery {
  constructor(
    private areaRepository: AreaRepository,
    private rankingRepository: RankingRepository
  ) {}

  async searchByRanking(rankingKey: string, topN: number): Promise<AreaProfileSummaryDto[]> {
    const rankings = await this.rankingRepository.findTopByRanking(rankingKey, topN);
    
    return rankings.map(ranking => ({
      areaCode: ranking.areaCode,
      areaName: ranking.areaName,
      rank: ranking.rank,
      value: ranking.value,
      percentile: ranking.percentile
    }));
  }

  async filterByCategory(categoryId: string): Promise<AreaProfileSummaryDto[]> {
    const rankings = await this.rankingRepository.findByCategory(categoryId);
    
    return rankings.map(ranking => ({
      areaCode: ranking.areaCode,
      areaName: ranking.areaName,
      categoryRankings: ranking.categoryRankings
    }));
  }
}
```

### DTO（Data Transfer Object）

アプリケーション層とプレゼンテーション層の間でデータを転送するためのオブジェクト。

```typescript
export interface AreaProfileDto {
  areaCode: string;
  basicInfo: AreaBasicInfoDto;
  topRankings: RankingSummaryDto[];
  trends: TimeSeriesTrendDto[];
  similarAreas: SimilarAreaDto[];
  geoInfo: GeoInfoDto;
}

export interface AreaComparisonDto {
  area1: AreaProfileDto;
  area2: AreaProfileDto;
  comparison: ComparisonResultDto;
}

export interface AreaProfileSummaryDto {
  areaCode: string;
  areaName: string;
  rank?: number;
  value?: number;
  percentile?: number;
  categoryRankings?: CategoryRankingDto[];
}
```

## Dashboard（ダッシュボード）

### 概要

ダッシュボード機能は、複数の統計データを統合して一覧表示する機能です。

### DashboardService

ダッシュボードのデータを統合して提供するアプリケーションサービス。

```typescript
export class DashboardService {
  constructor(
    private rankingService: RankingService,
    private timeSeriesService: TimeSeriesService,
    private comparisonService: ComparisonService
  ) {}

  async generateDashboard(categoryId: string): Promise<DashboardDto> {
    // 1. カテゴリ別ランキングを取得
    const rankings = await this.rankingService.getCategoryRankings(categoryId);
    
    // 2. 時系列トレンドを取得
    const trends = await this.timeSeriesService.getCategoryTrends(categoryId);
    
    // 3. 比較分析を実行
    const comparisons = await this.comparisonService.getCategoryComparisons(categoryId);

    return {
      categoryId,
      rankings,
      trends,
      comparisons,
      lastUpdated: new Date()
    };
  }
}
```

## RankingList（ランキング一覧）

### 概要

ランキング一覧機能は、特定の統計指標のランキングを表示する機能です。

### RankingListService

ランキング一覧のデータを提供するアプリケーションサービス。

```typescript
export class RankingListService {
  constructor(
    private rankingService: RankingService,
    private comparisonService: ComparisonService
  ) {}

  async getRankingList(rankingKey: string): Promise<RankingListDto> {
    // 1. ランキングデータを取得
    const rankings = await this.rankingService.getRankings(rankingKey);
    
    // 2. 統計情報を計算
    const statistics = await this.comparisonService.calculateRankingStatistics(rankings);

    return {
      rankingKey,
      rankings,
      statistics,
      totalCount: rankings.length
    };
  }
}
```

## ベストプラクティス

### 1. ドメインサービスの組み合わせ

複数のドメインサービスを適切に組み合わせてユースケースを実現する。

### 2. トランザクション管理

複数ドメインにまたがる操作では、適切なトランザクション管理を行う。

### 3. DTOの設計

プレゼンテーション層に必要な情報のみを含むDTOを設計する。

### 4. エラーハンドリング

各ドメインサービスからのエラーを適切にハンドリングし、ユーザーに分かりやすいエラーメッセージを提供する。

### 5. パフォーマンス最適化

必要に応じて並列処理やキャッシュを活用してパフォーマンスを最適化する。

## 依存関係

### ドメイン層への依存

- **Ranking ドメイン**: ランキングデータの取得
- **Comparison ドメイン**: 比較分析の実行
- **TimeSeries ドメイン**: 時系列データの取得
- **Area ドメイン**: 地域情報の取得
- **Geographic ドメイン**: 地理データの取得

### インフラ層への依存

- **Repository実装**: データの永続化
- **外部API**: 外部データの取得
