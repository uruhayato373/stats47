# コンポーネントアーキテクチャ設計

## 1. 概要

このドキュメントでは、stats47 プロジェクトのコンポーネントアーキテクチャの設計原則と構造について説明します。本プロジェクトは、e-Stat API から取得した統計データを視覚化し、ユーザーに直感的なデータ探索体験を提供することを目的としています。

### 設計原則

- **単一責任の原則**: 各コンポーネントは一つの明確な責任を持つ
- **再利用性**: 共通の UI パターンは共通コンポーネントとして実装
- **型安全性**: TypeScript の型定義を活用
- **アクセシビリティ**: WCAG 準拠の実装
- **パフォーマンス**: サーバーコンポーネントとクライアントコンポーネントの適切な分離
- **データ可視化**: 時系列分析と地図可視化の専門コンポーネント

### アーキテクチャ方針

- **Atomic Design**: UI コンポーネントを階層的に整理
- **コンポーネント解決システム**: 動的なコンポーネント選択
- **データ駆動**: データの構造に基づいたコンポーネント設計
- **レスポンシブファースト**: モバイルからデスクトップへの段階的拡張

## 2. アーキテクチャパターン

### 2.1 ページコンポーネント構造

#### カテゴリーページ

カテゴリーページ（`/[category]`）は、各統計カテゴリーのランディングページとして機能し、所属するサブカテゴリーへの導線を提供します。

**設計目標:**

- 明確なナビゲーション: サブカテゴリーへの分かりやすい導線
- 視覚的な整理: カテゴリーアイコンと色による識別
- 柔軟な対応: サブカテゴリーの有無に応じた表示
- 一貫性: 全カテゴリーで統一されたレイアウト

**アーキテクチャ:**

```
CategoryPageClient
├── CategoryHeader (アイコン、パンくず、タイトル)
├── SubcategoryGrid (サブカテゴリーカード群)
└── EmptyState (サブカテゴリーなしの場合)
```

#### サブカテゴリーページ

サブカテゴリーページ（`/[category]/[subcategory]`）は、カテゴリーごとに整理されたコンポーネント構造を採用し、各サブカテゴリーに最適化されたレイアウトとデータ可視化を提供します。

**設計目標:**

- 柔軟性: サブカテゴリーごとに異なるレイアウトとデータ表示
- 保守性: カテゴリー別のディレクトリ構造による整理
- 再利用性: 共通レイアウトとコンポーネントの活用
- 拡張性: 新しいサブカテゴリーの追加が容易

**ディレクトリ構造:**

```
src/components/subcategories/
├── SubcategoryLayout.tsx              # 共通レイアウトコンポーネント
├── index.tsx                          # コンポーネントマッピング
├── landweather/                       # 国土・気象カテゴリー
├── population/                        # 人口・世帯カテゴリー
├── laborwage/                         # 労働・賃金カテゴリー
└── [その他のカテゴリー]/
```

#### ダッシュボードページ

ダッシュボードページは、全国用と都道府県用に分離された新しいアーキテクチャを採用しています。各サブカテゴリーは 2 つの専用コンポーネントを持ち、`areaCode`に基づいて動的に選択されます。

**アーキテクチャパターン:**

- **NationalDashboard**: 全国専用ダッシュボード（`areaCode === "00000"`）
- **PrefectureDashboard**: 都道府県専用ダッシュボード（`areaCode !== "00000"`）

#### コンポーネント解決システム

動的なコンポーネント選択を実現するシステム:

```typescript
export const getDashboardComponentByArea = (
  subcategoryId: string,
  areaCode: string,
  categoryId?: string
): React.ComponentType<SubcategoryDashboardPageProps> => {
  const subcategory = getSubcategoryInfo(subcategoryId, categoryId);

  if (!subcategory) {
    return DefaultDashboardPage;
  }

  const ComponentMap = subcategory.components;

  if (areaCode === "00000") {
    return ComponentMap.national || DefaultDashboardPage;
  } else {
    return ComponentMap.prefecture || DefaultDashboardPage;
  }
};
```

## 3. データフロー設計

### 3.1 e-Stat API 統合パターン

**データ取得フロー:**

```
Component → Hook → API Service → e-Stat API
    ↓
State Management → UI Update
```

**主要な統合ポイント:**

- メタ情報取得: `useEstatMetaInfo`
- 統計データ取得: `useEstatStatsData`
- フィールド統計: `useFieldStats`

### 3.2 Cloudflare D1 統合パターン

**データ保存フロー:**

```
User Action → Component → Hook → Database Service → D1
    ↓
Optimistic Update → UI Update → Background Sync
```

**主要な統合ポイント:**

- メタ情報保存: `useMetaInfoSave`
- ランキングデータ: `useRankingData`
- エクスポート機能: 削除済み（未使用のため）

### 3.3 データ保存戦略

**階層的データ管理:**

1. **ローカル状態**: コンポーネント内の一時的な状態
2. **グローバル状態**: アプリケーション全体で共有される状態
3. **永続化**: Cloudflare D1 でのデータ永続化
4. **キャッシュ**: API レスポンスのキャッシュ戦略

## 5. パフォーマンス設計

### 5.1 コンポーネント分離戦略

**サーバーコンポーネント:**

- 静的コンテンツのレンダリング
- データ取得の最適化
- SEO 対応

**クライアントコンポーネント:**

- インタラクティブな要素
- 状態管理が必要な部分
- ブラウザ API の使用

### 5.2 レンダリング最適化

**メモ化戦略:**

- `useMemo`: 計算コストの高い処理
- `useCallback`: イベントハンドラーの最適化
- `React.memo`: コンポーネントの再レンダリング防止

**遅延読み込み:**

- 動的インポート: 必要な時のみコンポーネントを読み込み
- 画像の遅延読み込み: `loading="lazy"`
- データの段階的読み込み: ページネーション、無限スクロール

## 6. アクセシビリティ設計

### 6.1 WCAG 準拠

- スクリーンリーダー対応: 適切な ARIA ラベルとセマンティック HTML

## 7. 地図可視化コンポーネント

### 7.1 ChoroplethMap

コロプレス地図の表示を担当するコンポーネント。歴史的行政区域データセット（CODH）の行政区域データを使用。

**設計目標:**

- 都道府県レベルの地図表示（歴史的行政区域データセット使用）
- 統計値による色分け
- インタラクティブ機能（ズーム、パン、ホバー）
- カラースケールの選択
- 分類方法の選択（等間隔、分位数、自然分類）
- データソース表記（CODH、国土交通省への帰属表示）
- 将来の時系列アニメーション対応（Phase 3）

**使用ライブラリ:**

- react-leaflet: 地図表示
- topojson-client: TopoJSON のパース
- d3-scale: カラースケール生成
- d3-scale-chromatic: カラーパレット

**データソース:**

- 地理データ（行政区域）: 歴史的行政区域データセット β 版（CODH、CC BY 4.0）
- 地理データ（施設・災害等）: 国土数値情報（国土交通省、CC BY 4.0）※Phase 3 以降
- 統計データ: e-Stat API

**アーキテクチャ:**

```
ChoroplethMap
├── MapControls (ズーム、パン、リセット)
├── BaseMapSelector (ベースマップ切り替え)
├── MapCanvas (react-leaflet ベース)
│   ├── GsiTileLayer (国土地理院タイル)
│   ├── GeoShapes (都道府県ポリゴン)
│   ├── ColorScale (色分け)
│   └── Tooltip (ホバー情報)
├── MapLegend (凡例、カラーバー)
├── MapSettings (カラースキーム、分類方法)
└── Attribution (データソース表記)
```

**Props:**

```typescript
interface ChoroplethMapProps {
  data: ChoroplethData[];
  topoJsonUrl: string; // 歴史的行政区域データセットのTopoJSON
  year?: number; // 表示年度（デフォルト: 最新）
  colorScheme: "sequential" | "diverging";
  classificationMethod: "equal" | "quantile" | "natural";
  showLegend: boolean;
  showAttribution: boolean; // データソース表記の表示
  interactive: boolean;
  basemapType?: "std" | "pale" | "blank" | "photo"; // ベースマップタイプ
  showBasemapSelector?: boolean; // ベースマップ選択UIの表示
  onAreaClick: (areaCode: string) => void;
  onAreaHover: (areaCode: string | null) => void;
  onBasemapChange?: (type: string) => void; // ベースマップ変更コールバック

```

**実装例:**

```typescript
import { MapContainer, GeoJSON, TileLayer } from "react-leaflet";
import { feature } from "topojson-client";
import { GsiTileLayer } from "./GsiTileLayer";
import { BaseMapSelector } from "./BaseMapSelector";

export function ChoroplethMap({
  topoJsonUrl,
  year = 2023,
  basemapType = "std",
  showBasemapSelector = true,
  ...props
}: ChoroplethMapProps) {
  const [geoData, setGeoData] = useState(null);

  useEffect(() => {
    fetch(topoJsonUrl)
      .then((res) => res.json())
      .then((topology) => {
        // TopoJSONをGeoJSONに変換
        const features = feature(topology, topology.objects.municipalities);
        setGeoData(features);
      });
  }, [topoJsonUrl]);

  return (
    <div className="choropleth-map">
      {showBasemapSelector && (
        <BaseMapSelector
          currentType={basemapType}
          onChange={props.onBasemapChange}
        />
      )}
      <MapContainer center={[36, 138]} zoom={5}>
        <GsiTileLayer type={basemapType} />
        {geoData && (
          <GeoJSON
            data={geoData}
            style={getColorStyle}
            onEachFeature={addInteractivity}
          />
        )}
        {props.showAttribution && (
          <Attribution text="国土地理院 | 歴史的行政区域データセットβ版（CODH作成）" />
        )}
      </MapContainer>
    </div>
  );
}
```

- 複数のタイルタイプ対応（標準、淡色、白地図、航空写真）
- 出典表示の自動生成
- パフォーマンス最適化

**Props:**

```typescript
interface GsiTileLayerProps {
  type: "std" | "pale" | "blank" | "photo";
  maxZoom?: number;
  attribution?: string;
}
```

**実装例:**

```typescript
import { TileLayer } from "react-leaflet";

export function GsiTileLayer({
  type = "std",
  maxZoom = 18,
  attribution = "国土地理院",
}: GsiTileLayerProps) {
  const tileUrls = {
    std: "https://cyberjapandata.gsi.go.jp/xyz/std/{z}/{x}/{y}.png",
    pale: "https://cyberjapandata.gsi.go.jp/xyz/pale/{z}/{x}/{y}.png",
    blank: "https://cyberjapandata.gsi.go.jp/xyz/blank/{z}/{x}/{y}.png",
    photo: "https://cyberjapandata.gsi.go.jp/xyz/seamlessphoto/{z}/{x}/{y}.jpg",
  };

  return (
    <TileLayer
      url={tileUrls[type]}
      attribution={`&copy; <a href="https://maps.gsi.go.jp/development/ichiran.html">${attribution}</a>`}
      maxZoom={maxZoom}
    />
  );
}
```

### 7.3 BaseMapSelector

ベースマップの選択を担当するコンポーネント。

**設計目標:**

- ベースマップタイプの選択
- 視覚的なプレビュー

- レスポンシブ対応

**Props:**

```typescript
interface BaseMapSelectorProps {
  currentType: "std" | "pale" | "blank" | "photo";
  onChange: (type: string) => void;
  className?: string;
}
```

**実装例:**

```typescript
export function BaseMapSelector({
  currentType,
  onChange,
  className,
}: BaseMapSelectorProps) {
  const basemapOptions = [
    { value: "std", label: "標準地図", icon: "🗺️" },
    { value: "pale", label: "淡色地図", icon: "🌫️" },
    { value: "blank", label: "白地図", icon: "⬜" },
    { value: "photo", label: "航空写真", icon: "📸" },
  ];

  return (
    <div className={`basemap-selector ${className}`}>
      {basemapOptions.map((option) => (
        <button
          key={option.value}
          onClick={() => onChange(option.value)}
          className={`basemap-option ${
            currentType === option.value ? "active" : ""
          }`}
          aria-label={`${option.label}を選択`}
        >
          <span className="icon">{option.icon}</span>
          <span className="label">{option.label}</span>
        </button>
      ))}
    </div>
  );
}
```
