# 市区町村コロプレスマップ切替機能 設計書

> 決定日: 2026-03-24

## 概要

ランキングページ（例: `/ranking/total-population`）で、市区町村データが存在する場合にコロプレスマップの表示粒度を都道府県 ↔ 市区町村で切り替える機能。

## 設計方針：市区町村データの管理

### 結論：既存スキーマで管理。追加のフラグ・管理機構は不要

`ranking_items` の複合 PK `(ranking_key, area_type)` により、同じランキングキーで `"prefecture"` と `"city"` のレコードが共存できる。市区町村データの有無は既存関数で判定する：

```typescript
const cityRankingItem = await findRankingItem(rankingKey, "city");
const hasCityData = cityRankingItem !== null && cityRankingItem.isActive;
```

### 検討した代替案と不採用理由

| 案 | 概要 | 不採用理由 |
|---|---|---|
| `has_city_data` カラム追加 | prefecture レコードに boolean フラグを追加 | 市区町村ランキング追加・削除時に同期メンテが必要。既に ranking_items にデータがあるため冗長 |
| 静的リスト/定数 | コード内に市区町村データありのキーを列挙 | 手動更新が必要で陳腐化するリスク。DB の状態と乖離する可能性 |
| 新規リポジトリ関数 (`hasCityRanking`) | boolean を返す専用クエリ | `findRankingItem(key, "city")` で十分。city の `availableYears` も必要なため RankingItem 全体を取得する方が合理的 |

### データ状況（2026-03-25 時点）

| 項目 | 値 |
|---|---|
| アクティブな市区町村ランキング | 86 件（元28件から3倍増） |
| コロプレスマップ切替対象（都道府県版と同一キー） | 74 件 |
| `-city` 接尾辞付き | 0 件（全て統合完了） |
| データなし（無効化済み） | 782 件（市区町村レベルにデータが存在しない統計） |

## UI 設計

### トグル配置

地図カードヘッダーの年度セレクタ横に `ToggleGroup` を配置：

```
[年度セレクタ] [都道府県 | 市区町村] [ダウンロード]
```

- 市区町村データがないランキングではトグル非表示
- 切替時に年度セレクタの選択肢を市区町村側の `availableYears` に更新

### URL パラメータ

```
/ranking/total-population?areaType=city&year=2020
```

- `areaType` パラメータ省略時は都道府県（デフォルト）
- ブックマーク・URL 共有に対応

## 地図表示：全国日本地図での切替方式

### 採用した方式：同一日本地図上で TopoJSON を切り替え

都道府県コロプレスと市区町村コロプレスを同じ日本地図上で切り替える。トグル操作のみで全国の市区町村分布を俯瞰できる。

1. **デフォルト**: 都道府県 TopoJSON（約 1MB）で都道府県コロプレスを表示
2. **市区町村に切替**: 全国市区町村 TopoJSON をオンデマンド取得し、市区町村コロプレスを表示
3. **TopoJSON キャッシュ**: 一度取得した市区町村 TopoJSON はコンポーネント state にキャッシュし、再切替時は再取得しない

### ドリルダウン方式を採用しなかった理由

| | ドリルダウン方式 | 全国切替方式（採用） |
|---|---|---|
| UX | 2ステップ（トグル→県クリック） | 1ステップ（トグルのみ） |
| 全国俯瞰 | 不可（1県ずつ） | **可能（統計サイトとしての最大の価値）** |
| 初期ロード | 軽い（都道府県 1MB のみ） | 同じ（市区町村は切替時にオンデマンド取得） |

### TopoJSON サイズ

| データ | サイズ |
|---|---|
| 都道府県 | 約 1MB |
| 全国市区町村（47県合計） | 約 19MB（個別ファイル合計。統合 TopoJSON はアーク共有で圧縮される） |

市区町村 TopoJSON はページ初期ロード時には取得せず、ユーザーが市区町村トグルをクリックした時点でサーバーアクション経由で取得する。

### 既存資産の再利用

| 再利用先 | 用途 |
|---|---|
| `fetchAllCitiesTopology()` (packages/gis) | 全国市区町村 TopoJSON 取得 |
| `LeafletChoroplethMap` | 都道府県/市区町村の TopoJSON を `topology` props で切り替えて描画 |

## 新規ランキング追加時の手順

市区町村データ付きのランキングを追加する場合：

1. `/register-ranking` で `areaType: "city"` のランキングを登録（`rankingKey` は都道府県版と同一にする）
2. ランキングページに自動的にトグルが表示される（追加のコード変更不要）

市区町村専用（都道府県版なし）のランキングはトグル対象外。専用ページで対応する。
