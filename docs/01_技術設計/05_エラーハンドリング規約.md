# エラーハンドリング規約

> 2026-02-20 制定。monorepo 全体でエラーハンドリングを統一するための規約。

---

## 層別エラーハンドリング方針

```
┌─────────────────────────────────────────────────────────┐
│  Server Actions (apps/web)                               │
│  → ActionResult / Result<T, E>                          │
│  → try/catch で下位層の throw をキャッチし変換           │
├─────────────────────────────────────────────────────────┤
│  Service Layer (packages/*/services/)                    │
│  → ドメイン固有Result or Result<T, E>                   │
│  → throw をキャッチして Result に変換する境界層          │
│  → 計算系は値を返し、失敗時は [] + logger.warn          │
├─────────────────────────────────────────────────────────┤
│  Repository Layer (packages/*/repositories/)             │
│  → Result<T, Error> を統一使用                          │
│  → ok(data) / err(error) で返す                         │
│  → [] 返却パターンは禁止                                │
├─────────────────────────────────────────────────────────┤
│  External API Client (estat-api, gis, r2-storage)       │
│  → throw を使用（リトライ・中断との相性が良い）          │
│  → カスタムErrorクラスで分類                             │
│  → 呼び出し元（Service層）が catch して Result に変換    │
└─────────────────────────────────────────────────────────┘
```

| 層 | 返却パターン | エラー型 | throw の扱い |
|----|------------|---------|-------------|
| Server Actions | `ActionResult` / `Result<T, E>` | `string`（UI向け） | catch して変換 |
| Service（オーケストレーション） | `Result<T, E>` or ドメイン固有Result | `Error` or `string` | catch して変換 |
| Service（計算/変換） | 値の型そのもの（`T[]` 等） | `[]` + `logger.warn` | catch してログ + デフォルト値 |
| Repository | `Result<T, Error>` | `Error` | catch して `err()` |
| 外部APIクライアント | throw | カスタム`Error` | そのまま throw |

---

## 型の使い方

### `Result<T, E>` — `@stats47/types` から使用

```typescript
import { type Result, ok, err, isOk, isErr } from "@stats47/types";

// Repository での使用例
async function fetchData(): Promise<Result<Data, Error>> {
  try {
    const data = await db.query(...);
    return ok(data);
  } catch (error) {
    return err(error instanceof Error ? error : new Error(String(error)));
  }
}

// 呼び出し側
const result = await fetchData();
if (result.success) {
  // result.data が利用可能
} else {
  // result.error が利用可能
}
```

---

## 原則

- `Result<T, E>` は `@stats47/types` からインポート
- Repository は `[]` を返さない。必ず `Result` で wrap する
- Server Actions 内で `unwrap()` は使わない（throw するため）。`.success` チェックを使用
- `isOk`/`isErr` ガードは使用してもよいが、`.success` プロパティの直接チェックも許容（既存コードの大半がこのパターン）
- `SyncRankingResult` 等のドメイン固有Result型は、追加フィールド（進捗情報等）が必要な場合にのみ定義可
