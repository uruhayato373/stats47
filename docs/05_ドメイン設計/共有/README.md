# 共有ドメイン（Shared Kernel）

## 目的

複数ドメインに跨って再利用される契約・語彙・データ構造を一元管理し、重複と循環依存を防ぐ。

## 適用範囲

- 真に横断的な型・値オブジェクト・DTO・イベントスキーマ
- 共通ポリシー（エラー語彙、バリデーション規約）
- 2 つ以上のドメインで実利用されるもの

## 禁止事項

- 特定ドメイン実装への依存（Ranking/Area 等）
- 便宜的な一時退避（“なんでも shared”）

## 依存ポリシー

- 依存方向: 各ドメイン → 共有（可）／共有 → 各ドメイン（不可）
- 循環参照を禁止

## 昇格要件（ガバナンス）

1. 2 ドメイン以上での実利用実績
2. 仕様レビュー（影響分析・代替案・破壊的変更可否）
3. バージョニング指針に従う（破壊的変更時は移行ガイド付与）
