# SNSプラットフォーム仕様リファレンス

全PF同時投稿に必要な技術仕様。画像/動画サイズ・テキスト生成ルール・ファイル制限を定義する。

> 戦略・テーマ選定・投稿設計は [SNSコンテンツ設計](./01_SNSコンテンツ設計.md) を参照。

---

## 1. フィード投稿（静止画・カルーセル）

| プラットフォーム | 推奨サイズ | 比率 | 備考 |
|---|---|---|---|
| **X (Twitter)** | 1200 × 675 px | 16:9 | 正方形 1080×1080 も可。モバイル5MB/Web15MB |
| **Instagram** | 1080 × 1350 px | 4:5 | グリッドは3:4でクロップ |
| **YouTube（コミュニティ）** | 1280 × 720 px | 16:9 | サムネイルと同じ |
| **TikTok（カルーセル）** | 1080 × 1350 px | 4:5 | 9:16(1080×1920)も可。4〜35枚 |

## 2. 縦型動画（リール/Shorts/TikTok）

| プラットフォーム | サイズ | 比率 | 最適長 |
|---|---|---|---|
| **Instagram リール** | 1080 × 1920 px | 9:16 | 教育系：30〜90秒 |
| **YouTube Shorts** | 1080 × 1920 px | 9:16 | 15〜30秒 |
| **TikTok** | 1080 × 1920 px | 9:16 | 教育系：30〜60秒 |

## 3. 縦型動画のセーフゾーン

```
+--------------------+
|  ! 上部 ~250px     | <- アカウント名・フォローボタンが被る
|                    |
|  OK セーフゾーン    | <- 1080 x 1420 px（中央60〜70%）
|  MAP・ランキング   |    重要な情報はすべてここに配置
|  テキスト・数字    |
|                    |
|  ! 下部 ~400px     | <- キャプション・音楽情報・エンゲージボタンが被る
+--------------------+
```

## 4. プラットフォーム別テキスト生成仕様

各 `/post-*` スキルが参照するテキスト生成ルール。

### X (Twitter)

- **文字数**: 〜200字（URL含まず）
- **書き方**: データジャーナリスト視点を基本としつつ、フックの1行目は**主観的な断言**を許容する。断言→データ裏付けの構成がXでは最も拡散する
- **ハッシュタグ**: 0〜2個。トレンド参加時のみ（ハッシュタグが多いとスパム判定されやすい）
- **CTA**: 「引用で感想を」ではなく**立場を取らせる**誘導（後述）。引用RTがアルゴリズム上最も重い
- **リンク**: URL 直貼り（`https://stats47.jp/...`）
- **アルゴリズム重視**: リポスト → 引用RT → 返信
- **displayTitle**: 20字以内のSNS向けタイトル（統計用語を日常語に翻訳）
- **エンゲージメント戦略**: Xではリポスト・引用RTがリーチを直接決定する。「感想を」のような漠然としたCTAでは弱い。**立場を取らせる**投稿が引用RTを連鎖させる:
  - **断言→反論誘発型**（最も拡散力が高い）: 「〇〇県が最強、異論は引用RTで」。断言は賛否を生み、引用RTが連鎖する
  - **体感確認型**: 「〇〇県民、体感と合ってる？引用RTで教えて」。当事者意識が引用のハードルを下げる
  - **ニュース接続型**: トレンドニュースにデータで切り込む。「〇〇が話題だけど、データで見ると…」。時事性がリポストを加速する
  - **意見表明型**: 「このデータ、どう解釈する？」ではなく「このデータは〇〇を意味している」と主張する。反論が引用RTで来る
  - **穴埋め型**: 「〇〇県が1位の理由、一言で言うと____」。回答が自然と引用RTになる
- **トーン**: 200文字の制限内で「データ→主張→引用RT誘導」の3要素を入れる。硬い報道調ではなく、友達に「これ知ってた？」と話しかけるカジュアルさ

### Instagram

- **文字数**: 200〜500字。情報整理型（箇条書き活用）
- **書き方**: 教育者視点「1日1つ賢くなる」コンセプト
- **ハッシュタグ**: 3〜5個（#統計 #都道府県ランキング #<テーマ>）+ 上位県の地元タグ（#東京 #大阪 等）で発見タブ露出を増やす
- **CTA**: 保存誘導を最優先。「保存して見返そう」ではなく**保存する理由**を明示（後述）。DM共有誘導を2番目に配置
- **リンク**: 「プロフィールのリンクから🔗」（キャプション内URLは無効。👆は使わない）
- **アルゴリズム重視**: 保存 → DM共有 → コメント
- **hookText**: 15字以内（カルーセル表紙 / リール冒頭テロップ用）
- **displayTitle**: 20字以内
- **エンゲージメント戦略**: Instagramのアルゴリズムは保存数がリーチを直接決定する。「保存して」だけでは行動に至らない。**なぜ保存すべきか**の具体的シーンを提示する:
  - **保存促進型**（最も重要）: 「引っ越し検討中の人は保存必須📌」「子育て世代は保存して比較して」。保存する具体的シーンを提示することで保存率が上がる
  - **DM共有誘発型**: 「地元の友達に送って」「これ、〇〇県民の友達に見せたい」。DM共有はアルゴリズム上保存に次ぐ重みがある
  - **カルーセル完走型**: 1枚目で「意外な1位は最後のスライドに👉」と引っ張り、最終枚まで見させる。完走率が保存率と正の相関。最終枚にCTA（保存+プロフリンク誘導）を配置
  - **ストーリーズ連動型**: フィード投稿後にストーリーズで「どの県が意外だった？」投票を実施。ストーリーズからフィードへの遷移がエンゲージメントシグナルになる
  - **地元タグ型**: 「#東京 #大阪 #福岡」など上位県のタグをキャプションに含め、発見タブでの地域別露出を増やす
- **キャプション構成**: 冒頭2行に最もインパクトのある数字を配置（折りたたまれる前に見せる）。保存誘導は本文中に自然に埋め込む（末尾だけだと見られない）

### YouTube Shorts

- **タイトル**: 50字以内。SEOキーワードを先頭に配置（CTR最適化）
- **説明欄**: 250字以上。先頭125字が検索スニペットに表示されるためキーワード密度重要
- **ハッシュタグ**: 3〜5個（#Shorts 必須）
- **CTA**: 完視聴 > チャンネル登録 > コメント の優先順
- **リンク**: 説明欄にURL
- **アルゴリズム重視**: 視聴維持率（完視聴率） → CTR → コメント・いいね
- **hookText**: 15字以内（動画冒頭テロップ用）
- **displayTitle**: 20字以内
- **投稿頻度**: 1日2〜3本上限、3〜6時間間隔
- **エンゲージメント戦略**: YouTubeではコメント自体がアルゴリズムを直接動かす度合いはTikTokより低いが、コメントがあるとセッション時間が伸びる（コメント読み→次の動画視聴）ため間接的に効く。**完視聴率を最大化する構成が先で、その上でコメントを取りに行く二段構え**:
  - **予測→答え合わせ型**（最も相性がいい）: 冒頭「1位どの県だと思う？」→ 3位→2位→1位と引っ張る → 末尾「予想当たった？コメントで正解数教えて」。当たった/外れたの報告はコメントハードルが極めて低い
  - **比較提案型**: 末尾「〇〇と△△、どっちが意外だった？比較動画出してほしかったらコメントで教えて」。視聴者にネタを決めさせることで当事者意識が生まれ、公開時にもう一度見に来る
  - **シリーズ予告型**: 末尾「次は"実は貧しい県ランキング"を出します。見たい人はチャンネル登録して待ってて」。YouTubeの登録は「次の通知が来る」意識が強く、次回予告で登録動機を作る
  - **説明欄誘導型**: 「全47都道府県の順位は説明欄に載せてます。自分の県を確認してコメントで感想教えて」。説明欄に全データを載せているstats47の強みを活かす
- **ピン留めコメント**: YouTubeではピン留めコメントがTikTokより圧倒的に目立つ。TOP3+URLだけでなく**議論の種**を仕込む（例:「個人的に意外だったのは〇〇県。△△が理由だと思うけど、地元の方どう思いますか？」）。ピン留めへの返信はスレッドとして視認性が高く、他の視聴者も読む→滞在時間が伸びる

### TikTok

- **文字数**: 100〜300字。口語・会話型
- **書き方**: エンタメ視点。カジュアルなトーン
- **ハッシュタグ**: 5〜8個（#fyp は不要）。テーマ共通（#都道府県 #統計 #データで見る日本）+ トピック固有（#空き家 等）+ 上位県の地域タグ（#徳島 #沖縄 等）+ 発見系（#知らなかった #雑学 等）
- **CTA**: コメント誘導を最優先（後述）。フォロー誘導は末尾に添える程度
- **リンク**: 「プロフのリンクへ」
- **アルゴリズム重視**: コメント数・コメント滞在時間 → 完視聴率 → リピート → シェア
- **hookText**: 15字以内（動画冒頭テロップ用）
- **displayTitle**: 20字以内
- **コメント誘導戦略**: TikTokのレコメンドはコメント数・スレッド深度を重視する。「あなたの県は何位？」だけでは視聴完了には効くがコメント行動に至らない。以下のパターンでコメントを誘発する:
  - **理由質問型**: 「なぜ〇〇県が1位なのか、県民の方コメントで教えて」（当事者の体験談を引き出す）
  - **意外性×反論型**: 「東京が最下位、納得できる？」（賛否が割れてスレッドが伸びる）
  - **二択型**: 「住むなら1位の県？最下位の県？」（回答ハードルが低い）
  - **考察募集型**: 「この相関の理由、コメントで考察して」（知的好奇心層の参加）
  - **地元反応型**: 「1位の〇〇県民、自覚ある？」（地元愛でコメントが湧く）
- **コメント誘導の配置**: 動画の冒頭3秒以内にコメント誘導を含める（離脱前に「コメントで教えて」という目的を視聴者に与える）。キャプションにも同じ問いかけを入れる
- **ピン留めコメント**: 投稿後、自分から考察を1つピン留めコメントで書く。それに対する返信が連鎖しやすい
- **注意**: 煽りすぎない。「ヤバい県」のような攻撃的表現は避け、「なぜこうなるのか」という知的な問いかけのトーンを維持する（stats47のブランド＝データで客観的に見る）

## 5. ファイル形式・サイズ制限

| プラットフォーム | 画像 | 動画 |
|---|---|---|
| X | JPG/PNG/GIF/WebP 5MB(モバイル)/15MB(Web) | MP4/MOV 512MB以下 |
| Instagram | JPG/PNG 30MB以下 | MP4/MOV 4GB以下 |
| YouTube | JPG/PNG 2MB以下（サムネ） | MP4/MOV 256GB以下 |
| TikTok | JPG/PNG/WebP 〜100KB推奨 | MP4/MOV/WebM 500MB〜2GB |

## 7. YouTube Shorts 投稿頻度

- 1日2〜3本が上限（それ以上は逆効果）
- 最低3〜6時間の間隔を空ける
- アルゴリズムは視聴維持率 > 初動反応（投稿後1〜3時間）> 投稿頻度
- 短時間に連投すると通知が制限される場合がある
