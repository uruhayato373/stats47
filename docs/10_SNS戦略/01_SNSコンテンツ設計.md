## 0. 設計方針

### 0.1 原則

1. **全PF同時投稿、分析軸は X** — `/post-sns-captions` で4PF一括生成。追加の人的コストはほぼゼロ。ただし KPI 追跡・改善サイクルは X を主軸にする（引用RT・インプレッションが最も計測しやすい）
2. **テーマは人間の感情から逆算する** — 「数字の格差が大きい=面白い」ではない。金・ステータス・地元愛/自虐・子育ての不安など、人が反応する欲望・コンプレックスから逆算してデータを当てはめる
3. **AI をフル活用し、人間は判断と改善に集中する** — キャプション執筆・テーマ選定・トレンド分析を AI に任せ、人間は「何を投稿しないか」の判断とエンゲージメント分析に注力する

### 0.2 フェーズ設計

| フェーズ | 期間 | やること | やらないこと |
|---|---|---|---|
| **Phase 1: 検証** | 4週間 | 全PF同時投稿（X / Instagram / YouTube Shorts / TikTok）。週3投稿。反応を計測 | 投稿頻度の増加 |
| **Phase 2: 型の確立** | 4週間 | Phase 1 で伸びた型を量産。週5投稿に増。PF別の最適化 | — |

### 0.3 判定基準（Kill Criteria）

4週間・12投稿後に以下で判定する。数字は仮——重要なのは判定ロジックの存在。主に X のインプレッションで判定するが、他PFで突出した反応があればそれも考慮する。

| 判定 | 条件 | アクション |
|---|---|---|
| **成功** | 1投稿でもインプレッション 5,000 超（X基準） | その型を Phase 2 で量産 |
| **継続** | 平均インプレッション 1,000 以上 | テーマ・切り口を変えて追加4週間 |
| **撤退** | 全投稿インプレッション 500 未満 | SNS戦略自体を見直し、SEO/ブログに集中 |

### 0.4 投稿頻度ガイドライン

YouTube Shorts は1日2〜3本が上限。それ以上は逆効果（自分の動画同士でインプレッション食い合い、通知制限）。最低3〜6時間の間隔を空ける。アルゴリズムは投稿頻度よりも視聴維持率・初動反応（投稿後1〜3時間）を重視する。

## 1. AI 活用設計

### 1.1 投稿1本のワークフロー（30分以内）

```
1. /post-sns-captions スキルを起動、テーマ候補3件を AI に提案させる（5分）
   - domain パラメータで ranking / compare / correlation を指定
   - template パラメータで衝撃事実 / 対決 / 問いかけ / 逆説警告を指定
2. 候補から1件選択。AI が全PFのキャプションを一括生成（5分）
3. /render-sns-stills で画像・動画を生成（10分）
4. 最終チェック：炎上リスク・トーン・事実正確性の確認（5分）
5. 各PFに手動投稿（5分）
6. 投稿後3時間以内にリプライ対応（随時。AI ドラフト → 人間が送信）
```

### 1.2 AI が担う領域 vs 人間がやるべきこと

| 工程 | AI | 人間 |
|---|---|---|
| テーマ選定 | 感情トリガーに基づくテーマ候補提案（スコア付き） | 候補から選択。ボツの理由をフィードバック |
| キャプション執筆 | PF別にフック・本文・CTAを3パターン生成 | 微調整。トーンの最終確認 |
| 画像テキスト | サムネイル上のテキスト・煽りコピーを生成 | 品位チェック。炎上リスク判断 |
| エンゲージメント分析 | インプレッション・RT・引用の傾向を要約 | 勝ちパターンの言語化 |
| コメント返信 | 返信ドラフトを生成 | 最終確認して送信。炎上時は人間が判断 |
| **投稿しない判断** | — | **量より質。死にコンテンツを出さない勇気** |

### 1.3 動画制作方針（YouTube Shorts / TikTok / Reels）

- **AIナレーション × フェイスレス**: 教育系フェイスレスチャンネルの主流はAI音声によるデータ解説。顔出し不要で量産できる。Remotion レンダリング + AI ナレーションの組み合わせが stats47 の制作ラインに合致する
- **Bar Chart Race は差別化必須**: 単純な推移アニメーションだけでは既存動画に埋もれる。AIナレーション・視聴者投票（「次に落ちる県を予想して！」）・Split Screen 比較など、エンゲージメント施策と組み合わせること
- **「本物感」の補完**: Remotion の高品質レンダリングは強みだが、ショート動画では「作り込みすぎた感」が逆効果になりうる。ナレーションやテキストで人間味を補完する

### 1.4 スキル体系

| スキル | 用途 |
|---|---|
| `/post-sns-captions` | 全PF一括生成（個別スキルへの委譲）。通常はこれを使う |
| `/post-x` | X 向けテキスト+画像を生成。domain / template パラメータで全データドメイン対応 |
| `/post-instagram` | Instagram キャプション+カルーセル/リール生成 |
| `/post-youtube` | YouTube Shorts タイトル+説明生成 |
| `/post-tiktok` | TikTok キャプション生成 |
| `/post-note` | note.com 有料記事を作成（無料部+有料部） |
| `/render-sns-stills` | Remotion で画像/動画アセットを生成 |

## 2. テーマ選定戦略

### 2.1 感情トリガー分類

データの格差や統計的な面白さではなく、**人間の感情を動かすか**で選定する。

| 感情トリガー | テーマ例 | なぜ反応されるか |
|---|---|---|
| **財布・生活不安** | 「年収が高いのに貯金ゼロの県」 | 自分の生活に直結。保存・共有動機が強い |
| **地元愛・自虐** | 「実は全国1位の〇〇県」 | 地元民が反応。引用RTでマウント合戦 |
| **子育て・教育不安** | 「教育費と学力が比例しない県」 | 親世代の切実な関心。DM共有されやすい |
| **格差への怒り** | 「東京vs地方の医療格差」 | 社会的な議論を呼ぶ。引用RTが増える |
| **意外性・裏切り** | 「米消費量1位は新潟じゃない」 | 常識を裏切る。「マジで？」でシェア |

### 2.2 データドメイン × テンプレ型マトリクス

投稿コンテンツは **データドメイン**（素材）と **テンプレ型**（表現）の組み合わせで構成する。

| | 衝撃事実型 | 対決型 | 問いかけ型 | 逆説・警告型 |
|---|---|---|---|---|
| **ranking** | 「〇〇1位は意外な△△県」 | 「1位 vs 47位」 | 「あなたの県は何位？」 | 「〇〇なのに△△が低い県」 |
| **compare** | 「東京と大阪、意外な差」 | 「東京 vs 大阪 5番勝負」 | 「あなたはどっち派？」 | 「勝っているはずが負けている」 |
| **correlation** | 「所得と自殺率の衝撃的関係」 | 「相関する2指標の逆転県」 | 「なぜ〇〇と△△が連動する？」 | 「〇〇が高いほど△△が悪化する」 |

テンプレ型の詳細定義（構造・フック例・適合ドメイン）は各 `/post-*` スキル内の「テンプレート定義」セクションを参照。

### 2.3 Tier 設計

| Tier | 選定基準 | 投稿頻度 | データドメイン | テンプレ型 |
|---|---|---|---|---|
| **Tier 1** | 財布・人生に直結 + 意外性あり | 週2回 | correlation, ranking | 衝撃事実型, 逆説・警告型 |
| **Tier 2** | 地元愛・対決 + 議論を呼ぶ | 週1回 | compare, ranking | 対決型 |
| **Tier 3** | 雑学・豆知識として面白い | 週1回 | ranking | 問いかけ型 |

## 3. X（Twitter）投稿設計

### 3.1 投稿フォーマット

| 要素 | 仕様 |
|---|---|
| テキスト長 | 〜200字。事実凝縮型「ニュース速報」的 |
| 画像 | 1〜4枚。1200 × 675 px（16:9） |
| ハッシュタグ | 0〜2個。トレンド参加時のみ |
| CTA | 「引用で感想を」「RTで拡散」 |
| リンク | URL 直貼り（サイト誘導） |
| アルゴリズム重視指標 | リポスト → 引用RT → 返信 |

### 3.2 画像設計

| 要素 | 仕様 |
|---|---|
| サイズ | 1200 × 675 px（16:9） |
| テキスト | 4語以内。太字サンセリフ（Noto Sans JP Black） |
| 配色 | ダーク背景 + ヒートMAP配色（青→赤）+ 白テキスト |
| MAP | 左にヒートMAP日本地図、右に偏差値・数字を大きく |
| ファイル | JPG/PNG、5MB以下（モバイル） |

### 3.3 投稿カレンダー

週3回、曜日を固定:

| 曜日 | データドメイン | テンプレ型 | Tier |
|---|---|---|---|
| 火 | correlation / ranking | 衝撃事実型 / 逆説・警告型 | Tier 1 |
| 木 | compare | 対決型 | Tier 2 |
| 土 | ranking | 問いかけ型 | Tier 3 |
