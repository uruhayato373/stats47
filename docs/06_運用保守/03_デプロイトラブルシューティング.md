# デプロイトラブルシューティングガイド

## 概要

このドキュメントは、stats47プロジェクトのデプロイ時に発生する可能性のある問題とその解決方法をまとめています。

## よくある問題と解決方法

### 1. Windows環境でのビルドエラー（解決済み）

#### 旧問題（`@cloudflare/next-on-pages`使用時）

以前は`@cloudflare/next-on-pages`がbashスクリプトを必要としていたため、Windows環境で`spawn bash ENOENT`エラーが発生していました。

#### 現在の解決方法

現在、プロジェクトは**OpenNext Cloudflare Adapter**（`@opennextjs/cloudflare`）を使用しています。OpenNextはbashスクリプトへの依存がないため、Windows環境でも正常に動作します。

**ビルド方法**:

```bash
# Windows環境でも直接ビルド可能
npm run pages:build

# デプロイ
npx wrangler pages deploy .open-next --env staging
```

**注意**: GitHub Actionsを使用した自動デプロイも引き続き推奨されます。

### 2. ビルドエラー: Edge Runtime設定の不足

#### エラーメッセージ

```
ERROR: Failed to produce a Cloudflare Pages build from the project.
The following routes were not configured to run with the Edge Runtime:
  - /[category]/[subcategory]/dashboard/[areaCode]
  - /admin/blog
  ...
Please make sure that all your non-static routes export the following edge runtime route segment config:
  export const runtime = 'edge';
```

#### 解決方法

`@cloudflare/next-on-pages`を使用している場合、このエラーは発生しません。もし発生する場合は：

1. `@cloudflare/next-on-pages`が正しくインストールされているか確認
2. `package.json`の`pages:build`スクリプトが正しいか確認

### 3. デプロイエラー: 認証情報の不足

#### エラーメッセージ

```
Error: Missing required environment variable: CLOUDFLARE_API_TOKEN
```

#### 解決方法

1. **GitHub Secretsの設定**（CI環境の場合）:
   - GitHubリポジトリの「Settings」→「Secrets and variables」→「Actions」
   - 以下のSecretsを追加:
     - `CLOUDFLARE_API_TOKEN`
     - `CLOUDFLARE_ACCOUNT_ID`
     - `ESTAT_API_KEY`

2. **ローカル環境変数の設定**（ローカルデプロイの場合）:
   ```bash
   # PowerShell
   $env:CLOUDFLARE_API_TOKEN="your_token_here"
   $env:CLOUDFLARE_ACCOUNT_ID="your_account_id_here"
   
   # Git Bash / WSL
   export CLOUDFLARE_API_TOKEN=your_token_here
   export CLOUDFLARE_ACCOUNT_ID=your_account_id_here
   ```

### 4. デプロイエラー: データベースマイグレーションの失敗

#### エラーメッセージ

```
Error: Failed to apply migrations
```

#### 解決方法

1. **マイグレーションファイルの確認**:
   ```bash
   ls database/migrations/
   ```

2. **ローカルでマイグレーションをテスト**:
   ```bash
   npm run db:migrations:local
   ```

3. **ステージング環境でマイグレーションを適用**:
   ```bash
   npm run db:migrations:staging
   ```

4. **本番環境でマイグレーションを適用**（慎重に実行）:
   ```bash
   npm run db:migrations:production
   ```

### 5. デプロイ後のランタイムエラー

#### 症状

デプロイは成功したが、アプリケーションが正常に動作しない

#### 解決方法

1. **エラーログの確認**:
   ```bash
   npx wrangler pages deployment tail --env staging
   ```

2. **環境変数の確認**:
   - Cloudflare Pagesのダッシュボードで環境変数が正しく設定されているか確認
   - Secretsが正しく設定されているか確認

3. **データベース接続の確認**:
   ```bash
   npx wrangler d1 execute stats47_staging --env staging --remote --command "SELECT name FROM sqlite_master WHERE type='table';"
   ```

## デプロイ手順（推奨）

### ステージング環境へのデプロイ

1. **変更をコミット・プッシュ**:
   ```bash
   git add .
   git commit -m "Your commit message"
   git push origin develop
   ```

2. **GitHub Actionsで自動デプロイ**:
   - `develop`ブランチへのプッシュで自動的にデプロイが開始されます
   - GitHub Actionsのログで進行状況を確認

3. **手動デプロイ（必要な場合）**:
   - GitHubリポジトリの「Actions」タブ
   - 「Deploy to Staging」ワークフローを選択
   - 「Run workflow」ボタンをクリック

### 本番環境へのデプロイ

1. **ステージング環境で動作確認**

2. **mainブランチにマージ**:
   ```bash
   git checkout main
   git pull origin main
   git merge develop
   git push origin main
   ```

3. **GitHub Actionsで自動デプロイ**:
   - `main`ブランチへのプッシュで自動的に本番環境にデプロイされます

## チェックリスト

### デプロイ前

- [ ] ローカルで`npm run dev`が正常に動作する
- [ ] テストがすべて合格する（`npm test`）
- [ ] 型チェックが合格する（`npx tsc --noEmit`）
- [ ] 変更内容をコミット・プッシュしている
- [ ] GitHub Secretsが正しく設定されている

### デプロイ後

- [ ] デプロイが成功している（GitHub Actionsのログを確認）
- [ ] ステージング/本番環境のURLにアクセスできる
- [ ] 主要な機能が正常に動作する
- [ ] エラーログにエラーがない
- [ ] データベース接続が正常に動作する

## 緊急時の対応

### ロールバック

1. **前のデプロイメントを確認**:
   ```bash
   npx wrangler pages deployment list --env production
   ```

2. **特定のデプロイメントにロールバック**:
   ```bash
   npx wrangler pages deployment rollback <deployment-id> --env production
   ```

問題が解決しない場合は、以下を確認してください：
