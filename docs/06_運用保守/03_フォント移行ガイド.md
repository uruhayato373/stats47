# フォント移行ガイド

## 1. 移行の背景

### 1.1 移行の目的

stats47プロジェクトでは、現在使用しているGeistフォントから、Inter + Noto Sans JPのハイブリッド構成に移行します。

**移行理由:**
- **視認性の向上**: 統計数字とグラフの可読性向上
- **ブランディング**: テック企業らしいモダンな印象
- **パフォーマンス**: 変数フォントによる最適化
- **保守性**: 広く使われているフォントの採用

### 1.2 移行の影響範囲

**影響するファイル:**
- `src/app/layout.tsx`
- `src/app/globals.css`
- `tailwind.config.ts`（オプション）

**影響する機能:**
- 全ページのテキスト表示
- 統計データの表示
- グラフとチャートのラベル
- ナビゲーションとボタン

## 3. 移行手順

### Step 1: パッケージの確認

**現在のパッケージを確認:**
```bash
# geistパッケージの確認
npm list geist

# 必要に応じて削除（段階的）
npm uninstall geist
```

**注意:** geistパッケージは段階的に削除可能です。まずは新しいフォントを追加してから削除することを推奨します。

### Step 2: layout.tsxの更新

**変更前:**
```tsx
// src/app/layout.tsx
import { GeistSans } from "geist/font/sans";
import { GeistMono } from "geist/font/mono";

export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html lang="ja" className={`${GeistSans.variable} ${GeistMono.variable}`}>
      <body className={GeistSans.className}>{children}</body>
    </html>
  );
}
```

**変更後:**
```tsx
// src/app/layout.tsx
import { Inter } from "next/font/google";
import { Noto_Sans_JP } from "next/font/google";
import { Geist_Mono } from "next/font/google";

const inter = Inter({
  subsets: ["latin"],
  variable: "--font-inter",
  display: "swap",
  preload: true,
});

const notoSansJP = Noto_Sans_JP({
  weight: ["400", "500", "600", "700"],
  subsets: ["latin"],
  variable: "--font-noto-sans-jp",
  display: "swap",
  preload: true,
});

const geistMono = Geist_Mono({
  subsets: ["latin"],
  variable: "--font-geist-mono",
  display: "swap",
});

export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html 
      lang="ja" 
      className={`${inter.variable} ${notoSansJP.variable} ${geistMono.variable}`}
    >
      <body className={inter.className}>{children}</body>
    </html>
  );
}
```

### Step 3: globals.cssの更新

**変更前:**
```css
/* src/app/globals.css */
@theme inline {
  --font-sans: var(--font-geist-sans);
  --font-mono: var(--font-geist-mono);
}
```

**変更後:**
```css
/* src/app/globals.css */
@theme inline {
  --font-sans: var(--font-inter), var(--font-noto-sans-jp);
  --font-mono: var(--font-geist-mono);
}

/* フォントフォールバック */
body {
  font-family: var(--font-inter), var(--font-noto-sans-jp), 
    -apple-system, BlinkMacSystemFont, "Segoe UI",
    "Hiragino Sans", "Yu Gothic", "Meiryo", sans-serif;
}
```

### Step 4: Tailwind CSSの設定更新（オプション）

**tailwind.config.tsの更新:**
```ts
// tailwind.config.ts
module.exports = {
  theme: {
    extend: {
      fontFamily: {
        sans: ['var(--font-inter)', 'var(--font-noto-sans-jp)', 'sans-serif'],
        mono: ['var(--font-geist-mono)', 'monospace'],
      },
    },
  },
}
```

## 5. トラブルシューティング

### 5.1 よくある問題

**問題1: フォントが表示されない**

**症状:** フォールバックフォントが表示される

**原因:** フォントの読み込みエラー

**解決策:**
```tsx
// 1. インポートを確認
import { Inter } from "next/font/google";
import { Noto_Sans_JP } from "next/font/google";

// 2. 変数名を確認
const inter = Inter({
  variable: "--font-inter", // 正しい変数名
});

// 3. HTMLクラスを確認
<html className={`${inter.variable} ${notoSansJP.variable}`}>
```

**問題2: 日本語が正しく表示されない**

**症状:** 日本語がシステムフォントで表示される

**原因:** 日本語フォントの設定不備

**解決策:**
```tsx
// 1. 日本語フォントの設定を確認
const notoSansJP = Noto_Sans_JP({
  weight: ["400", "500", "600", "700"],
  subsets: ["latin"],
  variable: "--font-noto-sans-jp",
});

// 2. CSS変数の順序を確認
font-family: var(--font-inter), var(--font-noto-sans-jp), sans-serif;
```

**問題3: パフォーマンスが悪化した**

**症状:** ページ読み込み時間が増加

**原因:** 不要なウェイトの読み込み

**解決策:**
```tsx
// 必要なウェイトのみを指定
const notoSansJP = Noto_Sans_JP({
  weight: ["400", "700"], // 最小限のウェイト
  subsets: ["latin"],
  display: "swap",
});
```

**問題4: ビルドエラーが発生**

**症状:** ビルド時にエラーが発生

**原因:** インポートパスの間違い

**解決策:**
```tsx
// 正しいインポートパスを使用
import { Inter } from "next/font/google";
import { Noto_Sans_JP } from "next/font/google";
import { Geist_Mono } from "next/font/google";
```

### 5.2 デバッグ方法

**フォントの確認:**
```tsx
// デバッグ用スタイル
<div style={{ fontFamily: "var(--font-inter)" }}>
  Inter フォント
</div>

<div style={{ fontFamily: "var(--font-noto-sans-jp)" }}>
  Noto Sans JP フォント
</div>
```

**ブラウザ開発者ツールでの確認:**
1. Elements タブでフォントファミリーを確認
2. Computed タブで実際のフォントサイズを確認
3. Network タブでフォントファイルの読み込み状況を確認

**コンソールでの確認:**
```javascript
// フォントの読み込み状況を確認
document.fonts.ready.then(() => {
  console.log('Fonts loaded');
  console.log(document.fonts.check('16px Inter'));
  console.log(document.fonts.check('16px Noto Sans JP'));
});
```

## 6. ロールバック手順

### 6.1 緊急ロールバック

**問題が発生した場合の迅速なロールバック:**

1. **layout.tsx を元に戻す:**
```tsx
// 元のGeist設定に戻す
import { GeistSans } from "geist/font/sans";
import { GeistMono } from "geist/font/mono";

export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html lang="ja" className={`${GeistSans.variable} ${GeistMono.variable}`}>
      <body className={GeistSans.className}>{children}</body>
    </html>
  );
}
```

2. **globals.css を元に戻す:**
```css
@theme inline {
  --font-sans: var(--font-geist-sans);
  --font-mono: var(--font-geist-mono);
}
```

3. **キャッシュをクリア:**
```bash
# Next.jsのキャッシュをクリア
rm -rf .next

# ブラウザのキャッシュをクリア
# Chrome: Ctrl+Shift+R
# Firefox: Ctrl+F5
```

4. **再ビルド・再デプロイ:**
```bash
npm run build
npm run deploy:production
```

### 6.2 ロールバックトリガー

**以下の条件でロールバックを実行:**

- Lighthouseスコアが20%以上低下
- ユーザーからのフォント表示に関するバグ報告が3件以上
- 読み込み時間が2倍以上に増加
- 重要なページでフォントが表示されない

### 6.3 ロールバック後の対応

1. **問題の原因を調査**
2. **修正版の準備**
3. **再テストの実施**
4. **段階的な再デプロイ**
