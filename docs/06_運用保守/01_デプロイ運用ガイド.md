# デプロイ・運用ガイド

## 概要

このドキュメントは、stats47 プロジェクトの本番環境およびステージング環境へのデプロイと日常的な運用手順を提供します。

## 1. デプロイ手順

### 1.1 ステージング環境へのデプロイ

#### 事前確認

```bash
# 1. ブランチの確認
git branch

# 2. ビルドの動作確認
npm run build:staging

# 3. テストの実行
npm test

# 4. 型チェック
npx tsc --noEmit
```

#### デプロイ実行

```bash
# Cloudflare Pagesへデプロイ
npx wrangler pages deploy --env staging

# デプロイ完了後、URLを確認
# 出力例: https://staging.stats47.pages.dev
```

#### デプロイ後の確認

```bash
# 1. ヘルスチェック
curl https://staging.stats47.pages.dev/api/health

# 2. データベース接続確認
npx wrangler d1 execute stats47_staging --env staging --remote --command "SELECT COUNT(*) FROM users;"

# 3. ログ確認
npx wrangler pages deployment list --env staging
```

### 1.2 本番環境へのデプロイ

#### 事前確認（必須）

```bash
# 1. ステージング環境での動作確認完了
# 2. PRレビュー完了
# 3. 全テスト合格
npm test

# 4. ビルド確認
npm run build:production

# 5. セキュリティチェック
npm audit

# 6. パフォーマンステスト
npm run lighthouse
```

#### デプロイ実行

```bash
# 1. mainブランチへのマージ
git checkout main
git pull origin main

# 2. Cloudflare Pagesへデプロイ
npx wrangler pages deploy --env production

# 3. デプロイ完了の確認
# 出力例: https://stats47.pages.dev
```

#### デプロイ後の確認（重要）

```bash
# 1. 本番環境の動作確認
curl https://stats47.pages.dev/api/health

# 2. Core Web Vitalsの確認
npm run lighthouse -- --url=https://stats47.pages.dev

# 3. エラーログの確認
npx wrangler pages deployment tail --env production

# 4. データベース状態の確認
npx wrangler d1 execute stats47 --remote --command "SELECT name FROM sqlite_master WHERE type='table';"
```

## 2. データベース運用

### 2.1 マイグレーションの実行

#### ステージング環境

```bash
# 1. マイグレーションファイルの作成
npx wrangler d1 migrations create stats47_staging "add_new_column"

# 2. マイグレーションファイルの編集
# database/migrations/0001_add_new_column.sql

# 3. ローカルでテスト
npx wrangler d1 migrations apply stats47 --local

# 4. ステージングへ適用 ⚠️ --remote必須
npx wrangler d1 migrations apply stats47_staging --env staging --remote

# 5. 適用結果の確認
npx wrangler d1 execute stats47_staging --env staging --remote --command "SELECT name FROM sqlite_master WHERE type='table';"
```

#### 本番環境（慎重に実行）

```bash
# ⚠️ 本番環境では必ずバックアップを取得してから実行

# 1. バックアップの取得
./scripts/backup-production-db.sh

# 2. マイグレーション適用 ⚠️ --remote必須
npx wrangler d1 migrations apply stats47 --remote

# 3. 適用結果の確認
npx wrangler d1 execute stats47 --remote --command "SELECT COUNT(*) FROM users;"

# 4. アプリケーションの動作確認
curl https://stats47.pages.dev/api/users
```

### 2.2 データベースバックアップ

#### 定期バックアップ（推奨：日次）

```bash
# バックアップスクリプトの実行
./scripts/backup-production-db.sh

# または手動でエクスポート
npx wrangler d1 execute stats47 --remote --command "SELECT * FROM ranking_items;" --json > database/backups/$(date +%Y%m%d)/ranking_items.json
```

#### バックアップの確認

```bash
# バックアップファイルの一覧
ls -lh database/backups/

# バックアップの内容確認
cat database/backups/20250120/ranking_items.json | jq '.'
```

### 2.3 データ同期（本番 → ステージング）

```bash
# 1. 本番データのエクスポート
npx wrangler d1 execute stats47 --remote --command "SELECT * FROM ranking_items;" --json > database/backups/$(date +%Y%m%d)/ranking_items.json

# 2. SQLへの変換
node scripts/import-to-staging.js

# 3. ステージングへインポート ⚠️ --remote必須
npx wrangler d1 execute stats47_staging --env staging --remote --file=database/backups/$(date +%Y%m%d)/import_to_staging.sql

# 4. データ確認
npx wrangler d1 execute stats47_staging --env staging --remote --command "SELECT COUNT(*) FROM ranking_items;"
```

## 3. Secrets 管理

### 3.1 Secrets の設定

#### ステージング環境

```bash
# e-Stat APIキーの設定
wrangler secret put ESTAT_API_KEY --env staging
# プロンプトが表示されるので、APIキーを入力

# Cloudflare APIトークンの設定
wrangler secret put CLOUDFLARE_API_TOKEN --env staging
```

#### 本番環境

```bash
# e-Stat APIキーの設定
wrangler secret put ESTAT_API_KEY --env production

# Cloudflare APIトークンの設定
wrangler secret put CLOUDFLARE_API_TOKEN --env production
```

### 3.2 Secrets の確認

```bash
# Secretsの一覧表示
wrangler secret list --env staging
wrangler secret list --env production
```

### 3.3 Secrets の削除

```bash
# 古いSecretの削除
wrangler secret delete OLD_API_KEY --env staging
```

## 5. トラブルシューティング

### 5.1 デプロイ失敗

#### 症状

```bash
Error: Deployment failed
```

#### 対処法

```bash
# 1. ビルドログの確認
npx wrangler pages deployment list

# 2. ローカルでビルド確認
npm run build:production

# 3. 環境変数の確認
# Cloudflare Dashboardで環境変数を確認

# 4. 再デプロイ
npx wrangler pages deploy --env production
```

### 5.2 データベース接続エラー

#### 症状

```bash
Error: D1_ERROR: Failed to connect to database
```

#### 対処法

```bash
# 1. データベースの状態確認
npx wrangler d1 list --remote

# 2. データベースIDの確認
cat wrangler.toml | grep database_id

# 3. 環境変数の確認
wrangler secret list --env production

# 4. マイグレーション状態の確認
npx wrangler d1 migrations list stats47 --remote
```

### 5.3 パフォーマンス低下

#### 症状

- ページロードが遅い
- API レスポンスが遅い

#### 対処法

```bash
# 1. キャッシュのクリア
# Cloudflare Dashboardでキャッシュをパージ

# 2. データベースの最適化
npx wrangler d1 execute stats47 --remote --command "VACUUM;"

# 3. インデックスの確認
npx wrangler d1 execute stats47 --remote --command "SELECT * FROM sqlite_master WHERE type='index';"

# 4. クエリのパフォーマンス確認
npx wrangler d1 execute stats47 --remote --command "EXPLAIN QUERY PLAN SELECT * FROM users WHERE id = 1;"
```

## 6. ロールバック手順

### 6.1 アプリケーションのロールバック

```bash
# 1. 前のデプロイメントIDを確認
npx wrangler pages deployment list --env production

# 2. 特定のデプロイメントにロールバック
npx wrangler pages deployment rollback <deployment-id> --env production
```

### 6.2 データベースのロールバック

```bash
# ⚠️ データベースのロールバックは慎重に実行

# 1. バックアップからリストア
# database/backups/から適切なバックアップを選択

# 2. データのインポート
npx wrangler d1 execute stats47 --remote --file=database/backups/20250120/restore.sql

# 3. データの確認
npx wrangler d1 execute stats47 --remote --command "SELECT COUNT(*) FROM users;"
```
