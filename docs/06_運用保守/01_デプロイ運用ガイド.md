# デプロイ・運用ガイド

## 概要

このドキュメントは、stats47 プロジェクトの本番環境およびステージング環境へのデプロイと日常的な運用手順を提供します。

**重要**: このプロジェクトでは、**GitHub Actions**を使用してデプロイを実行します。Cloudflare Pagesの自動デプロイ（GitHub連携）は無効化してください。重複デプロイを避けるためです。

## 1. デプロイ手順

### 1.1 ステージング環境へのデプロイ

#### 自動デプロイ（GitHub Actions）

`develop`ブランチへのプッシュ時に、GitHub Actions が自動的にステージング環境へのデプロイとデータベースマイグレーションを実行します。`main`ブランチへのプッシュ時は本番環境へデプロイされます。

**ワークフローファイル**: `.github/workflows/deploy.yml`

**実行される処理**:

1. コードチェックアウト
2. Node.js 環境セットアップ
3. 依存関係のインストール
4. テスト実行（失敗しても続行）
5. 型チェック
6. ビルド（OpenNextを使用）
7. データベースマイグレーション適用（`--remote`フラグ付き）
8. マイグレーション結果の確認
9. Cloudflare Pages へデプロイ
10. ヘルスチェック

**手動実行方法**:

- GitHub リポジトリの「Actions」タブから「🚀 Deploy Next.js + Cloudflare D1/R2 Dashboard」ワークフローを選択
- 「Run workflow」ボタンをクリックして手動実行可能（ブランチを選択可能）

**注意**: Cloudflare Pagesの自動デプロイ（GitHub連携）は無効化してください。GitHub Actionsでデプロイを実行するため、重複デプロイを避けるためです。

**必要な GitHub Secrets**:

- `CLOUDFLARE_API_TOKEN`: Cloudflare API トークン
- `CLOUDFLARE_ACCOUNT_ID`: Cloudflare Account ID

GitHub Secrets の設定方法については、[3.4 GitHub Secrets の設定](#34-github-secretsの設定)を参照してください。

#### 手動デプロイ

自動デプロイを使用しない場合、以下の手順で手動デプロイを実行できます。

```bash
npm run pages:build
```

```bash
# デプロイ完了後、URLを確認
```

## 2. データベース運用

### 2.1 マイグレーションの実行

#### ステージング環境

```bash
# 1. マイグレーションファイルの作成
npx wrangler d1 migrations create stats47_staging "add_new_column"

# 2. マイグレーションファイルの編集
# database/migrations/0001_add_new_column.sql

# 3. ローカルでテスト
npx wrangler d1 migrations apply stats47 --local

# 4. ステージングへ適用 ⚠️ --remote必須
npx wrangler d1 migrations apply stats47_staging --env staging --remote

# 5. 適用結果の確認
npx wrangler d1 execute stats47_staging --env staging --remote --command "SELECT name FROM sqlite_master WHERE type='table';"
```

#### 本番環境（慎重に実行）

```bash
# ⚠️ 本番環境では必ずバックアップを取得してから実行

# 1. バックアップの取得
./scripts/backup-production-db.sh

# 2. マイグレーション適用 ⚠️ --remote必須
npx wrangler d1 migrations apply stats47 --remote

# 3. 適用結果の確認
npx wrangler d1 execute stats47 --remote --command "SELECT COUNT(*) FROM users;"

# 4. アプリケーションの動作確認
curl https://stats47.pages.dev/api/users
```

### 2.2 データベースバックアップ

#### 定期バックアップ（推奨：日次）

```bash
# バックアップスクリプトの実行
./scripts/backup-production-db.sh

# または手動でエクスポート
npx wrangler d1 execute stats47 --remote --command "SELECT * FROM ranking_items;" --json > database/backups/$(date +%Y%m%d)/ranking_items.json
```

#### バックアップの確認

```bash
# バックアップファイルの一覧
ls -lh database/backups/

# バックアップの内容確認
cat database/backups/20250120/ranking_items.json | jq '.'
```

### 2.3 データ同期（本番 → ステージング）

```bash
# 1. 本番データのエクスポート
npx wrangler d1 execute stats47 --remote --command "SELECT * FROM ranking_items;" --json > database/backups/$(date +%Y%m%d)/ranking_items.json

# 2. SQLへの変換
node scripts/import-to-staging.js

# 3. ステージングへインポート ⚠️ --remote必須
npx wrangler d1 execute stats47_staging --env staging --remote --file=database/backups/$(date +%Y%m%d)/import_to_staging.sql

# 4. データ確認
npx wrangler d1 execute stats47_staging --env staging --remote --command "SELECT COUNT(*) FROM ranking_items;"
```

```bash

wrangler secret put ESTAT_API_KEY --env production

# Cloudflare APIトークンの設定

```

### 3.2 Secrets の確認

```bash
wrangler secret list --env production
```

### 3.3 Secrets の削除

```bash
# 古いSecretの削除
wrangler secret delete OLD_API_KEY --env staging
```

### 3.4 GitHub Secrets の設定

GitHub Actions で自動デプロイを使用する場合、以下の Secrets を GitHub リポジトリに設定する必要があります。

#### 設定手順

1. GitHub リポジトリの「Settings」を開く
2. 左メニューから「Secrets and variables」→「Actions」を選択
3. 「New repository secret」をクリック
4. 以下の Secrets を追加：

**CLOUDFLARE_API_TOKEN**:

- Name: `CLOUDFLARE_API_TOKEN`
- Secret: Cloudflare API トークン（Cloudflare Dashboard で生成）

**CLOUDFLARE_ACCOUNT_ID**:

- Name: `CLOUDFLARE_ACCOUNT_ID`
- Secret: Cloudflare Account ID（Cloudflare Dashboard の右上に表示）

#### Cloudflare API トークンの生成方法

1. Cloudflare Dashboard にログイン
2. 「My Profile」→「API Tokens」を選択
3. 「Create Token」をクリック
4. 「Edit Cloudflare Workers」テンプレートを選択
5. 必要な権限を設定：
   - Account: Cloudflare Pages: Edit
   - Account: Account: Read
   - Zone: Zone: Read
   - Account: D1: Edit
6. 「Continue to summary」→「Create Token」をクリック
7. 生成されたトークンをコピー（再表示不可のため注意）

#### Secrets の確認

```bash
# GitHub CLIを使用してSecretsの一覧を確認（名前のみ）
gh secret list

# または、GitHubリポジトリのSettings → Secrets and variables → Actionsで確認
```

## 8. Cloudflare Pagesの自動デプロイ設定

### 8.1 自動デプロイの無効化

このプロジェクトでは、GitHub Actionsを使用してデプロイを実行するため、Cloudflare Pagesの自動デプロイ（GitHub連携）を無効化してください。重複デプロイを避けるためです。

**無効化手順**:

1. Cloudflare Dashboardにログイン
2. **Workers & Pages** → **Pages** を選択
3. プロジェクトを選択（`stats47`、`stats47-staging`など）
4. **Settings** → **Builds & deployments** を選択
5. **Build configuration** セクションで、**GitHub連携**を無効化
   - または、**Build configuration** → **Build command** と **Output directory** を空欄にする

**注意**: プレビューデプロイメント（PRごとのデプロイ）が必要な場合は、GitHub Actionsの`preview`ジョブを使用してください。

### 8.2 環境ごとのURL設定

Cloudflare Pagesでは、各プロジェクトが異なるURLを持ちます：

- **本番環境**: `https://stats47.pages.dev`（プロジェクト名: `stats47`）
- **ステージング環境**: `https://stats47-staging.pages.dev`（プロジェクト名: `stats47-staging`）
- **プレビュー環境**: `https://<branch-name>-<project-name>.pages.dev`（PRごとに自動生成）

カスタムドメインを設定する場合は、Cloudflare Dashboardから設定できます。

## 10. Cloudflare Pages設定

### 10.1 Edge Runtime設定

Cloudflare Pagesでは、すべてのページとAPIルートがEdge Runtimeで実行される必要があります。

詳細については、[Cloudflare Pages Edge Runtime設定ガイド](./02_Cloudflare_Pages_Edge_Runtime設定.md)を参照してください。

**主な設定**:
- `next.config.ts`の`output: "standalone"`を設定（OpenNextの要件）
- `next.config.ts`の`experimental.useCache`を無効化
- `open-next.config.ts`を作成（OpenNextの設定）
- OpenNextが自動的に最適化するため、各ページに`export const runtime = "edge";`を追加する必要はありません

## 11. R2ストレージ同期

### 11.1 同期の概要

開発環境で作成・更新した静的データ（地域マスタ、地理データ、ブログコンテンツ）をステージング環境、本番環境へ同期します。

**同期フロー**:
```
開発環境 (stats47-development)
    ↓
ステージング環境 (stats47-staging)
    ↓
本番環境 (stats47)
```

### 11.2 同期対象データ

以下のデータのみを環境間で同期します：

- `area/` - 地域マスタデータ（都道府県・市区町村）
- `geoshape/` - 地理データ（TopoJSON）
- `content/blog/` - ブログコンテンツ（MDX関連ファイル）

以下のデータは同期しません（環境ごとに独立）：

- `estat-api/` - APIキャッシュ（各環境でランタイムに生成）
- `ranking/` - ランキングデータ（各環境でランタイムに生成）

### 11.3 同期手順

#### 開発→ステージング同期

```bash
# ドライランで変更内容を確認
npm run sync:r2:dev-to-staging:dry

# 実際に同期を実行
npm run sync:r2:dev-to-staging
```

#### ステージング→本番同期

```bash
# ドライランで変更内容を確認
npm run sync:r2:staging-to-prod:dry

# 実際に同期を実行（本番環境への同期は慎重に）
npm run sync:r2:staging-to-prod
```

### 11.4 デプロイ時の同期フロー

1. **開発環境での作業**
   - ローカルでMDXファイルを更新
   - ローカルで静的データを更新
   - `npm run sync:mdx` でD1データベースに同期

2. **ステージング環境へのデプロイ**
   - コードをデプロイ
   - `npm run db:sync:local-to-staging` でD1データベースを同期
   - `npm run sync:r2:dev-to-staging` でR2ストレージを同期

3. **本番環境へのデプロイ**
   - ステージング環境でテスト完了後
   - コードをデプロイ
   - `npm run sync:r2:staging-to-prod` でR2ストレージを同期

### 11.5 環境変数の設定

同期スクリプトを実行するには、以下の環境変数が必要です：

```bash
CLOUDFLARE_ACCOUNT_ID=your_account_id
CLOUDFLARE_R2_ACCESS_KEY_ID=your_access_key_id
CLOUDFLARE_R2_SECRET_ACCESS_KEY=your_secret_access_key
```

詳細については、[R2ストレージ同期ガイド](./04_R2ストレージ同期ガイド.md)を参照してください。
