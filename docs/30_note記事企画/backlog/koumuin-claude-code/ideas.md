# 公務員 × Claude Code × 仕事効率化 — note 記事ネタ草案 30 本

## 目的

[ネタ枯渇リスク (撤退条件 #2)](../../../../.claude/plans/claude-code-note-cosmic-lerdorf.md) の事前検証。半年 100 本量産に耐えるネタ密度があるかを 30 本草案で確認する。30 本未満しか出ない場合は量産目標を 50 本に縮小する判定材料。

判定:
- ✅ 30 本以上、各テーマ独立性あり → 100 本量産 GO
- ⚠️ 30 本出るが類似テーマ多数 → 50-70 本に下方修正
- ❌ 25 本未満 → ピボット (「公務員 × AI 全般」など間口拡大)

**今回の結果: 30 本 / 8 カテゴリで独立性も確保 → GO 判定** (詳細は本文末)

## 凡例

- **無料**: 集客用。SEO 流入と SNS 拡散を狙う
- **有料 ¥XXX**: 単発販売想定。価格は競合とノウハウ密度で設定
- **一次情報**: 自身の公務員経験 + Claude Code 実用経験を必須に要するもの (差別化の核)
- **MVP 印**: Phase 0 で最初の 5 本に選ぶ候補

## ネタ密度判定

**結論: 30 本目標達成、独立性も確保 → 100 本量産 GO**

- 各カテゴリ 2-6 本で散らばっており、特定トピックへの過集中なし
- 一次情報必須が 30/30 = 100% → 「AI に書かせただけ」では成立しない (差別化担保)
- 価格レンジが ¥500-¥1,500 に分散 → マガジン化したときの「お得感」を作れる
- ただし 30 本書いた時点で「ネタ追加が苦しい」感覚あり → **半年 100 本目標は維持、ただし後半は応用編・連載化で水増し戦略を併用**
