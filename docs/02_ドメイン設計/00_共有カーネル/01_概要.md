# 共有カーネル概要

## 共有カーネルの目的

共有カーネルは、複数のドメインで共通して使用される要素を管理します。これにより、ドメイン間の重複を避け、一貫性を保つことができます。

## 主要な責務

1. **共通の値オブジェクト定義**: 複数ドメインで使用される値オブジェクト
2. **共通の型定義**: プロジェクト全体で使用される型
3. **共通のユーティリティ関数**: 複数ドメインで使用されるヘルパー関数
4. **共通の定数定義**: プロジェクト全体で使用される定数
5. **共通の例外定義**: プロジェクト全体で使用される例外

## ユビキタス言語

### 基本概念

- **値オブジェクト**: 不変の値表現
- **型定義**: 共通の型情報
- **ユーティリティ**: 共通のヘルパー関数
- **定数**: 共通の定数値
- **例外**: 共通のエラー処理

### 値オブジェクト関連

- **ID**: エンティティの識別子
- **Date**: 日付の表現
- **Money**: 金額の表現
- **Email**: メールアドレスの表現

### 型関連

- **Result**: 成功・失敗の結果型
- **Option**: 値の有無を表現する型
- **Either**: 2 つの値のいずれかを表現する型

## 境界づけられたコンテキスト

共有カーネルは以下の境界を持ちます：

### 含まれるもの

- 複数ドメインで共通して使用される要素
- プロジェクト全体で使用される型定義
- 共通のユーティリティ関数
- 共通の定数・設定値

### 含まれないもの

- 特定ドメインに固有の要素
- ビジネスロジック
- ドメイン固有のルール

## 他ドメインとの関係性

### 提供するサービス

- **全ドメイン**: 共通の値オブジェクト、型、ユーティリティ

### 依存関係

- **なし**: 共有カーネルは他のドメインに依存しない

## 共有カーネルの価値

1. **重複排除**: ドメイン間の重複コードを排除
2. **一貫性**: プロジェクト全体での一貫した実装
3. **保守性**: 共通要素の変更が一箇所で済む
4. **再利用性**: 共通要素の再利用による開発効率向上
5. **品質**: 共通要素の品質向上による全体品質向上
