# ドメイン設計

このディレクトリには、stats47 プロジェクトのドメイン駆動設計（DDD）に基づくドメイン別のドキュメントが格納されています。

## ドメイン分類

### 00\_共有カーネル

複数のドメインで共通して使用される要素

- **値オブジェクト**: 共通の値オブジェクト定義
- **共通型定義**: プロジェクト全体で使用される型
- **ユーティリティ**: 共通のヘルパー関数

### 01\_コアドメイン

ビジネスの競争優位性を生み出す中核機能

- **Analytics（統計分析）**: ランキング、比較分析、時系列分析、相関分析
- **Visualization（可視化）**: ダッシュボード、グラフ、地図

### 02\_支援ドメイン

コアドメインを支える重要機能

- **Area（地域管理）**: 都道府県・市区町村の管理
- **Category（カテゴリ管理）**: 統計カテゴリの管理
- **DataIntegration（データ統合）**: e-Stat API 連携

### 03\_汎用ドメイン

標準的な機能（既製品で代替可能）

- **Auth（認証）**: ユーザー認証・認可
- **Content（コンテンツ管理）**: ブログ等のコンテンツ管理
- **Export（エクスポート）**: データエクスポート機能
- **Infrastructure（インフラ）**: データベース等のインフラ機能

## 各ドメインの標準構成

各ドメインは以下の標準的な構成を持ちます：

- **01\_概要.md**: ドメインの責務と目的、ユビキタス言語
- **02\_モデル/**: エンティティ、値オブジェクト、集約
- **03\_サービス/**: ドメインサービス、アプリケーションサービス、ユースケース
- **04\_インフラ/**: リポジトリ、データアクセス
- **05\_サブドメイン/**: 必要に応じて細分化されたサブドメイン
