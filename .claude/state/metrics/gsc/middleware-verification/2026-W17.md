| 結果 | 件数 / 30 |
|---|---|
| 410 | 30 |
| その他 | 0 |

**判定**: ✅ middleware が完全に 410 を返している。GSC W17 で 5xx 計上されている真値 ~2,041 件は、PHASE-9 の middleware で全件 410 化済み。Google が再クロール後に dropping する性質。

**取得コマンド**:
```bash
grep '/correlation/' .claude/state/metrics/gsc/coverage-drilldown/2026-W17/5xx-urls.csv | head -30 | cut -d, -f1 | while read url; do
  curl -s -o /dev/null -w "%{http_code}\n" -A "Mozilla/5.0 (compatible; Googlebot/2.1; +http://www.google.com/bot.html)" "$url"
done | sort | uniq -c
```

## D-2 結論: /ranking canonical 修正は不要

実装確認: `apps/web/src/features/ranking/utils/generate-meta-data.ts:121-123`

```ts
alternates: {
  canonical: `/ranking/${rankingItem.rankingKey}`,
},
```

**判定**: ✅ canonical は完全に正しく設定済み。

**含意**: PHASE-10 の D-2 候補も **取り消し**。GSC の dup-no-canonical 517 件（/ranking/）は以下のいずれかが原因と推定:

- [仮説 1] Google が PHASE-9 デプロイ後の最新 metadata を再クロールしていない（→ 5/9 までに観測）

- [仮説 3] クエリパラメータ付き URL（`/ranking/{key}?year=...` 等）が重複扱いになっている
