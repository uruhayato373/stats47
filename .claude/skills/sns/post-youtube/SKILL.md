YouTube Shorts 投稿用コンテンツ（タイトル・説明）を生成してローカル R2 に保存する。

## 引数

ユーザーから以下を確認すること:

| パラメータ | 必須 | デフォルト | 説明 |
|---|---|---|---|
| **domain** | - | `ranking` | `ranking` / `compare` / `correlation` |
| **template** | - | ドメインによる | `shock` / `versus` / `question` / `paradox` |

### ranking ドメイン
- **rankingKey**: ランキングキー（必須）

### compare ドメイン
- **areaA**: 地域A のエリアコード（必須）
- **areaB**: 地域B のエリアコード（必須）
- **categoryKey**: カテゴリキー（必須）

### correlation ドメイン
- **rankingKeyX**: X軸ランキングキー（必須）
- **rankingKeyY**: Y軸ランキングキー（必須）

### デフォルトテンプレ

| ranking | `shock` |
| compare | `versus` |

## テンプレート定義

template パラメータに応じて構造とトーンを切り替える。

### shock（衝撃事実型）
- 目的: リポスト・引用RTによる拡散
- 適合: ranking, correlation
- 構造: [衝撃の1行フック — 数字で驚かせる] → [データハイライト 2〜3行] → [URL] → [ハッシュタグ]
- フック例: 「年収が高いのに『実質貧困』な県がある。」

### versus（対決型）
- 目的: 引用RTによる議論
- 適合: compare, ranking（1位 vs 47位）
- 構造: [A vs B 対決フレーム] → [比較データ 3〜4項目] → [あなたはどっち派？] → [URL] → [ハッシュタグ]
- フック例: 「東京 vs 大阪、経済力で圧勝するのはどっち？」

### question（問いかけ型）
- 目的: 引用RTでの回答を誘発
- 適合: ranking, correlation
- 構造: [問いかけ1行] → [意外なデータ 2〜3行] → [回答促進CTA]
- フック例: 「あなたの県の自殺率、全国何位か知っていますか？」

### paradox（逆説・警告型）
- 目的: 保存・DM共有による深い関与
- 適合: correlation, ranking
- 構造: [逆説の1行 — 「〇〇なのに△△」構文] → [データで証明 2〜3行] → [警告的CTA] → [URL] → [ハッシュタグ]
- フック例: 「所得が高い県ほど幸福度が低い。この逆説にはワケがある。」

## テキスト生成

### ペルソナ

あなたは stats47（都道府県統計データの可視化サービス）の統計解説 YouTuber です。検索から流入する視聴者に「もっと見たい」と思わせる解説者として振る舞ってください。

### タイトル最適化ルール

以下のルールを厳守する（競合分析・離脱分析に基づく実証済みパターン）:

- **「都道府県別」をタイトル先頭に配置**（再生数 4 倍の効果）
- **感情フック語を含める**（「衝撃」「意外」「ワースト」「知らない」等で再生数 8 倍）
- **トップ訴求 + 疑問形の複合技**で 1 万再生超を狙う（例: 「日本で一番〜なのはどこ？」）
- フレーミング優先順: トップ訴求 ≒ 衝撃数値 > ネガティブ訴求 > ストレート > 疑問形

### YouTube プラットフォームルール

- SEOキーワード型: タイトル50文字以内、検索キーワードを先頭に配置。タイトル末尾のハッシュタグは0〜1個に抑える（フックを目立たせるため）。**タイトル末尾に `#Shorts` を付ける場合は、その後にスペースなしでハッシュタグを連結しないこと**（`#Shorts#地方税` → `#Shorts #地方税`）
- 説明欄250字以上
- **説明欄の1行目にハッシュタグを3〜5個配置**（YouTube が動画タイトル上部に最大3個を青文字表示する。検索流入・関連動画表示に効果あり。例: `#公務員退職金 #都道府県ランキング #Shorts #地方公務員`）
- ハッシュタグ行の後に本文を開始、冒頭125文字にキーワード集中
- CTA優先順: **完視聴 > チャンネル登録 > コメント**。完視聴率を最大化する構成が最優先
- 本文冒頭に top3 + URL（UTMパラメータ付き）を配置
- 説明欄末尾に「自分の県を確認してコメントで感想教えて」と誘導（説明欄→コメントの動線）
- 全47都道府県データは維持（YouTube は独立コンテンツ）
- #Shorts 必須（説明欄冒頭のハッシュタグに含める）、ハッシュタグ計3〜5個。**各ハッシュタグの間には必ずスペースを入れること**（`#Shorts #都道府県ランキング` ○ / `#Shorts#都道府県ランキング` ×）
- **投稿頻度: 1日最大2本まで**。3本以上/日を連日投稿すると YouTube がスパム判定し、Shorts フィードへのインプレッション配信が停止するリスクがある
- コメント誘導パターン（データの特性に応じて選択）:
  - 予測→答え合わせ型: 冒頭「1位どの県だと思う？」→ 末尾「予想当たった？正解数コメントで教えて」
  - 説明欄誘導型: 「全47都道府県の順位は説明欄に。自分の県を確認してコメントで感想教えて」
  - 比較提案型: 「〇〇と△△、どっちが意外？比較動画出してほしかったらコメントで」
  - シリーズ予告型: 「次は"実は貧しい県ランキング"。見たい人はチャンネル登録して待ってて」
- **ピン留めコメントが最大の武器**: TOP3+URLに加え、データから読み取れる意外な事実や地元民への問いかけを1つ仕込む。返信スレッドが伸びるとコメント欄の滞在時間が増え間接的にアルゴリズムに効く
- コメント誘導は**動画の末尾**に配置する（TikTokは冒頭だが、YouTubeは完視聴させることが先）
- hookText: 15文字以内（動画冒頭テロップ用）。常識の逆転・意外性を優先する（例:「都会ほど仕事がない？」）。単なる数字の差（例:「○倍の地域差」）は弱いので避ける。**予測型も有効**（例:「1位の県わかる？」）
- displayTitle: 20文字以内。**「ランキング」を含めないこと**（Remotion テンプレートがサブタイトルに「〇〇年度 都道府県ランキング」を自動表示するため、重複する）。例: ×「緑茶消費量ランキング」→ ○「緑茶消費額」
- description がある場合、説明欄の冒頭125文字以内に指標の定義を含める（SEO効果）

### クイズ生成ルール

YouTube Studio の「クイズを追加」機能用データを生成する。投稿時にコピペで設定できるようにする。

- クイズは **2〜3問** 生成する（多すぎると煩わしい）
- 4択（A/B/C/D）、正解は1つ
- データから導ける客観的な問題にする（主観・意見系は避ける）
- `timestamp` は動画の構成に合わせた目安秒数を指定する

**クイズパターン（データに応じて選択）:**

| パターン | 例 | 配置タイミング |
|---|---|---|
| 1位予測 | 「1位はどの都道府県？」 | 動画冒頭（5〜10秒） |
| 意外な順位 | 「○○県は何位？」 | 中盤（動画の40〜60%地点） |
| TOP10入り | 「TOP10に入るのはどっち？」 | 10位発表前 |
| 数値クイズ | 「1位と47位の差はいくら？」 | 3位発表前 |
| ワースト予測 | 「最下位はどの都道府県？」 | 動画冒頭（5〜10秒） |

**選択肢の作り方:**
- 正解 + もっともらしい不正解3つ（ランダム順で配置）
- 不正解には「多くの人が間違えそうな県」を選ぶ（例: 人口が多い県、イメージと実態が乖離する県）
- `explanation` に正解の根拠を1文で添える（回答後に表示される）

### JSON 出力形式

```json
{
  "title": "50字以内SEOタイトル（末尾にハッシュタグ）",
  "description": "250字以上の説明欄",
  "pinnedComment": "ピン留めコメント用テキスト",
  "hashtags": ["#Shorts", ...],
  "hookText": "15字以内",
  "displayTitle": "20字以内",
  "quizzes": [
    {
      "timestamp": "0:08",
      "question": "公務員の退職金が最も高い都道府県は？",
      "choices": ["大阪府", "東京都", "神奈川県", "愛知県"],
      "correctIndex": 1,
      "explanation": "東京都が約2,483万円で1位。地域手当の高さが退職金にも反映されています。"
    }
  ]
}
```

## 出力先

| ドメイン | ディレクトリ |
|---|---|
| ranking | `.local/r2/sns/ranking/<rankingKey>/youtube-short/` |
| compare | `.local/r2/sns/compare/<areaA>-vs-<areaB>/youtube-short/` |
| correlation | `.local/r2/sns/correlation/<keyX>--<keyY>/youtube-short/` |

各ディレクトリに `shorts.json`、`shorts.txt`、`pinned_comment.txt`、`quizzes.txt` を保存。

**注意**: `youtube/` ディレクトリは通常動画（`RankingYouTube-Normal`）用に予約されている。ショート動画コンテンツは必ず `youtube-short/` に保存すること。

### YouTube Short A/B テスト

動画は2バージョンを同時生成し、A/B テストで最適な尺を判断する:

| バージョン | コンポジション | 尺 | 構成 | ファイル名 |
|---|---|---|---|---|
| **A版** | `RankingYouTube-Short` | ~32秒 | 上位5件 RankCard → タイルマップ → 全47件テーブル → CTA | `shorts-a.mp4` |
| **B版** | `RankingYouTube-Short-Full` | ~55秒 | 全47件を高速表示（モーション無し即表示）→ CTA | `shorts-b.mp4` |

キャプション（`shorts.json` / `shorts.txt` / `pinned_comment.txt`）は A/B 共通で1セット。タイトルも同一にして尺だけ異なる状態で投稿し、維持率を比較する。

## 手順

1. data.json を読み込み、算出値を計算
2. テンプレ定義とPFルールに基づき、Claude が直接 JSON を生成
   → ユーザーに提示して確認・修正
3. `shorts.json` を保存
4. コピペ投稿用の `shorts.txt` を保存:
   - ranking: title + description + 全47都道府県データ（偏差値付き）+ URL（UTM付き）+ ハッシュタグ
   - compare: title + description + 比較指標一覧 + URL（UTM付き）+ ハッシュタグ
   - correlation: title + description + 散布データ上位/下位5県 + URL（UTM付き）+ ハッシュタグ
   ※ 偏差値 = 50 + 10 × (value - 平均) / 標準偏差（小数第1位）
   ※ ランキングデータは等幅フォーマットで出力: 順位2桁右寄せ、都道府県名CJK幅パディング、値は小数第1位統一（13% → 13.0%）
5. `pinned_comment.txt` を生成して保存:
   ```
   ── ランキング TOP3 ──
   1位 ○○県 ○○
   2位 ○○県 ○○
   3位 ○○県 ○○

   【議論の種（データに基づく考察・問いかけを1つ）】
   例:「個人的に意外だったのは〇〇県。△△が理由だと思うけど、地元の方どう思いますか？」

   あなたの県は何位だった？全47都道府県ランキングはこちら👇
   <URL（UTM付き、utm_content=<template>-pinned）>

   📢 チャンネル登録で最新ランキングを見逃さない！
   ```
   ※ `utm_content` に `-pinned` を付加して説明欄リンクとのCTR差を計測可能にする
   ※ 議論の種はTOP3+URLの間に配置する。データから読み取れる意外な事実や、地元民に聞きたい疑問を1つ書く。返信スレッドが伸びるとコメント欄の滞在時間が増える
6. `quizzes.txt` を生成して保存（YouTube Studio でのコピペ用）:
   ```
   ── クイズ 1（0:08）──
   Q: 公務員の退職金が最も高い都道府県は？
   A: 大阪府
   B: 東京都 ✅
   C: 神奈川県
   D: 愛知県
   → 東京都が約2,483万円で1位。地域手当の高さが退職金にも反映されています。

   ── クイズ 2（2:30）──
   ...
   ```
7. テキストをユーザーに報告
