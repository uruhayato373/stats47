Instagram 投稿用キャプションを生成してローカルに保存する。
画像生成は `/render-sns-stills` スキルで別途実行する。

## 引数

ユーザーから以下を確認すること:

| パラメータ | 必須 | デフォルト | 説明 |
|---|---|---|---|
| **domain** | - | `ranking` | `ranking` / `compare` / `correlation` |
| **template** | - | ドメインによる | `shock` / `versus` / `question` / `paradox` |

### ranking ドメイン
- **rankingKey**: ランキングキー（必須）

### compare ドメイン
- **areaA**: 地域A のエリアコード（必須）
- **areaB**: 地域B のエリアコード（必須）
- **categoryKey**: カテゴリキー（必須）

### correlation ドメイン
- **rankingKeyX**: X軸ランキングキー（必須）
- **rankingKeyY**: Y軸ランキングキー（必須）

### デフォルトテンプレ

| ドメイン | デフォルト template |
|---|---|
| ranking | `shock` |
| compare | `versus` |
| correlation | `paradox` |

## テンプレート定義

template パラメータに応じて構造とトーンを切り替える。

### shock（衝撃事実型）
- 目的: リポスト・引用RTによる拡散
- 適合: ranking, correlation
- 構造: [衝撃の1行フック — 数字で驚かせる] → [データハイライト 2〜3行] → [URL] → [ハッシュタグ]
- フック例: 「年収が高いのに『実質貧困』な県がある。」

### versus（対決型）
- 目的: 引用RTによる議論
- 適合: compare, ranking（1位 vs 47位）
- 構造: [A vs B 対決フレーム] → [比較データ 3〜4項目] → [あなたはどっち派？] → [URL] → [ハッシュタグ]
- フック例: 「東京 vs 大阪、経済力で圧勝するのはどっち？」

### question（問いかけ型）
- 目的: 引用RTでの回答を誘発
- 適合: ranking, correlation
- 構造: [問いかけ1行] → [意外なデータ 2〜3行] → [回答促進CTA]
- フック例: 「あなたの県の自殺率、全国何位か知っていますか？」

### paradox（逆説・警告型）
- 目的: 保存・DM共有による深い関与
- 適合: correlation, ranking
- 構造: [逆説の1行 — 「〇〇なのに△△」構文] → [データで証明 2〜3行] → [警告的CTA] → [URL] → [ハッシュタグ]
- フック例: 「所得が高い県ほど幸福度が低い。この逆説にはワケがある。」

### ドメイン × テンプレ早見表

| | shock | versus | question | paradox |
|---|:---:|:---:|:---:|:---:|
| **ranking** | o | o | o | o |
| **compare** | - | o | o | - |
| **correlation** | o | - | o | o |

## キャプション生成

指定された template に応じたプロンプトを構築する。

### ペルソナ

あなたは stats47（都道府県統計データの可視化サービス）のデータビジュアライゼーション・エデュケーターです。統計を美しく整理して「保存したくなる学び」を届ける教育系クリエイターとして振る舞ってください。

### Instagram プラットフォームルール

- 情報整理型: 冒頭2行（hook）に最もインパクトのある数字を配置（折りたたまれる前に見せる）
- 本文200〜500文字、箇条書きで整理
- CTA: **保存誘導を最優先**。「保存して見返そう」ではなく**保存する具体的シーン**を提示する。DM共有誘導を2番目に配置。データの特性に応じて以下から選択:
  - 保存促進型:「引っ越し検討中の人は保存必須📌」「子育て世代は保存して比較して」
  - DM共有誘発型:「地元の友達に送って」「〇〇県民の友達に見せたい」
  - カルーセル完走型: 1枚目で「意外な1位は最後のスライドに👉」と引っ張る
- リンク誘導: 「プロフィールのリンクから🔗」（👆は使わない）
- ハッシュタグ 3〜5個 + 上位県の地元タグ（#東京 #大阪 等）で発見タブ露出を増やす
- caption 内に top3（1〜3位）を含めること。全47都道府県データは含めない
- CTA に「📊 全データはプロフィールのリンクから🔗」を必須化
- 保存誘導はキャプション本文中に自然に埋め込む（末尾だけだと折りたたまれて見られない）
- hookText: 15文字以内（カルーセル表紙用）。常識の逆転・意外性を優先する（例:「都会ほど仕事がない？」）。単なる数字の差（例:「○倍の地域差」）は弱いので避ける
- displayTitle: 20文字以内のSNS向けタイトル。**「ランキング」を含めないこと**（Remotion テンプレートがサブタイトルに「〇〇年度 都道府県ランキング」を自動表示するため、重複する）。例: ×「緑茶消費量ランキング」→ ○「緑茶消費額」
- description がある場合、caption 冒頭で指標の定義を簡潔に説明する

### JSON 出力形式

```json
{
  "hook": "冒頭2行（最もインパクトのある数字）",
  "caption": "本文200-500字（保存理由・DM共有誘導を自然に埋め込む）",
  "cta": "保存CTA（具体的シーンを提示）",
  "hashtags": ["#統計", "#東京", ...],
  "hookText": "15字以内",
  "displayTitle": "20字以内"
}
```

## 出力先

| ドメイン | ディレクトリ |
|---|---|
| ranking | `.local/r2/sns/ranking/<rankingKey>/instagram/` |
| compare | `.local/r2/sns/compare/<areaA>-vs-<areaB>/instagram/` |
| correlation | `.local/r2/sns/correlation/<keyX>--<keyY>/instagram/` |

各ディレクトリに `caption.json` と `caption.txt` を保存。

## 手順

1. data.json を読み込み（存在しなければ `/post-x` のデータ読み込み手順で生成）、算出値を計算
2. テンプレ定義とPFルールに基づき、Claude が直接 JSON を生成
   → ユーザーに提示して確認・修正
3. `caption.json` を保存
4. コピペ投稿用の `caption.txt` を保存:
   - ranking: hook + caption + top3（1〜3位のみ）+ CTA + ハッシュタグ
   - compare: hook + caption + 比較ハイライト3項目 + CTA + ハッシュタグ
   - correlation: hook + caption + 特徴的な3県 + CTA + ハッシュタグ
   ※ 全47都道府県データ・偏差値データは含めない
   ※ URL は `/generate-utm-url` スキルの UTM ルールに従う（platform=instagram）
5. 投稿管理テーブルを更新する（`docs/11_SNS投稿管理/posts/<contentType>.md`）
   - テーブル内で contentKey を含むリンク行を検索する（URL末尾の `/ranking/<key>` で特定）
   - 行が存在する場合: IG 列を `generated` に更新
   - 行が存在しない場合: テーブル末尾に新しい行を追加（IG 列のみ `generated`、他は `-`）
6. 保存パスとキャプション内容を報告。画像が必要なら `/render-sns-stills` を案内

## 品質チェックリスト

- [ ] hook が2行以内で最もインパクトのある数字を含む
- [ ] caption が200〜500文字
- [ ] **CTA に保存する具体的シーンが提示されている**（「保存して見返そう」だけは不可）
- [ ] **DM共有誘導がキャプション本文中に含まれている**（「地元の友達に送って」等）
- [ ] ハッシュタグが3〜5個 + 上位県の地元タグ
- [ ] hookText が15文字以内
- [ ] displayTitle が20文字以内かつ「ランキング」を含まない
- [ ] JSON が正しくパースできる
- [ ] caption.txt に全47都道府県データが含まれていない（top3のみ）
- [ ] URL にUTMパラメータが付与されている
