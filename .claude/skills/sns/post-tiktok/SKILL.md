TikTok 投稿用コンテンツ（キャプション）を生成してローカル R2 に保存する。

## 引数

ユーザーから以下を確認すること:

| パラメータ | 必須 | デフォルト | 説明 |
|---|---|---|---|
| **domain** | - | `ranking` | `ranking` / `compare` / `correlation` |
| **template** | - | ドメインによる | `shock` / `versus` / `question` / `paradox` |

### ranking ドメイン
- **rankingKey**: ランキングキー（必須）

### compare ドメイン
- **areaA**: 地域A のエリアコード（必須）
- **areaB**: 地域B のエリアコード（必須）
- **categoryKey**: カテゴリキー（必須）

### correlation ドメイン
- **rankingKeyX**: X軸ランキングキー（必須）
- **rankingKeyY**: Y軸ランキングキー（必須）

### デフォルトテンプレ

| ドメイン | デフォルト template |
|---|---|
| ranking | `shock` |
| compare | `versus` |
| correlation | `paradox` |

`/post-x` スキルの「データ読み込み」セクションと同じ手順でデータを読み込む。

## テンプレート定義

template パラメータに応じて構造とトーンを切り替える。

### shock（衝撃事実型）
- 目的: リポスト・引用RTによる拡散
- 適合: ranking, correlation
- 構造: [衝撃の1行フック — 数字で驚かせる] → [データハイライト 2〜3行] → [ハッシュタグ]
- フック例: 「年収が高いのに『実質貧困』な県がある。」

### versus（対決型）
- 目的: 引用RTによる議論
- 適合: compare, ranking（1位 vs 47位）
- 構造: [A vs B 対決フレーム] → [比較データ 3〜4項目] → [あなたはどっち派？] → [ハッシュタグ]
- フック例: 「東京 vs 大阪、経済力で圧勝するのはどっち？」

### question（問いかけ型）
- 目的: 回答を誘発
- 適合: ranking, correlation
- 構造: [問いかけ1行] → [意外なデータ 2〜3行] → [回答促進CTA]
- フック例: 「あなたの県の自殺率、全国何位か知っていますか？」
- **注意**: TikTok は全47都道府県データを含むため、「何位だと思う？」「予想して」系のCTAは矛盾する。代わりに「意外だった？」「納得できる？」など、データを見た上での反応を求めるCTAを使う

### paradox（逆説・警告型）
- 目的: 保存・DM共有による深い関与
- 適合: correlation, ranking
- 構造: [逆説の1行 — 「〇〇なのに△△」構文] → [データで証明 2〜3行] → [警告的CTA] → [ハッシュタグ]
- フック例: 「所得が高い県ほど幸福度が低い。この逆説にはワケがある。」

## キャプション生成

### ペルソナ

あなたは stats47（都道府県統計データの可視化サービス）のデータ系エンタメクリエイターです。統計を「えっ？マジ？」のネタに変える、テンポ重視の語り手として振る舞ってください。

### TikTok プラットフォームルール

- 口語・会話型: 100〜300文字、テンポよく
- CTA: **コメント誘導を最優先**。フォロー誘導は末尾に添える程度
- コメント誘導: キャプション冒頭3行以内に問いかけを配置。データの特性に応じて以下から選択:
  - 理由質問型:「なぜ〇〇県が1位？県民の方コメントで教えて」
  - 意外性×反論型:「〇〇が最下位、納得できる？」
  - 二択型:「住むなら1位の県？最下位の県？」
  - 考察募集型:「この相関の理由、コメントで考察して」
  - 地元反応型:「1位の〇〇県民、自覚ある？」
- リンク誘導: 不要（TikTok はキャプション内リンクがクリッカブルにならないため）。プロフィール bio にサイトURLを設定しておく
- ハッシュタグ 5〜8個（#fyp 不要）。構成: テーマ共通2〜3個（#都道府県 #統計 #データで見る日本）+ トピック固有1〜2個（#空き家 #少子化 等）+ 上位県の地域タグ1〜2個（#徳島 #沖縄 等）+ 発見系1個（#知らなかった #雑学 等）
- caption.txt に全47都道府県データを含めること（独立コンテンツとして完結させる）
- hookText: 15文字以内（動画冒頭テロップ用）。常識の逆転・意外性を優先する（例:「都会ほど仕事がない？」）。単なる数字の差（例:「○倍の地域差」）は弱いので避ける。**コメント誘導要素を含めるとさらに効果的**（例:「〇〇県民怒らないで」「理由わかる？」）
- displayTitle: 20文字以内。**「ランキング」を含めないこと**（Remotion テンプレートがサブタイトルに「〇〇年度 都道府県ランキング」を自動表示するため、重複する）。例: ×「緑茶消費量ランキング」→ ○「緑茶消費額」
- description がある場合、口語で指標の意味を冒頭に盛り込む
- 注意: 煽りすぎない。「ヤバい県」のような攻撃的表現は避け、「なぜこうなるのか」という知的な問いかけのトーンを維持する

### JSON 出力形式

```json
{
  "caption": "100-300字の口語キャプション（冒頭3行以内にコメント誘導）",
  "hashtags": ["#都道府県", ...],
  "hookText": "15字以内（コメント誘導要素を含めると効果的）",
  "displayTitle": "20字以内",
  "pinnedComment": "投稿後にピン留めする考察コメント（返信の連鎖を誘発）"
}
```

## 出力先

| ドメイン | ディレクトリ |
|---|---|
| ranking | `.local/r2/sns/ranking/<rankingKey>/tiktok/` |
| compare | `.local/r2/sns/compare/<areaA>-vs-<areaB>/tiktok/` |
| correlation | `.local/r2/sns/correlation/<keyX>--<keyY>/tiktok/` |

各ディレクトリに `caption.json` と `caption.txt` を保存。

## 手順

1. data.json を読み込み、算出値を計算
2. テンプレ定義とPFルールに基づき、Claude が直接 JSON を生成
   → ユーザーに提示して確認・修正
3. `caption.json` を保存
4. コピペ投稿用の `caption.txt` を保存:
   - ranking: caption + 全47都道府県データ（順位・値）+ ハッシュタグ
   - compare: caption + 比較ハイライト項目 + ハッシュタグ
   - correlation: caption + 特徴的な県 + ハッシュタグ
   ※ 独立コンテンツとして全データを含める（リンクは不要）
5. 投稿管理テーブルを更新する（`docs/11_SNS投稿管理/posts/<contentType>.md`）
   - テーブル内で contentKey を含むリンク行を検索する（URL末尾の `/ranking/<key>` で特定）
   - 行が存在する場合: TT 列を `generated` に更新
   - 行が存在しない場合: テーブル末尾に新しい行を追加（TT 列のみ `generated`、他は `-`）
6. キャプションをユーザーに報告

## 品質チェックリスト

- [ ] caption が100〜300文字
- [ ] 口語的でテンポのよい文体
- [ ] **キャプション冒頭3行以内にコメント誘導の問いかけがある**
- [ ] **pinnedComment が含まれている（考察的な内容で返信を誘発する）**
- [ ] ハッシュタグが5〜8個（#fyp なし、テーマ共通+トピック固有+地域タグ+発見系の構成）
- [ ] hookText が15文字以内
- [ ] displayTitle が20文字以内かつ「ランキング」を含まない
- [ ] JSON が正しくパースできる
- [ ] caption.txt に全47都道府県データが含まれている（独立コンテンツ）
- [ ] キャプション内にクリッカブルURLが含まれていない（TikTokではリンク不可）
- [ ] 煽り表現がない（「ヤバい県」等の攻撃的表現を避ける）
