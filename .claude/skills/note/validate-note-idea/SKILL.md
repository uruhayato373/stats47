note 記事（B/C/D シリーズ）のアイデアを、書き始める前に需要・独自性・信頼性の3軸で検証する。

> **A シリーズ（ランキング記事・量産型）はこの4ステップワークフローの対象外です。** A シリーズは自動生成スキルで処理するため、validate → design → write → edit の手順は不要です。

## 用途

- 記事のテーマが市場で通用するか判断したいとき
- タイトル・切り口・ターゲットを決めたいとき
- 書く前に「読まれない企画」を弾きたいとき

## フロー

```
★/validate-note-idea → /investigate-note-data → /design-note-structure → /write-note-section → /edit-note-draft
```

**このスキルで 3 軸すべて合格になるまで次に進まないこと。**

## 引数

ユーザーから以下を確認すること:

- **articleId**: 記事ID（必須）— 例: `B-1`, `C-1`, `D-1`
- **シリーズ**: B（ランキング読み物）/ C（統計調査ガイド）/ D（深掘り分析）
- **テーマ**: 記事テーマ（1文で）
- **ターゲット読者**: 誰の、どんな課題を解決するか
- **専門性**: このテーマに関する発信実績やデータ資産
- **目標**: 流入獲得 / ブランディング / フォロワー獲得

## 手順

### 1. note 戦略の確認

`docs/30_note記事企画/note戦略.md` から該当 articleId のセクションを読み、以下を把握する:
- 骨子（記事構成）
- stats47.jp へのリンク候補
- 想定読者
- シリーズの位置づけ（ファネルのどの段階か）

### 2. stats47 データ資産の棚卸し

テーマに関連するランキング・ブログ記事を確認する:

```bash
# 関連ランキングの検索
npx wrangler d1 execute stats47_static --local --persist-to .local/d1 --command \
  "SELECT ranking_key, title, category_key FROM ranking_items WHERE title LIKE '%キーワード%' AND is_active = 1 LIMIT 20"

# 関連ブログ記事の検索
ls .local/r2/blog/ | grep -i "キーワード"
```

### 3. stats47.jp リンクの実在確認

リンク候補として挙がっている stats47.jp のページが実在するか確認する:

```bash
# ブログ記事の存在確認
ls .local/r2/blog/

# ランキングページの存在確認
npx wrangler d1 execute stats47_static --local --persist-to .local/d1 --command \
  "SELECT key, title FROM ranking_items WHERE key IN ('候補1', '候補2') AND is_active = 1"
```

存在しないリンク候補は記事から除外する。実在するリンクのみ使用すること。

### 4. 企画検証の実行

以下のフレームワークで検証する:

```
あなたは note.com のコンテンツストラテジストです。
以下の情報をもとに、記事のアイデアを検証してください。

シリーズ: [B/C/D]
テーマ: [1文]
専門性: [発信実績やデータ資産]
ターゲット読者: [誰の、どんな課題を解決するか]
stats47 の関連データ: [手順2で見つけたランキング・記事]
目標: [流入獲得 / ブランディング / フォロワー獲得]
```

**検証する3軸:**

#### 需要分析
- このテーマに対する読者需要の根拠
- note / ブログ / SNS で類似コンテンツのトップ5とその弱点
- 自分の記事が独占できる切り口（データ × 切り口の掛け合わせ）

#### コンセプト設計
- 「読みたい」と思わせる1文コンセプト
- 読者が体験するビフォー → アフターの変化
- 類似コンテンツにない独自の価値

#### タイトル・設計
- タイトル候補を5つ（数字 or 問いかけ or 意外性を含む）
- stats47.jp への誘導設計（各記事 5〜10 本のリンク計画）
- SEO カニバリ防止の確認（読み物型タイトルにする）

#### 出版可能性スコア
- 需要: /10
- 独自性: /10
- 信頼性: /10

### 5. SEO カニバリ防止チェック

- title に「都道府県 〇〇 ランキング」形式を使わない
- stats47.jp = ファクト検索 + ランキング、note = ストーリー + 考察 + 調査ガイド
- 全47都道府県データを掲載しない（具体例は 3〜5 個まで）
- note 記事のターゲットキーワード:
  - B シリーズ: 「なぜ〜」「意外な〜」「〜の正体」等の読み物型
  - C シリーズ: 「〇〇調査 わかること」
  - D シリーズ: 社会課題系キーワード

### 6. 結果の保存

```
docs/31_note記事原稿/<slug>/_data/plan.md
```

検証結果（スコア・タイトル候補・コンセプト・リンク候補リスト）を保存する。

## 判定基準

- 3軸すべて **5点以上** → `/design-note-structure` へ進む
- いずれか **5点未満** → テーマ・切り口を修正して再検証
- 全記事無料のため、収益性より **stats47.jp への誘導設計の質** を重視する
