note 記事（B/C/D シリーズ）の構成を設計する。シリーズに応じた構成テンプレートを適用する。

> **A シリーズ（ランキング記事・量産型）はこの4ステップワークフローの対象外です。** A シリーズは自動生成スキルで処理するため、validate → design → write → edit の手順は不要です。

## 用途

- `/validate-note-idea` で検証済みの企画を、書ける状態まで構造化したいとき
- 章構成と stats47.jp 誘導設計を決めたいとき

## フロー

```
/validate-note-idea → /investigate-note-data → ★/design-note-structure → /write-note-section → /edit-note-draft
```

## 引数

ユーザーから以下を確認すること:

- **articleId**: 記事ID（例: `B-1`, `C-1`, `D-1`）
- **シリーズ**: B / C / D
- **タイトル**: `/validate-note-idea` で決定したタイトル
- **読者への約束**: 読者にどんな変化を届けるか（1文）
- **ターゲット読者**: 誰が読むか
- **トーン**: 会話的 / 権威的 / データドリブン（全セクション統一）
- **活用する stats47 データ**: ランキングキーや統計指標

## 手順

note 戦略（`docs/30_note記事企画/note戦略.md`）の骨子に基づき、シリーズ別テンプレートで構成を設計する。

### 0. research.md の読み込み

`/investigate-note-data` で生成した `docs/31_note記事原稿/<slug>/_data/research.md` を読み込み、相関分析・地域パターンの結果をもとに章構成を設計する。チャート候補リストを構成に反映し、どの章にどのチャートを配置するかを決定すること。

### B シリーズ: ランキング読み物（1,500〜2,500 字）

1. **Frontmatter** — title（「なぜ〜」「意外な〜」等の読み物型）、description（120文字）、tags（5〜7個）
2. **フック** — 意外な事実・読者の常識を揺さぶる問い
3. **ランキング結果の要約** — 上位5〜10県を提示（stats47.jp のスクリーンショット推奨）。stats47.jp リンク
4. **なぜこの結果になるのか** — 背景・歴史・地理・産業構造を考察
5. **元行政職員の視点** — データの定義の落とし穴、調査方法の注意点
6. **もっと詳しく知りたい方へ** — stats47.jp リンク集（5本以上）
7. **CTA** — `---` 下に stats47.jp リンク

### C シリーズ: 統計調査ガイド（2,000〜5,000 字）

1. **Frontmatter** — title（「〇〇調査でわかること」系）、description（120文字）、tags（5〜7個）
2. **この調査とは** — 概要（実施機関・周期・対象・歴史）、重要性を読者目線で
3. **どんなデータが取れるか** — 主要指標をカテゴリ別に整理
4. **都道府県別に見るとここが面白い** — 3〜5 個の具体的発見。各トピックに stats47.jp リンク（UTM: `utm_content=topic-N`）
5. **元行政職員の視点** — 調査の裏側・データの注意点・現場での活用例
6. **活用シーン** — ビジネス / 研究・政策 / 日常の意思決定
7. **e-Stat での探し方** — 検索手順 → 「加工が大変 → stats47 なら整形済み」の誘導
8. **関連する stats47 のデータ** — stats47.jp リンク集（UTM: `utm_content=links`）、5〜10本
9. **CTA** — `---` 下に stats47.jp トップへの誘導

### D シリーズ: 深掘り分析（3,000〜5,000 字）

1. **Frontmatter** — title（社会課題系）、description（120文字）、tags（5〜7個）
2. **問題提起** — 社会課題としてのフレーミング、読者の関心を引く導入
3. **データで見る現状** — 複数指標のクロス分析。stats47.jp リンク（UTM: `utm_content=data-N`）
4. **なぜこうなったのか** — 歴史的経緯・政策・社会構造の変化
5. **元行政職員の視点** — 現場で見てきた実態、データだけでは見えないこと
6. **これからどうなるか** — データから読み取れるトレンド
7. **stats47 リンク集** — 関連ランキング・ブログ記事（5本以上）
8. **CTA** — `---` 下に stats47.jp リンク

### 全シリーズ共通で決めること

- 各章の目的と読者への価値
- stats47.jp リンクの配置箇所（最低5本）
- 具体例として取り上げるトピック（3〜5個）
- 章ごとの文字数目安
- データリテラシーの知識（人口当たり補正、相関≠因果など）を自然に織り込む箇所

## 出力

構成を `plan.md` に追記保存する:

```
docs/31_note記事原稿/<slug>/_data/plan.md
```

## 注意

- **トーンは最初に決めて全章統一すること。** 指定しないと章ごとにテンションがバラバラになる。
- B シリーズは短くまとめて頻度で勝負。迷ったら 1,500〜2,000 字で十分。
- D シリーズは複数指標の横断分析が肝。データの深さで差別化する。
