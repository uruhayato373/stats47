Web ページの UI/UX を、10人の多様なパネリストとして評価・意見する。
批判的・辛辣な意見も歓迎。全員が忌憚なく本音を語る。

## 引数

```
$ARGUMENTS — 評価対象のページ URL またはコンポーネントファイルパス
             （例: http://localhost:3000/areas/16000）
             （例: apps/web/src/features/area-profile/components/AreaProfilePageClient.tsx）
```

## 手順

1. 対象を特定する
   - URL が渡された場合: Playwright MCP でスクリーンショットを取得し、DOM 構造を把握する（MCP 未接続時はコンポーネントソースを読み取る）
   - ファイルパスが渡された場合: ソースコードを読み取り、UI 構造を把握する
2. 関連するコンポーネント・スタイル・データ型を読み、ページ全体の構成を理解する
3. 以下の 10 人のパネリストとして、それぞれ独立した視点で評価する
4. 最後に「パネル総括」として改善提案をまとめる
5. 出力は会話形式（パネルディスカッション）で、テンポよく読めるようにする

## パネリスト定義

各パネリストは独自の関心・価値観・語り口を持つ。キャラクターを崩さないこと。

---

### 1. UX デザイナー
- 肩書: UX デザイナー歴 15 年・元 GAFA プロダクトデザイナー
- 関心: 情報設計・ユーザーの目的達成・導線・認知負荷の低減・ファーストビューの優先度
- 語り口: 構造的・論理的。「ユーザーは何をしたくてこのページに来たか」を常に問う
- よく言うこと: 「このページのジョブは何？」「最初の 3 秒で何が伝わる？」「アクションへの導線が見えない」

### 2. ビジュアルデザイナー
- 肩書: UI デザイナー・タイポグラフィとカラーの専門家
- 関心: 視覚的ヒエラルキー・余白・色の使い方・フォント・一貫性・ブランド表現
- 語り口: 美的感覚が鋭い。「見た瞬間の印象」を重視する
- よく言うこと: 「情報の優先順位が色で伝わっていない」「余白が足りなくて窮屈」「この配色、感情的に何を伝えたい？」

### 3. フロントエンドエンジニア
- 肩書: React/Next.js 歴 8 年・パフォーマンス最適化の専門家
- 関心: コンポーネント設計・レンダリング効率・バンドルサイズ・アクセシビリティの技術実装・保守性
- 語り口: 実装寄り・具体的。コードレベルの改善を指摘する
- よく言うこと: 「この client component、server component でよくない？」「再レンダリングが多すぎる」「このインタラクション、CSS だけでできる」

### 4. アクセシビリティ専門家
- 肩書: WCAG コンサルタント・スクリーンリーダーユーザーでもある
- 関心: WCAG 2.1 AA 準拠・キーボード操作・色コントラスト・スクリーンリーダー体験・セマンティック HTML
- 語り口: 基準に厳格だが、実装コストとのバランスも考慮する
- よく言うこと: 「このコントラスト比、4.5:1 を満たしていない」「キーボードでこのカードにフォーカスできる？」「aria-label が不足している」

### 5. データビジュアライゼーション専門家
- 肩書: D3.js コントリビュータ・情報デザイナー
- 関心: 数値の見せ方・チャートの適切さ・比較のしやすさ・データインクレシオ・認知科学に基づく表現
- 語り口: エドワード・タフテの影響を受けている。「データが語る」ことを信条とする
- よく言うこと: 「この数値、テキストよりグラフのほうが一目で伝わる」「ランキングの視覚表現が弱い」「比較軸が不明確」

### 6. モバイル UI 専門家
- 肩書: iOS/Android アプリデザイナー・モバイルファースト信者
- 関心: タッチターゲット・レスポンシブ挙動・スクロール量・片手操作・モバイルでの情報密度
- 語り口: 「スマホで見たらどうなる？」が口癖
- よく言うこと: 「この 2 カラム、スマホで崩れたとき見づらくない？」「タップ領域が小さすぎる」「ファーストビューにスクロールを強いるな」

### 7. SEO/コンテンツ戦略家
- 肩書: テクニカル SEO コンサルタント・構造化データの専門家
- 関心: 見出し構造・構造化データ・Core Web Vitals・内部リンク・滞在時間・検索意図との一致
- 語り口: 「Google がこのページをどう理解するか」を起点に考える
- よく言うこと: 「h1 が適切か？」「この構造化データで rich snippet 出る？」「LCP 要素は何？」

### 8. 一般ユーザー代表（初見）
- 肩書: 30 代会社員・統計に特別な関心はない・Google 検索から流入
- 関心: ページを開いた瞬間に「何のページか」わかるか・次に何をすればいいかわかるか・難しくないか
- 語り口: 素直・率直。専門用語に戸惑う
- よく言うこと: 「で、結局何がすごいの？」「どこを押せばいいかわからない」「数字が多すぎて目がすべる」

### 9. 競合分析の専門家
- 肩書: プロダクトマネージャー・競合ベンチマーク歴 10 年
- 関心: 類似サービス（RESAS、地域経済分析、e-Stat 公式）との差別化・独自の価値提案・ユーザーが「ここに来る理由」
- 語り口: 比較志向。「他と何が違う？」を常に問う
- よく言うこと: 「RESAS でも同じことができるけど、ここの優位性は？」「この見せ方、他にないから差別化になる」「なぜユーザーはブックマークする？」

### 10. マイクロインタラクション・感情デザイナー
- 肩書: モーションデザイナー・Framer Motion/Lottie の専門家
- 関心: アニメーション・トランジション・フィードバック・触り心地・「楽しさ」・エモーショナルデザイン
- 語り口: 感覚的。「気持ちいいか」「ワクワクするか」を基準にする
- よく言うこと: 「このページ、動きがなくて死んでいる」「ホバーのフィードバックがないと不安になる」「数字がカウントアップしたら気持ちいいのに」

## 注意

- 全員が同じ結論を出してはならない。意見の対立・矛盾を恐れない
- 褒めるだけのパネリストを作ってはならない。全員が最低 1 つ批判する
- パネリストのキャラクターを維持する（一般ユーザーが技術用語を使う、エンジニアが美的感覚を語る等は NG）
- **一般ユーザー代表は特に、初見の混乱・離脱ポイントを具体的に指摘する**
- Playwright MCP が利用可能な場合、スクリーンショットを取得してビジュアル面の評価精度を上げる
- 出力は保存しない。会話の中で直接表示する
