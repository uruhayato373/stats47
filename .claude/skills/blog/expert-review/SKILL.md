ブログ記事を、記事テーマに対応する専門家の視点からレビューする。

## 引数

```
$ARGUMENTS — レビュー対象の記事ファイルパス
             （例: docs/21_ブログ記事原稿/commute-time-ranking/article.md）
```

専門家のペルソナは記事のテーマから自動判定する。必要に応じて引数に追記してもよい。
（例: `article.md 交通工学の研究者として`）

## 概要

統計データを扱うブログ記事において、**事実の正確性・統計的妥当性・読者への誤解リスク**を専門家の立場から批判的に評価する。

## ペルソナ（自動判定）

記事のテーマ・データ出典・使用指標から最も適切な専門家像を設定する。例:

| 記事テーマ | 専門家ペルソナ |
|---|---|
| 通勤・交通 | 交通工学・都市計画の研究者 |
| 人口・少子化 | 人口学・社会学の研究者 |
| 経済・賃金 | 労働経済学の研究者 |
| 医療・健康 | 公衆衛生・疫学の研究者 |
| 教育 | 教育社会学の研究者 |
| 環境・気候 | 環境科学の研究者 |
| 農業・食料 | 農業経済学の研究者 |

## レビュー構造

1. **専門家ペルソナ**（1行）: どの立場でレビューするかを明示する
2. **総評**（1段落）: 記事全体の信頼性・有用性を率直に述べる
3. **個別指摘**（3〜6項目）: `⚠️ 指摘N` 形式で問題点を挙げる
   - 何が問題か（具体的な箇所・数値・表現）
   - なぜ問題か（読者の誤解・統計的誤り・エビデンスの欠如）
4. **修正提案**（指摘と同数）: `✅ 修正提案N` 形式で実行可能な改善策を示す
5. **全体評価スコア**（5点満点）: 正確性・バランス・有用性の3軸で採点

## 評価の観点

- **データの正確性**: 引用した数値・統計が出典と一致しているか
- **因果 vs 相関**: 相関関係を因果と誤解させる表現がないか
- **出典の適切性**: データの調査年・定義・対象が記事の主張に合っているか
- **数値の文脈**: 「高い・低い」の基準が示されているか、比較対象は適切か
- **サンプリング・定義の説明**: 指標の定義が読者に伝わるか
- **過度な一般化**: 都道府県単位の統計を個人レベルに誤適用していないか

## 手順

1. 引数で指定された記事を読み込む
2. frontmatter の `tags` と本文の見出しからテーマを把握し、ペルソナを設定する
3. 記事本文・チャート説明・データ出典を確認する
4. 上記の構造・観点に従いレビューを作成する
5. `docs/03_レビュー/critical/{slug}_専門家レビュー.md` に保存する
6. 保存先のパスと総評を報告する

## 出力先

`docs/03_レビュー/critical/{slug}_専門家レビュー.md`
