stats47 の収益化戦略を 3-5 案ブレインストームし、適合度・リスク・検証実験を設計する。

原典: [phuryn/pm-skills](https://github.com/phuryn/pm-skills) (MIT License) の `monetization-strategy` を stats47 向けにカスタマイズ。

## 引数

```
/monetization-strategy [constraint]
```

- `constraint`（任意）: 制約条件（例: `no-ads`, `affiliate-only`, `月5万円目標`）

## プロジェクトコンテキスト

stats47 は都道府県統計データの可視化サイト。以下の特性を前提に分析すること:

- **プロダクト**: 47都道府県の統計ランキング・チャート・比較を無料提供
- **ユーザー**: 統計好き一般層、学生、ライター、メディア関係者、地方自治体関係者
- **現在の収益**: なし（広告・アフィリエイト検討中）
- **トラフィック**: SEO + SNS 経由
- **技術基盤**: Cloudflare Pages（サーバーコスト極小）
- **運営**: 個人開発
- **コンテンツ資産**: ランキングページ、ブログ記事、SNS 投稿（動画含む）

## 手順

### Step 1: 現状データ収集

以下を調査する:
- 公開記事数・ランキングページ数（DB articles テーブル、ranking_items テーブル）
- SNS フォロワー数・エンゲージメント（sns_metrics テーブル）
- 直近のアクセス規模（既知のデータがあれば使用）

### Step 2: 候補戦略のブレインストーム

stats47 に適した収益化モデルを 3-5 案生成する。以下のカテゴリを考慮:

| カテゴリ | stats47 での例 |
|---|---|
| **広告** | Google AdSense、記事内ネイティブ広告 |
| **アフィリエイト** | 統計関連書籍、ふるさと納税、引越し比較 |
| **データ販売** | CSV/API での統計データ提供（プレミアム） |
| **スポンサー** | 自治体・企業からの記事スポンサー |
| **コンテンツ課金** | 詳細分析レポートの有料化 |
| **ライセンス** | チャート埋め込みウィジェットの法人利用 |
| **コンサル** | 自治体向けデータ分析レポート作成 |

### Step 3: 各戦略の評価

各候補について以下を分析する:

#### 3.1 戦略概要
- 収益モデルの仕組み
- 誰が支払い、何を得るか

#### 3.2 ユーザー適合度
- ターゲットユーザーの支払い意思
- ユーザー体験への影響（離脱リスク）

#### 3.3 ユニットエコノミクス推定
- 想定月間収益レンジ（悲観/標準/楽観）
- 必要トラフィック量
- 実装・運用コスト

#### 3.4 リスクと課題
- 市場リスク（需要不確実性）
- 実装複雑度
- ユーザー体験の毀損リスク
- 競合との価格競争リスク

#### 3.5 検証実験
- 低コストで検証する方法
- 成功基準と判断指標
- 実験期間

### Step 4: 優先度マトリクス

| 戦略 | 収益ポテンシャル | 実装容易性 | UX影響 | 推奨順位 |
|---|---|---|---|---|

### Step 5: 実装ロードマップ

- **Phase 1**（今すぐ）: 最も低リスク・低コストな戦略を実装
- **Phase 2**（1-2ヶ月後）: データ蓄積後に第2戦略を追加
- **Phase 3**（3ヶ月後）: 結果に基づき最適化

## 注意事項

- 収益化はユーザー体験を損なわない範囲で実施する
- 個人開発のため運用負荷が低い戦略を優先する
- 統計データの公共性を損なわない（基本データは無料維持）
- 将来的なハイブリッドモデル（広告 + アフィリエイト + データ）を見据える
