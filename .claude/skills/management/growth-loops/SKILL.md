stats47 の持続的成長ループ（フライホイール）を設計・評価する。

原典: [phuryn/pm-skills](https://github.com/phuryn/pm-skills) (MIT License) の `growth-loops` を stats47 向けにカスタマイズ。

## 引数

```
/growth-loops [focus]
```

- `focus`（任意）: 特定のループタイプに絞る場合（例: `viral`, `content`, `seo`）

## プロジェクトコンテキスト

stats47 は都道府県統計データの可視化サイト。以下の特性を前提に分析すること:

- **プロダクト**: 47都道府県の統計ランキング・チャート・比較機能を無料提供
- **コンテンツ**: ブログ記事、SNS 投稿（X/Instagram/YouTube/TikTok）、note.com 記事
- **現在の成長チャネル**: SEO（検索流入）、SNS 投稿、note.com からの流入
- **収益モデル**: 広告（将来的にアフィリエイト）
- **ユーザー**: 統計に興味がある一般層、学生、ライター、メディア関係者
- **技術基盤**: Next.js + Cloudflare Pages、D1、R2

## 手順

### Step 1: 現状の成長チャネル調査

以下を並列サブエージェントで収集する:

#### Agent A: トラフィックデータ
```
- DB (sns_metrics) から直近のSNSパフォーマンス
- 公開記事数（articles テーブル）
- .local/r2/blog/ の記事数
```

#### Agent B: コンテンツ資産
```
- D1 sns_posts テーブルから投稿状況（SELECT domain, platform, status, COUNT(*) FROM sns_posts GROUP BY domain, platform, status）
- .local/r2/sns/ の生成済みコンテンツ
- ブログ記事の企画状況（docs/90_課題管理/ブログ記事一括企画.md）
```

### Step 2: 5つの成長ループを評価

stats47 の特性に合わせて以下の5ループを評価する:

#### 1. コンテンツ SEO ループ
記事公開 → 検索流入 → ユーザーがデータ引用・シェア → 被リンク増加 → ドメイン権威向上 → 検索順位上昇

#### 2. SNS バイラルループ
統計データ投稿 → 「意外な事実」がシェアされる → 新規ユーザー流入 → サイトでデータ発見 → 自発的にシェア

#### 3. UGC（ユーザー生成コンテンツ）ループ
ユーザーがランキングデータを引用 → ブログ・記事・レポートで stats47 にリンク → SEO 向上 → 新規ユーザー

#### 4. データ引用ループ
CSV ダウンロード・埋め込みチャート提供 → メディア・研究者が利用 → クレジット付き引用 → 権威性向上

#### 5. クロスプラットフォームループ
YouTube → Instagram → X → note.com → サイト → 各 SNS のフォロワー増 → リーチ拡大

### Step 3: 各ループの評価基準

| 基準 | 説明 |
|---|---|
| **適合度** | stats47 のプロダクト特性との整合性（1-5） |
| **現在の成熟度** | 既に機能しているか、これから構築か（1-5） |
| **実装コスト** | 必要な開発・運用コスト（S/M/L） |
| **成長速度** | 効果が出るまでの期間（短期/中期/長期） |
| **複利効果** | 時間とともに加速するか |

### Step 4: ループ係数の推定

最も有望なループについて:
- ユーザー1人あたりのシェア/引用数
- シェア → 新規ユーザーへの変換率
- サイクル1回あたりの期間
- 複利的な成長予測

### Step 5: 実装ロードマップ

| 期間 | アクション |
|---|---|
| 30日 | 最優先ループの仕組みを実装 |
| 60日 | 計測・最適化、第2ループの準備 |
| 90日 | 第2ループ稼働、複合効果の検証 |
