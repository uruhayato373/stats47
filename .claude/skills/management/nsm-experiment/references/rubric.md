# NSM 実験 優先順位評価ルーブリック

`/nsm-experiment propose` が候補を採点するときに使う評価軸。**加重合計で実験提案の順位を決定**する。

## 4 軸評価（各 0-3 点）

### 1. インパクト (重み 40%)

目標 metric への期待 delta。

| 点 | 条件 |
|---|---|
| 3 | 対象クエリ/ページのインプレッションが過去 30 日で 100+ / 期待 delta が現状値の 2 倍以上 |
| 2 | インプレッション 30-100 / 期待 delta 1.5 倍 |
| 1 | インプレッション 10-30 / 期待 delta 1.2 倍 |
| 0 | インプレッション 10 未満 / 期待 delta 不明 |

**原則**: インパクト 0 の候補は提案リストに含めない（サンプルサイズが小さすぎる）。

### 2. 工数 (重み 30%)

実装からデプロイまでの総作業時間。

| 点 | 条件 |
|---|---|
| 3 | 30 分以内（単一ページの title/description 書き換え） |
| 2 | 30 分-2 時間（複数ページ編集、1 つの新規セクション追加） |
| 1 | 2 時間-半日（新規ページ作成、ランキング追加、リファクタリング） |
| 0 | 半日以上（大規模変更、技術的 SEO 実装、新規エージェント開発） |

**原則**: 工数 0 でも学習価値が高ければ提案可、ただし rubric では低く出る。

### 3. 学習価値 (重み 20%)

結果が他ページ/他施策に転用できる度合い。

| 点 | 条件 |
|---|---|
| 3 | 結果が全カテゴリ（ランキング / テーマ / ブログ）に転用可能（例: title パターンの発見） |
| 2 | 結果が同カテゴリ内で転用可能 |
| 1 | 結果が単一ページ改善のみ |
| 0 | 結果が再現性なし（偶然依存） |

### 4. 確実性 (重み 10%)

過去の成功パターン踏襲度合い。

| 点 | 条件 |
|---|---|
| 3 | `playbook.md` に記載された確立パターンそのまま |
| 2 | playbook パターンの軽微な応用 |
| 1 | 新規パターンだが仮説が明確 |
| 0 | 未検証・勘に近い手法 |

## 加重合計の計算

```
加重合計 = impact * 0.40 + effort * 0.30 + learning * 0.20 + certainty * 0.10
```

**スケール**: 0.00 - 3.00

**しきい値**:

| 加重合計 | 判定 | 扱い |
|---|---|---|
| ≥ 2.5 | 最優先 | 即提案、次週に実施推奨 |
| 2.0 - 2.5 | 優先 | 提案リスト上位、選択肢に入れる |
| 1.5 - 2.0 | 候補 | 提案リスト下位、他に優先度低いものがなければ |
| < 1.5 | 不採用 | 提案リストから除外 |

## 原則

### 同時 active 実験 ≤ 2 件

複数実験を同時に動かすと原因分離できなくなる。`listActive()` が 2 件以上返す場合、`/nsm-experiment start` が警告を出す。例外: 対象クエリが完全に独立している場合（別カテゴリ・別ページ・別メトリクス）。

### 計測期間の最低ライン

- GSC 指標: 10 日（3 日遅延 + 7 日評価）
- GA4 指標: 7 日（即日反映、ノイズ除去に 1 週間）
- コンテンツ追加: 14-28 日（インデックス待ち）
- CWV / PSI: 28 日（CrUX は 28 日移動平均）

この期間未満での `/nsm-experiment measure` は警告を出す。

### 失敗実験の記録

効果が出なかった実験も `close --result no-effect` or `negative` で記録する。「やっても効かないこと」を playbook に追記することで将来の無駄提案を防ぐ。

### rubric 自体の改訂

このファイルは固定ではなく、実験サイクルを回す中で調整する。例えば:

- 「インパクト 2 点」の基準が現実とズレていたら閾値を変更
- 新しい評価軸（SEO の複雑度など）が必要なら追加
- 不要な軸は削除

改訂は実験 10 件程度を経てから検討する。

## 計算例

**例 1**: 「神奈川県 統計」クエリの title 改善
- impact: 3 (月間 100+ impressions 見込み)
- effort: 3 (30 分)
- learning: 2 (title パターンを発見できれば他の県にも転用)
- certainty: 2 (playbook 1-A 応用)
- **加重合計 = 3*0.4 + 3*0.3 + 2*0.2 + 2*0.1 = 1.2 + 0.9 + 0.4 + 0.2 = 2.7** → 最優先

**例 2**: 新規ランキング追加 + AI コンテンツ生成
- impact: 2 (中規模、月 30 impressions 予想)
- effort: 1 (半日: e-Stat 調査 + register + AI 生成 + デプロイ)
- learning: 3 (新規ランキング登録フローの検証になる)
- certainty: 2 (既存の `/register-ranking` パターン踏襲)
- **加重合計 = 2*0.4 + 1*0.3 + 3*0.2 + 2*0.1 = 0.8 + 0.3 + 0.6 + 0.2 = 1.9** → 候補

**例 3**: 単一ページの微修正
- impact: 1 (月 10 impressions)
- effort: 3 (10 分)
- learning: 1 (他転用なし)
- certainty: 3 (明確)
- **加重合計 = 1*0.4 + 3*0.3 + 1*0.2 + 3*0.1 = 0.4 + 0.9 + 0.2 + 0.3 = 1.8** → 候補

**例 4**: サイト全体のナビ リデザイン
- impact: 3 (全体)
- effort: 0 (数日)
- learning: 3 (UX 全体に転用可)
- certainty: 0 (未検証)
- **加重合計 = 3*0.4 + 0*0.3 + 3*0.2 + 0*0.1 = 1.2 + 0 + 0.6 + 0 = 1.8** → 候補

**教訓**: 工数 0 の実験は「学習価値と確実性」の両方が 3 でないと加重が上がらない。スコープ分割を検討すべきシグナル。
