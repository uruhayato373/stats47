## DB 実装状況

### テーブル

| テーブル | 行数 | 内容 |
|---|---|---|
| `ports` | 171+α | 甲種港湾171港 + 乙種港湾（港名・都道府県・緯度経度・港格・管理者） |
| `port_statistics` | ~25,000 | 年度別統計データ（港×年×指標、甲種+乙種） |

### 位置情報

国土数値情報 C02（港湾データ 2014年）から甲種 171/171 港の緯度・経度を取得済み。
乙種港湾の緯度・経度は未取得（C02 データに全国994港が含まれるため取得可能だが未実装）。

### データ投入スクリプト

```bash
# 乙種港湾マスタを投入（初回のみ）
npx tsx packages/database/scripts/import-otsu-ports.ts

# 全指標を投入（甲種＋乙種）
npx tsx packages/database/scripts/populate-port-statistics.ts

npx tsx packages/database/scripts/populate-port-statistics.ts --metric passengers

# 甲種のみ / 乙種のみ
npx tsx packages/database/scripts/populate-port-statistics.ts --class 甲種

npx tsx packages/database/scripts/populate-port-statistics.ts --dry-run
```

## 港湾コード体系

```
PPNNN
│ └── 港湾連番（001～）
└──── 都道府県コード（01～47）
```

例: `01001` = 北海道_稚内, `23001` = 愛知県_名古屋, `47008` = 沖縄県_那覇

**注意**: e-Stat の港湾コードと国土数値情報 C02 の港湾コードは一部不一致。DB には e-Stat のコードを採用。

## クエリ例

```sql
-- 港別ランキング（2023年 貨物量）
SELECT p.port_name, p.prefecture_name, ps.value
FROM port_statistics ps
JOIN ports p ON ps.port_code = p.port_code
WHERE ps.metric_key = 'cargo_total' AND ps.year = '2023'
ORDER BY ps.value DESC;

-- 年度推移
SELECT year, value FROM port_statistics
WHERE port_code = '23001' AND metric_key = 'cargo_total'
ORDER BY year;

-- 都道府県別集計
SELECT p.prefecture_name, SUM(ps.value) as total
FROM port_statistics ps
JOIN ports p ON ps.port_code = p.port_code
WHERE ps.metric_key = 'cargo_total' AND ps.year = '2023'
GROUP BY p.prefecture_code
ORDER BY total DESC;

-- 旅客数ランキング（2023年）
SELECT p.port_name, p.prefecture_name, ps.value
FROM port_statistics ps
JOIN ports p ON ps.port_code = p.port_code
WHERE ps.metric_key = 'passengers_total' AND ps.year = '2023'
ORDER BY ps.value DESC
LIMIT 20;
```
