# フォント色統一設定ガイド

## 現状分析

### プロジェクト構成
- **フレームワーク**: Next.js 15.5.2 + React 19
- **スタイリング**: Tailwind CSS v4
- **テーマ管理**: カスタムThemeContext
- **状態管理**: Jotai（一部）

### 現在のテーマ設定状況
プロジェクトには既にダークモード・ライトモード切り替え機能が実装されている：

1. **テーマ管理**: `src/contexts/ThemeContext.tsx` でテーマ状態を管理
2. **グローバルCSS**: `src/app/globals.css` でライト/ダーク両モードの基本色定義済み
3. **Tailwind設定**: `tailwind.config.ts` でカスタムカラーパレット定義済み
4. **スタイルフック**: `src/hooks/useStyles.ts` でコンポーネント別スタイル定義済み

## フォント色統一の実装方針

### 1. 基本方針
- **統一性**: 既存のカラーシステムを活用し、一貫したフォント色階層を構築
- **保守性**: useStylesフックを中心とした管理で、変更時の影響範囲を最小化
- **アクセシビリティ**: WCAG 2.1 AA基準を満たすコントラスト比を確保

### 2. フォント色階層定義

#### ライトモード
```
Primary Text:    #1f2937 (gray-800)  - 見出し、重要なテキスト
Secondary Text:  #374151 (gray-700)  - 本文
Tertiary Text:   #6b7280 (gray-500)  - 補助テキスト、説明文
Muted Text:      #9ca3af (gray-400)  - 非活性、ヒント文字
```

#### ダークモード
```
Primary Text:    #f9fafb (gray-50)   - 見出し、重要なテキスト
Secondary Text:  #e5e7eb (gray-200)  - 本文
Tertiary Text:   #d1d5db (gray-300)  - 補助テキスト、説明文
Muted Text:      #9ca3af (gray-400)  - 非活性、ヒント文字
```

### 3. 実装アプローチ

#### A. useStylesフック拡張（推奨）
既存の`src/hooks/useStyles.ts`を拡張し、フォント色のスタイルセットを統一

#### B. CSS変数活用
globals.cssでフォント色専用のCSS変数を定義し、Tailwindクラスでマッピング

#### C. Tailwindカスタムクラス
tailwind.config.tsでセマンティックなフォント色クラスを定義

## 具体的な実装手順

### Step 1: useStylesフックの拡張
`src/hooks/useStyles.ts`にフォント色定義を追加：

```typescript
// テキスト色の統一定義
text: {
  primary: "text-gray-800 dark:text-gray-50",      // 見出し、重要
  secondary: "text-gray-700 dark:text-gray-200",   // 本文
  tertiary: "text-gray-500 dark:text-gray-300",    // 補助
  muted: "text-gray-400 dark:text-gray-400",       // 非活性
  
  // 特殊用途
  brand: "text-indigo-600 dark:text-indigo-400",   // ブランドカラー
  success: "text-green-600 dark:text-green-400",   // 成功状態
  warning: "text-amber-600 dark:text-amber-400",   // 警告状態
  error: "text-red-600 dark:text-red-400",         // エラー状態
},
```

### Step 2: 既存コンポーネントの移行
テキスト色を使用している全コンポーネントを段階的に移行：

1. **見出し系コンポーネント**: H1, H2, H3 → `text.primary`
2. **本文系コンポーネント**: P, div → `text.secondary`  
3. **補助テキスト**: label, caption → `text.tertiary`
4. **非活性テキスト**: disabled, placeholder → `text.muted`

### Step 3: Tailwindカスタムクラスの定義（オプション）
`tailwind.config.ts`にセマンティッククラスを追加：

```typescript
theme: {
  extend: {
    textColor: {
      'content-primary': 'rgb(31 41 55 / <alpha-value>)',
      'content-secondary': 'rgb(55 65 81 / <alpha-value>)',
      'content-tertiary': 'rgb(107 114 128 / <alpha-value>)',
      'content-muted': 'rgb(156 163 175 / <alpha-value>)',
    }
  }
}
```

### Step 4: Storybookでの確認
各コンポーネントのStoryでライト/ダーク両モードでの表示を確認

### Step 5: テストの実装
アクセシビリティテスト含めて、コントラスト比の確認テストを実装

## メンテナンス方針

### 1. 新規コンポーネント作成時
- useStylesフックのテキストスタイルを必ず使用
- 直接的なTailwindカラークラスは禁止

### 2. 既存コンポーネント修正時
- テキスト色変更時は統一されたスタイルセットを使用
- レビュー時にコンソーシング確認

### 3. デザイン変更時
- useStylesフックの定義のみ変更
- 個別コンポーネントへの影響を最小化

## 注意事項

### 実装時の注意点
1. **段階的移行**: 全コンポーネントを一度に変更せず、段階的に移行
2. **テスト確認**: 各段階でテーマ切り替えが正常に動作することを確認
3. **アクセシビリティ**: コントラスト比が基準を満たしていることを確認

### 避けるべき実装
1. **ハードコーディング**: コンポーネント内での直接的な色指定
2. **不統一なクラス名**: 同じ意味でも異なるクラス名の使用
3. **CSS変数の重複定義**: globals.cssとTailwindでの重複定義
