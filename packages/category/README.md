# @stats47/category

統計データの分類・管理・ナビゲーションを担当するドメインパッケージ。

> [!IMPORTANT]
> **移行期間中**: 現在、カテゴリ・サブカテゴリ構造から**タグベース構造**への移行フェーズです。
> `categories`, `subcategories` テーブルは互換性のために維持されていますが、推奨される新しい構造は `tags` テーブルです。

## 概要

Category & Tag ドメインは、stats47 プロジェクトにおける統計データの分類・管理・ナビゲーション機能を提供します。
従来の厳格な「カテゴリ/サブカテゴリ」階層構造に加え、より柔軟な「タグ」システムを導入し、多面的なデータ分類を実現します。

### ビジネス価値

- **多面的な分類**: ランキング項目やダッシュボードを、階層に縛られずに柔軟に分類可能
- **情報の構造化**: ユーザーが目的のデータを発見しやすくする
- **ナビゲーション支援**: 関連コンテンツ（タグ）による回遊性の向上
- **拡張性**: 新しいトピックや地域タグの追加が容易

## 責務

| 責務 | 説明 |
| :--- | :--- |
| **タグ管理** | タグ（カテゴリ、トピック等）の定義 |
| **階層管理** | タグ間の親子関係の管理 |
| **関連付け** | コンテンツ（ランキング等）とタグの紐付け |
| **表示順序制御** | 表示順序管理 |
| **ナビゲーション** | UI コンポーネントでのタグ表示 |

## ドメインモデル

### 主要テーブル (Active)

#### `tags`（タグマスター）
タグの一意識別子、表示名、種類（category, topic等）、親タグID、アイコン等を保持します。

#### `ranking_item_tags`（ランキング・タグ関連）
ランキング項目（ranking_key, area_type）とタグの紐付けを管理します。

#### `dashboard_tags`（ダッシュボード・タグ関連）
ダッシュボードとタグの紐付けを管理します。
