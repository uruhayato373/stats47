# repositories

外部データソースへのアクセスを担当するレイヤー。

## 責務

- 外部 API、データベース、ストレージへの CRUD 操作
- データソース固有の処理（リクエスト構築、レスポンス検証など）
- 低レベルのエラーハンドリング

## サブディレクトリ

| ディレクトリ | データソース | 説明 |
|-------------|-------------|------|
| `api/` | e-Stat API | 外部 API との HTTP 通信 |
| `cache/` | R2 | オブジェクトストレージへのキャッシュ操作 |
| `d1/` | D1 | リレーショナルデータベースへの操作 |

## 原則

- 1ファイル1関数
- ビジネスロジックを含まない
- データソースの詳細を上位レイヤーから隠蔽
