# services

ビジネスロジックとオーケストレーションを担当するレイヤー。

## 責務

- 複数のリポジトリや utils を組み合わせた処理フロー
- キャッシュ戦略の実装（キャッシュ確認 → API取得 → キャッシュ保存）
- エラーハンドリングとドメイン固有エラーへの変換

## 原則

- 1ファイル1関数
- 外部データソースへの直接アクセスは行わない（repositories を使用）
- データ変換ロジックは utils に委譲

## ファイル一覧

| ファイル | 関数 | 説明 |
|----------|------|------|
| `fetch-meta-info.ts` | `fetchMetaInfo` | メタ情報取得（キャッシュ優先） |
