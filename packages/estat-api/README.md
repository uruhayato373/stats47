# @stats47/estat-api

## 概要

e-Stat API との統合を担当するコアパッケージです。
統計データ、メタ情報、統計リストの取得と、アプリケーション内での正規化された形式への変換を提供します。

## サブドメイン構成

本パッケージは以下の 3 つの主要機能（サブドメイン）で構成されています：

1.  **[統計データ (stats-data)](./src/stats-data/README.md)**: `getStatsData` エンドポイント。多次元の統計値の取得と整形。
2.  **[メタ情報 (meta-info)](./src/meta-info/README.md)**: `getMetaInfo` エンドポイント。統計表の定義（分類・地域・時間軸）の取得。
3.  **[統計表リスト (stats-list)](./src/stats-list/README.md)**: `getStatsList` エンドポイント。利用可能な統計表の検索・探索。

## 設計原則

- **Fetcher-Formatter パターン**: データの「取得（HTTP通信）」と「加工（正規化）」を明確に分離します。
- **R2 キャッシュ戦略**: API のレート制限を回避し高速化するため、e-Stat API の生レスポンスを R2 ストレージにキャッシュします。
- **純粋関数**: 加工ロジックは副作用のない純粋関数として実装し、高いテスト容易性を確保します。

## 共通レイヤー

- `core/`: 共通の HTTP クライアント、型定義、エラーハンドリング、レート制限、リトライロジックを含みます。

## 利用可能な機能

- 統計表のキーワード検索
- 分類・地域・年度情報を含む詳細なメタデータの取得
- 都道府県別、市区町村別等の統計データの取得と正規化
- R2 ストレージを活用した永続キャッシュ
